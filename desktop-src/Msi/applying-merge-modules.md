# <a name="applying-merge-modules"></a>Application de modules de fusion

Les modules de fusion doivent être fusionnés dans un package d’installation à l’aide d’un outil de fusion. La meilleure solution consiste à obtenir un outil de fusion distribué librement ou à acheter l’un des outils de fusion à la disposition des éditeurs de logiciels indépendants. Par exemple, vous pouvez utiliser la fonctionnalité fournie par Mergemod.dll.

Les modules de fusion à plusieurs langages peuvent être ouverts dans des langages différents, puis fusionnés dans une cible pour produire une version de langue différente du package. Pour plus d’informations, consultez :

-   [Utilisation de l’automatisation pour fusionner un module de fusion dans une base de données](using-automation-to-merge-a-merge-module-into-a-database.md)
-   [Utilisation de l’API pour fusionner un module de fusion dans une base de données](using-the-api-to-merge-a-merge-module-into-a-database.md)
-   [Ouverture d’un module de fusion Multiple-Language dans un langage spécifique](opening-a-multiple-language-merge-module-in-a-specific-language.md)
-   [Fusion multiple d’un module multiple Language dans le même package](merging-a-multiple-language-module-into-the-same-package-multiple-times.md)
-   [Connexion d’un module de fusion à plusieurs fonctionnalités](connecting-a-merge-module-to-multiple-features.md)
