# <a name="using-isolated-components"></a>Utilisation des composants isolés

En utilisant des [composants isolés](isolated-components.md), les créateurs de packages d’installation peuvent spécifier que le programme d’installation copie les fichiers partagés d’une application directement dans le dossier de l’application plutôt que dans un emplacement partagé. Ce jeu de fichiers privé est ensuite utilisé exclusivement par l’application.

Pour plus d’informations, consultez [installation d’un composant COM dans un emplacement privé](installing-a-com-component-to-a-private-location.md), [installation d’un composant non-com dans un emplacement privé](installing-a-non-com-component-to-a-private-location.md), [rendre un composant COM dans un package existant privé](make-a-com-component-in-an-existing-package-private.md) ou [rendre un composant non-com dans un package existant privé](make-a-non-com-component-in-an-existing-package-private.md).
