# <a name="machine-policies"></a>Stratégies d’ordinateur

Les stratégies d’ordinateur suivantes peuvent être configurées sous :

**HKEY \_ \_** \\ **stratégies logicielles** de l’ordinateur LOCAL \\  \\ **Microsoft** \\ **Windows** \\ **Installer**



| Policy                                                                                    | Type de données valeur           | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
|-------------------------------------------------------------------------------------------|---------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [AlwaysInstallElevated a](alwaysinstallelevated.md)<br/>                             | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie est définie sur 1 et que la valeur utilisateur correspondante est également définie, le programme d’installation installe toujours avec des privilèges élevés.<br/> Dans le cas contraire, le programme d’installation utilise des privilèges élevés pour installer les applications managées et utilise le niveau de privilège de l’utilisateur actuel pour les applications non managées.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| [AllowLockdownBrowse](allowlockdownbrowse.md)<br/>                                 | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie est définie sur 1, les utilisateurs non-administrateurs peuvent rechercher de nouvelles sources lors de l’exécution d’une installation avec des privilèges élevés. La valeur par défaut est que seuls les administrateurs peuvent rechercher des sources au cours d’une installation avec élévation de privilèges. La définition de cette stratégie permet également aux utilisateurs non-administrateurs d’exécuter des programmes sur des privilèges LocalSystem au cours d’une installation avec élévation de privilèges.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| [AllowLockdownMedia](allowlockdownmedia.md)<br/>                                   | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie est définie sur 1, les utilisateurs non-administrateurs peuvent utiliser des sources multimédias, comme un CD-ROM, lors de l’exécution d’une installation avec des privilèges élevés. La valeur par défaut est que seuls les administrateurs peuvent utiliser des sources multimédias au cours d’une installation avec élévation de privilèges. La définition de cette stratégie permet également aux utilisateurs non-administrateurs d’exécuter des programmes sur des privilèges LocalSystem au cours d’une installation avec élévation de privilèges.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| [AllowLockdownPatch](allowlockdownpatch.md)<br/>                                   | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie système par ordinateur n’est pas définie, seuls les administrateurs peuvent corriger les produits existants qui ont été installés avec des privilèges élevés. Si cette valeur de stratégie est définie sur 1, les utilisateurs non-administrateurs peuvent, dans certains cas, appliquer des correctifs aux produits lors de l’exécution d’une installation à l’aide de privilèges élevés. Avec la stratégie définie, le correctif peut installer des [mises à niveau mineures](minor-upgrades.md) lors de l’exécution d’une installation à l’aide de privilèges élevés. le correctif ne peut pas installer les [mises à niveau majeures](major-upgrades.md). La définition de cette stratégie permet également aux utilisateurs non-administrateurs d’exécuter des programmes sur des privilèges LocalSystem au cours d’une installation avec élévation de privilèges.<br/>                                                                                                                                                                                                                                                                                      |
| [Déboguer](debug.md)<br/>                                                             | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie existe et qu’elle a la valeur 1, le programme d’installation écrit des messages de débogage communs dans le débogueur à l’aide de la fonction [**OutputDebugString**](/windows/win32/api/debugapi/nf-debugapi-outputdebugstringa) . Si cette valeur existe et est définie sur 2, le programme d’installation écrit tous les messages de débogage valides dans le débogueur à l’aide de la fonction **OutputDebugString** .<br/> cette stratégie est à des fins de débogage uniquement et peut ne pas être prise en charge dans les futures versions de Windows Installer.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

| [DisableBrowse](disablebrowse.md)<br/>                                             | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie existe et qu’elle est définie sur 1, les utilisateurs ne peuvent pas parcourir pour localiser les sources du programme d’installation. La zone de liste déroulante **utiliser des fonctionnalités de** pour l’entrée directe est verrouillée et le bouton **Parcourir** est désactivé. Pour plus d’informations sur la navigation source, consultez [résilience source](source-resiliency.md).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| [DisableFlyWeightPatching](disableflyweightpatching.md)<br/>                       | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie système par ordinateur est définie sur 1, toutes les options d' [optimisation des correctifs](patch-optimization.md) sont désactivées pendant l’installation.<br/> **Windows Installer 2,0 :** Non pris en charge.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| [DisableLUAPatching](disableluapatching.md)<br/>                                   | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie système par ordinateur est définie sur 1, le programme d’installation empêche les non-administrateurs d’utiliser une mise à jour corrective de compte à privilèges limités vers n’importe quelle application installée sur l’ordinateur. Lorsque cette valeur n’est pas définie ou égale à 0, les non-administrateurs peuvent appliquer des correctifs LUA à une application compatible LUA.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| [DisableMSI](disablemsi.md)<br/>                                                   | **\_valeur DWORD reg**<br/> | si la valeur de cette stratégie est définie sur 0, si elle est absente ou n’est pas un nombre autre que 1 ou 2, l’effet sur le Windows Installer dépend du système d’exploitation. sur Windows Server 2003, Windows Installer est activée pour les applications gérées et désactivée pour les installations d’applications non gérées. sur Windows XP, le Windows Installer est activé pour toutes les applications.<br/> si cette valeur de stratégie est définie sur 0, Windows Installer est activé pour toutes les applications. Toutes les opérations d’installation sont autorisées.<br/> si cette valeur de stratégie est définie sur 1, Windows Installer est désactivée pour les applications non managées, mais est toujours activée pour les applications managées. Les installations par utilisateur non élevées sont bloquées. Les installations par utilisateur avec élévation de privilèges et par ordinateur sont autorisées.<br/> si cette valeur de stratégie est définie sur 2, Windows Installer est toujours désactivée pour toutes les applications. Aucune installation n’est autorisée, y compris les réparations, les réinstallations ou les installations à la demande.<br/> |
| [DisablePatch](disablepatch.md)<br/>                                               | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie est définie sur 1, le programme d’installation n’applique pas les correctifs. | [DisableSharedComponent](disablesharedcomponent.md)<br/>                           | **\_valeur DWORD reg**<br/> | Si cette [stratégie système](system-policy.md) par ordinateur est définie sur 1, aucun package sur le système n’obtient la fonctionnalité de composant partagé activée par l’attribut **msidbComponentAttributesShared** dans la [table des composants](component-table.md).<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| [DisableUserInstalls](disableuserinstalls.md)<br/>                                 | **\_valeur DWORD reg**<br/> | Si cette valeur de stratégie n’est pas définie, le programme d’installation recherche les produits dans le registre dans l’ordre suivant : produits gérés inscrits en tant que produits par utilisateur, produits non gérés inscrits en tant qu’utilisateur et enfin produits inscrits en tant qu’ordinateur.<br/> Si cette valeur de stratégie est définie sur 1, le programme d’installation ignore tous les produits inscrits en tant qu’utilisateur et recherche uniquement les produits inscrits en tant qu’ordinateur. Si vous tentez d’effectuer une installation par utilisateur, le programme d’installation affiche un message d’erreur et arrête l’installation.<br/>                                                                                                                                                                                                                                                                                                                                                              |

Lorsque cette stratégie n’est pas définie, la valeur par défaut est 10%.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
