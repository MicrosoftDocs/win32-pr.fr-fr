# <a name="programming-with-the-system-registry-provider"></a>Programmation avec le fournisseur de Registre système

Les rubriques suivantes décrivent comment les applications peuvent utiliser le fournisseur de Registre système pour fournir des données pour les classes que vous créez.



| Rubrique                                                                                                                      | Description                                                                                                                                                                                                                    |
|----------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Définition des classes pour le fournisseur de Registre système](defining-classes-for-the-system-registry-provider.md)                 | Pour accéder au registre système, vous devez créer des classes Windows qui stockent les informations de Registre dans le référentiel Windows Management Instrumentation (WMI).                                                                     |
| [Utilisation du fournisseur de Registre système en tant que fournisseur de propriétés](using-the-system-registry-provider-as-a-property-provider.md) | Le fournisseur de Registre système requiert que vous utilisiez plusieurs qualificateurs pour afficher les propriétés du Registre.                                                                                                                             |
| [Description d’une ressource pour le registre](describing-a-resource-for-the-registry.md)                                       | Vous pouvez accéder aux données du Registre qui décrivent une ressource système. Pour modifier les informations, le fournisseur de Registre système requiert une syntaxe spécifique.                                                                      |
| [Accès à une valeur de registre sans nom](accessing-an-unnamed-registry-value.md)                                             | Pour accéder à la plupart des valeurs de Registre, créez une classe pour recevoir les valeurs dans WMI, ou vous pouvez examiner des propriétés de Registre spécifiques. Toutefois, le fournisseur de Registre système requiert une syntaxe spéciale pour afficher les valeurs de registre sans nom. |
