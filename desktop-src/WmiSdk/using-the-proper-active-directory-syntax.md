# <a name="using-the-proper-active-directory-syntax"></a>Utilisation de la syntaxe de Active Directory appropriée

Vous fournissez l’espace de noms LDAP (Lightweight Directory Access Protocol) et les chemins d’accès de l’interface Active Directory Services (ADSI) lors de l’appel du [fournisseur Active Directory](/previous-versions/windows/desktop/dsprov/active-directory-provider) via WMI.

Les rubriques suivantes décrivent l’espace de noms LDAP et le chemin d’accès ADSI :

-   [Description de l’espace de noms LDAP](describing-the-ldap-namespace.md)

    Un chemin d’accès d’espace de noms décrit l’emplacement d’un espace de noms distant. Utilisez le chemin d’accès de l’espace de noms pour vous connecter à distance à un espace de noms Active Directory.

-   [Description du chemin d’accès ADSI](describing-the-adsi-path.md)

    Le chemin d’accès ADSI décrit l’emplacement d’un objet à l’aide de l’API ADSI. Utilisez le chemin d’accès ADSI dans les paramètres que le fournisseur passe sur Active Directory.

> [!Note]  
> Pour plus d’informations sur la prise en charge et l’installation de ce composant sur un système d’exploitation spécifique, consultez [disponibilité du système d’exploitation des composants WMI](operating-system-availability-of-wmi-components.md).
