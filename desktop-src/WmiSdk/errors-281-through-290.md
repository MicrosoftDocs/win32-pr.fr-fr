## <a name="fatal-error-282"></a>Erreur irrécupérable 282

<dl> <dt>

<span id="_282__Fatal_____fileName___line____Syntax_error._Last_token_read_is__token__"></span><span id="_282__fatal_____filename___line____syntax_error._last_token_read_is__token__"></span><span id="_282__FATAL_____FILENAME___LINE____SYNTAX_ERROR._LAST_TOKEN_READ_IS__TOKEN__"></span>**<282,> fatale : " <fileName> : <\#> de ligne : erreur de syntaxe. La dernière lecture de jeton est <token> «**
</dt> <dd>

Cette erreur se produit lorsque le compilateur sait qu’une erreur de syntaxe s’est produite, mais n’est pas en mesure de fournir des informations plus spécifiques sur l’erreur de syntaxe.

</dd> </dl>

## <a name="fatal-error-283"></a>Erreur irrécupérable 283

<dl> <dt>

<span id="_283__Fatal_____fileName___line_____Unrecognized_character__character__in_input_"></span><span id="_283__fatal_____filename___line_____unrecognized_character__character__in_input_"></span><span id="_283__FATAL_____FILENAME___LINE_____UNRECOGNIZED_CHARACTER__CHARACTER__IN_INPUT_"></span>**<283,> irrécupérable : " <fileName> : <ligne \# \#> : caractère non reconnu <character> dans l’entrée"**
</dt> <dd>

L’entrée contient un caractère non reconnu. En règle générale, les caractères non reconnus sont ignorés, et ce message d’erreur indique le premier caractère non reconnu.

</dd> </dl>
