# <a name="i-wmi"></a>I (WMI)

<dl> <dt>

<span id="wmi.gloss_impersonation"></span><span id="WMI.GLOSS_IMPERSONATION"></span>**emprunt d’identité**
</dt> <dd>

Capacité d’un thread dans un processus à assumer l’identité de sécurité et les privilèges d’un autre processus.

</dd> <dt>

<span id="wmi.gloss_instance_provider"></span><span id="WMI.GLOSS_INSTANCE_PROVIDER"></span>**fournisseur d’instance**
</dt> <dd>

Serveur COM qui fournit des instances de classes et implémente des méthodes de l’interface [**IWbemServices**](/windows/desktop/api/WbemCli/nn-wbemcli-iwbemservices) pour prendre en charge la récupération, la modification, la suppression, l’énumération ou le traitement des requêtes de données.

</dd> <dt>

<span id="wmi.gloss_intrinsic_event"></span><span id="WMI.GLOSS_INTRINSIC_EVENT"></span>**événement intrinsèque**
</dt> <dd>

Événement fourni par WMI, qui est une notification de modification du modèle de données WMI standard. Par exemple, la création d’une nouvelle instance de [**\_ processus Win32**](/windows/desktop/CIMWin32Prov/win32-process) est un type [**\_ \_ InstanceCreationEvent**](--instancecreationevent.md) d’événement intrinsèque. Consultez également [*événement extrinsèque*](gloss-e.md).

</dd> <dt>

<span id="wmi.gloss_instrumentation"></span><span id="WMI.GLOSS_INSTRUMENTATION"></span>**Instrumentation**
</dt> <dd>

Mécanisme utilisé pour signaler l’état du matériel et des logiciels du système. WMI implémente l’instrumentation en modélisant un système à l’aide de classes basées sur le [*Common Information Model (CIM)*](gloss-c.md). WMI dans le .NET Framework crée des classes instrumentées en ajoutant des attributs.

</dd> </dl>
