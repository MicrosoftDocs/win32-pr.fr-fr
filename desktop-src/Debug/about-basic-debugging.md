# <a name="about-basic-debugging"></a>À propos du débogage de base

Les [fonctions de débogage](debugging-functions.md) peuvent être utilisées pour créer un débogueur de base basé sur les événements. *Piloté par événement* signifie que le débogueur est averti chaque fois que certains événements se produisent dans le processus en cours de débogage. La notification permet au débogueur de prendre les mesures appropriées en réponse aux événements.

Pour obtenir une vue d’ensemble des termes du débogage, consultez la [terminologie relative au débogage](debugging-terminology.md).

La [prise en charge du débogage à partir des fonctions de processus, de thread et d’exception](debug-support-from-process-thread-and-exception-functions.md) décrit les fonctionnalités spécifiques au débogage de certains processus, threads et fonctions de gestion des exceptions.

La [création d’un débogueur de base](creating-a-basic-debugger.md) décrit l’utilisation des fonctions de débogage de base pour créer un débogueur simple. Ces fonctions permettent à une application d’attendre les événements de débogage, de provoquer des exceptions de point d’arrêt, de transférer le contrôle d’exécution au débogueur, et ainsi de suite.

Les [événements de débogage](debugging-events.md) décrivent le mécanisme d’événement de débogage. Les événements de débogage entraînent le système à notifier le débogueur.
