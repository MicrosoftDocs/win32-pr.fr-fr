# <a name="exception-handling-error-handling"></a><span data-ttu-id="67d49-104">Gestion des exceptions (gestion des erreurs)</span><span class="sxs-lookup"><span data-stu-id="67d49-104">Exception Handling (Error Handling)</span></span>

<span data-ttu-id="67d49-105">Les exceptions peuvent être lancées par le matériel ou les logiciels et peuvent se produire en mode noyau et en mode utilisateur.</span><span class="sxs-lookup"><span data-stu-id="67d49-105">Exceptions can be initiated by hardware or software, and can occur in kernel-mode as well as user-mode code.</span></span> <span data-ttu-id="67d49-106">La gestion structurée des exceptions fournit un mécanisme unique pour la gestion des exceptions en mode noyau et en mode utilisateur.</span><span class="sxs-lookup"><span data-stu-id="67d49-106">Structured exception handling provides a single mechanism for the handling of kernel-mode and user-mode exceptions.</span></span>

<span data-ttu-id="67d49-107">L’exécution de certaines séquences d’instructions peut entraîner des exceptions qui sont lancées par le matériel.</span><span class="sxs-lookup"><span data-stu-id="67d49-107">The execution of certain instruction sequences can result in exceptions that are initiated by hardware.</span></span> <span data-ttu-id="67d49-108">Par exemple, une violation d’accès est générée par le matériel lorsqu’un processus tente de lire ou d’écrire sur une adresse virtuelle pour laquelle il ne dispose pas de l’accès approprié.</span><span class="sxs-lookup"><span data-stu-id="67d49-108">For example, an access violation is generated by the hardware when a process attempts to read from or write to a virtual address to which it does not have the appropriate access.</span></span>

<span data-ttu-id="67d49-109">Les événements qui requièrent une gestion des exceptions peuvent également se produire pendant l’exécution d’une routine logicielle (par exemple, lorsqu’une valeur de paramètre non valide est spécifiée).</span><span class="sxs-lookup"><span data-stu-id="67d49-109">Events that require exception handling may also occur during execution of a software routine (for example, when an invalid parameter value is specified).</span></span> <span data-ttu-id="67d49-110">Dans ce cas, un thread peut lancer une exception explicitement en appelant la fonction [**RaiseException**](/windows/win32/api/errhandlingapi/nf-errhandlingapi-raiseexception) .</span><span class="sxs-lookup"><span data-stu-id="67d49-110">When this happens, a thread can initiate an exception explicitly by calling the [**RaiseException**](/windows/win32/api/errhandlingapi/nf-errhandlingapi-raiseexception) function.</span></span> <span data-ttu-id="67d49-111">Cette fonction permet au thread appelant de spécifier des informations qui décrivent l’exception.</span><span class="sxs-lookup"><span data-stu-id="67d49-111">This function enables the calling thread to specify information that describes the exception.</span></span>

<span data-ttu-id="67d49-112">Une exception peut être continue ou non continue.</span><span class="sxs-lookup"><span data-stu-id="67d49-112">An exception can be continuable or noncontinuable.</span></span> <span data-ttu-id="67d49-113">Une exception qui ne peut pas être poursuivie se produit lorsque l’événement n’est pas continuable dans le matériel, ou si la continuation n’est pas appropriée.</span><span class="sxs-lookup"><span data-stu-id="67d49-113">A noncontinuable exception arises when the event is not continuable in the hardware, or if continuation makes no sense.</span></span> <span data-ttu-id="67d49-114">Une exception non continuable ne met pas fin à l’application.</span><span class="sxs-lookup"><span data-stu-id="67d49-114">A noncontinuable exception does not terminate the application.</span></span> <span data-ttu-id="67d49-115">Par conséquent, une application peut être en mesure d’intercepter l’exception et de l’exécuter.</span><span class="sxs-lookup"><span data-stu-id="67d49-115">Therefore, an application may be able to catch the exception and run.</span></span> <span data-ttu-id="67d49-116">Toutefois, une exception non viable se produit généralement à la suite d’une pile endommagée ou d’un autre problème sérieux, ce qui rend difficile la récupération à partir de l’exception.</span><span class="sxs-lookup"><span data-stu-id="67d49-116">However, a noncontinuable exception typically arises as a result of a corrupted stack or other serious problem, making it difficult to recover from the exception.</span></span>
