<dl> <dt>

<span id="_security_object_identifier_gly"></span><span id="_SECURITY_OBJECT_IDENTIFIER_GLY"></span>**identificateur d’objet**
</dt> <dd>

OID Numéro qui identifie de façon unique une classe d’objet ou un attribut. Un identificateur d’objet est représenté sous la forme d’une chaîne décimale séparée par des points, telle que 1.2.3.4. Les identificateurs d’objets sont organisés dans une hiérarchie globale. Les autorités d'immatriculation nationales communiquent des identificateurs d'objet racine aux individus ou organisations, qui gèrent la hiérarchie sous leur identificateur d'objet racine.

Voir aussi [*notation de syntaxe abstraite 1*](a-gly.md), [*attribut*](a-gly.md).

</dd> <dt>

<span id="_security_ocsp_gly"></span><span id="_SECURITY_OCSP_GLY"></span>**OCSP**
</dt> <dd>

Consultez *protocole d’état de certificat en ligne*.

</dd> <dt>

<span id="_security_ofb_gly"></span><span id="_SECURITY_OFB_GLY"></span>**OFB**
</dt> <dd>

Consultez les *Commentaires de sortie*.

</dd> <dt>

<span id="_security_oid_gly"></span><span id="_SECURITY_OID_GLY"></span>**OID**
</dt> <dd>

Consultez *identificateur d’objet*.

</dd> <dt>

<span id="_security_online_certificate_status_protocol_gly"></span><span id="_SECURITY_ONLINE_CERTIFICATE_STATUS_PROTOCOL_GLY"></span>**protocole d’état de certificat en ligne**
</dt> <dd>

OCSP Protocole Internet utilisé pour obtenir l’état de révocation d’un certificat numérique X. 509.

</dd> <dt>

<span id="_security_opaque_blob_gly"></span><span id="_SECURITY_OPAQUE_BLOB_GLY"></span>**objet BLOB opaque**
</dt> <dd>

Séquence d’octets utilisée pour stocker les clés de session. Les objets blob opaque contiennent le matériel de clé de base et toutes les informations d’état actuelles. Cela comprend des informations telles que la valeur salt, le vecteur d’initialisation et la table clé. Le format des objets BLOB opaques n’est pas publié. Chaque fournisseur de services de chiffrement détermine son propre format BLOB, qui doit inclure le chiffrement des objets BLOB opaques avec une certaine sorte de clé symétrique.

</dd> <dt>

<span id="_security_outer_content_gly"></span><span id="_SECURITY_OUTER_CONTENT_GLY"></span>**contenu externe**
</dt> <dd>

Améliorations de certaines données encapsulées. Ce terme est principalement utilisé lors de l’étude des données améliorées (contenu interne) dans les \# messages PKCS 7.

</dd> <dt>

<span id="_security_output_feedback_gly"></span><span id="_SECURITY_OUTPUT_FEEDBACK_GLY"></span>**Commentaires de sortie**
</dt> <dd>

OFB Mode de chiffrement par bloc qui utilise des commentaires similaires au mode CFB (Cipher Feedback). La seule différence entre les deux modes est la façon dont le registre à décalage est rempli.

Voir aussi [*Commentaires sur le chiffrement*](c-gly.md).

</dd> </dl>
