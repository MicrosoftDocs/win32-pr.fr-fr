# <a name="file-access-rights-constants"></a>Constantes de droits d’accès au fichier

Les droits d’accès valides pour les fichiers et les répertoires sont les suivants : **supprimer**, lire le contrôle, **écrire \_ DAC**, écrire **\_ le** **\_ propriétaire** et **synchroniser** les [droits d’accès standard](/windows/desktop/SecAuthZ/standard-access-rights). Le tableau suivant répertorie les droits d’accès spécifiques aux fichiers et aux répertoires.



| Constante/valeur                                                                                                                                                                                                                                             | Description                                                                                                                                                                                                                                                                                                                                                                        |
|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="FILE_ADD_FILE"></span><span id="file_add_file"></span><dl> <dt>**Fichier \_ Ajouter le \_ fichier**</dt> <dt>2</dt> </dl>                                      | Pour un répertoire, droit de créer un fichier dans le répertoire.<br/>                                                                                                                                                                                                                                                                                                           |
| <span id="FILE_ADD_SUBDIRECTORY"></span><span id="file_add_subdirectory"></span><dl> <dt>**Fichier \_ Ajouter un \_ sous-répertoire**</dt> <dt>4</dt> </dl>              | Pour un répertoire, droit de créer un sous-répertoire.<br/>                                                                                                                                                                                                                                                                                                                    |
| <span id="FILE_ALL_ACCESS"></span><span id="file_all_access"></span><dl> <dt>**Fichier \_ TOUT \_ accès**</dt><dt></dt> </dl>                                 | Tous les droits d’accès possibles pour un fichier.<br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="FILE_APPEND_DATA"></span><span id="file_append_data"></span><dl> <dt>**Fichier \_ Ajouter des \_ données**</dt> <dt>4</dt> </dl>                             | Pour un objet fichier, droit d’ajouter des données au fichier. (Pour les fichiers locaux, les opérations d’écriture ne remplacent pas les données existantes si cet indicateur est spécifié sans **\_ \_ données d’écriture de fichier**.) Pour un objet d’annuaire, droit de créer un sous-répertoire (**fichier \_ Ajouter un \_ sous-répertoire**).<br/>                                                                                        |
| <span id="FILE_CREATE_PIPE_INSTANCE"></span><span id="file_create_pipe_instance"></span><dl> <dt>**Fichier \_ CRÉER \_ une \_ instance de canal**</dt> <dt>4</dt> </dl> | Pour un canal nommé, droit de créer un canal.<br/>                                                                                                                                                                                                                                                                                                                           |
| <span id="FILE_DELETE_CHILD"></span><span id="file_delete_child"></span><dl> <dt>**Fichier \_ SUPPRIMER l' \_ enfant**</dt> <dt>64 (0x40)</dt> </dl>                  | Pour un répertoire, droit de supprimer un répertoire et tous les fichiers qu’il contient, y compris les fichiers en lecture seule.<br/>                                                                                                                                                                                                                                                              |
| <span id="FILE_EXECUTE"></span><span id="file_execute"></span><dl> <dt>**Fichier \_ EXÉCUTER**</dt> <dt>32 (0x20)</dt> </dl>                                  | Pour un fichier de code natif, droit d’exécuter le fichier. Ce droit d’accès donné aux scripts peut entraîner l’exécution du script, en fonction de l’interpréteur de script.<br/>                                                                                                                                                                                                   |
| <span id="FILE_LIST_DIRECTORY"></span><span id="file_list_directory"></span><dl> <dt>**Fichier \_ RÉPERTORIer le \_ répertoire**</dt> <dt>1</dt> </dl>                    | Pour un répertoire, droit de répertorier le contenu du répertoire.<br/>                                                                                                                                                                                                                                                                                                       |
| <span id="FILE_READ_ATTRIBUTES"></span><span id="file_read_attributes"></span><dl> <dt>**Fichier \_ \_Attributs de lecture**</dt> <dt>128 (0x80)</dt> </dl>        | Droit de lire les attributs du fichier.<br/>                                                                                                                                                                                                                                                                                                                                      |
| <span id="FILE_READ_DATA"></span><span id="file_read_data"></span><dl> <dt>**Fichier \_ LIRE les \_ données**</dt> <dt>1</dt> </dl>                                   | Pour un objet fichier, droit de lire les données de fichier correspondantes. Pour un objet d’annuaire, droit de lire les données d’annuaire correspondantes.<br/>                                                                                                                                                                                                                           |
| <span id="FILE_READ_EA"></span><span id="file_read_ea"></span><dl> <dt>**Fichier \_ LIRE \_ EA**</dt> <dt>8</dt> </dl>                                         | Droit de lire les attributs de fichier étendus.<br/>                                                                                                                                                                                                                                                                                                                             |
| <span id="FILE_TRAVERSE"></span><span id="file_traverse"></span><dl> <dt>**Fichier \_ TRAVERSER**</dt> <dt>32 (0x20)</dt> </dl>                               | Pour un répertoire, droit de traverser le répertoire. Par défaut, les utilisateurs se voient attribuer le [privilège](/windows/desktop/SecAuthZ/privileges) **contourner la \_ \_ vérification de parcours** , qui ignore le [droit d’accès](/windows/desktop/SecAuthZ/access-rights-and-access-masks) **\_ Parcourir les fichiers** . Pour plus d’informations, consultez les notes relatives à la [sécurité des fichiers et aux droits d’accès](file-security-and-access-rights.md) .<br/> |
| <span id="FILE_WRITE_ATTRIBUTES"></span><span id="file_write_attributes"></span><dl> <dt>**Fichier \_ \_Attributs d’écriture**</dt> <dt>256 (0x100)</dt> </dl>    | Droit d’écrire des attributs de fichier.<br/>                                                                                                                                                                                                                                                                                                                                     |
| <span id="FILE_WRITE_DATA"></span><span id="file_write_data"></span><dl> <dt>**Fichier \_ ÉCRITURE de \_ données**</dt> <dt>2</dt> </dl>                                | Pour un objet fichier, droit d’écrire des données dans le fichier. Pour un objet d’annuaire, droit de créer un fichier dans le répertoire (fichier d'**Ajout de fichier \_ \_**).<br/>                                                                                                                                                                                                                      |
| <span id="FILE_WRITE_EA"></span><span id="file_write_ea"></span><dl> <dt>**Fichier \_ ÉCRIRE \_ EA**</dt> <dt>16 (0x10)</dt> </dl>                              | Droit d’écrire des attributs de fichier étendus.<br/>                                                                                                                                                                                                                                                                                                                            |
| <span id="STANDARD_RIGHTS_READ"></span><span id="standard_rights_read"></span><dl> <dt>**Standard \_ DROITS \_ lus**</dt><dt></dt> </dl>                  | Comprend **le \_ contrôle de lecture**, qui est le droit de lire les informations dans le descripteur de sécurité de l’objet de fichier ou de répertoire. Cela n’inclut pas les informations dans la liste SACL.<br/>                                                                                                                                                                                        |
| <span id="STANDARD_RIGHTS_WRITE"></span><span id="standard_rights_write"></span><dl> <dt>**Standard \_ DROITS d' \_ écriture**</dt><dt></dt> </dl>               | Identique à **la \_ \_ lecture des droits standard**.<br/>                                                                                                                                                                                                                                                                                                                                     |
