# <a name="terms-used-in-the-print-schema"></a>Termes utilisés dans le schéma d’impression

Cette rubrique n’est pas à jour. Pour obtenir les informations les plus récentes, consultez la [spécification du schéma d’impression](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

La liste suivante contient des termes spécifiques au schéma d’impression et aux technologies associées.



| Terme                                                                                                                                                                 | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="attribute_name"></span><span id="ATTRIBUTE_NAME"></span>nom de l’attribut<br/>                                                                          | Un nom d’attribut est le jeton situé à gauche du signe égal dans un attribut XML standard. La représentation standard d’un attribut XML est le *nom d’attribut*= « valeur d'*attribut*». Bien que similaire, le terme « nom d’attribut » ne doit pas être confondu avec l’attribut de nom défini dans l’infrastructure de schéma d’impression. L’attribut Name défini dans l’infrastructure du schéma d’impression est une instance spécifique d’un nom d’attribut. <br/>                                                                                                        |
| <span id="candidate"></span><span id="CANDIDATE"></span>finales<br/>                                                                                         | Option. Toute option qui appartient à la liste évaluée pendant le processus de notation. Consultez les termes, l’option de notation ou de correspondance, ainsi que l’option de référence.<br/>                                                                                                                                                                                                                                                                                                                                                                                |
| <span id="Configuration"></span><span id="configuration"></span><span id="CONFIGURATION"></span>Configuré<br/>                                             | Décrit l’un des États qu’un appareil peut supposer ou placer. L’appareil effectue sa tâche différemment dans chaque État. Il existe une différence distincte et identifiable entre le comportement de l’appareil dans chacun de ces États. Par exemple, dans un État, le support est retiré d’un plateau et se termine dans un autre tiroir. Une modification apportée à l’un ou à l’autre de ces bacs détermine un état différent.<br/>                                                                                                                               |
| <span id="Constraint"></span><span id="constraint"></span><span id="CONSTRAINT"></span>Restrictions<br/>                                                         | Informations nécessaires pour déterminer la validité d’une configuration. Une contrainte se présente souvent sous la forme : la fonctionnalité a définie sur option1 n’est pas compatible avec FeatureB défini sur option2.<br/>                                                                                                                                                                                                                                                                                                                                                            |
| <span id="corresponding_elements"></span><span id="CORRESPONDING_ELEMENTS"></span>éléments correspondants<br/>                                                  | Deux éléments qui ont le même emplacement par rapport à une paire d’éléments d’origine spécifiée. Les éléments Origin sont souvent deux éléments option qui sont comparés.<br/>                                                                                                                                                                                                                                                                                                                                                              |
| <span id="default"></span><span id="DEFAULT"></span>valeurs<br/>                                                                                               | Option. Option désignée par le schéma d’impression ou le fournisseur PrintCapabilities comme option assignée à une fonctionnalité quand PrintTicket ne contient pas de paramètre explicite pour cette fonctionnalité. Chaque fonctionnalité dispose d’une option par défaut désignée.<br/>                                                                                                                                                                                                                                                               |
| <span id="device_configuration_attribute"></span><span id="DEVICE_CONFIGURATION_ATTRIBUTE"></span>attribut de configuration de l’appareil<br/>                          | Attribut d’appareil qui contribue à la définition de la configuration de l’appareil. Un attribut de configuration d’appareil est représenté dans l’infrastructure du schéma d’impression par des constructions de type fonctionnalité ou option ou. Notez que l’attribut utilisé dans ce contexte n’a pas de relation avec les attributs XML ou pour imprimer les éléments d’attribut définis par l’infrastructure de schéma.<br/>                                                                                                                                                                          |
| <span id="element_type"></span><span id="ELEMENT_TYPE"></span>type d’élément<br/>                                                                                | Le type de chaque élément XML défini dans l’infrastructure du schéma d’impression est identifié par sa balise d’élément XML.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                               |
| <span id="elements_in_common"></span><span id="ELEMENTS_IN_COMMON"></span>éléments en commun<br/>                                                              | Consultez les éléments correspondants.<br/> Fonctionnalité. Construction définie par l’infrastructure de schéma d’impression qui représente un attribut de configuration d’appareil spécifique. Chaque instance de fonctionnalité représente un attribut d’appareil. Une instance de fonctionnalité est définie en définissant explicitement chacun de ses États.<br/> Option Identity. Option qui n’a aucun effet observable quand elle est sélectionnée ; en fait, un « no-op ». Le résultat est toujours le même, quel que soit le nombre de fois que cette option est sélectionnée.<br/>                                               |
| <span id="instance_name"></span><span id="INSTANCE_NAME"></span>nom de l’instance<br/>                                                                             | Attribut XML qui identifie l’instance spécifique d’un type d’élément. Consultez attribut Name.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| <span id="interface_client"></span><span id="INTERFACE_CLIENT"></span>client d’interface<br/>                                                                    | Entité qui utilise des documents PrintCapabilities et qui construit et consomme des PrintTicket. Les exemples courants de clients d’interface incluent les modules d’interface utilisateur (IU) et les modules de rendu.<br/>                                                                                                                                                                                                                                                                                                                                   |
| <span id="interface_provider"></span><span id="INTERFACE_PROVIDER"></span>fournisseur d’interface<br/>                                                              | Entité qui implémente et expose les fonctionnalités au moyen d’une interface. Dans le schéma d’impression, cela permet de créer des documents PrintCapabilities et de valider des des PrintTicket. <br/>                                                                                                                                                                                                                                                                                                                                                 |
| <span id="location_of_an_element"></span><span id="LOCATION_OF_AN_ELEMENT"></span>emplacement d’un élément<br/>                                                  | Position d’une instance particulière d’un élément dans la hiérarchie d’éléments. L’emplacement est défini à la fois par l’élément particulier et par tous les ancêtres de cet élément. En d’autres termes, l’emplacement est l’équivalent d’un chemin d’accès qualifié complet dans un système de fichiers. <br/>                                                                                                                                                                                                                                                                    |
| <span id="name_attribute"></span><span id="NAME_ATTRIBUTE"></span>attribut Name<br/>                                                                          | Désignation donnée à l’attribut XML avec le nom d’attribut « Name ». Cet attribut apparaît dans la plupart des types d’éléments définis dans l’infrastructure du schéma d’impression. Il identifie et définit des instances de chaque type d’élément. Les deux termes suivants font référence aux attributs de nom : la règle d’unicité sœur et la Convention d’espace de noms.<br/>                                                                                                                                                                                         |
| <span id="Namespace_Convention"></span><span id="namespace_convention"></span><span id="NAMESPACE_CONVENTION"></span>Convention d’espace de noms<br/>                 | Indique que les attributs de nom définis par schéma, ainsi que tout objet défini par schéma, se trouvent dans un espace de noms dédié, l’espace de noms public et que les attributs de nom définis de manière privée doivent exister dans un espace de noms différent, un qui peut être identifié comme appartenant au tiers en cours de définition.<br/> Option. Construction de l’infrastructure du schéma d’impression utilisée pour représenter les États que chaque fonctionnalité peut supposer ou être placée dans. Par exemple, si la fonctionnalité représente une taille de support, les valeurs d’option standard peuvent être letter, Legal, A4, entre autres.<br/> |
| <span id="Parameter"></span><span id="parameter"></span><span id="PARAMETER"></span>Paramètre<br/>                                                             | Le schéma d’impression prend en charge la possibilité de décrire des données paramétrables à des fins d’entrée utilisateur. Il existe trois types d’éléments d’infrastructure de schéma d’impression qui jouent des rôles dans l’entrée paramétrique : ParameterDef, ParameterInit et ParameterRef. Un élément ParameterDef caractérise un paramètre (sa plage de valeurs, le type de ses données et d’autres caractéristiques). Un élément ParameterInit est utilisé pour initialiser un paramètre. Un élément ParameterRef est utilisé pour référencer une instance particulière d’un élément ParameterInit.<br/>         |
| <span id="parameterized"></span><span id="PARAMETERIZED"></span>paramétrable<br/>                                                                             | Option. Instance option qui contient au moins un élément ScoredProperty qui, à son tour, contient un élément ParameterRef plutôt qu’un élément value.<br/>                                                                                                                                                                                                                                                                                                                                                                                 |
| <span id="perfectly-matching"></span><span id="PERFECTLY-MATCHING"></span>correspondance parfaite<br/>                                                              | Options. Deux instances d’option sont considérées comme parfaitement identiques si pour chaque élément ScoredProperty dans une instance d’option, il existe une instance ScoredProperty correspondante dans l’autre instance d’option, et les deux éléments de valeur des instances ScoredProperty correspondantes sont identiques.<br/>                                                                                                                                                                                                                                        |
| <span id="PrintCapabilities_Schema"></span><span id="printcapabilities_schema"></span><span id="PRINTCAPABILITIES_SCHEMA"></span>Schéma PrintCapabilities<br/> | Schéma qui définit la syntaxe et la sémantique utilisées par un document PrintCapabilities. Le schéma PrintCapabilities et le schéma PrintTicket sont des sous-ensembles du schéma d’impression. Voir aussi structure du schéma d’impression.<br/>                                                                                                                                                                                                                                                                                                                       |
| <span id="PrintTicket_Schema"></span><span id="printticket_schema"></span><span id="PRINTTICKET_SCHEMA"></span>Schéma PrintTicket<br/>                         | Schéma qui définit la syntaxe et la sémantique utilisées par PrintTicket. Le schéma PrintTicket et le schéma PrintCapabilities sont des sous-ensembles du schéma d’impression. Voir aussi structure du schéma d’impression.<br/>                                                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Framework"></span><span id="print_schema_framework"></span><span id="PRINT_SCHEMA_FRAMEWORK"></span>Structure du schéma d’impression<br/>         | Schéma qui définit la syntaxe et la sémantique utilisées par les technologies de schéma d’impression. Par exemple, l’infrastructure du schéma d’impression définit une collection de types d’éléments et leurs utilisations, mais n’énumère pas toutes les instances d’élément.<br/>                                                                                                                                                                                                                                                                                                    |
| <span id="Print_Schema_Keywords"></span><span id="print_schema_keywords"></span><span id="PRINT_SCHEMA_KEYWORDS"></span>Mots clés du schéma d’impression<br/>             | Schéma qui définit des instances individuelles des éléments de propriété, de fonctionnalité et d’option couramment utilisés pour décrire les appareils et la mise en forme des tâches. Ce schéma définit un langage commun qui rend les documents PrintCapabilities et les des PrintTicket portables. Également connu sous le nom de schéma public.<br/>                                                                                                                                                                                                                                         |
| <span id="privately-defined"></span><span id="PRIVATELY-DEFINED"></span>défini en privé<br/>                                                                 | Toutes les instances d’éléments qui apparaissent dans un document PrintCapabilities ou PrintTicket qui ne sont pas définies explicitement dans les mots clés du schéma d’impression. Ces instances d’élément doivent être qualifiées explicitement avec un espace de noms privé par fournisseur.<br/>                                                                                                                                                                                                                                                                                           |
| <span id="reference"></span><span id="REFERENCE"></span>faire<br/>                                                                                         | Option. Option à laquelle chaque option candidate est comparée. Une option de référence provient généralement de PrintTicket. Consultez les termes, l’option de notation ou de mappage et l’option candidat.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="root_level"></span><span id="ROOT_LEVEL"></span>niveau de la racine<br/>                                                                                      | Le niveau racine est la base de la structure hiérarchique de l’infrastructure du schéma d’impression. Un élément de niveau racine est défini en tant qu’enfant d’un élément PrintCapabilities ou d’un élément PrintTicket.<br/>                                                                                                                                                                                                                                                                                                                                            |
| <span id="scoping_prefix"></span><span id="SCOPING_PREFIX"></span>préfixe d’étendue<br/>                                                                          | Une étiquette textuelle pré-ajoutée à un mot clé de schéma pour fournir une étendue contextuelle. Les préfixes d’étendue de schéma d’impression autorisés sont Job, doc et page. Par exemple, le préfixe d’étendue pour JobInputBin est « Job ».<br/>                                                                                                                                                                                                                                                                                                                               |
| <span id="ScoredProperty"></span><span id="scoredproperty"></span><span id="SCOREDPROPERTY"></span>ScoredProperty<br/>                                         | Construction d’infrastructure de schéma d’impression utilisée pour représenter les qualités qui distinguent ou différencient une instance d’option d’une autre. Par exemple, la différence entre les lettres et les papiers de taille légale.<br/>                                                                                                                                                                                                                                                                                                                              |
| <span id="scoring_or_mapping"></span><span id="SCORING_OR_MAPPING"></span>notation ou mappage<br/>                                                              | Option. Processus de sélection dans une liste de l’instance d’option (candidate) qui correspond le mieux à une instance d’option donnée (référence). Les termes étroitement liés sont l’option candidat et la référence. <br/>                                                                                                                                                                                                                                                                                                                                  |
| <span id="Sibling_Uniqueness_Rule"></span><span id="sibling_uniqueness_rule"></span><span id="SIBLING_UNIQUENESS_RULE"></span>Règle d’unicité sœur<br/>     | Indique que tous les frères du même type d’élément doivent avoir des attributs de nom uniques. Certains types d’éléments sont exemptés de cette règle. Ces exceptions sont définies dans l’infrastructure du schéma d’impression.<br/>                                                                                                                                                                                                                                                                                                                                          |
| <span id="single-valued_versus_multivalued"></span><span id="SINGLE-VALUED_VERSUS_MULTIVALUED"></span>à valeur unique et à valeurs multiples<br/>                    | Propriété. Une propriété dont la valeur est indifférente (ne dépend pas de la configuration) est considérée comme étant à valeur unique. Une telle propriété n’a qu’une seule valeur dans toutes les configurations possibles. Une propriété qui dépend de la configuration est considérée comme ayant plusieurs valeurs.<br/>                                                                                                                                                                                                                                                        |
| <span id="suffix_suffixes"></span><span id="SUFFIX_SUFFIXES"></span>suffixe/suffixe<br/>                                                                       | Mot clé Schema représentant le nom de la fonctionnalité ou du paramètre. Par exemple, le suffixe de JobInputBin est «InputBin'.<br/>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| <span id="Validation"></span><span id="validation"></span><span id="VALIDATION"></span>Métrage<br/>                                                         | Processus qui consiste à vérifier qu’un PrintTicket est conforme au schéma et représente une configuration qui peut être exécutée par un appareil spécifique.<br/>                                                                                                                                                                                                                                                                                                                                                                                           |
| <span id="_Undefined_"></span><span id="_undefined_"></span><span id="_UNDEFINED_"></span>\_Non défini\_<br/>                                                   | Les données spécifiques qui sont représentées ici sont un espace réservé pour le pilote, l’appareil ou l’application spécifique à produire. Elle ne communique pas si les données doivent être une chaîne.<br/>                                                                                                                                                                                                                                                                                                                                                            |
