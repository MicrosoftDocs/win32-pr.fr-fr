# <a name="when-to-use-multitasking"></a><span data-ttu-id="f503b-103">Quand utiliser le multitâche</span><span class="sxs-lookup"><span data-stu-id="f503b-103">When to Use Multitasking</span></span>

<span data-ttu-id="f503b-104">Il existe deux façons d’implémenter le multitâche : en tant que processus unique avec plusieurs threads ou en plusieurs processus, chacun avec un ou plusieurs threads.</span><span class="sxs-lookup"><span data-stu-id="f503b-104">There are two ways to implement multitasking: as a single process with multiple threads or as multiple processes, each with one or more threads.</span></span> <span data-ttu-id="f503b-105">Une application peut placer chaque thread qui requiert un espace d’adressage privé et des ressources privées dans son propre processus, afin de le protéger des activités d’autres threads de processus.</span><span class="sxs-lookup"><span data-stu-id="f503b-105">An application can put each thread that requires a private address space and private resources into its own process, to protect it from the activities of other process threads.</span></span>

<span data-ttu-id="f503b-106">Un processus multithread peut gérer des tâches mutuellement exclusives à l’aide de threads, tels que la fourniture d’une interface utilisateur et l’exécution de calculs en arrière-plan.</span><span class="sxs-lookup"><span data-stu-id="f503b-106">A multithreaded process can manage mutually exclusive tasks with threads, such as providing a user interface and performing background calculations.</span></span> <span data-ttu-id="f503b-107">La création d’un processus multithread peut également être un moyen pratique de structurer un programme qui exécute plusieurs tâches similaires ou identiques simultanément.</span><span class="sxs-lookup"><span data-stu-id="f503b-107">Creating a multithreaded process can also be a convenient way to structure a program that performs several similar or identical tasks concurrently.</span></span> <span data-ttu-id="f503b-108">Par exemple, un serveur de canaux nommés peut créer un thread pour chaque processus client qui se connecte au canal.</span><span class="sxs-lookup"><span data-stu-id="f503b-108">For example, a named pipe server can create a thread for each client process that attaches to the pipe.</span></span> <span data-ttu-id="f503b-109">Ce thread gère la communication entre le serveur et le client.</span><span class="sxs-lookup"><span data-stu-id="f503b-109">This thread manages the communication between the server and the client.</span></span> <span data-ttu-id="f503b-110">Votre processus peut utiliser plusieurs threads pour effectuer les tâches suivantes :</span><span class="sxs-lookup"><span data-stu-id="f503b-110">Your process could use multiple threads to accomplish the following tasks:</span></span>

-   <span data-ttu-id="f503b-111">Gérer l’entrée pour plusieurs fenêtres.</span><span class="sxs-lookup"><span data-stu-id="f503b-111">Manage input for multiple windows.</span></span>
-   <span data-ttu-id="f503b-112">Gérer l’entrée à partir de plusieurs appareils de communication.</span><span class="sxs-lookup"><span data-stu-id="f503b-112">Manage input from several communications devices.</span></span>
-   <span data-ttu-id="f503b-113">différencier des tâches de priorités différentes</span><span class="sxs-lookup"><span data-stu-id="f503b-113">Distinguish tasks of varying priority.</span></span> <span data-ttu-id="f503b-114">(par exemple, un thread prioritaire gère les tâches soumises à des contraintes de temps, et un thread de priorité inférieure effectue d’autres tâches) ;</span><span class="sxs-lookup"><span data-stu-id="f503b-114">For example, a high-priority thread manages time-critical tasks, and a low-priority thread performs other tasks.</span></span>
-   <span data-ttu-id="f503b-115">autoriser l’interface utilisateur à rester réactive, tout en allouant du temps aux tâches en arrière-plan.</span><span class="sxs-lookup"><span data-stu-id="f503b-115">Allow the user interface to remain responsive, while allocating time to background tasks.</span></span>

<span data-ttu-id="f503b-116">Il est généralement plus efficace pour une application d’implémenter le multitâche en créant un processus multithread unique, plutôt que de créer plusieurs processus, pour les raisons suivantes :</span><span class="sxs-lookup"><span data-stu-id="f503b-116">It is typically more efficient for an application to implement multitasking by creating a single, multithreaded process, rather than creating multiple processes, for the following reasons:</span></span>

-   <span data-ttu-id="f503b-117">Le système peut effectuer un basculement de contexte plus rapidement pour les threads que les processus, car un processus a plus de charge mémoire qu’un thread ne le fait (le contexte de processus est plus grand que le contexte de thread).</span><span class="sxs-lookup"><span data-stu-id="f503b-117">The system can perform a context switch more quickly for threads than processes, because a process has more overhead than a thread does (the process context is larger than the thread context).</span></span>
-   <span data-ttu-id="f503b-118">Tous les threads d’un processus partagent le même espace d’adressage et peuvent accéder aux variables globales du processus, ce qui peut simplifier la communication entre les threads.</span><span class="sxs-lookup"><span data-stu-id="f503b-118">All threads of a process share the same address space and can access the process's global variables, which can simplify communication between threads.</span></span>
-   <span data-ttu-id="f503b-119">Tous les threads d’un processus peuvent partager des handles ouverts aux ressources, tels que des fichiers et des canaux.</span><span class="sxs-lookup"><span data-stu-id="f503b-119">All threads of a process can share open handles to resources, such as files and pipes.</span></span>

<span data-ttu-id="f503b-120">Vous pouvez utiliser d’autres techniques à la place du multithreading.</span><span class="sxs-lookup"><span data-stu-id="f503b-120">There are other techniques you can use in the place of multithreading.</span></span> <span data-ttu-id="f503b-121">Les plus importantes sont les suivantes : les entrées et sorties asynchrones, les ports de terminaison d’e/s, les appels de procédure asynchrones (APC) et la possibilité d’attendre plusieurs événements.</span><span class="sxs-lookup"><span data-stu-id="f503b-121">The most significant of these are as follows: asynchronous input and output (I/O), I/O completion ports, asynchronous procedure calls (APC), and the ability to wait for multiple events.</span></span>

<span data-ttu-id="f503b-122">Un thread unique peut lancer plusieurs demandes d’e/s consommatrices de temps qui peuvent s’exécuter simultanément à l’aide d’e/s asynchrones.</span><span class="sxs-lookup"><span data-stu-id="f503b-122">A single thread can initiate multiple time-consuming I/O requests that can run concurrently using asynchronous I/O.</span></span> <span data-ttu-id="f503b-123">Les e/s asynchrones peuvent être effectuées sur des fichiers, des canaux et des appareils de communication série.</span><span class="sxs-lookup"><span data-stu-id="f503b-123">Asynchronous I/O can be performed on files, pipes, and serial communication devices.</span></span> <span data-ttu-id="f503b-124">Pour plus d’informations, consultez [synchronisation et entrée et sortie avec chevauchement](../sync/synchronization-and-overlapped-input-and-output.md).</span><span class="sxs-lookup"><span data-stu-id="f503b-124">For more information, see [Synchronization and Overlapped Input and Output](../sync/synchronization-and-overlapped-input-and-output.md).</span></span>

<span data-ttu-id="f503b-125">Un seul thread peut bloquer sa propre exécution en attendant qu’un ou plusieurs événements se produisent.</span><span class="sxs-lookup"><span data-stu-id="f503b-125">A single thread can block its own execution while waiting for any one or all of several events to occur.</span></span> <span data-ttu-id="f503b-126">Cela est plus efficace que l’utilisation de plusieurs threads, chacun en attente d’un événement unique et plus efficace que l’utilisation d’un thread unique qui consomme le temps processeur en recherchant continuellement les événements à effectuer.</span><span class="sxs-lookup"><span data-stu-id="f503b-126">This is more efficient than using multiple threads, each waiting for a single event, and more efficient than using a single thread that consumes processor time by continually checking for events to occur.</span></span> <span data-ttu-id="f503b-127">Pour plus d’informations, consultez [fonctions Wait](../sync/wait-functions.md).</span><span class="sxs-lookup"><span data-stu-id="f503b-127">For more information, see [Wait Functions](../sync/wait-functions.md).</span></span>
