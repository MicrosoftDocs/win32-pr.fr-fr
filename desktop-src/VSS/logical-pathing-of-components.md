# <a name="logical-pathing-of-components"></a><span data-ttu-id="cb5d5-103">Chemin logique des composants</span><span class="sxs-lookup"><span data-stu-id="cb5d5-103">Logical Pathing of Components</span></span>

<span data-ttu-id="cb5d5-104">Le [*chemin d’accès logique*](vssgloss-l.md) permet d’organiser les composants gérés par un writer en groupes bien définis.</span><span class="sxs-lookup"><span data-stu-id="cb5d5-104">[*Logical pathing*](vssgloss-l.md) is used to organize components managed by a writer into well-defined groups.</span></span>

<span data-ttu-id="cb5d5-105">Le chemin d’accès logique est analogue à la structure du chemin d’accès de fichier traditionnel, à l’aide de la barre oblique inverse « \\ » pour séparer les éléments dans le chemin d’accès.</span><span class="sxs-lookup"><span data-stu-id="cb5d5-105">The logical path is analogous in structure to traditional file pathing, using the backslash "\\" to separate elements in the path.</span></span> <span data-ttu-id="cb5d5-106">Contrairement aux chemins d’accès de fichier, la racine d’un chemin d’accès logique est **null** au lieu de « \\ ».</span><span class="sxs-lookup"><span data-stu-id="cb5d5-106">Unlike file paths, the root of a logical path is **NULL**, instead of "\\".</span></span>

<span data-ttu-id="cb5d5-107">Le chemin d’accès logique est exprimé sous la forme d’une chaîne terminée par le caractère **null**, et il n’existe aucune autre restriction sur les caractères que le chemin d’accès peut contenir.</span><span class="sxs-lookup"><span data-stu-id="cb5d5-107">The logical path is expressed as a **NULL**-terminated string, and there are no other restrictions on the characters the path can contain.</span></span>

<span data-ttu-id="cb5d5-108">L’utilisation la plus importante du chemin logique consiste à définir des [*jeux de composants*](vssgloss-c.md), où l' [*inclusion explicite*](vssgloss-e.md) d’un composant dans une opération de sauvegarde ou de restauration d’un composant sélectionnable nécessite l’inclusion d’un certain nombre d’autres composants (sous-[*composant*](vssgloss-s.md)).</span><span class="sxs-lookup"><span data-stu-id="cb5d5-108">The most important use of logical pathing is in defining [*component sets*](vssgloss-c.md), where the [*explicit component inclusion*](vssgloss-e.md) in a backup or restore operation of one selectable component requires the inclusion of a number of other components ([*subcomponent*](vssgloss-s.md)).</span></span> <span data-ttu-id="cb5d5-109">Le chemin d’accès logique du composant de définition d’un jeu de composants est un parent des chemins logiques de ses sous-composants et :</span><span class="sxs-lookup"><span data-stu-id="cb5d5-109">The logical path of the defining component of a component set is a parent to the logical paths of its subcomponents and:</span></span>

-   <span data-ttu-id="cb5d5-110">Les sous-composants doivent partager en tant que chemin d’accès racine le chemin d’accès logique du composant sélectionnable qui définit le jeu de composants.</span><span class="sxs-lookup"><span data-stu-id="cb5d5-110">Subcomponents must share as a root path the logical path of the selectable component that defines the component set.</span></span>
-   <span data-ttu-id="cb5d5-111">Le chemin d’accès racine **null** est valide.</span><span class="sxs-lookup"><span data-stu-id="cb5d5-111">A root path of **NULL** is valid.</span></span>
-   <span data-ttu-id="cb5d5-112">Le nom du composant sélectionnable de définition doit être le premier élément de chemin d’accès logique après le chemin d’accès racine pour chaque sous-composant non sélectionnable du jeu de composants.</span><span class="sxs-lookup"><span data-stu-id="cb5d5-112">The name of the defining selectable component must be the first logical path element after the root path for every nonselectable subcomponent of the component set.</span></span>
