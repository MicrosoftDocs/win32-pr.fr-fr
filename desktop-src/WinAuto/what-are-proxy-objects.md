# <a name="what-are-proxy-objects"></a><span data-ttu-id="c8ec4-104">Que sont les objets proxy ?</span><span class="sxs-lookup"><span data-stu-id="c8ec4-104">What Are Proxy Objects?</span></span>

<span data-ttu-id="c8ec4-105">Un objet *proxy* joue le rôle d’intermédiaire entre le client et un objet accessible.</span><span class="sxs-lookup"><span data-stu-id="c8ec4-105">A *proxy* object acts as an intermediary between the client and an accessible object.</span></span> <span data-ttu-id="c8ec4-106">L’objectif de l’objet proxy est de surveiller l’étendue de la durée de vie de l’objet accessible et de transférer les appels à l’objet accessible uniquement s’il n’est pas détruit.</span><span class="sxs-lookup"><span data-stu-id="c8ec4-106">The purpose of the proxy object is to monitor the life span of the accessible object and to forward calls to the accessible object only if it is not destroyed.</span></span>

<span data-ttu-id="c8ec4-107">Lorsqu’un client appelle une propriété [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) pour obtenir des informations sur un objet, l’objet proxy doit vérifier si l’objet accessible est toujours disponible.</span><span class="sxs-lookup"><span data-stu-id="c8ec4-107">When a client calls an [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) property to get information about an object, the proxy object must check whether the accessible object is still available.</span></span> <span data-ttu-id="c8ec4-108">Si c’est le cas, l’objet proxy transmet la requête du client à l’objet accessible.</span><span class="sxs-lookup"><span data-stu-id="c8ec4-108">If it is, the proxy object passes the client's request to the accessible object.</span></span> <span data-ttu-id="c8ec4-109">Si l’objet accessible est détruit (par exemple, lorsqu’une boîte de dialogue avec des contrôles personnalisés est fermée), l’objet proxy retourne une erreur.</span><span class="sxs-lookup"><span data-stu-id="c8ec4-109">If the accessible object is destroyed (for example, when a dialog box with custom controls is closed), the proxy object returns an error.</span></span> <span data-ttu-id="c8ec4-110">Pour indiquer que l’objet a été détruit, il est recommandé que les serveurs retournent le code d’erreur **co \_ E \_ OBJNOTCONNECTED** , car cette erreur est retournée par le modèle COM (Component Object Model) après qu’un serveur a appelé [CoDisconnectObject](/windows/win32/api/combaseapi/nf-combaseapi-codisconnectobject).</span><span class="sxs-lookup"><span data-stu-id="c8ec4-110">To indicate that the object has been destroyed, it is recommended that servers return the error code **CO\_E\_OBJNOTCONNECTED** because this error is returned by Component Object Model (COM) after a server calls [CoDisconnectObject](/windows/win32/api/combaseapi/nf-combaseapi-codisconnectobject).</span></span>

<span data-ttu-id="c8ec4-111">L’objet proxy est transparent pour le client.</span><span class="sxs-lookup"><span data-stu-id="c8ec4-111">The proxy object is transparent to the client.</span></span> <span data-ttu-id="c8ec4-112">Lorsque le client appelle [**AccessibleObjectFromEvent**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfromevent), [**AccessibleObjectFromPoint**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint)ou [**AccessibleObjectFromWindow**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfromwindow), il reçoit un pointeur vers une interface [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) .</span><span class="sxs-lookup"><span data-stu-id="c8ec4-112">When the client calls [**AccessibleObjectFromEvent**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfromevent), [**AccessibleObjectFromPoint**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint), or [**AccessibleObjectFromWindow**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfromwindow), it receives back a pointer to an [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) interface.</span></span> <span data-ttu-id="c8ec4-113">Toutefois, lorsque le client utilise ce pointeur pour appeler l’une des propriétés ou méthodes **IAccessible** , le code exécuté se trouve dans l’objet proxy.</span><span class="sxs-lookup"><span data-stu-id="c8ec4-113">However, when the client uses this pointer to call any of the **IAccessible** properties or methods, the code that is executed is within the proxy object.</span></span>
