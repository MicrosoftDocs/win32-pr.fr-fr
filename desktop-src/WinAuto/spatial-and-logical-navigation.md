## <a name="establishing-the-order-in-logical-navigation"></a><span data-ttu-id="44ab4-125">Établissement de la commande dans la navigation logique</span><span class="sxs-lookup"><span data-stu-id="44ab4-125">Establishing the Order in Logical Navigation</span></span>

<span data-ttu-id="44ab4-126">Dans la navigation logique, les développeurs qui conçoivent les objets établissent les relations entre eux.</span><span class="sxs-lookup"><span data-stu-id="44ab4-126">In logical navigation, the developers who design the objects establish the relationships between them.</span></span> <span data-ttu-id="44ab4-127">La navigation logique est plus subjective que la navigation spatiale.</span><span class="sxs-lookup"><span data-stu-id="44ab4-127">Logical navigation is more subjective than spatial navigation.</span></span> <span data-ttu-id="44ab4-128">En outre, l’ordre dans la navigation logique n’est pas le même que celui utilisé avec les ID enfants.</span><span class="sxs-lookup"><span data-stu-id="44ab4-128">Also, the order in logical navigation is not the same as the order used with child IDs.</span></span>

<span data-ttu-id="44ab4-129">Pour les objets qui ont des emplacements d’écran, les développeurs de serveurs doivent établir l’ordre de navigation de la façon dont la plupart des utilisateurs considèrent logiquement.</span><span class="sxs-lookup"><span data-stu-id="44ab4-129">For objects that have screen locations, server developers should establish the navigation order in the way that most users would consider logical.</span></span> <span data-ttu-id="44ab4-130">Dans les pays/régions anglophones, par exemple, il s’agit d’un ordre de gauche à droite et de haut en bas.</span><span class="sxs-lookup"><span data-stu-id="44ab4-130">In English-speaking countries/regions, for instance, this means a left-to-right, top-to-bottom ordering.</span></span>

<span data-ttu-id="44ab4-134">Pour les objets qui n’ont pas d’emplacements d’écran définis, l’ordre logique est déterminé par les développeurs de serveurs et les développeurs de clients ne doivent pas faire d’hypothèses.</span><span class="sxs-lookup"><span data-stu-id="44ab4-134">For objects that do not have defined screen locations, the logical order is decided by server developers, and client developers should make no assumptions about it.</span></span> <span data-ttu-id="44ab4-135">Par exemple, il est acceptable pour les objets non visibles, tels que les objets qui sont temporairement masqués, d’être mêlés à des objets visibles.</span><span class="sxs-lookup"><span data-stu-id="44ab4-135">For instance, it is acceptable for non-visible objects, such as objects that are only temporarily hidden, to be interspersed with visible objects.</span></span>
