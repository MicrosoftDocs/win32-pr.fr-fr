# <a name="toolbar-control-msaa-ui-element-reference"></a>Toolbar, contrôle (référence des éléments d’interface utilisateur MSAA)

> [!Note]  
> Cette rubrique décrit les objets de **contrôle ToolBar** à des fins de référence des éléments d’interface utilisateur MSAA. La procédure de création d’objets de **contrôle ToolBar** dans différentes infrastructures d’interface utilisateur n’est pas décrite ici. Consultez la documentation de référence sur les API pour l’infrastructure d’interface utilisateur que vous utilisez.

 

Un contrôle ToolBar contient des boutons qui exécutent des commandes de menu et est généralement contenu dans une fenêtre sous la barre de menus.

Le nom de la classe de fenêtre d’un contrôle ToolBar est TOOLBARCLASSNAME, qui est défini en tant que « ToolbarWindow32 » dans commctrl. h.

|---------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**accDoDefaultAction**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accdodefaultaction) | La barre d’outils ne prend pas en charge la méthode [**accDoDefaultAction**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accdodefaultaction) . Pour les boutons de la barre d’outils, **accDoDefaultAction** appelle [**PostMessage**](/windows/desktop/api/winuser/nf-winuser-postmessagea) avec le message [**BM \_ Click**](/windows/desktop/Controls/bm-click) pour cliquer sur le bouton spécifié. |
