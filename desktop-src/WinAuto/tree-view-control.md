# <a name="tree-view-control-msaa-ui-element-reference"></a>Contrôle Tree View (référence des éléments de l’interface utilisateur MSAA)

> [!Note]  
> Cette rubrique décrit les objets de **contrôle Tree View** à des fins de référence d’élément d’interface utilisateur MSAA. La procédure de création d’objets de **contrôle Tree View** dans différentes infrastructures d’interface utilisateur n’est pas décrite ici. Consultez la documentation de référence sur les API pour l’infrastructure d’interface utilisateur que vous utilisez.

 

Un contrôle Tree View affiche une liste hiérarchique d’éléments à l’aide de lignes pour connecter des éléments associés dans une hiérarchie. Chaque élément se compose d’une étiquette et d’une image bitmap facultative. L’Explorateur Windows utilise un contrôle arborescence pour afficher les répertoires.

Le nom de la classe de fenêtre pour un contrôle Tree View est WC \_ TreeView, qui est défini comme « SysTreeView » dans commctrl. h.

|---------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|

En fonction de la propriété **State** d’un élément dans l’arborescence, **accDoDefaultAction** développe ou réduit l’élément spécifié. |

## <a name="notes"></a>Notes

La propriété **accLocation** des éléments de la vue arborescence retourne l’emplacement uniquement du texte de l’élément. Les coordonnées retournées par **accLocation** pour les éléments de l’arborescence n’incluent pas l’icône située à gauche du texte.
