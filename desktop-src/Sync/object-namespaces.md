<span data-ttu-id="47596-106">Un processus peut créer un espace de noms privé à l’aide de la fonction [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) .</span><span class="sxs-lookup"><span data-stu-id="47596-106">A process can create a private namespace using the [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) function.</span></span> <span data-ttu-id="47596-107">Cette fonction requiert que vous spécifiiez une *limite* qui définit la façon dont les objets de l’espace de noms doivent être isolés.</span><span class="sxs-lookup"><span data-stu-id="47596-107">This function requires that you specify a *boundary* that defines how the objects in the namespace are to be isolated.</span></span> <span data-ttu-id="47596-108">L’appelant doit se trouver dans la limite spécifiée pour que l’opération de création aboutisse.</span><span class="sxs-lookup"><span data-stu-id="47596-108">The caller must be within the specified boundary for the create operation to succeed.</span></span> <span data-ttu-id="47596-109">Pour spécifier une limite, utilisez les fonctions [**CreateBoundaryDescriptor**](/windows/desktop/api/WinBase/nf-winbase-createboundarydescriptora) et [**AddSIDToBoundaryDescriptor**](/windows/win32/api/namespaceapi/nf-namespaceapi-addsidtoboundarydescriptor) .</span><span class="sxs-lookup"><span data-stu-id="47596-109">To specify a boundary, use the [**CreateBoundaryDescriptor**](/windows/desktop/api/WinBase/nf-winbase-createboundarydescriptora) and [**AddSIDToBoundaryDescriptor**](/windows/win32/api/namespaceapi/nf-namespaceapi-addsidtoboundarydescriptor) functions.</span></span>

<span data-ttu-id="47596-110">Le paramètre *lpAliasPrefix* de [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) sert de nom de l’espace de noms.</span><span class="sxs-lookup"><span data-stu-id="47596-110">The *lpAliasPrefix* parameter of [**CreatePrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-createprivatenamespacea) serves as the name of the namespace.</span></span> <span data-ttu-id="47596-111">Chaque espace de noms est identifié de manière unique par son nom et ses limites.</span><span class="sxs-lookup"><span data-stu-id="47596-111">Each namespace is uniquely identified by its name and boundaries.</span></span> <span data-ttu-id="47596-112">Le système prend en charge plusieurs espaces de noms privés portant le même nom, à condition qu’ils spécifient des limites différentes.</span><span class="sxs-lookup"><span data-stu-id="47596-112">The system supports multiple private namespaces with the same name, as long as they specify different boundaries.</span></span>

<span data-ttu-id="47596-113">Supposons qu’un processus demande la création d’un espace de noms, NS1, qui définit une limite contenant deux éléments : le SID d’administrateur et le numéro de session actuel.</span><span class="sxs-lookup"><span data-stu-id="47596-113">Suppose that a process requests the creation of a namespace, NS1, that defines a boundary containing two elements: the data-ttu-id="47596-115">Un autre processus peut accéder à cet espace de noms à l’aide de la fonction [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) .</span><span class="sxs-lookup"><span data-stu-id="47596-115">Another process can access this namespace using the <span data-ttu-id="47596-121">Le processus qui a créé l’espace de noms peut utiliser la fonction [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) pour fermer le descripteur de l’espace de noms.</span><span class="sxs-lookup"><span data-stu-id="47596-121">The process that created the namespace can use the [**ClosePrivateNamespace**](/windows/win32/api/namespaceapi/nf-namespaceapi-closeprivatenamespace) function to close the handle to the namespace.</span></span> <span data-ttu-id="47596-122">Le handle est également fermé lorsque le processus qui a créé l’espace de noms se termine.</span><span class="sxs-lookup"><span data-stu-id="47596-122">The handle is also closed when the process that created the namespace terminates.</span></span> <span data-ttu-id="47596-123">Une fois le descripteur d’espace de noms fermé, les appels suivants à [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) échouent, mais toutes les opérations sur les objets de l’espace de noms réussissent.</span><span class="sxs-lookup"><span data-stu-id="47596-123">After the namespace handle is closed, subsequent calls to [**OpenPrivateNamespace**](/windows/desktop/api/WinBase/nf-winbase-openprivatenamespacea) fail, but all operations on objects in the namespace succeed.</span></span>
