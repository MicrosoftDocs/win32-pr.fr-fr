# <a name="about-the-control-point-api"></a>À propos de l’API de point de contrôle

La technologie UPnP permet la mise en réseau dynamique d’appareils intelligents, de PC et d’autres points de contrôle. La mise en réseau dynamique est disponible sur un large éventail de supports physiques. L’implémentation Microsoft de l’API de point de contrôle avec la technologie UPnP se compose d’un ensemble d’objets COM qui sont utilisés pour écrire des applications et des pages Web qui recherchent et contrôlent des appareils basés sur UPnP. Par exemple, un magnétoscope compatible UPnP peut être contrôlé à partir d’une page Web conçue spécifiquement pour le contrôler. Pour étendre ce concept, les développeurs peuvent contrôler de nombreux appareils à partir d’un emplacement central, distant ou homologue.

Les applications créées à l’aide de l’API de point de contrôle avec la technologie UPnP sont comprises entre les scripts incorporés dans les pages HTML et les programmes C++ à part entière. l’API de Point de contrôle avec la technologie UPnP prend en charge le système de développement microsoft Visual Basic, microsoft Visual Basic scripting Edition (VBScript) et C++.

Les sections suivantes expliquent en général comment fonctionne l’API du point de contrôle avec la technologie UPnP :

-   [Notions de base des points de contrôle](control-point-basics.md)
-   [Recherche d’appareils](finding-devices.md)
-   [Description des appareils](describing-devices.md)
-   [Contrôle des appareils](controlling-devices.md)

Tous les exemples de l’API de point de contrôle avec la documentation sur la technologie UPnP font référence à des appareils fictifs, des types d’appareils et des chaînes.
