# <a name="s-network-monitor"></a>S (Moniteur réseau)

<dl> <dt>

<span id="_netmon_sap_gly"></span><span id="_NETMON_SAP_GLY"></span>**SAP**
</dt> <dd>

Consultez point d’accès au service.

</dd> <dt>

<span id="_netmon_server_message_block_gly"></span><span id="_NETMON_SERVER_MESSAGE_BLOCK_GLY"></span>**SMB (Server Message Block)**
</dt> <dd>

Protocole au niveau du fichier qui contient des primitives et est utilisé pour des opérations telles que l' **ouverture**, la **fermeture**, la **lecture** et l' **écriture**.

</dd> <dt>

<span id="_netmon_service_access_point_gly"></span><span id="_NETMON_SERVICE_ACCESS_POINT_GLY"></span>**point d’accès au service (SAP)**
</dt> <dd>

Identificateur de frame.

</dd> <dt>

<span id="_netmon_session_gly"></span><span id="_NETMON_SESSION_GLY"></span>**session**
</dt> <dd>

Ensemble de paquets de données qu’une station envoie à une autre station pendant une capture. Une session est un échange unidirectionnel de paquets dans lequel chaque session a un propriétaire (la station envoyant les paquets de données) et un partenaire (la station recevant les paquets de données). Voir aussi informations de session \_ .

</dd> <dt>

<span id="_netmon_session_information_gly"></span><span id="_NETMON_SESSION_INFORMATION_GLY"></span>**informations de session**
</dt> <dd>

Informations qui décrivent une session. Moniteur réseau enregistre les informations de session lors de la capture des données. Les informations de session sont appelées statistiques de conversation et peuvent être récupérées par un appel à la méthode **GetConversationStatistics** . Voir aussi [*\_ statistiques*](c.md)sur les sessions, les [*captures*](c.md)et les conversations.

</dd> <dt>

<span id="_netmon_smb_gly"></span><span id="_NETMON_SMB_GLY"></span>**ELLE**
</dt> <dd>

Consultez Server Message Block.

</dd> <dt>

<span id="_netmon_source_address_gly"></span><span id="_NETMON_SOURCE_ADDRESS_GLY"></span>**adresse source**
</dt> <dd>

Nombre hexadécimal unique qui identifie l’ordinateur à partir duquel les données sont envoyées.

</dd> <dt>

<span id="_netmon_special_blobs_gly"></span><span id="_NETMON_SPECIAL_BLOBS_GLY"></span>**objet BLOB spécial**
</dt> <dd>

Un fournisseur de paquets réseau (NPP) ou un objet BLOB de filtre qui contient des informations supplémentaires que d’autres objets BLOB ne fournissent pas. Voir aussi [*fournisseur de paquets réseau*](n.md) et [*objet BLOB*](b.md).

</dd> <dt>

<span id="_netmon_station_gly"></span><span id="_NETMON_STATION_GLY"></span>**station**
</dt> <dd>

Une carte réseau spécifique installée sur un ordinateur qui capture des données. Voir aussi informations sur la station.

</dd> <dt>

<span id="_netmon_station_information_gly"></span><span id="_NETMON_STATION_INFORMATION_GLY"></span>**informations sur la station**
</dt> <dd>

Informations qui décrivent une station. Moniteur réseau enregistre les informations de station pendant une capture de données. Les informations de station sont également appelées statistiques de conversation que vous pouvez récupérer à l’aide d’un appel à la méthode **GetConversationStatistics** . Voir aussi statistiques sur les stations et les [*conversations*](c.md).

</dd> <dt>

<span id="_netmon_summary_property_gly"></span><span id="_NETMON_SUMMARY_PROPERTY_GLY"></span>**Summary, propriété**
</dt> <dd>

Propriété définie par l’analyseur, utilisée pour regrouper les propriétés d’un protocole à afficher dans l’interface utilisateur Moniteur réseau. Une structure **PROPERTYINFO** définit une propriété Summary de la même façon que les autres propriétés définies par l’analyseur. La propriété Summary est généralement la première définition de propriété attachée à un frame lorsque Moniteur réseau appelle la fonction **AttachProperties** .

</dd> <dt>

<span id="_netmon_summary_pane_gly"></span><span id="_NETMON_SUMMARY_PANE_GLY"></span>**volet Résumé**
</dt> <dd>

Volet d’une fenêtre de l’observateur de frame Moniteur réseau qui répertorie les frames contenus dans un fichier de capture. Pour chaque frame, le volet Résumé répertorie le nombre de frames, l’heure à laquelle le cadre est capturé, l’adresse source et la destination du cadre, les protocoles utilisés pour envoyer le frame et une brève description du cadre. Si une adresse brute est associée à une adresse IP ou IPX/XNS, cette adresse est affichée dans **src Other addr** et **DST Other addr**. Le type d’adresse (IP ou IPX/XNS) apparaît dans **type autre ADR**.

</dd> </dl>
