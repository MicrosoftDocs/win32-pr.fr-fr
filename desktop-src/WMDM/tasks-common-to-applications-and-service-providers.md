# <a name="tasks-common-to-applications-and-service-providers"></a>Tâches communes aux applications et aux fournisseurs de services

Cette section décrit les tâches suivantes qui s’appliquent aux applications et aux fournisseurs de services basés sur les Gestionnaire de périphériques Windows Media.

-   [La compilation des fichiers IDL fournis avec le kit de développement logiciel (SDK)](compiling-the-idl-files-supplied-with-the-sdk.md) décrit les fichiers IDL utilisés pour créer les fichiers d’en-tête, si vous choisissez de ne pas utiliser les fichiers d’en-tête fournis avec le kit de développement logiciel (SDK).
-   [L’activation de la journalisation](enabling-logging.md) décrit un objet de journalisation fourni par Windows Media gestionnaire de périphériques pour enregistrer des données à partir d’un objet en cours d’exécution.
-   [L’utilisation de canaux authentifiés sécurisés](using-secure-authenticated-channels.md) décrit les canaux authentifiés sécurisés, qui sont utilisés pour le chiffrement, le déchiffrement et l’authentification des composants et des messages. Tout ce qui repose sur les Gestionnaire de périphériques Windows Media doit au moins être en mesure d’effectuer l’authentification des composants.
-   [Gestion du contenu protégé](handling-protected-content.md) décrit comment votre application peut gérer le contenu protégé par la gestion des droits numériques (DRM) Windows Media.
-   [Obtention et définition des métadonnées et des attributs](getting-and-setting-metadata-and-attributes.md) décrit comment une application peut récupérer et définir des attributs et des métadonnées pour un stockage ou un appareil.
