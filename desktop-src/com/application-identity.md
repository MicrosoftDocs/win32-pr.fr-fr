# <a name="application-identity"></a>Identité de l'application

L’identité d’une application est le compte utilisé pour exécuter l’application. L’identité peut être celle de l’utilisateur actuellement connecté (l’utilisateur interactif), l’utilisateur qui a lancé le serveur, un utilisateur spécifié ou un service. Pour savoir comment définir l’identité de l’application à l’aide de Dcomcnfg.exe, consultez Définition de la [sécurité échelle à l’aide de DCOMCNFG](setting-processwide-security-using-dcomcnfg.md).

Les rubriques suivantes décrivent chacun des quatre types d’identité d’application et expliquent les limitations de chaque type :

-   [Utilisateur interactif](interactive-user.md)
-   [Lancement de l’utilisateur](launching-user.md)
-   [Utilisateur spécifié](specified-user.md)
-   [Identité du service](service-identity.md)
