# <a name="security-in-com"></a><span data-ttu-id="41129-103">Sécurité dans COM</span><span class="sxs-lookup"><span data-stu-id="41129-103">Security in COM</span></span>

<span data-ttu-id="41129-104">La sécurité dans COM est fermement basée sur la sécurité fournie par Windows et les mécanismes de sécurité RPC sous-jacents.</span><span class="sxs-lookup"><span data-stu-id="41129-104">Security in COM is firmly based on the security provided by Windows and the underlying RPC security mechanisms.</span></span> <span data-ttu-id="41129-105">La sécurité COM repose sur l' *authentification* (le processus de vérification de l’identité d’un appelant) et l' *autorisation* (le processus de détermination si un appelant est autorisé à faire ce qu’il demande).</span><span class="sxs-lookup"><span data-stu-id="41129-105">COM security relies on *authentication* (the process of verifying a caller's identity) and *authorization* (the process of determining whether a caller is authorized to do what it is asking to do).</span></span> <span data-ttu-id="41129-106">Il existe deux principaux types de sécurité dans COM : la *sécurité de l’activation* et la sécurité de l' *appel*.</span><span class="sxs-lookup"><span data-stu-id="41129-106">There are two main types of security in COM: *activation security* and *call security*.</span></span> <span data-ttu-id="41129-107">La sécurité d’activation détermine si un client peut lancer un serveur.</span><span class="sxs-lookup"><span data-stu-id="41129-107">Activation security determines whether a client can launch a server at all.</span></span> <span data-ttu-id="41129-108">Une fois le serveur lancé, vous pouvez utiliser la sécurité de l’appel pour contrôler l’accès aux objets d’un serveur.</span><span class="sxs-lookup"><span data-stu-id="41129-108">After a server has been launched, you can use call security to control access to a server's objects.</span></span>

<span data-ttu-id="41129-109">Dans ce modèle de sécurité, les serveurs gèrent et aident à protéger les objets, les clients obtiennent l’accès aux objets via des serveurs, et les serveurs peuvent tenter d’accéder au client en usurpant l’identité du client.</span><span class="sxs-lookup"><span data-stu-id="41129-109">In this security model, servers manage and help protect objects, clients get access to objects through servers, and servers can attempt access while impersonating the client.</span></span>

<span data-ttu-id="41129-110">Le système implémente le protocole d’authentification Kerberos V5 et le package de sécurité Schannel.</span><span class="sxs-lookup"><span data-stu-id="41129-110">The system implements the Kerberos v5 authentication protocol and the Schannel security package.</span></span> <span data-ttu-id="41129-111">Il comprend également des fonctionnalités telles que l’emprunt d’identité au niveau du délégué, l’authentification mutuelle, la possibilité de définir des niveaux d’authentification pour une AppID dans le registre et le masquage.</span><span class="sxs-lookup"><span data-stu-id="41129-111">It also includes features such as delegate-level impersonation, mutual authentication, the ability to set authentication levels for an AppID in the registry, and cloaking.</span></span> <span data-ttu-id="41129-112">À l’aide de la sécurité COM, vous pouvez implémenter des objets qui peuvent effectuer des opérations privilégiées sans compromettre la sécurité.</span><span class="sxs-lookup"><span data-stu-id="41129-112">Using COM security, you can implement objects that can perform privileged operations without compromising security.</span></span>

<span data-ttu-id="41129-113">Étant donné que de nombreuses fonctionnalités de sécurité COM sont disponibles, il est utile de déterminer à l’origine le type de sécurité dont votre application a besoin.</span><span class="sxs-lookup"><span data-stu-id="41129-113">Because there is a wide range of COM security features available, it is helpful to initially determine what kind of security your application needs.</span></span> <span data-ttu-id="41129-114">Pour la plupart des applications, la définition d’un niveau de sécurité acceptable peut être un processus simple, mais vous pouvez également utiliser la sécurité COM pour prendre en charge des scénarios de sécurité très complexes.</span><span class="sxs-lookup"><span data-stu-id="41129-114">For most applications, setting an acceptable level of security can be a painless process, but you can also use COM security to support very complex security scenarios.</span></span>

<span data-ttu-id="41129-115">Vous pouvez définir la sécurité échelle à l’aide de Dcomcnfg.exe pour définir le registre ou en appelant [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity).</span><span class="sxs-lookup"><span data-stu-id="41129-115">You can set security processwide, either by using Dcomcnfg.exe to set the registry or by calling [**CoInitializeSecurity**](/windows/desktop/api/combaseapi/nf-combaseapi-coinitializesecurity).</span></span> <span data-ttu-id="41129-116">Deux interfaces principales, [**IClientSecurity**](/windows/desktop/api/ObjIdl/nn-objidl-iclientsecurity) et [**IServerSecurity**](/windows/win32/api/objidlbase/nn-objidlbase-iserversecurity) (et les fonctions d’assistance associées), vous permettent de définir la sécurité au niveau des appels au sein de votre programme.</span><span class="sxs-lookup"><span data-stu-id="41129-116">Two primary interfaces, [**IClientSecurity**](/windows/desktop/api/ObjIdl/nn-objidl-iclientsecurity) and [**IServerSecurity**](/windows/win32/api/objidlbase/nn-objidlbase-iserversecurity) (and associated helper functions), allow you to set call-level security within your program.</span></span>
