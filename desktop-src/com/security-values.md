# <a name="security-values"></a>Valeurs de sécurité

Plusieurs valeurs affectent la sécurité des applications COM, notamment le niveau d’authentification, le niveau d’emprunt d’identité, le masquage, l’identité de l’application, la configuration de la stratégie de restriction logicielle et le suivi des références.

Pour plus d'informations, voir les rubriques suivantes :

-   [Niveau d’authentification](authentication-level.md) décrit comment contrôler la quantité de protection pour la communication entre les objets com.
-   [Délégation et emprunt d’identité](delegation-and-impersonation.md) explique comment le masquage affecte l’identité présentée au serveur lors de l’emprunt d’identité et comment le client accorde au serveur l’autorité pour agir en son nom.
-   L’identité de l' [application](application-identity.md) identifie et explique les identités que votre application peut utiliser.
-   La [stratégie de restriction logicielle](software-restriction-policy.md) explique comment protéger votre système contre les dangers associés à l’exécution de code inconnu.
-   Le [suivi des références](reference-tracking.md) explique comment vous pouvez vous assurer que les objets ne sont pas libérés trop tôt.
