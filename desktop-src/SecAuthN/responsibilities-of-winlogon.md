<span data-ttu-id="5da90-104">[*Winlogon*](../secgloss/w-gly.md) a les responsabilités suivantes :</span><span class="sxs-lookup"><span data-stu-id="5da90-104">[*Winlogon*](../secgloss/w-gly.md) has the following responsibilities:</span></span>

-   <span data-ttu-id="5da90-105">Station Windows et protection des postes de travail</span><span class="sxs-lookup"><span data-stu-id="5da90-105">Window station and desktop protection</span></span>

    <span data-ttu-id="5da90-106">Winlogon définit la protection de la station Windows et des postes de travail correspondants pour s’assurer que chacun d’eux est correctement accessible.</span><span class="sxs-lookup"><span data-stu-id="5da90-106">Winlogon sets the protection of the window station and corresponding desktops to ensure that each is properly accessible.</span></span> <span data-ttu-id="5da90-107">En général, cela signifie que le système local aura un accès complet à ces objets et qu’un utilisateur connecté de manière interactive aura un accès en lecture à l’objet de station Windows et un accès complet à l’objet de bureau de l’application.</span><span class="sxs-lookup"><span data-stu-id="5da90-107">In general, this means that the local system will have full access to these objects and that an interactively logged-on user will have read access to the window station object and full access to the application desktop object.</span></span>

-   <span data-ttu-id="5da90-108">Reconnaissance SAS standard</span><span class="sxs-lookup"><span data-stu-id="5da90-108">Standard SAS recognition</span></span>

    <span data-ttu-id="5da90-109">Winlogon dispose de hooks spéciaux dans le serveur User32 qui lui permettent de surveiller les événements de séquence de touches de [*sécurité*](../secgloss/s-gly.md) (SAS) Ctrl + Alt + Suppr.</span><span class="sxs-lookup"><span data-stu-id="5da90-109">Winlogon has special hooks into the User32 server that allow it to monitor CTRL+ALT+DEL [*secure attention sequence*](../secgloss/s-gly.md) (SAS) events.</span></span> <span data-ttu-id="5da90-110">Winlogon rend ces informations d’événement SAP accessibles aux GINA pour les utiliser en tant que SAP, ou dans le cadre de leur SAS.</span><span class="sxs-lookup"><span data-stu-id="5da90-110">Winlogon makes this SAS event information available to GINAs to use as their SAS, or as part of their SAS.</span></span> <span data-ttu-id="5da90-111">En règle générale, les GINA doivent surveiller les propres SASss.     <span data-ttu-id="5da90-113">Lorsque Winlogon rencontre un événement SAS ou lorsqu’une signature d’accès partagé est remise à Winlogon par le GINA, Winlogon définit l’État en conséquence, les modifications apportées au bureau Winlogon et appelle l’une des fonctions de traitement SAP du GINA.</span><span class="sxs-lookup"><span data-stu-id="5da90-113">When Winlogon encounters a SAS event or when a SAS is delivered to Winlogon by the GINA, Winlogon sets the state accordingly, changes to the Winlogon desktop, and calls one of the SAS processing functions of the GINA.</span></span>

-   <span data-ttu-id="5da90-114">Chargement du profil utilisateur</span><span class="sxs-lookup"><span data-stu-id="5da90-114">User profile loading</span></span>

    <span data-ttu-id="5da90-115">Lorsque les utilisateurs ouvrent une session, leurs profils utilisateur sont chargés dans le registre.</span><span class="sxs-lookup"><span data-stu-id="5da90-115">When users log on, their user profiles are loaded into the registry.</span></span> <span data-ttu-id="5da90-116">De cette façon, les processus de l’utilisateur peuvent utiliser la clé de Registre spéciale HKEY \_ Current \_ User.</span><span class="sxs-lookup"><span data-stu-id="5da90-116">In this way, the processes of the user can use the special registry key HKEY\_CURRENT\_USER.</span></span> <span data-ttu-id="5da90-117">Winlogon effectue cette opération automatiquement après une ouverture de session réussie, mais avant l’activation de l’interpréteur de commandes pour l’utilisateur qui vient d’être connecté.</span><span class="sxs-lookup"><span data-stu-id="5da90-117">Winlogon does this automatically after a successful logon but before activation of the shell for the newly logged-on user.</span></span>

-   <span data-ttu-id="5da90-118">Attribution de la sécurité à l’interpréteur de commandes utilisateur</span><span class="sxs-lookup"><span data-stu-id="5da90-118">Assignment of security to user shell</span></span>

-   <span data-ttu-id="5da90-122">Contrôle de l’écran de veille</span><span class="sxs-lookup"><span data-stu-id="5da90-122">Screen saver control</span></span>
