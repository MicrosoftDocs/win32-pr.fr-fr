# <a name="relation-to-other-services"></a>Relation avec d’autres services

D’autres parties de Microsoft Internet Security Framework utilisent le [*sous-système de carte à puce*](../secgloss/s-gly.md), comme indiqué dans l’illustration suivante. (En raison des restrictions d’exportation américaines, un [*fournisseur de services de chiffrement*](../secgloss/c-gly.md) (CSP) qui utilise un SCSP pour communiquer des demandes liées au chiffrement doit signer et vérifier le SCSP.)

![relation entre Microsoft Internet Security Framework et le sous-système de carte à puce](images/smartovr3.png)

Pour plus d’informations sur le rôle du sous-système de carte à puce dans Microsoft Internet Security Framework, consultez [https://www.microsoft.com/whdc/device/input/smartcard/default.mspx](https://www.microsoft.com/whdc/device/input/smartcard/default.mspx) .
