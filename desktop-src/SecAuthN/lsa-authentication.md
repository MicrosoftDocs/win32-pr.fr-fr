# <a name="lsa-authentication"></a>Authentification LSA

L’authentification LSA décrit les parties de l' [*autorité de sécurité locale*](../secgloss/l-gly.md) (LSA) que les applications peuvent utiliser pour authentifier et connecter des utilisateurs au système local. Elle explique également comment créer et appeler des packages d’authentification et des packages de sécurité.

Les fonctions d’authentification LSA vous permettent d’écrire un package d’authentification, un package de sous-authentification ou un fournisseur de support de sécurité combiné (SSP/AP).

Les rubriques suivantes fournissent des informations supplémentaires sur l’authentification LSA :

-   [Modèle d’authentification LSA](lsa-authentication-model.md)
-   [Sessions LSA Logon](lsa-logon-sessions.md)
-   [Authentification de l’ouverture de session utilisateur LSA](lsa-user-logon-authentication.md)
-   [Packages d’authentification](authentication-packages.md)

Pour plus d’informations sur les fonctions d’authentification LSA, consultez [référence de l’authentification](authentication-reference.md).

Pour plus d’informations sur les technologies associées, consultez [LSA Policy](../secmgmt/lsa-policy.md) et [Winlogon et fournisseurs d’informations d’identification](winlogon-and-credential-providers.md).
