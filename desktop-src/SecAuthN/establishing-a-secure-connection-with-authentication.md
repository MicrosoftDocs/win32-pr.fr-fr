<span data-ttu-id="68714-104">Dans un [*protocole d’application*](/windows/desktop/SecGloss/a-gly)client/serveur, un serveur est lié à un port de communication, tel qu’un socket ou une interface RPC.</span><span class="sxs-lookup"><span data-stu-id="68714-104">In a client/server [*application protocol*](/windows/desktop/SecGloss/a-gly), a server binds to a communication port such as a socket or an RPC interface.</span></span> <span data-ttu-id="68714-105">Le serveur attend ensuite qu’un client se connecte et demande le service.</span><span class="sxs-lookup"><span data-stu-id="68714-105">The server then waits for a client to connect and request service.</span></span> <span data-ttu-id="68714-106">Le rôle de sécurité au niveau de la configuration de la connexion est bidirectionnel dans le cas de l’authentification mutuelle :</span><span class="sxs-lookup"><span data-stu-id="68714-106">The role of security at connection setup is two-fold in the case of mutual authentication:</span></span>

<span data-ttu-id="68714-109">Les composants client et serveur d’une application de transport utilisent un [*package de sécurité*](/windows/desktop/SecGloss/s-gly) pour établir une connexion sécurisée pour la transmission des messages.</span><span class="sxs-lookup"><span data-stu-id="68714-109">The client and server components of a transport application use a [*security package*](/windows/desktop/SecGloss/s-gly) to establish a secure connection for transmitting messages.</span></span> <span data-ttu-id="68714-110">La première étape de l’établissement d’une connexion sécurisée consiste à créer un [*contexte de sécurité*](/windows/desktop/SecGloss/s-gly). autrement dit, une structure de données opaque qui contient les données de sécurité relatives à une connexion, telles qu’une [*clé de session*](/windows/desktop/SecGloss/s-gly) et la durée de la session.</span><span class="sxs-lookup"><span data-stu-id="68714-110">The first step in establishing a secure connection is to create a [*security context*](/windows/desktop/SecGloss/s-gly); that is, an opaque data structure that contains the security data relevant to a connection, such as a [*session key*](/windows/desktop/SecGloss/s-gly) and the duration of the session.</span></span>

<span data-ttu-id="68714-111">Un contexte de sécurité est essentiellement un message du package de sécurité associé au client au package de sécurité associé au serveur.</span><span class="sxs-lookup"><span data-stu-id="68714-111">A security context is essentially a message from the security package associated with the client to the security package associated with the server.</span></span> <span data-ttu-id="68714-112">Par conséquent, la création d’un contexte de sécurité nécessite généralement que le client et le serveur effectuent des appels à leurs packages de sécurité respectifs.</span><span class="sxs-lookup"><span data-stu-id="68714-112">Consequently, creating a security context typically requires both client and server to make calls to their respective security packages.</span></span> <span data-ttu-id="68714-113">Pour plus d’informations sur les fonctions de contexte, consultez [gestion du contexte](authentication-functions.md).</span><span class="sxs-lookup"><span data-stu-id="68714-113">For more information on context functions, see [Context Management](authentication-functions.md).</span></span>

<span data-ttu-id="68714-114">Le protocole utilisé pour établir une connexion authentifiée sécurisée implique l’échange d’un ou plusieurs « jetons de sécurité » entre le client et le package then informs the server of the client authentication or failure to authenticate.</span></span>

<span data-ttu-id="68714-120">Le client reçoit le jeton du serveur dans un message, récupère le jeton à partir du message reçu et utilise ce jeton dans un appel à son package de sécurité.</span><span class="sxs-lookup"><span data-stu-id="68714-120">The client receives the server's token in a message, retrieves the token from the received message, and uses that token in a call to its security
