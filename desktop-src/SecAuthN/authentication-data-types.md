# <a name="authentication-data-types"></a>Types de données d’authentification

## <a name="lsa-data-types"></a>Types de données LSA

L' [*autorité de sécurité locale*](/windows/desktop/SecGloss/l-gly) (LSA) utilise le type de données suivant.



| Type de données                                            | Description                                                                                                              |
|------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------|
| [**\_demande du client PLSA \_**](plsa-client-request.md) | Type de données opaque utilisé en interne par l’autorité de certification locale pour tenir à jour les informations du client relatives aux demandes individuelles.<br/> |



 

## <a name="sspi-data-types-and-values"></a>Valeurs et types de données SSPI

[SSPI](sspi.md) utilise les données et les types énumérés suivants.



| Type de données ou valeur                         | Description                                                                      |
|--------------------------------------------|----------------------------------------------------------------------------------|
| [Handles SSPI](sspi-handles.md)           | Les types de handles SSPI et les pointeurs vers ces handles.<br/>                      |
| [Codes d’État SSPI](sspi-status-codes.md) | Codes d’État retournés dans les applications SSPI.<br/>                           |
| [Confirmé](timestamp.md)                 | Type défini pour contenir des informations sur la validité des jetons de sécurité.<br/> |
