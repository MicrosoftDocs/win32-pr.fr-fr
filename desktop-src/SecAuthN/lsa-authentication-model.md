# <a name="lsa-authentication-model"></a><span data-ttu-id="4cb71-103">Modèle d’authentification LSA</span><span class="sxs-lookup"><span data-stu-id="4cb71-103">LSA Authentication Model</span></span>

<span data-ttu-id="4cb71-104">Le modèle d’authentification de l' [*autorité de sécurité locale*](../secgloss/l-gly.md) (LSA) présente les fonctionnalités suivantes :</span><span class="sxs-lookup"><span data-stu-id="4cb71-104">The [*Local Security Authority*](../secgloss/l-gly.md) (LSA) authentication model has the following features:</span></span>

-   <span data-ttu-id="4cb71-105">L’authentification LSA prend en charge les [packages d’authentification](authentication-packages.md)personnalisés.</span><span class="sxs-lookup"><span data-stu-id="4cb71-105">LSA authentication supports custom [authentication packages](authentication-packages.md).</span></span> <span data-ttu-id="4cb71-106">Cela permet aux clients finaux et aux éditeurs de logiciels indépendants (ISV) de personnaliser ou de remplacer les routines d’authentification pour répondre aux exigences qui dépassent celles fournies par les packages d’authentification Microsoft standard.</span><span class="sxs-lookup"><span data-stu-id="4cb71-106">This allows end-customers and independent software vendors (ISVs) to customize or replace authentication routines to meet requirements beyond those provided by the standard Microsoft authentication packages.</span></span> <span data-ttu-id="4cb71-107">Bien que les packages d’authentification fournis par Microsoft requièrent un nom d’utilisateur et des données de connexion par mot de passe, un package d’authentification personnalisé peut prendre d’autres formes de données d’ouverture de session, telles que les informations de carte ATM et un numéro d’identification personnel (PIN).</span><span class="sxs-lookup"><span data-stu-id="4cb71-107">While the authentication packages provided by Microsoft require a user name and password logon data, a custom authentication package can take other forms of logon data, such as ATM card information and a personal identification number (PIN).</span></span> <span data-ttu-id="4cb71-108">Un package d’authentification personnalisé peut également être utilisé pour implémenter un nouveau [*protocole de sécurité*](../secgloss/s-gly.md).</span><span class="sxs-lookup"><span data-stu-id="4cb71-108">A custom authentication package can also be used to implement a new [*security protocol*](../secgloss/s-gly.md).</span></span>
-   <span data-ttu-id="4cb71-109">Le LSA prend en charge les packages de sécurité personnalisés, qui fonctionnent comme des fournisseurs de support de sécurité pour les applications distribuées et comme des packages d’authentification pour les applications qui requièrent des services d’authentification.</span><span class="sxs-lookup"><span data-stu-id="4cb71-109">The LSA supports custom security packages, which function as security support providers for distributed applications and as authentication packages for applications that require authentication services.</span></span> <span data-ttu-id="4cb71-110">La fonctionnalité d’authentification est accessible à l’aide de la même interface que celle d’un package d’authentification autonome, tandis que la fonctionnalité du fournisseur de support de sécurité est accessible à l’aide de SSPI ( [Security Support Provider Interface](sspi.md) ).</span><span class="sxs-lookup"><span data-stu-id="4cb71-110">The authentication functionality is accessed using the same interface as a stand-alone authentication package, while the security support provider functionality is accessed using [Security Support Provider Interface](sspi.md) (SSPI).</span></span> <span data-ttu-id="4cb71-111">L’ensemble des fonctions de prise en charge LSA disponibles pour les packages de sécurité personnalisés leur permet de fournir des fonctionnalités de sécurité avancées, telles que la création de jetons et la gestion des [*informations d’identification supplémentaires*](../secgloss/s-gly.md) .</span><span class="sxs-lookup"><span data-stu-id="4cb71-111">The set of LSA support functions available to custom security packages allows them to supply advanced security features, such as token creation and [*supplemental credentials*](../secgloss/s-gly.md) management.</span></span>
-   <span data-ttu-id="4cb71-112">L’autorité LSA prend en charge la gestion des informations d’identification hétérogènes pour interagir avec des produits non-Microsoft, tels que les réseaux et les bases de données.</span><span class="sxs-lookup"><span data-stu-id="4cb71-112">The LSA supports heterogeneous credentials management to interface with non-Microsoft products, such as networks and databases.</span></span> <span data-ttu-id="4cb71-113">Étant donné que ces produits ont souvent leurs propres informations d’identification de sécurité, le LSA fournit des fonctions que les packages d’authentification peuvent utiliser pour associer des informations d’identification non-Microsoft à des processus Windows.</span><span class="sxs-lookup"><span data-stu-id="4cb71-113">Because such products often have their own security credentials, the LSA provides functions that authentication packages can use to associate non-Microsoft credentials with Windows processes.</span></span> <span data-ttu-id="4cb71-114">Les packages d’authentification personnalisés peuvent fournir des services pour interroger ces informations d’identification si nécessaire, par exemple lorsqu’un redirecteur réseau tente d’établir une connexion à un système distant.</span><span class="sxs-lookup"><span data-stu-id="4cb71-114">Custom authentication packages can provide services for querying these credentials when needed, such as when a network redirector attempts to establish a connection to a remote system.</span></span>
-   <span data-ttu-id="4cb71-115">Chaque classe d’appareil de connexion installée sur un système peut avoir son propre processus d’ouverture de session.</span><span class="sxs-lookup"><span data-stu-id="4cb71-115">Each class of logon device installed on a system can have its own logon process.</span></span> <span data-ttu-id="4cb71-116">Les classes d’appareils incluent généralement des appareils tels que les lecteurs de carte à puce. Toutefois, dans le cadre de l’authentification [*LSA*](../secgloss/l-gly.md) , les réseaux connectés sont également traités comme des appareils.</span><span class="sxs-lookup"><span data-stu-id="4cb71-116">Device classes typically include devices such as smart card readers; however, for the purposes of [*LSA*](../secgloss/l-gly.md) authentication, connected networks are also treated as devices.</span></span> <span data-ttu-id="4cb71-117">Par défaut, le système d’exploitation Windows fournit le processus d’ouverture de session qui prend en charge les ouvertures de session par nom d’utilisateur et mot de passe à l’aide d’un clavier.</span><span class="sxs-lookup"><span data-stu-id="4cb71-117">By default, the Windows operating system provides the logon process that supports user name and password logons using a keyboard.</span></span> <span data-ttu-id="4cb71-118">Les développeurs peuvent ajouter la prise en charge d’autres appareils, tels que les [*lecteurs*](../secgloss/r-gly.md)de [*carte à puce*](../secgloss/s-gly.md) .</span><span class="sxs-lookup"><span data-stu-id="4cb71-118">Developers can add support for other devices, such as [*smart card*](../secgloss/s-gly.md) [*readers*](../secgloss/r-gly.md).</span></span> <span data-ttu-id="4cb71-119">Pour plus d’informations sur les cartes à puce, consultez [authentification par carte à puce](smart-card-authentication.md).</span><span class="sxs-lookup"><span data-stu-id="4cb71-119">For more information about smart cards, see [Smart Card Authentication](smart-card-authentication.md).</span></span> <span data-ttu-id="4cb71-120">Pour plus d’informations sur la prise en charge des périphériques d’ouverture de session, consultez [Winlogon et Gina](winlogon-and-gina.md).</span><span class="sxs-lookup"><span data-stu-id="4cb71-120">For more information about logon device support, see [Winlogon and GINA](winlogon-and-gina.md).</span></span>
