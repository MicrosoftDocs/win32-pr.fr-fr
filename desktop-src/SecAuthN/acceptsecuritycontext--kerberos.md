## <a name="remarks"></a><span data-ttu-id="46733-213">Notes</span><span class="sxs-lookup"><span data-stu-id="46733-213">Remarks</span></span>

<span data-ttu-id="46733-215">Lorsque le serveur reçoit une demande d’un client, le serveur utilise le paramètre *fContextReq* pour spécifier ce qu’il requiert de la session.</span><span class="sxs-lookup"><span data-stu-id="46733-215">When the server receives a request from a client, the server uses the *fContextReq* parameter to specify what it requires of the session.</span></span> <span data-ttu-id="46733-216">De cette manière, un serveur peut spécifier que les clients doivent être en mesure d’utiliser une session confidentielle ou contrôlée par [*l’intégrité*](../secgloss/i-gly.md), et il peut rejeter des clients qui ne répondent pas à cette demande.</span><span class="sxs-lookup"><span data-stu-id="46733-216">In this fashion, a server can specify that clients must be capable of using a confidential or [*integrity*](../secgloss/i-gly.md)-checked session, and it can reject clients that cannot meet that demand.</span></span> <span data-ttu-id="46733-217">En guise d’alternative, un serveur ne peut nécessiter rien, et tout ce que le client peut fournir ou nécessite est retourné dans le paramètre *pfContextAttr* .</span><span class="sxs-lookup"><span data-stu-id="46733-217">Alternatively, a server can require nothing, and whatever the client can provide or requires is returned in the *pfContextAttr* parameter.</span></span>

<span data-ttu-id="46733-218">Pour un package qui prend en charge l’authentification à plusieurs jambes, telle que l’authentification mutuelle, la séquence d’appel est la suivante :</span><span class="sxs-lookup"><span data-stu-id="46733-218">For a package that supports multiple-leg authentication, such as mutual authentication, the calling sequence is as follows:</span></span>
