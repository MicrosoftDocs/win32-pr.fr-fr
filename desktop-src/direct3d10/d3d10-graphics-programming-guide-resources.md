# <a name="resources-direct3d-10"></a><span data-ttu-id="3b195-103">Ressources (Direct3D 10)</span><span class="sxs-lookup"><span data-stu-id="3b195-103">Resources (Direct3D 10)</span></span>

<span data-ttu-id="3b195-104">Une ressource est une zone de mémoire accessible au pipeline Direct3D.</span><span class="sxs-lookup"><span data-stu-id="3b195-104">A resource is an area in memory that can be accessed by the Direct3D pipeline.</span></span> <span data-ttu-id="3b195-105">Pour que le pipeline puisse accéder efficacement à la mémoire, les données fournies au pipeline (telles que la géométrie d’entrée, les ressources du nuanceur, les textures, etc.) doivent être stockées dans une ressource.</span><span class="sxs-lookup"><span data-stu-id="3b195-105">In order for the pipeline to access memory efficiently, data that is provided to the pipeline (such as input geometry, shader resources, textures etc) must be stored in a resource.</span></span> <span data-ttu-id="3b195-106">Il existe deux types de ressource dont dérivent l’ensemble des ressources Direct3D : la mémoire tampon et la texture.</span><span class="sxs-lookup"><span data-stu-id="3b195-106">There are two types of resources from which all Direct3D resources derive: a buffer or a texture.</span></span> <span data-ttu-id="3b195-107">Jusqu’à 128 ressources peuvent être actives à chaque phase du pipeline.</span><span class="sxs-lookup"><span data-stu-id="3b195-107">Up to 128 resources can be active for each pipeline stage.</span></span>

<span data-ttu-id="3b195-108">Chaque application crée généralement de nombreuses ressources.</span><span class="sxs-lookup"><span data-stu-id="3b195-108">Each application will typically create many resources.</span></span> <span data-ttu-id="3b195-109">Exemples de ressources : mémoires tampons de vertex, tampon d’index, mémoire tampon constante, textures et ressources de nuanceur.</span><span class="sxs-lookup"><span data-stu-id="3b195-109">Examples of resource include: vertex buffers, index buffer, constant buffer, textures, and shader resources.</span></span> <span data-ttu-id="3b195-110">Il existe plusieurs options qui déterminent la façon dont les ressources peuvent être utilisées.</span><span class="sxs-lookup"><span data-stu-id="3b195-110">There are several options that determine how resources can be used.</span></span> <span data-ttu-id="3b195-111">Vous pouvez créer des ressources fortement typées ou de type inférieur ; vous pouvez contrôler si les ressources ont un accès en lecture et en écriture. vous pouvez rendre les ressources accessibles uniquement au processeur, au GPU ou aux deux.</span><span class="sxs-lookup"><span data-stu-id="3b195-111">You can create resources that are strongly typed or type less; you can control whether resources have both read and write access; you can make resources accessible to only the CPU, GPU, or both.</span></span> <span data-ttu-id="3b195-112">Naturellement, il y aura un compromis entre vitesse et fonctionnalité : plus le nombre de fonctionnalités que vous autorisez à disposer d’une ressource est faible, plus les performances attendues sont élevées.</span><span class="sxs-lookup"><span data-stu-id="3b195-112">Naturally, there will be speed vs. functionality tradeoff - the more functionality you allow a resource to have, the less performance you should expect.</span></span>

<span data-ttu-id="3b195-113">Étant donné qu’une application utilise souvent de nombreuses textures, Direct3D introduit également le concept d’un tableau de texture pour simplifier la gestion des textures.</span><span class="sxs-lookup"><span data-stu-id="3b195-113">Since an application often uses many textures, Direct3D also introduces the concept of a texture array to simplify texture management.</span></span> <span data-ttu-id="3b195-114">Un tableau de texture contient une ou plusieurs textures (toutes des mêmes types et dimensions) qui peuvent être indexées à partir d’une application ou par des nuanceurs.</span><span class="sxs-lookup"><span data-stu-id="3b195-114">A texture array contains one or more textures (all of the same type and dimensions) that can be indexed from within an application or by shaders.</span></span> <span data-ttu-id="3b195-115">Les tableaux de texture vous permettent d’utiliser une seule interface avec plusieurs index pour accéder à de nombreuses textures.</span><span class="sxs-lookup"><span data-stu-id="3b195-115">Texture arrays allow you to use a single interface with multiple indexes to access many textures.</span></span> <span data-ttu-id="3b195-116">Vous pouvez créer autant de tableaux de textures que nécessaire pour gérer différents types de texture.</span><span class="sxs-lookup"><span data-stu-id="3b195-116">You can create as many texture arrays to manage different texture types as you need.</span></span>

<span data-ttu-id="3b195-117">Une fois que vous avez créé les ressources que votre application utilisera, vous connectez ou liez chaque ressource aux étapes de pipeline qui vont les utiliser.</span><span class="sxs-lookup"><span data-stu-id="3b195-117">Once you have created the resources that your application will use, you connect or bind each resource to the pipeline stages that will use them.</span></span> <span data-ttu-id="3b195-118">Pour ce faire, vous devez appeler une API de liaison, qui prend un pointeur vers la ressource.</span><span class="sxs-lookup"><span data-stu-id="3b195-118">This is accomplished by calling a bind API, which takes a pointer to the resource.</span></span> <span data-ttu-id="3b195-119">Étant donné que plusieurs étapes de pipeline peuvent avoir besoin d’accéder à la même ressource, Direct3D 10 introduit le concept d’affichage des ressources.</span><span class="sxs-lookup"><span data-stu-id="3b195-119">Since more than one pipeline stage may need access to the same resource, Direct3D 10 introduces the concept of a resource view.</span></span> <span data-ttu-id="3b195-120">Une vue identifie la partie d’une ressource qui est accessible.</span><span class="sxs-lookup"><span data-stu-id="3b195-120">A view identifies the portion of a resource that can be accessed.</span></span> <span data-ttu-id="3b195-121">Vous pouvez créer des vues m ou une ressource et les lier à n étapes de canalisation, en supposant que vous suivez les règles de liaison pour les ressources partagées (le runtime génère des erreurs au moment de la compilation, si ce n’est pas le cas).</span><span class="sxs-lookup"><span data-stu-id="3b195-121">You can create m views or a resource and bind them to n pipeline stages, assuming you follow binding rules for shared resource (the runtime will generate errors at compile time if you don't).</span></span>

<span data-ttu-id="3b195-122">Un affichage des ressources fournit un modèle général pour l’accès à une ressource (textures, mémoires tampons, etc.).</span><span class="sxs-lookup"><span data-stu-id="3b195-122">A resource view provides a general model for access to a resource (textures, buffers, etc.).</span></span> <span data-ttu-id="3b195-123">Étant donné que vous pouvez utiliser une vue pour indiquer à l’exécution les données auxquelles accéder et comment y accéder, les vues de ressources vous permettent de créer moins de ressources.</span><span class="sxs-lookup"><span data-stu-id="3b195-123">Because you can use a view to tell the runtime what data to access and how to access it, resource views allow you create type less resources.</span></span> <span data-ttu-id="3b195-124">Autrement dit, vous pouvez créer des ressources pour une taille donnée au moment de la compilation, puis déclarer le type de données dans la ressource lorsque la ressource est liée au pipeline.</span><span class="sxs-lookup"><span data-stu-id="3b195-124">That is, you can create resources for a given size at compile time, and then declare the data type within the resource when the resource gets bound to the pipeline.</span></span> <span data-ttu-id="3b195-125">Les vues présentent de nombreuses nouvelles fonctionnalités permettant d’utiliser des ressources, telles que la capacité à lire des surfaces de stencil/profondeur dans le nuanceur, de générer des cubemaps dynamiques en une seule passe et de les afficher simultanément sur plusieurs secteurs d’un volume.</span><span class="sxs-lookup"><span data-stu-id="3b195-125">Views expose many new capabilities for using resources, such as the ability to read back depth/stencil surfaces in the shader, generating dynamic cubemaps in a single pass, and rendering simultaneously to multiple slices of a volume.</span></span>
