# <a name="midl-and-com"></a>MIDL et COM

Tous les objets COM implémentent une ou plusieurs interfaces. Lorsque vous créez un objet COM personnalisé, vous devez décrire l’interface ou les interfaces dans un fichier IDL. Les rubriques de cette section présentent des informations sur l’utilisation de MIDL pour définir des interfaces COM :

-   [Fichiers générés pour une interface COM](files-generated-for-a-com-interface.md)
-   [Marshaling des types de données OLE](marshaling-ole-data-types.md)

Pour plus d’informations sur la définition d’interfaces COM avec MIDL, consultez [définition des interfaces com](../com/defining-com-interfaces.md). Pour plus d’informations sur la génération de fichiers générés pour différentes plateformes, consultez [ciblage de stubs pour des plateformes 32 bits ou 64 bits spécifiques](targeting-stubs-for-specific-32-bit-or-64-bit-platforms.md).
