<dl> <dt>

<span id="_rras_aged_out_gly"></span><span id="_RRAS_AGED_OUT_GLY"></span>**obsolète**
</dt> <dd>

Par ailleurs, *anciennement dans le cache*. Dans RRAS, un itinéraire est obsolète lorsqu’il est supprimé de la table de routage après une période spécifiée.

</dd> <dt>

<span id="_rras_authentication_gly"></span><span id="_RRAS_AUTHENTICATION_GLY"></span>**identification**
</dt> <dd>

L’authentification est le processus qui consiste à vérifier qu’un utilisateur est bien celui qu’il prétend être. L’authentification s’effectue généralement à l’aide d’un nom d’utilisateur et d’un mot de passe.

</dd> <dt>

<span id="_rras_autonomous_system_gly"></span><span id="_RRAS_AUTONOMOUS_SYSTEM_GLY"></span>**système autonome (AS)**
</dt> <dd>

Un système autonome est un groupe de routeurs qui partagent une stratégie d’administration unique. Ces routeurs utilisent tous le même protocole de routage pour communiquer (appelé [*protocole de passerelle interne*](i-gly.md)). Voir aussi [*protocole de passerelle externe*](e-gly.md).

</dd> <dt>

<span id="_rras_autostatic_route_gly"></span><span id="_RRAS_AUTOSTATIC_ROUTE_GLY"></span>**Itinéraire autostatique**
</dt> <dd>

Un itinéraire autostatique est un itinéraire qui est obtenu de manière dynamique à partir d’un protocole de routage. Une fois qu’un itinéraire est obtenu de cette manière, il se comporte comme un [*STATICROUTE*](s-gly.md).

</dd> </dl>
