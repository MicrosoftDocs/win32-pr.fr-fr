# <a name="peer-networking-scenarios"></a>Scénarios de mise en réseau pair à pair

L’infrastructure d’égal à égal vous offre de nombreuses options pour le développement d’applications de réseau pair à pair. Cette rubrique contient des informations sur les scénarios de mise en réseau pair à pair suivants :

-   Applications de collaboration
-   Applications de messagerie instantanée ou de conversation
-   Applications de distribution de contenu

## <a name="collaborative-applications"></a>Applications de collaboration

Vous pouvez utiliser l’infrastructure d’égal à égal pour créer des applications collaboratives permettant aux utilisateurs de partager interactivement des idées, qu’elles se trouvent dans la même salle de conférence ou sur Internet. Par exemple, vous pouvez créer une application de type tableau blanc partagé.

> [!Note]  
> L’infrastructure d’égal à égal ne prend pas en charge le streaming audio ou vidéo.

 

## <a name="instant-messaging-or-chat-applications"></a>Applications de messagerie instantanée ou de conversation

La messagerie instantanée et les applications de conversation qui utilisent des serveurs centraux peuvent être mises à jour pour utiliser l’infrastructure d’égal à égal. Par exemple, une application de conversation peut utiliser l’infrastructure d’égal à égal comme alternative à un serveur centralisé. L’infrastructure d’égal à égal améliore l’évolutivité et permet aux utilisateurs de participer à des activités de messagerie et de conversation sans serveur.

## <a name="content-distribution-applications"></a>Applications de distribution de contenu

Vous pouvez utiliser l’infrastructure d’égal à égal comme un mécanisme de distribution pour le contenu. Par exemple, vous pouvez distribuer un bulletin d’informations à des collègues qui s’abonnent à un service. Dans la mesure où les données de l’infrastructure d’égal à égal sont synchronisées, les abonnés reçoivent le bulletin d’informations même si le serveur de publication n’est pas en ligne.
