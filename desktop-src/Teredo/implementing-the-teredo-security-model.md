# <a name="implementing-the-teredo-security-model"></a><span data-ttu-id="dfbd9-104">Implémentation du modèle de sécurité Teredo</span><span class="sxs-lookup"><span data-stu-id="dfbd9-104">Implementing the Teredo Security Model</span></span>

<span data-ttu-id="dfbd9-105">Le modèle de sécurité Teredo est basé sur la technologie de [plateforme de filtrage Windows](/windows/desktop/FWP/windows-filtering-platform-start-page) (WFP) intégrée à Windows Vista.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-105">The Teredo security model is based on the [Windows Filtering Platform](/windows/desktop/FWP/windows-filtering-platform-start-page) (WFP) technology built into Windows Vista.</span></span> <span data-ttu-id="dfbd9-106">Par conséquent, il est recommandé que les pare-feu tiers utilisent la plateforme WFP pour appliquer le modèle de sécurité [Teredo](about-teredo.md) .</span><span class="sxs-lookup"><span data-stu-id="dfbd9-106">As a result, it is recommended that third-party firewalls use WFP to enforce the [Teredo](about-teredo.md) security model.</span></span>

<span data-ttu-id="dfbd9-107">L’implémentation générale du [modèle de sécurité Teredo](the-teredo-security-model.md) requiert les éléments suivants :</span><span class="sxs-lookup"><span data-stu-id="dfbd9-107">The general implementation of the [Teredo Security Model](the-teredo-security-model.md) requires the following:</span></span>

-   <span data-ttu-id="dfbd9-108">Un pare-feu hôte compatible IPv6 doit être inscrit auprès de Windows Security Center (WSC) sur l’ordinateur.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-108">An IPv6-capable host firewall must be registered with Windows Security Center (WSC) on the machine.</span></span> <span data-ttu-id="dfbd9-109">En l’absence d’un pare-feu basé sur l’hôte ou de WSC, l’interface Teredo n’est pas disponible pour utilisation.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-109">In the absence of a host-based firewall, or WSC itself, the Teredo interface will not be available for use.</span></span> <span data-ttu-id="dfbd9-110">Il s’agit de la seule exigence de recevoir le trafic sollicité d’Internet via l’interface Teredo.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-110">This is the only requirement to receive solicited traffic from the Internet over the Teredo interface.</span></span>
-   <span data-ttu-id="dfbd9-111">Toute application qui reçoit le trafic non sollicité d’Internet via l’interface Teredo doit s’inscrire auprès d’un pare-feu compatible IPv6, comme le [pare-feu Windows](/previous-versions/windows/desktop/ics/windows-firewall-start-page), avant de recevoir le trafic non sollicité.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-111">Any application that receives unsolicited traffic from the Internet over the Teredo interface must register with an IPv6-capable firewall, like [Windows Firewall](/previous-versions/windows/desktop/ics/windows-firewall-start-page), before receiving the unsolicited traffic.</span></span>

<span data-ttu-id="dfbd9-112">Une adresse Teredo devient dormante si le trafic n’a pas été envoyé ou reçu via l’interface Teredo pendant une heure et si les applications qui répondent aux critères de sécurité requis pour le trafic non sollicité ne sont pas en cours d’exécution ou si les sockets d’écoute ne sont pas ouverts.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-112">A Teredo address will become dormant if traffic has not been sent or received over the Teredo interface for one hour and if applications that meet the required security criteria for unsolicited traffic are not running or do not have listening sockets open.</span></span>

<span data-ttu-id="dfbd9-113">Après le redémarrage d’un ordinateur, ou si l’adresse Teredo perd ses qualifications, Windows Vista qualifie automatiquement l’adresse et le rend disponible pour une utilisation dès qu’une application est liée à des sockets compatibles IPv6.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-113">After the reboot of a machine, or if the Teredo address loses its qualifications, Windows Vista will automatically qualify the address and makes it available for use as soon as an application binds to IPv6-capable sockets.</span></span> <span data-ttu-id="dfbd9-114">Il est important de noter que l’option de traversée latérale du pare-feu Windows définie par une application pour autoriser le trafic non sollicité via Teredo est persistante entre les redémarrages.</span><span class="sxs-lookup"><span data-stu-id="dfbd9-114">It is important to note that the Windows Firewall "Edge Traversal" option set by an application to allow unsolicited traffic via Teredo is persistent across reboots.</span></span>
