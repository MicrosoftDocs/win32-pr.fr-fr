<span data-ttu-id="688af-104">Les handles d’objet de noyau sont spécifiques au processus.</span><span class="sxs-lookup"><span data-stu-id="688af-104">Kernel object handles are process specific.</span></span> <span data-ttu-id="688af-105">Autrement dit, un processus doit créer l’objet ou ouvrir un objet existant pour obtenir un handle d’objet de noyau.</span><span class="sxs-lookup"><span data-stu-id="688af-105">That is, a process must either create the object or open an existing object to obtain a kernel object handle.</span></span> <span data-ttu-id="688af-106">La limite par processus sur les handles de noyau est de 2 ^ 24.</span><span class="sxs-lookup"><span data-stu-id="688af-106">The per-process limit on kernel handles is 2^24.</span></span> <span data-ttu-id="688af-107">Toutefois, les handles étant stockés dans la réserve paginée, le nombre réel de handles que vous pouvez créer est basé sur la mémoire disponible.</span><span class="sxs-lookup"><span data-stu-id="688af-107">However, handles are stored in the paged pool, so the actual number of handles you can create is based on available memory.</span></span> <span data-ttu-id="688af-108">Le nombre de descripteurs que vous pouvez créer sur des fenêtres 32 bits est nettement inférieur à 2 ^ 24.</span><span class="sxs-lookup"><span data-stu-id="688af-108">The number of handles that you can create on 32-bit Windows is significantly lower than 2^24.</span></span>

<span data-ttu-id="688af-109">Tout processus peut créer un nouveau descripteur vers un objet de noyau existant (même créé par un autre processus), à condition que le processus sache le nom de l’objet indicate the actions that can be granted or denied to a process.</span></span> <span data-ttu-id="688af-111">Une application spécifie des droits d’accès quand elle crée un objet ou obtient un handle d’objet existant.</span><span class="sxs-lookup"><span data-stu-id="688af-111">An application specifies access rights when it creates an object or obtains an existing object handle.</span></span> <span data-ttu-id="688af-112">Chaque type d’objet de noyau prend en charge son propre ensemble de droits d’accès.</span><span class="sxs-lookup"><span data-stu-id="688af-112">Each type of kernel object supports its own set of access rights.</span></span> <span data-ttu-id="688af-113">Par exemple, les handles d’événements peuvent avoir un accès set ou Wait (ou les deux), les descripteurs de Objects](/windows/desktop/SecAuthZ/securable-objects).</span></span>

![application créant un objet d’événement avec plusieurs handles](images/cshob-04.png)

<span data-ttu-id="688af-125">Si un autre processus connaît le nom de l’événement et possède un accès de sécurité à l’objet, il peut créer son propre handle d’objet d’événement à l’aide de [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa).</span><span class="sxs-lookup"><span data-stu-id="688af-125">If another process knows the event name and has security access to the object, it can create its own event object handle by using [**OpenEvent**](/windows/desktop/api/synchapi/nf-synchapi-openeventa).</span></span> <span data-ttu-id="688af-126">L’application de création peut également dupliquer l’un de ses descripteurs dans le même processus ou dans un autre processus à l’aide de la fonction [**DuplicateHandle**](/windows/win32/api/handleapi/nf-handleapi-duplicatehandle) .</span><span class="sxs-lookup"><span data-stu-id="688af-126">The creating application could also duplicate one of its handles into the same process or into another process by using the [**DuplicateHandle**](/windows/win32/api/handleapi/nf-handleapi-duplicatehandle) function.</span></span>

<span data-ttu-id="688af-127">Un objet reste en mémoire tant qu’il existe au moins un handle d’objet.</span><span class="sxs-lookup"><span data-stu-id="688af-127">An object remains in memory as long as at least one object handle exists.</span></span> <span data-ttu-id="688af-128">Dans l’illustration suivante, les applications utilisent la fonction [**CloseHandle**](/windows/win32/api/handleapi/nf-handleapi-closehandle) pour fermer leurs handles d’objet d’événement.</span><span class="sxs-lookup"><span data-stu-id="688af-128">In the following illustration, the applications use the [**CloseHandle**](/windows/win32/api/handleapi/nf-handleapi-closehandle) function to close their event object handles.</span></span> <span data-ttu-id="688af-129">En l’absence de descripteurs d’événements, le système supprime l’objet de la mémoire, comme indiqué dans l’illustration suivante.</span><span class="sxs-lookup"><span data-stu-id="688af-129">When there are no event handles, the system removes the object from memory, as shown in the following illustration.</span></span>
