Consultez également [exemples de code](https://developer.microsoft.com/windows/samples/).

## <a name="find-a-sample-for-the-api-youre-interested-in"></a>Trouver un exemple pour l’API qui vous intéresse

Vous pouvez utiliser Microsoft Visual Studio pour rechercher le code source d’un référentiel entier afin de déterminer si l’utilisation d’une API Windows particulière est démontrée.

* Clonez les dépôts figurant dans les sections ci-dessous (ou téléchargez le fichier ZIP) dans votre système de fichiers local.
* **Recherchez ensuite dans les fichiers** dans Visual Studio. Définissez **regarder dans** le dossier dans lequel vous avez cloné ou téléchargé, puis recherchez le nom de l’API qui vous intéresse. La vérification de la correspondance de la **casse** donne de meilleurs résultats.
* Veillez à vérifier le **mot entier** si l’API peut être appelée sous différentes formes &mdash; , par exemple **CreateMutex**, **CreateMutexA** et **CreateMutexW**. Dans ce cas, recherchez **CreateMutex** avec l’option **mot entier** désactivé.

> [!NOTE]
> Vous pouvez installer [Visual Studio](https://visualstudio.microsoft.com/downloads/) sans dépense. Une édition Community est disponible et gratuite pour les étudiants, les contributeurs open source et les particuliers. Bien sûr, vous pouvez faire la même chose avec n’importe quel exemple de référentiel.
