# <a name="local-security-policy"></a>Stratégie de sécurité locale

La stratégie de sécurité locale d’un système est un ensemble d’informations sur la sécurité d’un ordinateur local.

Les informations de stratégie de sécurité locale incluent les éléments suivants :

-   Les domaines approuvés pour authentifier les tentatives de connexion.
-   Les comptes d’utilisateurs qui peuvent accéder au système et comment. Par exemple, de manière interactive, par le biais d’un réseau ou d’un service.
-   Les droits et les [*privilèges*](/windows/desktop/SecGloss/p-gly) attribués aux comptes.
-   Stratégie d’audit de sécurité.

L' [*autorité de sécurité locale*](/windows/desktop/SecGloss/l-gly) (LSA) stocke les informations de stratégie locale dans un ensemble d' [objets de stratégie LSA](lsa-policy-objects.md).
