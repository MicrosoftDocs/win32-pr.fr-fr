# <a name="policy-object"></a><span data-ttu-id="a57b7-103">Objet de stratégie</span><span class="sxs-lookup"><span data-stu-id="a57b7-103">Policy Object</span></span>

<span data-ttu-id="a57b7-104">L’objet de **stratégie** est utilisé pour contrôler l’accès à la base de données de l' [*autorité de sécurité locale*](/windows/desktop/SecGloss/l-gly) (LSA) et contient des informations qui s’appliquent à l’ensemble du système ou qui établissent des valeurs par défaut pour le système.</span><span class="sxs-lookup"><span data-stu-id="a57b7-104">The **Policy** object is used to control access to the [*Local Security Authority*](/windows/desktop/SecGloss/l-gly) (LSA) database and contains information that applies to the entire system or establishes defaults for the system.</span></span> <span data-ttu-id="a57b7-105">Chaque système n’a qu’un seul objet de **stratégie** .</span><span class="sxs-lookup"><span data-stu-id="a57b7-105">Each system has only one **Policy** object.</span></span> <span data-ttu-id="a57b7-106">Cet objet de **stratégie** est créé par l’autorité LSA au démarrage du système, et les applications ne peuvent pas la créer ou la détruire.</span><span class="sxs-lookup"><span data-stu-id="a57b7-106">This **Policy** object is created by the LSA when the system starts up, and applications cannot create or destroy it.</span></span>

<span data-ttu-id="a57b7-107">Les informations stockées dans un objet de **stratégie** incluent :</span><span class="sxs-lookup"><span data-stu-id="a57b7-107">The information stored in a **Policy** object includes:</span></span>

-   <span data-ttu-id="a57b7-108">Quota de mémoire par défaut du système.</span><span class="sxs-lookup"><span data-stu-id="a57b7-108">System default memory quota.</span></span> <span data-ttu-id="a57b7-109">Sauf indication contraire, chaque utilisateur ouvrant une session sur le système se verra attribuer ce quota de mémoire.</span><span class="sxs-lookup"><span data-stu-id="a57b7-109">Unless otherwise specified, each user logging on to the system will be assigned this memory quota.</span></span> <span data-ttu-id="a57b7-110">Des quotas de mémoire spéciaux peuvent être attribués à des personnes ou à des membres de groupes ou groupes locaux par le biais d’un objet de [**compte**](account-object.md) .</span><span class="sxs-lookup"><span data-stu-id="a57b7-110">Special memory quotas can be assigned to individuals or members of groups or local groups through an [**Account**](account-object.md) object.</span></span>
-   <span data-ttu-id="a57b7-111">Exigences d’audit de sécurité à l’ensemble du système.</span><span class="sxs-lookup"><span data-stu-id="a57b7-111">System-wide security auditing requirements.</span></span>
-   <span data-ttu-id="a57b7-112">Nom et SID du domaine du compte de ce système.</span><span class="sxs-lookup"><span data-stu-id="a57b7-112">The name and SID of the account domain of this system.</span></span>
-   <span data-ttu-id="a57b7-113">Informations sur le domaine principal de ce système.</span><span class="sxs-lookup"><span data-stu-id="a57b7-113">Information about the primary domain of this system.</span></span> <span data-ttu-id="a57b7-114">Ces informations incluent le nom et le SID du domaine principal, le nom du compte dans le domaine principal à utiliser pour les demandes d’authentification, le nom et les traductions de SID, et l’obtention des noms des contrôleurs de domaine au sein du domaine.</span><span class="sxs-lookup"><span data-stu-id="a57b7-114">This information includes the name and SID of the primary domain, the name of the account within the primary domain that is to be used for authentication requests, name and SID translations, and obtaining the names of domain controllers within the domain.</span></span> <span data-ttu-id="a57b7-115">Ces noms peuvent être obsolètes et ne doivent être pris en considération qu’en tant qu’indicateur.</span><span class="sxs-lookup"><span data-stu-id="a57b7-115">These names may be out of date and should be taken only as a hint.</span></span> <span data-ttu-id="a57b7-116">L’ordre de cette liste est supposé être significatif et sera conservé.</span><span class="sxs-lookup"><span data-stu-id="a57b7-116">The order of this list is assumed to be significant and will be maintained.</span></span> <span data-ttu-id="a57b7-117">Cela permet, par exemple, que le prénom de la liste représente le dernier contrôleur de domaine principal connu.</span><span class="sxs-lookup"><span data-stu-id="a57b7-117">This allows, for example, the first name in the list to represent the last-known primary domain controller.</span></span>

<span data-ttu-id="a57b7-120">Les champs **AccountDomain** et **PrimaryDomain** de l’objet de **stratégie** sont utilisés à des fins différentes selon le type de relations système et d’approbation :</span><span class="sxs-lookup"><span data-stu-id="a57b7-120">The **AccountDomain** and **PrimaryDomain** fields of the **Policy** object are used for different purposes depending upon the type of system and trust relationships:</span></span>

-   <span data-ttu-id="a57b7-121">Sur un système qui n’a pas de domaine principal, le champ **AccountDomain** contient le nom et le SID du domaine du compte local du système, qui est le même que le nom de l’ordinateur.</span><span class="sxs-lookup"><span data-stu-id="a57b7-121">On a system that does not have a primary domain, the **AccountDomain** field contains the name and SID of the system's local account domain, which is the same as the computer name.</span></span> <span data-ttu-id="a57b7-122">Le champ **PrimaryDomain** contient le nom du groupe de travail dont cet ordinateur est membre.</span><span class="sxs-lookup"><span data-stu-id="a57b7-122">The **PrimaryDomain** field contains the name of the workgroup this machine is a member of.</span></span> <span data-ttu-id="a57b7-123">Les objets [**trustedDomain**](trusteddomain-object.md) sont ignorés à une exception près : il ne peut pas y avoir d’objet **trustedDomain** portant le même nom que le groupe de travail, car il s’affiche comme s’il s’agissait du domaine principal de l’ordinateur.</span><span class="sxs-lookup"><span data-stu-id="a57b7-123">[**TrustedDomain**](trusteddomain-object.md) objects are ignored with one exception—there cannot be a **TrustedDomain** object with the same name as the workgroup because it will appear as though it is the machine's primary domain.</span></span>
-   <span data-ttu-id="a57b7-124">Sur un système qui a un domaine principal, le champ **AccountDomain** identifie le nom et le SID du domaine de compte local, comme précédemment.</span><span class="sxs-lookup"><span data-stu-id="a57b7-124">On a system that has a primary domain, the **AccountDomain** field identifies the name and SID of the local account domain, as before.</span></span> <span data-ttu-id="a57b7-125">Toutefois, le champ **PrimaryDomain** contient le nom et le SID du domaine principal pour le système.</span><span class="sxs-lookup"><span data-stu-id="a57b7-125">However, the **PrimaryDomain** field contains the name and SID of the primary domain for the system.</span></span> <span data-ttu-id="a57b7-126">En outre, il doit exister un objet [**trustedDomain**](trusteddomain-object.md) avec le nom et le SID identifiés dans le champ **PrimaryDomain** .</span><span class="sxs-lookup"><span data-stu-id="a57b7-126">Additionally, there should be a [**TrustedDomain**](trusteddomain-object.md) object with the name and SID identified in the **PrimaryDomain** field.</span></span> <span data-ttu-id="a57b7-127">Cet objet **trustedDomain** contient les informations relatives au compte et au serveur nécessaires pour établir un canal sécurisé vers un contrôleur de domaine dans le domaine principal.</span><span class="sxs-lookup"><span data-stu-id="a57b7-127">This **TrustedDomain** object contains the account and server information necessary to establish a secure channel to a domain controller in the primary domain.</span></span> <span data-ttu-id="a57b7-128">Tous les autres objets **trustedDomain** sont ignorés.</span><span class="sxs-lookup"><span data-stu-id="a57b7-128">Any other **TrustedDomain** objects are ignored.</span></span>
-   <span data-ttu-id="a57b7-129">Sur les contrôleurs de domaine, le champ **AccountDomain** identifie le domaine de compte local pour le système. Toutefois, le nom du compte est attribué à l’utilisateur plutôt qu’à un nom connu.</span><span class="sxs-lookup"><span data-stu-id="a57b7-129">On domain controllers, the **AccountDomain** field identifies the local account domain for the system; however, the account name is user assigned rather than being a well-known name.</span></span> <span data-ttu-id="a57b7-130">Étant donné que le domaine principal est le même que le domaine du compte, le champ **PrimaryDomain** doit contenir la même valeur que le champ **AccountDomain** .</span><span class="sxs-lookup"><span data-stu-id="a57b7-130">Since the primary domain is the same as the account domain, the **PrimaryDomain** field must contain the same value as the **AccountDomain** field.</span></span> <span data-ttu-id="a57b7-131">En outre, tous les objets [**trustedDomain**](trusteddomain-object.md) sont supposés être valides et représentent des relations d’approbation avec d’autres domaines.</span><span class="sxs-lookup"><span data-stu-id="a57b7-131">Furthermore, all [**TrustedDomain**](trusteddomain-object.md) objects are expected to be valid and represent trust relationships with other domains.</span></span> <span data-ttu-id="a57b7-132">Si le système n’approuve aucun autre domaine, il ne doit pas y avoir d’objets **trustedDomain** .</span><span class="sxs-lookup"><span data-stu-id="a57b7-132">If the system does not trust any other domains, there should not be any **TrustedDomain** objects.</span></span>
