<span data-ttu-id="e3663-105">Bien que les genres de propriétés d’exécution offertes par Automation et les contrôles ActiveX Microsoft soient importants, ils ne répondent pas directement à la nécessité de stocker des informations avec des objets stockés de manière permanente dans le système de fichiers.</span><span class="sxs-lookup"><span data-stu-id="e3663-105">While the kinds of run-time properties that Automation and Microsoft ActiveX Controls offer are important, they do not directly address the need to store information with objects persistently stored in the file system.</span></span> <span data-ttu-id="e3663-106">Ces entités peuvent inclure des fichiers (structurés, composés, etc.), des répertoires et des catalogues de résumé.</span><span class="sxs-lookup"><span data-stu-id="e3663-106">These entities could include files (structured, compound, and so forth), directories, and summary catalogs.</span></span> <span data-ttu-id="e3663-107">COM fournit un format sérialisé standard pour ces propriétés persistantes, ainsi qu’un ensemble d’interfaces et de fonctions qui vous permettent de créer et de manipuler les jeux de propriétés et leurs propriétés.</span><span class="sxs-lookup"><span data-stu-id="e3663-107">COM provides both a standard serialized format for these persistent properties, and a set of interfaces and functions that allow you to create and manipulate the property sets and their properties.</span></span>

<span data-ttu-id="e3663-122">L’interface [**IPropertySetStorage**](/windows/desktop/api/Propidl/nn-propidl-ipropertysetstorage) définit les méthodes qui créent et gèrent les jeux de
