# <a name="how-to-design-a-domain-shader"></a><span data-ttu-id="131cb-103">Comment : concevoir un nuanceur de domaine</span><span class="sxs-lookup"><span data-stu-id="131cb-103">How To: Design a Domain Shader</span></span>

<span data-ttu-id="131cb-104">Un nuanceur de domaine est le troisième des trois étapes qui fonctionnent ensemble pour implémenter le [pavage](direct3d-11-advanced-stages-tessellation.md).</span><span class="sxs-lookup"><span data-stu-id="131cb-104">A domain shader is the third of three stages that work together to implement [tessellation](direct3d-11-advanced-stages-tessellation.md).</span></span> <span data-ttu-id="131cb-105">Le nuanceur de domaine génère la surface Geometry à partir des points de contrôle transformés à partir d’un nuanceur de coque et des coordonnées UV.</span><span class="sxs-lookup"><span data-stu-id="131cb-105">The domain shader generates the surface geometry from the transformed control points from a hull shader and the UV coordinates.</span></span> <span data-ttu-id="131cb-106">Cette rubrique montre comment concevoir un nuanceur de domaine.</span><span class="sxs-lookup"><span data-stu-id="131cb-106">This topics shows how to design a domain shader.</span></span>

<span data-ttu-id="131cb-107">Un nuanceur de domaine est appelé une fois pour chaque point généré par la fonction fixe du paveur.</span><span class="sxs-lookup"><span data-stu-id="131cb-107">A domain shader is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="131cb-108">Les entrées sont les \[ coordonnées UV W \] du point sur le correctif, ainsi que toutes les données de sortie du nuanceur de coque, y compris les points de contrôle et les constantes de patch.</span><span class="sxs-lookup"><span data-stu-id="131cb-108">The inputs are the UV\[W\] coordinates of the point on the patch, as well as all of the output data from the hull shader including control points and patch constants.</span></span> <span data-ttu-id="131cb-109">La sortie est un vertex défini d’une manière ou d’une autre.</span><span class="sxs-lookup"><span data-stu-id="131cb-109">The output is a vertex defined in whatever way is desired.</span></span> <span data-ttu-id="131cb-110">Si la sortie est envoyée au nuanceur de pixels, la sortie doit inclure une position (indiquée par une sémantique de position de SV \_ ).</span><span class="sxs-lookup"><span data-stu-id="131cb-110">If the output is being sent to the pixel shader, the output must include a position (denoted with a SV\_Position semantic).</span></span>

<span data-ttu-id="131cb-111">**Pour concevoir un nuanceur de domaine**</span><span class="sxs-lookup"><span data-stu-id="131cb-111">**To design a domain shader**</span></span>

1.  <span data-ttu-id="131cb-112">Définissez l’attribut de domaine.</span><span class="sxs-lookup"><span data-stu-id="131cb-112">Define the domain attribute.</span></span>

    ```
    [domain("quad")]
    ```

    

    <span data-ttu-id="131cb-113">Le domaine est défini pour un correctif Quad.</span><span class="sxs-lookup"><span data-stu-id="131cb-113">The domain is defined for a quad patch.</span></span>

2.  <span data-ttu-id="131cb-114">Déclarez l’emplacement sur la coque avec la valeur système d' [emplacement du domaine](/windows/desktop/direct3dhlsl/sv-domainlocation) .</span><span class="sxs-lookup"><span data-stu-id="131cb-114">Declare the location on the hull with the [domain location](/windows/desktop/direct3dhlsl/sv-domainlocation) system value.</span></span>

    -   <span data-ttu-id="131cb-115">Pour un patch à quatre cœurs, utilisez un float2.</span><span class="sxs-lookup"><span data-stu-id="131cb-115">For a quad patch, use a float2.</span></span>
    -   <span data-ttu-id="131cb-116">Pour un correctif triple, utilisez un float3 (pour les coordonnées Barycentric)</span><span class="sxs-lookup"><span data-stu-id="131cb-116">For a tri patch, use a float3 (for barycentric coordinates)</span></span>
    -   <span data-ttu-id="131cb-117">Pour une isoligne, utilisez un float2.</span><span class="sxs-lookup"><span data-stu-id="131cb-117">For an isoline, use a float2.</span></span>

    <span data-ttu-id="131cb-118">Par conséquent, l’emplacement du domaine pour un correctif Quad ressemble à ceci :</span><span class="sxs-lookup"><span data-stu-id="131cb-118">Therefore, the domain location for a quad patch looks like this:</span></span>

    ```
    float2 UV : SV_DomainLocation
    ```

    

3.  <span data-ttu-id="131cb-119">Définissez les autres entrées.</span><span class="sxs-lookup"><span data-stu-id="131cb-119">Define the other inputs.</span></span>

    <span data-ttu-id="131cb-120">Les autres entrées proviennent du nuanceur de coque et sont définies par l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="131cb-120">The other inputs come from the hull shader and are user defined.</span></span> <span data-ttu-id="131cb-121">Cela inclut les points de contrôle d’entrée pour le correctif, dont il peut exister entre 1 et 32 points, et les données de constante de correction d’entrée.</span><span class="sxs-lookup"><span data-stu-id="131cb-121">This includes the input control points for patch, of which there can be between 1 and 32 points, and input patch constant data.</span></span>

    <span data-ttu-id="131cb-122">Les points de contrôle sont définis par l’utilisateur, généralement avec une structure telle que celle-ci (définie dans [Comment : concevoir un nuanceur de coque](direct3d-11-advanced-stages-hull-shader-design.md)) :</span><span class="sxs-lookup"><span data-stu-id="131cb-122">The control points are user defined, usually with a structure such as this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch
    ```

    

    <span data-ttu-id="131cb-123">Les données de constante de patch sont également définies par l’utilisateur et peuvent ressembler à celle-ci (définie dans [Comment : concevoir un nuanceur de coque](direct3d-11-advanced-stages-hull-shader-design.md)) :</span><span class="sxs-lookup"><span data-stu-id="131cb-123">The patch constant data is also user defined, and might look like this one (defined in [How To: Design a Hull Shader](direct3d-11-advanced-stages-hull-shader-design.md)):</span></span>

    ```
    HS_CONSTANT_DATA_OUTPUT input
    ```

    

4.  <span data-ttu-id="131cb-124">Ajoutez du code défini par l’utilisateur pour calculer les sorties. Cela constitue le corps du nuanceur de domaine.</span><span class="sxs-lookup"><span data-stu-id="131cb-124">Add user-defined code to compute the outputs; this makes up the body of the domain shader.</span></span>

    <span data-ttu-id="131cb-125">Cette structure contient des sorties de nuanceur de domaine définies par l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="131cb-125">This structure contains user-defined domain shader outputs.</span></span>

    ```
    struct DS_OUTPUT
    {
        float3 vNormal    : NORMAL;
        float2 vUV        : TEXCOORD;
        float3 vTangent   : TANGENT;
        float3 vBiTangent : BITANGENT;
        
        float4 vPosition  : SV_POSITION;
    };
    ```

    

    <span data-ttu-id="131cb-126">La fonction prend chaque UV d’entrée (à partir du du paveur) et évalue le correctif de Bézier à cette position.</span><span class="sxs-lookup"><span data-stu-id="131cb-126">The function takes each input UV (from the tessellator) and evaluates the Bezier patch at this position.</span></span>

    ```
    [domain("quad")]
    DS_OUTPUT BezierEvalDS( HS_CONSTANT_DATA_OUTPUT input, 
                            float2 UV : SV_DomainLocation,
                            const OutputPatch<BEZIER_CONTROL_POINT, 16> bezpatch )
    {
        DS_OUTPUT Output;

        // Insert code to compute the output here.

        return Output;    
    }
    ```

    

    <span data-ttu-id="131cb-127">La fonction est appelée une fois pour chaque point généré par la fonction Fixed du paveur.</span><span class="sxs-lookup"><span data-stu-id="131cb-127">The function is invoked once for each point generated by the fixed function tessellator.</span></span> <span data-ttu-id="131cb-128">Dans la mesure où cet exemple utilise un correctif Quad, l’emplacement du domaine d’entrée ([SV \_ DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) est un float2 (UV); un correctif triple aura un emplacement d’entrée float3 (coordonnées Barycentric UVW) et une isoligne aura un emplacement de domaine d’entrée float2.</span><span class="sxs-lookup"><span data-stu-id="131cb-128">Since this example uses a quad patch, the input domain location ([SV\_DomainLocation](/windows/desktop/direct3dhlsl/sv-domainlocation)) is a float2 (UV); a tri patch would have a float3 input location (UVW barycentric coordinates), and an isoline would have a float2 input domain location.</span></span>

    <span data-ttu-id="131cb-129">Les autres entrées de la fonction proviennent directement du nuanceur de coque.</span><span class="sxs-lookup"><span data-stu-id="131cb-129">The other inputs for the function come from the hull shader directly.</span></span> <span data-ttu-id="131cb-130">Dans cet exemple, il s’agit de 16 points de contrôle, chacun étant un **\_ \_ point de contrôle Bézier**, ainsi que des données constantes de correctifs (**sortie de \_ \_ données \_ de constante HS**).</span><span class="sxs-lookup"><span data-stu-id="131cb-130">In this example it is 16 control points each one being a **BEZIER\_CONTROL\_POINT**, as well as patch constant data (**HS\_CONSTANT\_DATA\_OUTPUT**).</span></span> <span data-ttu-id="131cb-131">La sortie est un vertex contenant toutes les données de **\_ sortie** souhaitées-DS dans cet exemple.</span><span class="sxs-lookup"><span data-stu-id="131cb-131">The output is a vertex containing any data desired - **DS\_OUTPUT** in this example.</span></span>
