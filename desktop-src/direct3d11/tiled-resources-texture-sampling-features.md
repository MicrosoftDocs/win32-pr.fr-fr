<span data-ttu-id="9c161-134">Le matériel exécute [**Max**](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-max)(FShaderMinLODClamp, fComputedLOD) à la même place dans le calcul LOD dans lequel la bride MinLOD par ressource se produit, ce qui est également un **nombre max**().</span><span class="sxs-lookup"><span data-stu-id="9c161-134">The hardware performs [**max**](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-max)(fShaderMinLODClamp,fComputedLOD) in the same place in the LOD calculation where the per-resource MinLOD clamp occurs, which is also a **max**().</span></span>

<span data-ttu-id="9c161-135">Si le résultat de l’application de la bride LOD par échantillon et de tout autre clamp LOD défini dans l’échantillonneur est un jeu vide, le résultat est le même que le résultat de l’accès hors limites en tant que bride minLOD par ressource : 0 pour les composants dans le format de surface et les valeurs par défaut pour les composants manquants.</span><span class="sxs-lookup"><span data-stu-id="9c161-135">If the result of applying the per-sample LOD clamp and any other LOD clamps defined in the sampler is an empty set, the result is the same out of bounds access result as the per-resource minLOD clamp: 0 for components in the surface format and defaults for missing components.</span></span>

## <a name="minmax-reduction-filtering"></a><span data-ttu-id="9c161-139">Filtrage de réduction min/max</span><span class="sxs-lookup"><span data-stu-id="9c161-139">Min/Max reduction filtering</span></span>
