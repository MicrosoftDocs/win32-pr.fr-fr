<span data-ttu-id="d5470-108">Lorsqu’un pool de mosaïques est cultivé, des vignettes supplémentaires sont ajoutées à la fin via une ou plusieurs nouvelles allocations par le pilote d’affichage.</span><span class="sxs-lookup"><span data-stu-id="d5470-108">When a tile pool is grown, additional tiles are added to the end via one or more new allocations by the display driver.</span></span> <span data-ttu-id="d5470-109">Cette répartition des allocations n’est pas visible pour l’application.</span><span class="sxs-lookup"><span data-stu-id="d5470-109">This breakdown into allocations isn't visible to the application.</span></span> <span data-ttu-id="d5470-110">La mémoire existante dans le pool de mosaïques n’est pas touchée et les mappages de ressources en mosaïque existants dans cette mémoire restent intacts.</span><span class="sxs-lookup"><span data-stu-id="d5470-110">Existing memory in the tile pool is left untouched, and existing tiled resource mappings into that memory remain intact.</span></span>

<span data-ttu-id="d5470-111">Quand un pool de mosaïques est réduit, les vignettes sont supprimées de la fin.</span><span class="sxs-lookup"><span data-stu-id="d5470-111">When a tile pool is shrunk, tiles are removed from the end.</span></span> <span data-ttu-id="d5470-112">Les vignettes sont supprimées, même en dessous de la taille d’allocation initiale, jusqu’à 0, ce qui signifie que les <span data-ttu-id="d5470-116">Pour pouvoir économiser de la mémoire, une application doit non seulement réduire un pool de mosaïques, mais également supprimer/remapper des mappages existants au-delà de la fin de la nouvelle taille de pool de mosaïques plus petite.</span><span class="sxs-lookup"><span data-stu-id="d5470-116">To be able to save memory, an application has to not only shrink a tile pool but also remove/remap existing mappings past the end of the new smaller tile pool size.</span></span>
