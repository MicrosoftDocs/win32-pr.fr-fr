# <a name="tier-2"></a><span data-ttu-id="16eaf-103">Niveau 2</span><span class="sxs-lookup"><span data-stu-id="16eaf-103">Tier 2</span></span>

<span data-ttu-id="16eaf-104">Cette section décrit le support de niveau 2.</span><span class="sxs-lookup"><span data-stu-id="16eaf-104">This section describes tier 2 support.</span></span>

-   <span data-ttu-id="16eaf-105">Matériel au niveau de la fonctionnalité 11,1 minimum.</span><span class="sxs-lookup"><span data-stu-id="16eaf-105">Hardware at Feature Level 11.1 minimum.</span></span>
-   <span data-ttu-id="16eaf-106">Toutes les fonctionnalités du niveau précédent (sans limitations spécifiques au [niveau 1](tier-1.md) ), ainsi que les ajouts apportés aux éléments suivants :</span><span class="sxs-lookup"><span data-stu-id="16eaf-106">All features of the previous tier (without [Tier 1](tier-1.md) specific limitations) plus the additions in these following items:</span></span>
-   <span data-ttu-id="16eaf-107">Les instructions du nuanceur pour le déverrouillage du LOD et les commentaires d’État mappés sont disponibles.</span><span class="sxs-lookup"><span data-stu-id="16eaf-107">Shader instructions for clamping LOD and mapped status feedback are available.</span></span> <span data-ttu-id="16eaf-108">Pour plus d’informations, consultez [exposition des ressources en mosaïque HLSL](hlsl-tiled-resources-exposure.md).</span><span class="sxs-lookup"><span data-stu-id="16eaf-108">For more info, see [HLSL tiled resources exposure](hlsl-tiled-resources-exposure.md).</span></span>
-   <span data-ttu-id="16eaf-109">Les lectures à partir de vignettes non mappées retournent 0 dans tous les composants non manquants du format, et la valeur par défaut pour les composants manquants.</span><span class="sxs-lookup"><span data-stu-id="16eaf-109">Reads from non-mapped tiles return 0 in all non-missing components of the format, and the default for missing components.</span></span>
-   <span data-ttu-id="16eaf-110">Les écritures dans les vignettes non mappées sont arrêtées du passage à la mémoire, mais elles peuvent se trouver dans les caches que les lectures ultérieures vers la même adresse peuvent ou non être levées.</span><span class="sxs-lookup"><span data-stu-id="16eaf-110">Writes to non-mapped tiles are stopped from going to memory but might end up in caches that subsequent reads to the same address might or might not pick up.</span></span>
-   <span data-ttu-id="16eaf-111">Le filtrage de texture avec un encombrement qui chevauche les vignettes **null** et non **null** contribue à 0 (avec des valeurs par défaut pour les composants de format manquants) pour les texels sur les vignettes **null** dans l’opération de filtre globale.</span><span class="sxs-lookup"><span data-stu-id="16eaf-111">Texture filtering with a footprint that straddles **NULL** and non-**NULL** tiles contributes 0 (with defaults for missing format components) for texels on **NULL** tiles into the overall filter operation.</span></span> <span data-ttu-id="16eaf-112">Un matériel précoce ne répond pas à cette exigence et retourne 0 (avec des valeurs par défaut pour les composants de format manquants) pour le résultat de filtre complet si des texels (avec une pondération différente de zéro) tombent sur une vignette **null** .</span><span class="sxs-lookup"><span data-stu-id="16eaf-112">Some early hardware don't meet this requirement and returns 0 (with defaults for missing format components) for the full filter result if any texels (with nonzero weight) fall on a **NULL** tile.</span></span> <span data-ttu-id="16eaf-113">Aucun autre matériel ne sera autorisé à manquer l’obligation d’inclure tous les texels (avec une pondération différente de zéro) dans l’opération de filtre.</span><span class="sxs-lookup"><span data-stu-id="16eaf-113">No other hardware will be allowed to miss the requirement to include all (nonzero weighted) texels in the filter operation.</span></span>
-   <span data-ttu-id="16eaf-114">Les accès texels **null** entraînent l’opération [**CheckAccessFullyMapped**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) sur les commentaires d’État pour qu’une texture lue retourne la valeur false.</span><span class="sxs-lookup"><span data-stu-id="16eaf-114">**NULL** texel accesses cause the [**CheckAccessFullyMapped**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) operation on the status feedback for a texture read to return false.</span></span> <span data-ttu-id="16eaf-115">C’est quelle que soit la façon dont le résultat de l’accès à la texture peut obtenir un masque d’écriture dans le nuanceur et le nombre de composants dans le format de texture (la combinaison peut faire apparaître que la texture n’a pas besoin d’être accessible).</span><span class="sxs-lookup"><span data-stu-id="16eaf-115">This is regardless of how the texture access result might get write masked in the shader and how many components are in the texture format (the combination of which might make it appear that the texture does not need to be accessed).</span></span>
-   <span data-ttu-id="16eaf-116">Contraintes d’alignement pour les formes de mosaïques standard : les des mipmaps qui remplissent au moins une vignette standard dans toutes les dimensions sont assurés d’utiliser la mosaïque standard, le reste étant considéré comme une **unité** dans n vignettes (n signalées à l’application).</span><span class="sxs-lookup"><span data-stu-id="16eaf-116">Alignment constraints for standard tile shapes: Mipmaps that fill at least one standard tile in all dimensions are guaranteed to use the standard tiling, with the remainder considered packed as a **unit** into N tiles (N reported to the application).</span></span> <span data-ttu-id="16eaf-117">L’application peut mapper les N vignettes dans des emplacements arbitrairement disjoints dans un pool de mosaïques, mais doit mapper tout ou aucune des vignettes compressées.</span><span class="sxs-lookup"><span data-stu-id="16eaf-117">The application can map the N tiles into arbitrarily disjoint locations in a tile pool, but must either map all or none of the packed tiles.</span></span> <span data-ttu-id="16eaf-118">La compression MIP est un ensemble unique de vignettes compressées par tranche de tableau.</span><span class="sxs-lookup"><span data-stu-id="16eaf-118">The mip packing is a unique set of packed tiles per array slice.</span></span>
-   <span data-ttu-id="16eaf-119">Le filtrage de réduction min/max est pris en charge.</span><span class="sxs-lookup"><span data-stu-id="16eaf-119">Min/Max reduction filtering is supported.</span></span> <span data-ttu-id="16eaf-120">Pour plus d’informations sur le filtrage de réduction min/max, consultez [fonctionnalités d’échantillonnage de texture des ressources en mosaïque](tiled-resources-texture-sampling-features.md).</span><span class="sxs-lookup"><span data-stu-id="16eaf-120">For info about Min/Max reduction filtering, see [Tiled resources texture sampling features](tiled-resources-texture-sampling-features.md).</span></span>
-   <span data-ttu-id="16eaf-121">Les ressources en mosaïque avec un des mipmaps inférieur à la taille de mosaïque standard dans une dimension ne sont pas autorisées à avoir une taille de tableau supérieure à 1.</span><span class="sxs-lookup"><span data-stu-id="16eaf-121">Tiled resources with any mipmaps less than standard tile size in any dimension are not allowed to have an array size larger than 1.</span></span>
-   <span data-ttu-id="16eaf-122">Limitations sur la façon dont les vignettes sont accessibles lorsqu’il existe des mappages en double, décrits dans [limitations d’accès aux vignettes avec des mappages dupliqués](tile-access-limitations-with-duplicate-mappings-.md), continuez à appliquer.</span><span class="sxs-lookup"><span data-stu-id="16eaf-122">Limitations on how tiles can be accessed when there are duplicate mappings, described in [Tile access limitations with duplicate mappings](tile-access-limitations-with-duplicate-mappings-.md), continue to apply.</span></span>
