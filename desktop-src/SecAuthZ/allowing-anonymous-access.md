# <a name="allowing-anonymous-access"></a>Autorisation de l’accès anonyme

La stratégie de sécurité par défaut restreint l’accès local anonyme à aucun droit. Les administrateurs peuvent ensuite ajouter ou soustraire des droits tels qu’ils s’affichent.

Un groupe d’accès local existe pour les applications avec les mêmes droits d’accès que tout le monde. les administrateurs peuvent ensuite augmenter ou diminuer de manière appropriée le nombre d’utilisateurs de ce groupe, appelé le **groupe accès Compatible pré-Windows 2000**.

Pour plus d’informations, consultez les pages de référence pour les fonctions listées dans le tableau de la rubrique relative aux [fonctions de groupe local](/windows/desktop/NetMgmt/local-group-functions) .
