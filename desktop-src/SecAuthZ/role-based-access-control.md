# <a name="role-based-access-control"></a>Contrôle d’accès en fonction du rôle

L’API du gestionnaire d’autorisations et le composant logiciel enfichable MMC fournissent aux applications un Framework de contrôle d’accès basé sur les rôles. L’API du gestionnaire d’autorisations, également appelée AzMan, fournit un modèle de développement simplifié dans lequel gérer des groupes et des règles d’entreprise flexibles et stocker des stratégies d’autorisation. Pour plus d'informations, voir les rubriques suivantes :

-   [Avantages de l’autorisation basée sur les rôles](advantages-of-role-based-authorization.md)
-   [Modèle du gestionnaire d’autorisations](authorization-manager-model.md)

**Windows XP :** l’API du gestionnaire d’autorisations et le composant logiciel enfichable MMC sont disponibles en téléchargement (Pack des outils d’Administration de Windows Server 2003) à partir de [https://www.microsoft.com/downloads](https://www.microsoft.com/Downloads/) . Ce téléchargement prend en charge l’administration et le développement de stratégies du gestionnaire d’autorisations. il ne prend pas en charge le déploiement d’applications qui utilisent le gestionnaire d’autorisations pour contrôler l’accès.
