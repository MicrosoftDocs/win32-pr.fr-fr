<span data-ttu-id="85456-108">Avec la distribution au niveau du transport, le serveur HTTP sous-jacent (par exemple, l' [API http](/windows/desktop/Http/http-api-start-page)) est utilisé pour gérer le routage des demandes vers l’appareil et ses services.</span><span class="sxs-lookup"><span data-stu-id="85456-108">With transport level dispatch, the underlying HTTP server (such as the [HTTP API](/windows/desktop/Http/http-api-start-page)) is used to manage routing of requests to the device and its services.</span></span> <span data-ttu-id="85456-109">Le serveur fournit une URL différente pour chaque service et pour l’appareil, et des récepteurs différents sont enregistrés pour chaque URL.</span><span class="sxs-lookup"><span data-stu-id="85456-109">The server provides a different URL for each service and for the device and different sinks are registered for each URL.</span></span> <span data-ttu-id="85456-110">Cela permet à du code d’être conçu de telle sorte que chaque service soit isolé de l’autre, s’exécutant en tant que composants distincts au sein du même processus ou exécuté en tant que processus distincts.</span><span class="sxs-lookup"><span data-stu-id="85456-110">This allows code to be designed such that each service is isolated from the other, either running as separate components within the same process or running as separate processes.</span></span>
