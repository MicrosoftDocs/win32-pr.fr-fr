# <a name="wsd-device-development"></a>Développement de périphérique WSD

Les fabricants de périphériques et les autres implémenteurs d’appareils peuvent créer des appareils qui interagissent avec les clients et hôtes WSDAPI basés sur Windows sans utiliser WSDAPI.

Outre le développement de la fonctionnalité de l’appareil, les responsables de l’implémentation des appareils doivent implémenter le [profil des appareils pour les services Web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS).

Dans la mesure du possible, utilisez WSDAPI pour créer un client WSD Windows ou une application hôte d’appareil. L’utilisation de WSDAPI réduit la complexité du développement et économise du temps. Pour plus d’informations, consultez [développement d’applications WSD sur Windows](wsd-application-development-on-windows.md).
