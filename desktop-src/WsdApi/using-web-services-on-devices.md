# <a name="using-web-services-on-devices"></a>Utilisation de services Web sur des appareils

L’API Microsoft Web Services on Devices (WSDAPI) prend en charge l’implémentation d’appareils et de services contrôlés par le client, ainsi que les hôtes d’appareils conformes au [profil des appareils pour les services Web](https://specs.xmlsoap.org/ws/2006/02/devprof/) (DPWS). WSDAPI peut être utilisé pour le développement des implémentations du client et du serveur (périphérique).

Dans la mesure du possible, utilisez WSDAPI pour créer un client WSD Windows ou une application hôte d’appareil. L’utilisation de WSDAPI réduit la complexité du développement et économise du temps. Pour plus d’informations, consultez [développement d’applications WSD sur](wsd-application-development-on-windows.md) les outils de développement Windows et [wsdapi](wsdapi-development-tools.md).

Vous pouvez également créer des appareils WSD qui n’exécutent pas Windows et qui sont interopérables avec les clients WSD basés sur Windows et les hôtes d’appareils. Ces appareils doivent répondre à des exigences spécifiques pour maintenir l’interopérabilité. Pour plus d’informations, consultez [développement d’appareils WSD](wsd-device-development.md).
