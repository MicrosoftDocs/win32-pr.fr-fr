# <a name="preferredprefix-element"></a>élément preferredPrefix

Définit le préfixe auquel l’espace de noms doit être mappé pour rendre le XML plus lisible.

## <a name="usage"></a>Utilisation

``` syntax
<preferredPrefix/>
```

## <a name="attributes"></a>Attributs

Il n’y a pas d’attributs.

## <a name="child-elements"></a>Éléments enfants

Il n’y a pas d’éléments enfants.

## <a name="parent-elements"></a>Éléments parents



| Élément                                   | Description                                                        |
|-------------------------------------------|--------------------------------------------------------------------|
| [**Joint**](namespace.md)<br/> | Espace de noms à utiliser pour la génération de code.<br/> <br/> |



## <a name="remarks"></a>Notes

Cet élément remplace le préfixe d’URI par défaut utilisé pour le code généré. Par exemple, un espace de noms associé à un support peut avoir le préfixe « AV » préféré (pour l’audio/l’audiovisuel).

Par défaut, le code généré crée un préfixe préféré à partir de l’URI.
