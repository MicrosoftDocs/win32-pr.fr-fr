# <a name="rfcs-relevant-to-winsnmp"></a>RFC relatives à WinSNMP

Les normes TCP/IP sont définies dans les RFC (Request for Comments), qui sont publiées par l’IETF (Internet Engineering Task Force). Les RFC qui s’appliquent aux fonctionnalités de WinSNMP sont répertoriées dans le tableau suivant.



| Numéro RFC | Intitulé                                                                                                  |
|------------|--------------------------------------------------------------------------------------------------------|
| 1155       | « Structure et identification des informations de gestion pour les Internet TCP/IP »                    |
| 1157       | « Un protocole SNMP (simple Network Management Protocol) »                                                          |
| 1213       | « Base d’informations de gestion pour la gestion de réseau des réseaux Internet TCP/IP : MIB-II »                 |
| 1901       | « Présentation de SNMPv2 basé sur la communauté »                                                               |
| 1902       | « Structure des informations de gestion pour la version 2 du protocole SNMP (simple Network Management Protocol) » |
| 1903       | « Conventions textuelles pour la version 2 du protocole SNMP (simple Network Management Protocol) »                 |
| 1904       | « Instructions de conformité pour la version 2 du protocole SNMP (simple Network Management Protocol) »              |
| 1905       | « Opérations de protocole pour la version 2 du protocole SNMP (simple Network Management Protocol) »                 |
| 1906       | « Mappages de transport pour la version 2 du protocole SNMP (simple Network Management Protocol) »                  |
| 1907       | « Base d’informations de gestion pour la version 2 du protocole SNMP (simple Network Management Protocol) »         |
| 1908       | « Coexistence entre la version 1 et la version 2 de l’infrastructure de gestion de réseau Internet standard »    |
| 2089       | « V2ToV1 mappage de SNMPv2 sur SNMPv1 au sein d’un agent SNMP bidirectionnel »                                     |
