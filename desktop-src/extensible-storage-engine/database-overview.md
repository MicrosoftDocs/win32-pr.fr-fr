## <a name="database-overview"></a><span data-ttu-id="3eddd-105">Présentation de la base de données</span><span class="sxs-lookup"><span data-stu-id="3eddd-105">Database Overview</span></span>

<span data-ttu-id="3eddd-106">La base de données ESE est une méthode d’accès séquentiel indexé (ISAM) pour le stockage et la récupération des données.</span><span class="sxs-lookup"><span data-stu-id="3eddd-106">The ESE database is an indexed sequential access method (ISAM) for storing and retrieving data.</span></span> <span data-ttu-id="3eddd-107">Une base de données ESE est stockée dans un fichier unique et se compose d’une ou plusieurs tables définies par l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="3eddd-107">An ESE database is stored in a single file and consists of one or more user-defined tables.</span></span> <span data-ttu-id="3eddd-108">Les données sont organisées dans des enregistrements de la table avec une ou plusieurs colonnes définies par l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="3eddd-108">Data is organized in records in the table with one or more user-defined columns.</span></span> <span data-ttu-id="3eddd-109">Les index créés fournissent une organisation différente pour l’ensemble de l’ensemble ou un sous-ensemble d’enregistrements dans la table.</span><span class="sxs-lookup"><span data-stu-id="3eddd-109">Indexes that are created provide different organization for the entire set or a subset of records in the table.</span></span> <span data-ttu-id="3eddd-110">À l’aide de l’API ESE, les applications peuvent créer des curseurs qui parcourent les enregistrements de la base de données dans différents ordres séquentiels.</span><span class="sxs-lookup"><span data-stu-id="3eddd-110">Using the ESE API, applications can create cursors that navigate records in the database in different sequential orders.</span></span> <span data-ttu-id="3eddd-111">Les éléments de la table sont définis ci-dessous :</span><span class="sxs-lookup"><span data-stu-id="3eddd-111">The elements of the table are defined below:</span></span>

  - <span data-ttu-id="3eddd-112">**Colonne**: la colonne est un champ de la table qui stocke un type spécifique d’informations.</span><span class="sxs-lookup"><span data-stu-id="3eddd-112">**Column**: The column is a field in the table that stores a specific type of information.</span></span> <span data-ttu-id="3eddd-113">Les colonnes peuvent être fixes ou de longueur variable, en fonction du type de données qui y est stocké.</span><span class="sxs-lookup"><span data-stu-id="3eddd-113">Columns can be fixed, or variable length, depending on the data type stored in them.</span></span> <span data-ttu-id="3eddd-114">Certaines colonnes, telles que les colonnes avec balises, ne prennent pas d’espace quand **null** ou sont définies sur la valeur par défaut et peuvent contenir plusieurs valeurs.</span><span class="sxs-lookup"><span data-stu-id="3eddd-114">Some columns, such as tagged columns take no space when **NULL** or set to the default value, and can contain multiple values.</span></span>

  - <span data-ttu-id="3eddd-115">**Enregistrements**: un enregistrement est une collection de valeurs de colonnes qui ont une identité unique comme défini par la clé primaire.</span><span class="sxs-lookup"><span data-stu-id="3eddd-115">**Records**: A record is a collection of columns values that have a unique identity as defined by the primary key.</span></span>

  - <span data-ttu-id="3eddd-116">**Index**: l’index est une collection de colonnes clés qui définissent un classement stocké des enregistrements dans la table.</span><span class="sxs-lookup"><span data-stu-id="3eddd-116">**Index**: The index is a collection of key columns that define a stored ordering of records in the table.</span></span> <span data-ttu-id="3eddd-117">L’index cluster, ou principal, définit l’ordre dans lequel les enregistrements sont stockés dans la table.</span><span class="sxs-lookup"><span data-stu-id="3eddd-117">The clustered, or primary, index defines the order in which the records are stored within the table.</span></span> <span data-ttu-id="3eddd-118">Vous pouvez définir plusieurs index afin de spécifier des ordres différents de traversée dans les enregistrements de la table.</span><span class="sxs-lookup"><span data-stu-id="3eddd-118">Multiple indices can be defined in order to specify different orderings of traversal through records in the table.</span></span> <span data-ttu-id="3eddd-119">Un index peut également limiter l’ensemble des enregistrements visibles en fonction de critères simples tels que la présence ou l’absence d’une valeur de colonne clé particulière dans l’enregistrement.</span><span class="sxs-lookup"><span data-stu-id="3eddd-119">An index may also limit the set of records visible based on simple criteria such as the presence or absence of a particular key column value in the record.</span></span>

  - <span data-ttu-id="3eddd-120">**Curseur**: le curseur indique l’enregistrement en cours dans la table et navigue vers les enregistrements de la table à l’aide de l’index actuel.</span><span class="sxs-lookup"><span data-stu-id="3eddd-120">**Cursor**: The cursor indicates the current record in the table and navigates to records in the table using the current index.</span></span> <span data-ttu-id="3eddd-121">Le curseur contient également des informations sur l’état de la mise à jour actuellement préparée.</span><span class="sxs-lookup"><span data-stu-id="3eddd-121">The cursor also contains information on the state of the currently prepared update.</span></span>

<span data-ttu-id="3eddd-122">Les colonnes et les index peuvent être ajoutés ou supprimés de la table à tout moment.</span><span class="sxs-lookup"><span data-stu-id="3eddd-122">Columns and indices may be added-to or removed-from the table at any time.</span></span> <span data-ttu-id="3eddd-123">Bien qu’il soit possible de définir plusieurs index, les données de la table sont physiquement stockées et ordonnées de manière logique en fonction de la définition d’index primaire dans une arborescence B +.</span><span class="sxs-lookup"><span data-stu-id="3eddd-123">Although multiple indices may be defined, the data in the table is physically stored and logically clustered according to the primary index definition in a B+ tree.</span></span> <span data-ttu-id="3eddd-124">Chaque index secondaire est stocké dans une arborescence B + distincte qui contient uniquement des pointeurs logiques vers les données réelles stockées dans la table primaire.</span><span class="sxs-lookup"><span data-stu-id="3eddd-124">Each secondary index is stored in a separate B+ tree that contains only logical pointers to the actual data that is stored in the primary table.</span></span> <span data-ttu-id="3eddd-125">Si aucun index n’est défini, les enregistrements de la table sont stockés dans une arborescence B + dans l’ordre d’insertion et sont désignés par le terme « index séquentiel ».</span><span class="sxs-lookup"><span data-stu-id="3eddd-125">If no index is defined, the records in the table are stored in a B+ tree in the order of insertion and are referred to as the sequential index.</span></span>
