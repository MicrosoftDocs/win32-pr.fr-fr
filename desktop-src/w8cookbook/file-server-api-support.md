# <a name="file-server-api-support"></a>Prise en charge des API du serveur de fichiers

## <a name="platform"></a>Plateforme

**serveurs** – Windows Server 2012 

## <a name="description"></a>Description

La documentation sur les API prises en charge pour le protocole Server Message Block 2,2 (SMB 2,2) et les systèmes de fichiers résilients/Cluster Shared volumes (ReFS/CSVFS) sera mise à la disposition des développeurs à la ressource listée ci-dessous. La documentation est divisée en plusieurs catégories :

-   Fonctions de gestion de fichiers
-   Fonctions de gestion d’annuaire
-   Fonctions de gestion des volumes
-   Fonctions de sécurité
-   Codes de prise en charge de fichiers et de répertoires
-   Codes de contrôle du volume
-   Fichiers mémoire mappée

## <a name="usage"></a>Usage

Certaines API ne sont pas prises en charge avec tous les protocoles ou dans tous les scénarios, tandis que d’autres API sont dépréciées dans Windows Server 2012. Passez en revue la liste des API prises en charge pour garantir la prise en charge continue de vos applications.
