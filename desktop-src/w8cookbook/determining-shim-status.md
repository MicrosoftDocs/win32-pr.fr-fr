# <a name="determining-shim-status"></a>Détermination de l’état des shims

## <a name="platforms"></a>Plateformes

<dl> Clients-Windows 8  
Serveurs-Windows Server 2012  
</dl>

## <a name="description"></a>Description

Windows 8 journalise des événements lorsque les applications sont corrigées pour différentes raisons. Le moyen le plus simple de déterminer si votre application est corrigée consiste à consulter l’observateur d’événements. accédez à journaux des Applications et des Services \\ Microsoft Windows Application-Experience la \\ télémétrie du programme.

## <a name="mitigation"></a>Limitation des risques

La meilleure façon de vous sortir de la correction consiste à renommer l’exécutable ou à augmenter la version principale de 1 (recompiler l’exécutable).
