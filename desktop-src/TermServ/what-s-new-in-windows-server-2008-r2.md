# <a name="whats-new-in-windows-server-2008-r2"></a>Nouveautés de Windows Server 2008 R2

Windows Server 2008 R2 introduit les nouveaux éléments de programmation suivants pour Services Bureau à distance.



<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<tbody>
<tr class="odd">
<td>Les services Terminal Server sont désormais Services Bureau à distance<br/></td>
<td>Les services Terminal Server ont été renommés Services Bureau à distance. Un serveur sur lequel est installé le service de rôle hôte de session Bureau à distance (hôte de session Bureau à distance) est maintenant appelé serveur hôte de session Bureau à distance, au lieu d’un serveur Terminal Server.<br/>
<ul>
<li><a href="terminal-services-is-now-remote-desktop-services.md">Les services Terminal Server sont désormais Services Bureau à distance</a></li>
</ul></td>
</tr>
<tr class="even">
<td>API de virtualisation Bureau à distance<br/></td>
<td>L’API de virtualisation Bureau à distance fournit des énumérations, des interfaces et des structures que vous pouvez utiliser pour créer des plug-ins personnalisés qui remplacent la logique de redirection standard de Connexion Bureau à distance Broker (Service Broker pour les connexions Bureau à distance). Le Service Broker pour les connexions Bureau à distance (anciennement session Broker TS) a été étendu pour prendre en charge les connexions aux ordinateurs virtuels.<br/>
<ul>
<li><a href="using-the-remote-desktop-virtualization-api.md">Utilisation de l’API de virtualisation Bureau à distance</a></li>
<li><a href="terminal-services-virtualization-api-reference.md">Informations de référence sur l’API de virtualisation Bureau à distance</a></li>
</ul></td>
</tr>
<tr class="odd">
<td>Fournisseur protocole RDP (Remote Desktop Protocol)<br/></td>
<td>L’API du fournisseur protocole RDP (Remote Desktop Protocol) peut être utilisée pour créer un protocole qui personnalise l’interaction entre le service Services Bureau à distance et les clients de bureau.<br/>
<ul>
<li><a href="creating-a-custom-remote-protocol.md">Création d’un fournisseur de protocole RDP (Remote Desktop Protocol)</a></li>
<li><a href="custom-remote-protocol-reference.md">Référence du fournisseur protocole RDP (Remote Desktop Protocol)</a></li>
</ul></td>
</tr>
<tr class="even">
<td>API Services Bureau à distance AudioEndpoint<br/></td>
<td>L’API Services Bureau à distance AudioEndpoint prend en charge l’énumération, les interfaces et les structures pour l’inscription du point de terminaison audio et le transport des données.<br/>
<ul>
<li><a href="terminal-services-audioendpoint-api-reference.md">Informations de référence sur l’API Services Bureau à distance AudioEndpoint</a></li>
</ul></td>
</tr>
<tr class="odd">
<td>API du service Administration des connexions aux programmes RemoteApp et aux services Bureau à distance<br/></td>
<td>L’API de service Administration des connexions aux programmes RemoteApp et aux services Bureau à distance prend en charge des interfaces et des structures que vous pouvez utiliser pour effectuer un filtrage personnalisé des ressources et fournir une prise en charge des types de fichiers qui ne sont pas pris en charge en mode natif dans Windows Server 2008 R2.<br/>
<ul>
<li><a href="centralized-publishing-api-reference.md">Informations de référence sur l’API de service Administration des connexions aux programmes RemoteApp et aux services Bureau à distance</a></li>
</ul></td>
</tr>
</tbody>
</table>
