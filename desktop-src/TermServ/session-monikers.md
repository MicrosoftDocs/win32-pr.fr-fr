# <a name="session-monikers"></a>Monikers de session

DCOM (Distributed COM) permet l’activation d’objets pour chaque session à l’aide d’un moniker de session fourni par le système. Pour plus d’informations sur la création et l’utilisation de monikers de session, y compris un exemple de code qui montre comment activer un processus serveur local, consultez les rubriques suivantes.

## <a name="in-this-section"></a>Dans cette section

<dl> <dt>

[Activation de session à session avec un moniker de session](session-to-session-activation-with-a-session-moniker.md)
</dt> <dd>

L’activation de session à session (également appelée activation inter-sessions) permet à un processus client de démarrer (activer) un processus serveur local sur une session spécifiée.

</dd> <dt>

[Utilisation d’un moniker de session](using-a-session-moniker.md)
</dt> <dd>

L’activation de session à session permet à un processus client d’activer un processus serveur local sur une session spécifiée.

</dd> </dl>
