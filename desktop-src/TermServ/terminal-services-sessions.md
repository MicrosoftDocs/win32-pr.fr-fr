# <a name="remote-desktop-sessions"></a><span data-ttu-id="a012a-103">Sessions Bureau à distance</span><span class="sxs-lookup"><span data-stu-id="a012a-103">Remote Desktop Sessions</span></span>

<span data-ttu-id="a012a-104">Lorsqu’un utilisateur se connecte à un ordinateur Services Bureau à distance, une session est démarrée pour l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="a012a-104">When a user logs on to a Remote Desktop Services–enabled computer, a session is started for the user.</span></span> <span data-ttu-id="a012a-105">Chaque session est identifiée par un ID de session unique.</span><span class="sxs-lookup"><span data-stu-id="a012a-105">Each session is identified by a unique session ID.</span></span> <span data-ttu-id="a012a-106">Étant donné que chaque connexion à un client Connexion Bureau à distance (RDC) reçoit un ID de session distinct, l’expérience utilisateur est semblable à la connexion à plusieurs ordinateurs en même temps. par exemple, un ordinateur de bureau et un ordinateur privé.</span><span class="sxs-lookup"><span data-stu-id="a012a-106">Because each logon to a Remote Desktop Connection (RDC) client receives a separate session ID, the user-experience is similar to being logged on to multiple computers at the same time; for example, an office computer and a home computer.</span></span>

<span data-ttu-id="a012a-107">Chaque session Bureau à distance est associée à une station Windows Interactive.</span><span class="sxs-lookup"><span data-stu-id="a012a-107">Each remote desktop session is associated with an interactive window station.</span></span> <span data-ttu-id="a012a-108">Le seul nom de station de fenêtre pris en charge pour une station Windows Interactive est « WinSta0 ». par conséquent, chaque session est associée à sa propre station de fenêtre « WinSta0 ».</span><span class="sxs-lookup"><span data-stu-id="a012a-108">The only supported window station name for an interactive window station is "WinSta0"; therefore each session is associated with its own "WinSta0" window station.</span></span> <span data-ttu-id="a012a-109">Il existe trois bureaux standard pour chaque station Windows : le bureau Winlogon, le Bureau de l’économiseur d’écran et le bureau interactif.</span><span class="sxs-lookup"><span data-stu-id="a012a-109">There are three standard desktops for each window station: the Winlogon desktop, the screen saver desktop, and the interactive desktop.</span></span>

<span data-ttu-id="a012a-110">L’utilisateur associé à la station de fenêtre interactive pour une session est connu sous le nom d' *utilisateur interactif*.</span><span class="sxs-lookup"><span data-stu-id="a012a-110">The user associated with the interactive window station for a session is known as the *interactive user*.</span></span> <span data-ttu-id="a012a-111">Sur un client Connexion Bureau à distance (RDC), il peut y avoir plusieurs utilisateurs interactifs en plus de l’utilisateur interactif sur la console Services Bureau à distance.</span><span class="sxs-lookup"><span data-stu-id="a012a-111">On a Remote Desktop Connection (RDC) client there can be multiple interactive users in addition to the interactive user on the Remote Desktop Services console.</span></span> <span data-ttu-id="a012a-112">Pour récupérer l’identificateur de la session actuellement attachée à la console, utilisez la fonction [**WTSGetActiveConsoleSessionId**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) .</span><span class="sxs-lookup"><span data-stu-id="a012a-112">To retrieve the identifier of the session currently attached to the console, use the [**WTSGetActiveConsoleSessionId**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) function.</span></span>

<span data-ttu-id="a012a-113">Lorsqu’un utilisateur se déconnecte d’un client Connexion Bureau à distance (RDC), la session du client sur le serveur hôte de session Bureau à distance (anciennement appelé Terminal Server) est supprimée et les stations et postes de travail Windows associés à cette session sont supprimés.</span><span class="sxs-lookup"><span data-stu-id="a012a-113">When a user logs off from a Remote Desktop Connection (RDC) client, the session that the client has on the Remote Desktop Session Host (RD Session Host) server (formerly known as a terminal server) is deleted and the window stations and desktops associated with that session are removed.</span></span> <span data-ttu-id="a012a-114">Toutefois, étant donné que la session de console Services Bureau à distance n’est jamais supprimée, les stations de fenêtres associées à la session de console ne sont pas supprimées.</span><span class="sxs-lookup"><span data-stu-id="a012a-114">However, because the Remote Desktop Services console session is never deleted, the window stations associated with the console session are not deleted.</span></span> <span data-ttu-id="a012a-115">Cela affecte la façon dont les applications se comportent dans un environnement Services Bureau à distance lorsqu’elles sont configurées pour s’exécuter dans le contexte de sécurité de l’utilisateur interactif, également appelé mode d’activation d’objet « RunAs interactive User ».</span><span class="sxs-lookup"><span data-stu-id="a012a-115">This affects how applications behave in a Remote Desktop Services environment when they are configured to run in the security context of the interactive user, also known as the "RunAs Interactive User" object activation mode.</span></span>

<span data-ttu-id="a012a-116">Pour plus d’informations sur le démarrage d’un processus serveur local sur une session spécifiée, consultez Activation de session [à session à l’aide d’un moniker de session](session-to-session-activation-with-a-session-moniker.md) et utilisation d' [un moniker de session](using-a-session-moniker.md).</span><span class="sxs-lookup"><span data-stu-id="a012a-116">For more information about starting a local server process on a specified session, see [Session-to-Session Activation with a Session Moniker](session-to-session-activation-with-a-session-moniker.md) and [Using a Session Moniker](using-a-session-moniker.md).</span></span> <span data-ttu-id="a012a-117">Pour plus d’informations sur les contextes de sécurité, consultez [le contexte de sécurité du client](/windows/desktop/SecAuthZ/the-client-security-context).</span><span class="sxs-lookup"><span data-stu-id="a012a-117">For more information about security contexts, see [The Client's Security Context](/windows/desktop/SecAuthZ/the-client-security-context).</span></span>
