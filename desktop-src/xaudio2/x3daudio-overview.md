## <a name="listeners-and-emitters"></a><span data-ttu-id="cdf3d-106">Écouteurs et émetteurs</span><span class="sxs-lookup"><span data-stu-id="cdf3d-106">Listeners and Emitters</span></span>

-   <span data-ttu-id="cdf3d-119">X3DAudio utilise les coordonnées de la main gauche.</span><span class="sxs-lookup"><span data-stu-id="cdf3d-119">X3DAudio uses left-handed coordinates.</span></span> <span data-ttu-id="cdf3d-120">Pour une utilisation avec des coordonnées droitiers, les développeurs doivent nier l’élément. z des membres OrientTop, OrientFront, position et Velocity de l' \_ écouteur X3DAUDIO et de l' \_ émetteur X3DAUDIO.</span><span class="sxs-lookup"><span data-stu-id="cdf3d-120">To use with right-handed coordinates, developers need to negate the .z element of the OrientTop, OrientFront, Position, and Velocity members of X3DAUDIO\_LISTENER and X3DAUDIO\_EMITTER.</span></span>

<span data-ttu-id="cdf3d-121">En plus de la position, les écouteurs et les émetteurs peuvent inclure la vélocité.</span><span class="sxs-lookup"><span data-stu-id="cdf3d-121">In addition to position, listeners and emitters can include velocity.</span></span> <span data-ttu-id="cdf3d-122">Contrairement à un moteur de rendu 3D, X3DAudio utilise uniquement la vélocité pour calculer les effets Doppler (il n’est pas utilisé pour calculer la position).</span><span class="sxs-lookup"><span data-stu-id="cdf3d-122">Unlike a 3D rendering engine, X3DAudio only uses velocity to calculate Doppler effects (it is not used to calculate position).</span></span>
