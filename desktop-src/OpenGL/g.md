# <a name="g-opengl"></a>G (OpenGL)

<dl> <dt>

<span id="opengl_gamma_correction"></span><span id="OPENGL_GAMMA_CORRECTION"></span>**correction gamma**
</dt> <dd>

Fonction appliquée aux couleurs stockées dans la mémoire tampon de trame pour corriger la réponse non linéaire de l’œil (et parfois de l’analyse) aux changements linéaires dans les valeurs d’intensité des couleurs.

</dd> <dt>

<span id="opengl_geometric_model"></span><span id="OPENGL_GEOMETRIC_MODEL"></span>**modèle géométrique**
</dt> <dd>

Les vertex et les paramètres de la coordonnée d’objet qui décrivent un objet. Notez que OpenGL ne définit pas de syntaxe pour les modèles géométriques, mais plutôt une syntaxe et une sémantique pour le rendu des modèles géométriques.

</dd> <dt>

<span id="opengl_geometric_object"></span><span id="OPENGL_GEOMETRIC_OBJECT"></span>**objet géométrique**
</dt> <dd>

Modèle géométrique.

</dd> <dt>

<span id="opengl_geometric_primitive"></span><span id="OPENGL_GEOMETRIC_PRIMITIVE"></span>**primitive géométrique**
</dt> <dd>

Point, ligne ou Polygone.

</dd> <dt>

<span id="opengl_gouraud_shading"></span><span id="OPENGL_GOURAUD_SHADING"></span>**Ombrage Gouraud**
</dt> <dd>

Interpolation lisse des couleurs sur un polygone ou un segment de ligne. Les couleurs sont attribuées aux vertex et interpolées de manière linéaire sur la primitive pour produire une variation relativement lisse de la couleur. Également appelé ombrage lissé.

</dd> <dt>

<span id="opengl_group"></span><span id="OPENGL_GROUP"></span>**Communauté**
</dt> <dd>

Groupe d’un, deux, trois ou quatre éléments qui représente chaque pixel d’une image dans la mémoire du client. Ainsi, dans le contexte d’une image mémoire du client, un groupe et un pixel sont identiques.

</dd> </dl>
