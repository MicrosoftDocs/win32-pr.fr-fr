# <a name="i-opengl"></a>I (OpenGL)

<dl> <dt>

<span id="opengl_image"></span><span id="OPENGL_IMAGE"></span>**image**
</dt> <dd>

Tableau rectangulaire de pixels, dans la mémoire du client ou dans le trame.

</dd> <dt>

<span id="opengl_image_primitive"></span><span id="OPENGL_IMAGE_PRIMITIVE"></span>**primitive d’image** 
</dt> <dd>

Bitmap ou image.

</dd> <dt>

<span id="opengl_immediate_mode"></span><span id="OPENGL_IMMEDIATE_MODE"></span>**mode immédiat**
</dt> <dd>

Mode dans lequel une commande OpenGL est appelée directement, plutôt qu’à partir d’une liste d’affichage. Aucun bit en mode immédiat n’existe ; le mode en mode immédiat fait référence à l’utilisation de OpenGL, plutôt qu’à un bit spécifique de l’État OpenGL.

</dd> <dt>

<span id="opengl_index"></span><span id="OPENGL_INDEX"></span>**évaluer**
</dt> <dd>

Valeur unique qui est interprétée comme une valeur absolue, plutôt que comme une valeur normalisée dans une plage spécifiée (comme un composant). Les index de couleurs sont les noms des couleurs, qui sont déréférencées par le matériel d’affichage à l’aide de la carte des couleurs. Les index sont généralement masqués, plutôt que ancrés, lorsqu’ils sont hors limites. Par exemple, l’index 0xf7 est masqué en 0x7 lorsqu’il est écrit dans une mémoire tampon de 4 bits (couleur ou gabarit). Les index de couleurs et les index de stencil sont toujours traités comme des index, jamais comme des composants.

</dd> <dt>

<span id="opengl_iris_gl"></span><span id="OPENGL_IRIS_GL"></span>**IRIS GL**
</dt> <dd>

La bibliothèque graphique propriétaire de Silicon Graphics, développée de 1982 à 1992. OpenGL a été conçu avec IRIS GL comme point de départ.

</dd> </dl>
