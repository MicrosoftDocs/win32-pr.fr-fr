# <a name="devices"></a>Appareils

## <a name="in-this-section"></a>Contenu de cette section

| Rubrique | Description |
|---|---|
| [Accès de l’appareil](deviceaccess/device-access-broker-api-portal.md)<br/> | Les API d’accès aux appareils permettent aux développeurs d’écrire des applications du Windows Store pour des appareils personnalisés. L’API applique une stratégie pour accorder à une application du Windows Store le niveau d’accès approprié à un appareil et, si l’accès est accordé, fournit des méthodes pour envoyer des codes de contrôle afin d’utiliser l’appareil.<br/> |
| [API AllJoyn](/previous-versions/windows/desktop/alljoyn/alljoyn-api-portal)<br/> | AllJoyn est un Framework et un protocole de type DCOM permettant d’effectuer des appels de méthode à distance et/ou d’envoyer des signaux unidirectionnels entre des applications sur un bus distribué. Il est destiné à être utilisé principalement pour les scénarios de type Internet des objets. <br/> |
| [API point of service](/uwp/api/Windows.Devices.PointOfService)<br/> | L’API point of service permet aux développeurs d’applications d’accéder aux périphériques de point de service (POS). L’espace de noms fournit une interface indépendante du fournisseur pour accéder aux appareils PDS à partir d’une application UWP.<br/> |
| [Ressources de communication](/windows/desktop/DevIO/communications-resources)<br/>           | Une ressource de communication est un appareil physique ou logique qui fournit un seul flux de données asynchrone bidirectionnel. Les ports série, les ports parallèles, les télécopieurs et les modems sont des exemples de ressources de communication. Pour chaque ressource de communication, il existe un fournisseur de services, constitué d’une bibliothèque ou d’un pilote, qui permet aux applications d’accéder à la ressource. <br/> |
| [Gestion des appareils](/windows/desktop/DevIO/device-management)<br/>                         | La gestion des appareils permet d’informer uniformément toutes les applications et tous les composants système des modifications qui peuvent affecter leur fonctionnement ou l’accès aux ressources. Les applications et le système utilisent et traitent les événements des appareils pour tirer parti des nouvelles ressources lorsqu’elles deviennent disponibles et pour empêcher la perte de données lorsque des ressources existantes deviennent indisponibles. <br/>                 |
| [Stockage étendu](/previous-versions/windows/desktop/enstor/enhanced-storage-portal)<br/>                  | L’API de stockage améliorée permet une authentification de bout en bout cohérente pour les périphériques de stockage personnel qui diffèrent au niveau du facteur de forme. <br/>                                                                                                                                                                                                                                                 |
| [Détection de fonction](/previous-versions/windows/desktop/fundisc/fd-portal)<br/>                                 | La découverte des fonctions fournit une interface de programmation uniforme pour énumérer les ressources système, telles que les périphériques matériels, qu’elles soient locales ou connectées via un réseau.<br/>                                                                                                                                                                                                 |
| [PnP-X](/previous-versions/windows/desktop/fundisc/pnp-x)<br/>                                                  | Les extensions de Plug-and-Play (PnP-X) permettent à un système informatique de découvrir des appareils réseau et de les installer sur le système local à l’aide de Plug-and-Play (PnP). <br/>                                                                                                                                                                                                                     |
| [API d’emplacement](/windows/desktop/LocationAPI/windows-location-api-portal)<br/>             | L’API location permet de simplifier la programmation prenant en charge l’emplacement en fournissant un moyen standard de récupérer des données sur l’emplacement de l’utilisateur et de standardiser les formats des rapports de données d’emplacement. L’API location gère automatiquement les transitions entre les fournisseurs de données de localisation et choisit toujours le fournisseur le plus précis pour la situation actuelle.<br/>                                 |
| [API de capteur](/windows/desktop/SensorsAPI/portal)<br/>                                     | L’API de capteur permet aux applications d’obtenir et d’utiliser les données des capteurs de manière standardisée.<br/>                                                                                                                                                                                                                                                                                     |
| [API UPnP](/windows/desktop/UPnP/universal-plug-and-play-start-page)<br/>                | L’infrastructure UPnP permet la mise en réseau dynamique d’appareils intelligents, de périphériques sans fil et de PC. <br/>                                                                                                                                                                                                                                                                            |
| [Services Web sur les appareils](/windows/desktop/WsdApi/wsd-portal)<br/>                           | L’API Microsoft Web Services on Devices (WSDAPI) prend en charge l’implémentation d’appareils et de services contrôlés par le client, ainsi que les hôtes d’appareils conformes au profil des appareils pour les services Web (DPWS). <br/>                                                                                                                                                                          |
| [Kit de développement logiciel (SDK) Windows Media Gestionnaire de périphériques 11](./wmdm/windows-media-device-manager-11-sdk.md)<br/> | Les applications ou composants basés sur Windows Media Gestionnaire de périphériques ont une API cohérente pour communiquer avec un large éventail d’appareils, notamment le protocole MTP (Media Transfer Protocol), la classe de stockage de masse (MSC), l’RAPI et d’autres périphériques basés sur les versions les plus récentes et les plus anciennes de la technologie Windows Media.<br/>                                                                    |
| [Windows Mixed Reality](https://msdn.microsoft.com/library/Mt844799(v=WIN.10).aspx)<br/>          | Windows Mixed Reality permet la création d’une expérience de réalité mixte pour Microsoft HoloLens et d’autres casques immersifs. Pour plus d’informations, consultez le centre de développement de la [réalité mixte](https://developer.microsoft.com/windows/mixed-reality).<br/>                                                                                                                                    |
| [Appareils mobiles Windows](./windows-portable-devices.md)<br/>            | Les appareils mobiles Windows (WPD) permettent aux ordinateurs de communiquer avec les périphériques de stockage et les supports associés. WPD offre une méthode flexible et fiable pour les ordinateurs qui communiquent avec les lecteurs musicaux, les périphériques de stockage, les téléphones mobiles, les appareils photo et de nombreux autres types d’appareils connectés.<br/>                                                                                               |
