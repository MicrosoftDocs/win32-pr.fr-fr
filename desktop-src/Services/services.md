# <a name="services-services"></a>Services (services)

Une *application de service* est conforme aux règles d’interface du gestionnaire de contrôle des services (SCM). Il peut être démarré automatiquement au démarrage du système, par un utilisateur via l’applet services du panneau de configuration, ou par une application qui utilise les [fonctions de service](service-functions.md). Les services peuvent s’exécuter même si aucun utilisateur n’est connecté au système.

Un *service de pilote* est conforme aux protocoles de pilote de périphérique. Il est similaire à une application de service, mais il n’interagit pas avec le SCM. Par souci de simplicité, le terme *service* fait référence à une *application de service* dans cette vue d’ensemble.

Les déclencheurs peuvent maintenant être utilisés pour contrôler le démarrage du service. Pour plus d’informations, consultez [configuration du service](service-configuration.md).
