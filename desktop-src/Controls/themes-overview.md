# <a name="visual-styles"></a>Styles visuels

## <a name="purpose"></a>Objectif

Les *styles visuels* modifient l’apparence des contrôles communs en fonction du thème choisi par l’utilisateur. Avant Windows 8, vous devez configurer votre application pour qu’elle utilise des styles visuels spécifiques. Sinon, les contrôles communs de l’application sont toujours rendus dans le style associé au thème Windows classique, quel que soit le thème actuellement sélectionné. Dans Windows 8, les styles visuels ne peuvent pas être désactivés, le mode Windows classique n’existe plus et le mode contraste élevé a été modifié pour fonctionner avec les styles visuels.

Cette section fournit une vue d’ensemble des styles visuels et explique comment configurer votre application pour qu’elle les utilise.

## <a name="developer-audience"></a>Développeurs concernés

Les styles visuels sont conçus pour être utilisés par les développeurs C/C++ et les concepteurs d’interface utilisateur. En général, les développeurs ont besoin d’un niveau modéré de compréhension des concepts de programmation d’interface utilisateur, de la programmation d’API Windows et d’Unicode.

## <a name="run-time-requirements"></a>Conditions d’exécution

Windows Vista et systèmes d’exploitation ultérieurs.

> [!Note]  
> Les styles visuels ne sont pas pris en charge en mode 256-Color.
