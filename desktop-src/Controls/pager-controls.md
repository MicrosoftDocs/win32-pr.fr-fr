# <a name="about-pager-controls"></a>À propos des contrôles de pagineur

Un *contrôle de pagineur* est un conteneur de fenêtre qui est utilisé avec une fenêtre qui n’a pas assez de zone d’affichage pour afficher tout son contenu. Le contrôle de pagineur permet à l’utilisateur de faire défiler vers la zone de la fenêtre qui n’est pas actuellement affichée.

-   [À propos des contrôles de pagineur](#about-pager-controls)

## <a name="about-pager-controls"></a>À propos des contrôles de pagineur

Microsoft Internet Explorer version 4,0 (commctrl.dll version 4,71) introduit le contrôle de pagineur. Ce contrôle est utile dans les situations où une fenêtre ne dispose pas de suffisamment de zone pour afficher une fenêtre enfant. Par exemple, si votre application a une barre d’outils qui n’est pas assez grande pour afficher tous ses éléments, vous pouvez affecter la barre d’outils à un contrôle de pagineur et les utilisateurs pourront faire défiler vers la gauche ou vers la droite pour accéder à tous les éléments. Vous pouvez également créer des contrôles de pagineur qui défilent verticalement.

Une fenêtre assignée au contrôle pager est appelée *fenêtre contenue*.

La capture d’écran suivante montre une barre d’outils contenue dans un contrôle de pagineur. Le contrôle de pagineur est affiché en rouge pour montrer les zones du contrôle qui sont visibles.

> [!Note]  
> Le contrôle de pagineur est implémenté dans la version 4,71 et les versions ultérieures de Comctl32.dll.
