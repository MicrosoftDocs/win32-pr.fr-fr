<span data-ttu-id="c8ebb-115">Lorsqu’un service s’exécute sous le compte LocalSystem sur un ordinateur qui est membre d’un domaine, le service dispose de l’accès réseau accordé au compte d’ordinateur, ou à tous les groupes dont le compte d’ordinateur est membre.</span><span class="sxs-lookup"><span data-stu-id="c8ebb-115">When a service runs under the LocalSystem account on a computer that is a domain member, the service has whatever network access is granted to the computer account, or to any groups of which the computer account is a member.</span></span> <span data-ttu-id="c8ebb-116">N’oubliez pas que dans Windows 2000, un compte d’ordinateur de domaine est un principal de service, similaire à un compte d’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="c8ebb-116">Be aware that in Windows 2000, a domain computer account is a service principal, similar to a user account.</span></span> <span data-ttu-id="c8ebb-117">Cela signifie qu’un compte d’ordinateur peut être dans un groupe de sécurité et qu’une entrée du contrôle d’accès dans un descripteur de sécurité peut accorder l’accès à un compte d’ordinateur.</span><span class="sxs-lookup"><span data-stu-id="c8ebb-117">This means that a computer account can be in a security group, and an ACE in a security descriptor can grant access to a computer account.</span></span> <span data-ttu-id="c8ebb-118">N’oubliez pas que l’ajout de comptes d’ordinateur aux groupes n’est pas recommandé pour deux raisons :</span><span class="sxs-lookup"><span data-stu-id="c8ebb-118">Be aware that adding computer accounts to groups is not recommended for two reasons:</span></span>
