# <a name="characteristics-of-attributes"></a>Caractéristiques des attributs

Chaque attribut d’un objet Active Directory Domain Services est défini par un objet [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) dans le conteneur de schéma. Les propriétés d’un objet **attributeSchema** spécifient les caractéristiques de l’attribut, par exemple :

-   Identificateurs d’attributs. Les attributs ont plusieurs identificateurs, dont la plus intéressante du point de vue de la programmation est le [**lDAPDisplayName**](/windows/desktop/ADSchema/a-ldapdisplayname), qui est utilisé par les clients LDAP pour lire et écrire l’attribut, et le [**schemaIDGUID**](/windows/desktop/ADSchema/a-schemaidguid), qui est utilisé dans les descripteurs de sécurité pour contrôler l’accès à l’attribut.
-   Type de données contenues par les instances de l’attribut. Les propriétés de syntaxe d’un attribut déterminent le type de données, tel qu’un entier, une chaîne ou un binaire. Des propriétés supplémentaires peuvent spécifier la plage de valeurs autorisées pour l’attribut et si une instance de l’attribut peut avoir plusieurs valeurs.
-   Inclusion de l’attribut dans des groupes. Autres propriétés baliser un attribut à inclure dans un jeu de propriétés, qui est un ensemble de propriétés associées, ou à inclure dans l’ensemble d’attributs répliqués dans le catalogue global ou indexés pour optimiser les performances de recherche.

Le tableau suivant répertorie le [**lDAPDisplayName**](/windows/desktop/ADSchema/a-ldapdisplayname) et la description des propriétés de clé d’un objet [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) . Pour plus d’informations et pour obtenir la liste complète des propriétés obligatoires et facultatives d’un objet **attributeSchema** , consultez [**classe Schema-attribute**](/windows/desktop/ADSchema/c-attributeschema).

| [**rangeLower**](/windows/desktop/ADSchema/a-rangelower), [ **rangeUpper**](/windows/desktop/ADSchema/a-rangeupper)         | Paire d’entiers qui spécifient les limites inférieure et supérieure de la plage de valeurs pour cet attribut. Pour les attributs de syntaxe numérique, la plage spécifie la valeur minimale et la valeur maximale. Pour les attributs avec une syntaxe de chaîne, la plage spécifie la taille minimale et maximale, en caractères. Pour les attributs de syntaxe binaire, la plage spécifie le nombre d’octets.<br/> Si [**rangeLower**](/windows/desktop/ADSchema/a-rangelower) et [**rangeUpper**](/windows/desktop/ADSchema/a-rangeupper) sont définis, **rangeLower** doit être inférieur à **rangeUpper**. Par exemple, si **rangeLower** pour un entier est 3 et que **rangeUpper** est absent, cela signifie qu’il n’y a aucune contrainte supérieure sur l’attribut. De même, si rangeUpper pour une chaîne est 2000, et rangeLower est absent, cela indique qu’il n’y a aucune contrainte inférieure sur la longueur.<br/> |
