# <a name="security-issues-for-service-publication"></a><span data-ttu-id="0bb84-106">Problèmes de sécurité pour la publication de service</span><span class="sxs-lookup"><span data-stu-id="0bb84-106">Security Issues for Service Publication</span></span>

<span data-ttu-id="0bb84-107">Le système limite la possibilité de créer, de modifier ou de supprimer des objets de point de connexion.</span><span class="sxs-lookup"><span data-stu-id="0bb84-107">The system restricts the ability to create, modify, or delete connection point objects.</span></span> <span data-ttu-id="0bb84-108">Tenez compte de ces restrictions lorsque vous publiez un service.</span><span class="sxs-lookup"><span data-stu-id="0bb84-108">Be aware of, and handle, these restrictions when you publish a service.</span></span>

<span data-ttu-id="0bb84-109">Les clients doivent être en mesure d’approuver les données publiées dans un objet de point de connexion dans l’annuaire.</span><span class="sxs-lookup"><span data-stu-id="0bb84-109">Clients must be able to trust the data published in a connection point object in the directory.</span></span> <span data-ttu-id="0bb84-110">Pour cette raison, l’autorisation de créer un objet point de connexion est généralement limitée aux utilisateurs privilégiés, tels que les administrateurs de domaine.</span><span class="sxs-lookup"><span data-stu-id="0bb84-110">For this reason, permission to create a connection point object is typically restricted to privileged users, such as domain administrators.</span></span> <span data-ttu-id="0bb84-111">Cela empêche les utilisateurs non autorisés des clients conclure en créant des points de connexion non valides pour les services connus.</span><span class="sxs-lookup"><span data-stu-id="0bb84-111">This prevents unauthorized users from deceiving clients by creating invalid connection points for well-known services.</span></span>

<span data-ttu-id="0bb84-112">Les services ne doivent pas s’exécuter avec des privilèges d’administrateur de domaine.</span><span class="sxs-lookup"><span data-stu-id="0bb84-112">Services must not run with domain administrator privileges.</span></span> <span data-ttu-id="0bb84-113">Cela signifie qu’un service ne peut généralement pas créer son propre point de connexion.</span><span class="sxs-lookup"><span data-stu-id="0bb84-113">This means that a service typically cannot create its own connection point.</span></span> <span data-ttu-id="0bb84-114">Au lieu de cela, vous fournissez une application de configuration ou d’installation de service qui crée le point de connexion.</span><span class="sxs-lookup"><span data-stu-id="0bb84-114">Instead, you provide a service installation or configuration application that creates the connection point.</span></span> <span data-ttu-id="0bb84-115">Ce programme d’installation doit être exécuté par un utilisateur disposant des privilèges nécessaires.</span><span class="sxs-lookup"><span data-stu-id="0bb84-115">This installer must be run by a user with the necessary privileges.</span></span>

<span data-ttu-id="0bb84-116">Bien qu’un service ne puisse pas généralement créer son point de connexion, il doit être en mesure de mettre à jour les propriétés du point de connexion au moment de l’exécution.</span><span class="sxs-lookup"><span data-stu-id="0bb84-116">Although a service cannot typically create its connection point, it must be able to update the connection point properties at run time.</span></span> <span data-ttu-id="0bb84-117">Les propriétés des points de connexion contiennent les données de liaison utilisées par les clients pour se connecter au service.</span><span class="sxs-lookup"><span data-stu-id="0bb84-117">The connection point properties contain the binding data used by clients to connect to the service.</span></span> <span data-ttu-id="0bb84-118">Si les données de liaison changent, le service doit mettre à jour le point de connexion ; <span data-ttu-id="0bb84-121">Un service s’exécutant sous le compte LocalSystem peut créer un point de connexion en tant qu’objet enfant sous son propre objet ordinateur dans le répertoire.</span><span class="sxs-lookup"><span data-stu-id="0bb84-121">A service running under the LocalSystem account can create a connection point as a child object under its own computer object in the directory.</span></span> <span data-ttu-id="0bb84-122">Un tel service est une exception à la règle de services qui ne crée pas leurs propres points de connexion.</span><span class="sxs-lookup"><span data-stu-id="0bb84-122">Such a service is an exception to the rule of services not creating their own connection points.</span></span> <span data-ttu-id="0bb84-123">Un service LocalSystem a également l’autorisation de modifier les propriétés des objets de point de connexion sous son propre objet ordinateur.</span><span class="sxs-lookup"><span data-stu-id="0bb84-123">A LocalSystem service also has permission to modify the properties of connection point objects under its own computer object.</span></span> <span data-ttu-id="0bb84-124">N’oubliez pas qu’un service doit s’exécuter sous le compte LocalSystem uniquement si cela est absolument nécessaire.</span><span class="sxs-lookup"><span data-stu-id="0bb84-124">Be aware that a service should run under the LocalSystem account only if absolutely necessary.</span></span> <span data-ttu-id="0bb84-125">Pour plus d’informations, consultez [instructions de sélection d’un compte d’ouverture de session de service](guidelines-for-selecting-a-service-logon-account.md).</span><span class="sxs-lookup"><span data-stu-id="0bb84-125">For more information, see [Guidelines for Selecting a Service Logon Account](guidelines-for-selecting-a-service-logon-account.md).</span></span>

<span data-ttu-id="0bb84-126">L’application qui crée un objet point de connexion, ou tout objet, doit avoir les autorisations créer des enfants pour la classe d’objet à créer dans le conteneur où l’objet sera créé.</span><span class="sxs-lookup"><span data-stu-id="0bb84-126">The application that creates a connection point object, or any object, must have create child permissions for the object class to be created in the container where the object will be created.</span></span> <span data-ttu-id="0bb84-127">Pour supprimer un objet, le processus qui effectue l’opération doit avoir les autorisations supprimer les enfants pour la classe d’objet à supprimer sur le conteneur contenant l’objet, ou disposer des autorisations de suppression sur l’objet lui-même.</span><span class="sxs-lookup"><span data-stu-id="0bb84-127">To remove an object, the process performing the operation must have delete child permissions for the object class to be deleted on the container holding the object, or have delete permissions on the object itself.</span></span> <span data-ttu-id="0bb84-128">Pour mettre à jour un point de connexion, le processus effectuant l’opération doit disposer d’un accès en écriture aux propriétés à mettre à jour sur l’objet.</span><span class="sxs-lookup"><span data-stu-id="0bb84-128">To update a connection point the process performing the operation must have write-access to the properties to be updated on the object.</span></span>
