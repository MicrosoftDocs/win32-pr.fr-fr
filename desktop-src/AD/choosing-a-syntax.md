# <a name="choosing-a-syntax"></a>Choix d’une syntaxe

Il existe 23 syntaxes définies dans Active Directory Domain Services. Cette rubrique contient une liste de syntaxes recommandées à utiliser lors de la définition d’un nouvel attribut. Pour plus d’informations, consultez [syntaxes for Attributes in Active Directory Domain Services](syntaxes-for-attributes-in-active-directory-domain-services.md).

Le tableau suivant fournit une liste de recommandations.



| Données à stocker dans l’attribut      | Syntaxe à utiliser                                                      | Commentaire                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|---------------------------------|--------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Données binaires                     | [**String(Octet)**](/windows/desktop/ADSchema/s-string-octet)                       | Utilisez pour stocker des données binaires. Il s’agit d’un tableau d’octets.                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Données binaires avec une référence de DN | [**Object(DN-Binary)**](/windows/desktop/ADSchema/s-object-dn-binary)               | Contient une valeur binaire et un nom unique (DN). Le serveur Active Directory maintient à jour le DN.                                                                                                                                                                                                                                                                                                                                                                                                     |
| Boolean                         | [**Boolean**](/windows/desktop/ADSchema/s-boolean)                                  | À utiliser pour les valeurs booléennes.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Référence de DN                    | [**Object(DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn)                       | Utilisez pour stocker les noms uniques que vous souhaitez maintenir à jour par le serveur de Active Directory. Lorsqu’un attribut de syntaxe DN est créé avec un DN valide, le serveur traite l’attribut comme une référence à l’objet représenté par le DN qui a été défini. Si l’objet référencé est renommé ou déplacé, le serveur s’assure que l’attribut reflète la modification. Si l’attribut est réinitialisé avec un nouveau nom unique, l’attribut est une référence à l’objet représenté par le nouveau nom unique.                                |
| Integer                         | [**Integer**](/windows/desktop/ADSchema/s-integer)                                  | Utilisez pour les entiers.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Entier long (valeurs de 64 bits)   | [**LargeInteger**](/windows/desktop/ADSchema/s-largeinteger)                        | Utilisez pour les valeurs 64 bits.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| DN lié                       | [**Object(DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn)                       | Cette syntaxe de chaîne peut être utilisée pour les DNs liés. Les liens de retour doivent être de syntaxe DN. Les liens ascendants peuvent être de syntaxe DN, d’objet ( [**DN-String)**](/windows/desktop/ADSchema/s-object-dn-string), d' [**objet (DN-Binary)**](/windows/desktop/ADSchema/s-object-dn-binary), d' [**objet (point d’accès)**](/windows/desktop/ADSchema/s-object-access-point)ou d' [**objet (ou-Name)**](/windows/desktop/ADSchema/s-object-or-name). Les attributs liés doivent avoir un **LinkId** défini. Consultez la description du **LinkId** dans [**attributs-Schema**](/windows/desktop/ADSchema/c-attributeschema) Properties. |
| Descripteur de sécurité             | [**String(NT-Sec-Desc)**](/windows/desktop/ADSchema/s-string-nt-sec-desc)           | Chaîne d’octets contenant un descripteur de sécurité.                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Identificateur de sécurité (SID)       | [**Chaîne (SID)**](/windows/desktop/ADSchema/s-string-sid)                           | Chaîne d’octets contenant un identificateur de sécurité (SID). Utilisez cette syntaxe pour stocker uniquement les valeurs SID.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| String                          | [**String(Unicode)**](/windows/desktop/ADSchema/s-string-unicode)                   | À utiliser pour la plupart des attributs de chaîne. Il prend en charge le jeu de caractères Unicode. Lorsque le serveur Active Directory effectue des comparaisons par rapport à des attributs de cette syntaxe (tels que l’évaluation d’une requête), il effectue des comparaisons qui ne respectent pas la casse. Utilisez les autres syntaxes de chaîne ([**chaîne (IA5)**](/windows/desktop/ADSchema/s-string-ia5), [**chaîne (numérique)**](/windows/desktop/ADSchema/s-string-numeric), et ainsi de suite) pour stocker les chaînes qui doivent contenir uniquement les jeux de caractères spécifiques pris en charge par la syntaxe.<br/>                          |
| Données de chaîne avec une référence de DN | [**Object(DN-String)**](/windows/desktop/ADSchema/s-object-dn-string)               | Chaîne contenant une valeur de chaîne et un nom unique (DN). Le serveur Active Directory maintient à jour le DN.                                                                                                                                                                                                                                                                                                                                                                                            |
| Temps                            | [**String(Generalized-Time)**](/windows/desktop/ADSchema/s-string-generalized-time) | Utilisez la syntaxe de [**chaîne (temps général)**](/windows/desktop/ADSchema/s-string-generalized-time) pour stocker les valeurs d’heure plutôt que la syntaxe de [**chaîne (heure UTC)**](/windows/desktop/ADSchema/s-string-utc-time) , car la **chaîne (temps général** ) utilise quatre caractères pour l’année et **chaîne (heure UTC)** utilise uniquement deux.                                                                                                                                                                                                                 |
