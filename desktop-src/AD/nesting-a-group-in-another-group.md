# <a name="nesting-a-group-in-another-group"></a>Imbrication d’un groupe dans un autre groupe

L’ajout d’un groupe en tant que membre d’un autre groupe est appelé *imbrication*. Pour les groupes de distribution, l’imbrication est prise en charge en mode mixte et en mode natif. Pour les groupes de sécurité, l’imbrication est prise en charge uniquement pour les domaines s’exécutant en mode natif.

Pour imbriquer un groupe dans un autre groupe, utilisez les mêmes techniques que celles décrites dans [Ajout de membres à des groupes dans un domaine](adding-members-to-groups-in-a-domain.md). N’oubliez pas que, en fonction de l’étendue du groupe, le groupe peut contenir uniquement des types et des étendues spécifiques d’autres groupes.

Les options d’imbrication varient également selon que le domaine est en mode mixte ou en mode natif. Pour plus d’informations sur les groupes dans différents modes de domaine, consultez [imbrication en mode natif](nesting-in-native-mode.md) et [imbrication en mode mixte](nesting-in-mixed-mode.md).

Pour plus d’informations sur l’imbrication de groupes, consultez [Erreurs courantes](common-errors.md).
