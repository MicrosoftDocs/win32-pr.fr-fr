# <a name="directory-services-today"></a>Services d’annuaire aujourd’hui

Il est courant de trouver plusieurs répertoires d’administration déployés au sein d’une même organisation. ces répertoires incluent des répertoires de ressources réseau, tels que les annuaires LDAP tels que le service d’annuaire microsoft Active Directory, Windows service d’annuaire du système d’exploitation, ainsi que des répertoires spécifiques à l’application, tels que microsoft Exchange.

![déploiement de plusieurs annuaires](images/ds2chal.png)

## <a name="the-directory-challenge"></a>Le défi de l’annuaire

Dans l’organisation, plusieurs annuaires présentent des défis complexes pour les utilisateurs, les administrateurs et les développeurs. Ces problèmes ont un déploiement limité de répertoires étendus. Les utilisateurs sont confrontés à plusieurs ouvertures de session et à différentes interfaces pour les informations dans plusieurs annuaires. Les administrateurs sont confrontés à la complexité de la gestion de plusieurs annuaires. Les utilisateurs finaux et les administrateurs veulent que les développeurs d’applications utilisent un annuaire d’administration existant, mais les développeurs sont confrontés à un dilemme sur lequel utiliser. Chaque annuaire offre des interfaces d’application uniques. Un développeur doit choisir une implémentation d’annuaire spécifique ou prendre en charge plusieurs versions de son application. Par conséquent, les développeurs utilisent rarement des services d’annuaire existants.
