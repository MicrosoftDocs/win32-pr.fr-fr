# <a name="active-directory-service-interfaces-scripting"></a>Scripts des interfaces de service Active Directory

Les interfaces ADSI (Microsoft Active Directory Service Interfaces), lorsqu’elles sont combinées avec un langage de script tel que Microsoft Visual Basic Scripting Edition ou le logiciel de développement Microsoft JScript, fournissent à un administrateur système des outils puissants pour la gestion des utilisateurs, des services, des files d’attente à l’impression et d’autres parties d’un réseau d’ordinateurs exécutant des systèmes d’exploitation Microsoft Windows NT ou Windows 2000. Les scripts utilisant le modèle d’objet ADSI peuvent remplacer les opérations manuelles répétitives qui seraient normalement effectuées à l’aide des outils d’administration ou de la console MMC (Microsoft Management Console). Les scripts ADSI peuvent également automatiser des tâches de maintenance système régulières pour une opération mains libres et offrent une interface simple pour écrire des outils de gestion de système Web à l’aide de Active Server pages.

Pour plus d’informations sur l’écriture de scripts avec ADSI, avec des didacticiels de script, des exemples et des outils, consultez :

-   [Centre de scripts](https://www.microsoft.com/technet/community/scriptcenter/default.mspx) sur le site Web Microsoft TechNet
-   [Erreurs et interception des erreurs](errors-and-error-trapping.md)
