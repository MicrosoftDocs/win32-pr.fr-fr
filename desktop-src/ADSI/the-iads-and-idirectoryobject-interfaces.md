# <a name="the-iads-and-idirectoryobject-interfaces"></a><span data-ttu-id="e7e7d-106">Les interfaces IADs et IDirectoryObject</span><span class="sxs-lookup"><span data-stu-id="e7e7d-106">The IADs and IDirectoryObject Interfaces</span></span>

<span data-ttu-id="e7e7d-107">Les clients ADSI gèrent et manipulent des objets de service d’annuaire à l’aide de l’une des deux interfaces COM suivantes : [**IADs**](/windows/desktop/api/Iads/nn-iads-iads) ou [**IDirectoryObject**](/windows/desktop/api/Iads/nn-iads-idirectoryobject).</span><span class="sxs-lookup"><span data-stu-id="e7e7d-107">ADSI clients manage and manipulate directory service objects by using one of two COM interfaces: [**IADs**](/windows/desktop/api/Iads/nn-iads-iads) or [**IDirectoryObject**](/windows/desktop/api/Iads/nn-iads-idirectoryobject).</span></span> <span data-ttu-id="e7e7d-108">**IADs** est une interface [**IDispatch**](/windows/win32/api/oaidl/nn-oaidl-idispatch) destinée à être utilisée par des clients à liaison tardive tels que ceux écrits en Microsoft Visual Basic, Java et divers langages de script.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-108">**IADs** is an [**IDispatch**](/windows/win32/api/oaidl/nn-oaidl-idispatch) interface intended for use by late-bound clients such as those written in Microsoft Visual Basic, Java, and various scripting languages.</span></span> <span data-ttu-id="e7e7d-109">**IDirectoryObject** est une interface vtable qui fournit un accès direct aux objets par des clients à liaison anticipée, tels que ceux écrits en C et C++.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-109">**IDirectoryObject** is a vtable interface that provides direct access to objects by early bound clients such as those written in C and C++.</span></span>

<span data-ttu-id="e7e7d-110">Chaque objet ADSI doit implémenter à la fois [**IADs**](/windows/desktop/api/Iads/nn-iads-iads) et [**IDirectoryObject**](/windows/desktop/api/Iads/nn-iads-idirectoryobject).</span><span class="sxs-lookup"><span data-stu-id="e7e7d-110">Each ADSI object must implement both [**IADs**](/windows/desktop/api/Iads/nn-iads-iads) and [**IDirectoryObject**](/windows/desktop/api/Iads/nn-iads-idirectoryobject).</span></span> <span data-ttu-id="e7e7d-111">Les clients ADSI écrits dans des langages tels que C ou C++, qui sont en mesure d’accéder directement aux vtables, peuvent utiliser l’une ou l’autre interface, mais pas les deux dans la même application.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-111">ADSI clients written in languages such as C or C++, which are able to directly access vtables, can use either interface, but not both in the same application.</span></span> <span data-ttu-id="e7e7d-112">Les clients ADSI écrits en Visual Basic ou Java sont limités à l’utilisation de **IADs**.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-112">ADSI clients written in Visual Basic or Java are limited to using **IADs**.</span></span>

<span data-ttu-id="e7e7d-113">L’interface [**IADs**](/windows/desktop/api/Iads/nn-iads-iads) permet aux clients à liaison tardive de tirer parti des fonctionnalités de maintenance inhérentes du modèle d’objet ADSI.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-113">The [**IADs**](/windows/desktop/api/Iads/nn-iads-iads) interface enables late-bound clients to take advantage of the inherent housekeeping features of the ADSI object model.</span></span> <span data-ttu-id="e7e7d-114">Parmi ces fonctionnalités, citons le cache des propriétés, qui permet aux clients de lire et d’écrire des propriétés sans passer par le câble pour chaque appel.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-114">Among these features is the property cache, which enables clients to read and write properties without going over the wire for each call.</span></span> <span data-ttu-id="e7e7d-115">En outre, les applications clientes tirent parti de l’interface utilisateur et des bibliothèques de contrôles ActiveX puissantes, ainsi qu’un style de programmation plus simple.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-115">In addition, client applications gain the use of powerful UI and ActiveX control libraries and a simpler style of programming.</span></span> <span data-ttu-id="e7e7d-116">En retour, les clients à liaison tardive doivent utiliser le type de données **Variant** , qui exclut l’utilisation des types de données natifs plus riches fournis par ADSI.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-116">In return, late-bound clients must use the **VARIANT** data type, which precludes using the richer native data types provided by ADSI.</span></span>

<span data-ttu-id="e7e7d-117">L’interface [**IDirectoryObject**](/windows/desktop/api/Iads/nn-iads-idirectoryobject) permet aux clients à liaison anticipée de tirer pleinement parti des types de données du service d’annuaire natif au détriment d’un léger avantage en termes de performances par rapport à l’utilisation du cache de propriétés.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-117">The [**IDirectoryObject**](/windows/desktop/api/Iads/nn-iads-idirectoryobject) interface enables early bound clients to take full advantage of native directory-service data types at the cost of foregoing a slight performance advantage from using the property cache.</span></span> <span data-ttu-id="e7e7d-118">En retour, l’interface **IDirectoryObject** fournit un accès direct et direct aux propriétés de l’objet par le biais d’une requête unique, plutôt qu’à l’aide d’appels d' **extraction** et de **placement** individuels.</span><span class="sxs-lookup"><span data-stu-id="e7e7d-118">In return, the **IDirectoryObject** interface provides direct, on-the-wire access to object properties through a single request, rather than through individual **get** and **put** calls.</span></span>
