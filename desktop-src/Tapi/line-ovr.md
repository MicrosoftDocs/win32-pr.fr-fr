# <a name="line"></a><span data-ttu-id="0609b-104">Lignes</span><span class="sxs-lookup"><span data-stu-id="0609b-104">Line</span></span>

<span data-ttu-id="0609b-105">Le concept de ligne a évolué au fil du temps et a été partiellement remplacé par les concepts d’adresse et de terminal.</span><span class="sxs-lookup"><span data-stu-id="0609b-105">The line concept has evolved over time and been partly superseded by the Address and Terminal concepts.</span></span> <span data-ttu-id="0609b-106">TAPI 3 n’utilise pas directement le concept de ligne, mais TAPI 2 continue à incorporer ce paradigme.</span><span class="sxs-lookup"><span data-stu-id="0609b-106">TAPI 3 does not directly use the concept of line, but TAPI 2 continues to incorporate this paradigm.</span></span>

<span data-ttu-id="0609b-107">Un *périphérique de ligne* est un appareil physique, tel qu’un tableau de télécopie, un modem ou une carte RNIS connectée à un réseau.</span><span class="sxs-lookup"><span data-stu-id="0609b-107">A *line device* is a physical device such as a fax board, a modem, or an ISDN card that is connected to a network.</span></span> <span data-ttu-id="0609b-108">L’appareil peut ne pas être connecté physiquement à l’ordinateur sur lequel l’application TAPI est exécutée, par exemple un pool de modems sur un serveur.</span><span class="sxs-lookup"><span data-stu-id="0609b-108">The device may not be physically connected to the computer on which the TAPI application is running, such as a modem pool on a server.</span></span> <span data-ttu-id="0609b-109">Les appareils en ligne prennent en charge les fonctionnalités de communication en permettant aux applications d’envoyer des informations ou de recevoir des informations d’un réseau.</span><span class="sxs-lookup"><span data-stu-id="0609b-109">Line devices support communications capabilities by allowing applications to send information to or receive information from a network.</span></span> <span data-ttu-id="0609b-110">Un périphérique de ligne contient un ensemble d’un ou de plusieurs canaux homogènes qui peuvent être utilisés pour établir des appels.</span><span class="sxs-lookup"><span data-stu-id="0609b-110">A line device contains a set of one or more homogeneous channels that can be used to establish calls.</span></span>

<span data-ttu-id="0609b-111">Dans les applications TAPI 2. x, un appareil de ligne est la représentation logique d’un appareil téléphonique physique.</span><span class="sxs-lookup"><span data-stu-id="0609b-111">Within TAPI 2.x applications, a line device is the logical representation of a physical telephone device.</span></span> <span data-ttu-id="0609b-112">Bien que « Line » contourne souvent un élément avec deux points de terminaison, il est possible d’abstraire un appareil de ligne à un point unique, car l’interface TAPI l’affiche uniquement comme point d’entrée de la ligne menant au commutateur.</span><span class="sxs-lookup"><span data-stu-id="0609b-112">Although "line" often connotes something with two endpoints, it is possible to abstract a line device to a single point because TAPI views it only as a point of entry to the line that leads to the switch.</span></span>

<span data-ttu-id="0609b-114">Bien que les trois lignes de l’illustration précédente soient composées de matériel différent et utilisées pour différentes fonctions, elles sont extraites du même type de périphérique et régies par les mêmes règles.</span><span class="sxs-lookup"><span data-stu-id="0609b-114">Although the three lines in the preceding illustration are composed of different hardware and used for different functions, they are abstracted to the same device type and governed by the same rules.</span></span> <span data-ttu-id="0609b-115">Le téléphone ne représente pas un appareil téléphonique, mais un appareil de ligne utilisé pour les appels vocaux.</span><span class="sxs-lookup"><span data-stu-id="0609b-115">The telephone represents not a phone device but a line device used for voice calls.</span></span> <span data-ttu-id="0609b-116">Lors de l’utilisation de ce périphérique de ligne pour les appels entrants ou sortants, l’application doit également ouvrir et contrôler une instance de la classe de périphérique téléphonique, qui est décrite en détail dans les sections suivantes.</span><span class="sxs-lookup"><span data-stu-id="0609b-116">When using this line device for incoming or outgoing calls, the application would also need to open and control an instance of the phone-device class, which is described in detail in later sections.</span></span>

<span data-ttu-id="0609b-117">La classe de périphérique en ligne est une représentation indépendante du périphérique d’un périphérique de ligne physique, tel qu’un modem.</span><span class="sxs-lookup"><span data-stu-id="0609b-117">The line device class is a device-independent representation of a physical line device, such as a modem.</span></span> <span data-ttu-id="0609b-118">Il peut contenir un ou plusieurs canaux de communication identiques (utilisés pour la signalisation et/ou les informations) entre l’application et le commutateur ou le réseau.</span><span class="sxs-lookup"><span data-stu-id="0609b-118">It can contain one or more identical communications channels (used for signaling and/or information) between the application and the switch or network.</span></span> <span data-ttu-id="0609b-119">Étant donné que les canaux appartenant à une seule ligne ont des fonctionnalités identiques, ils sont interchangeables.</span><span class="sxs-lookup"><span data-stu-id="0609b-119">Because channels belonging to a single line have identical capabilities, they are interchangeable.</span></span> <span data-ttu-id="0609b-120">Dans de nombreux cas (comme avec les POTS), un fournisseur de services modélise une ligne comme n’ayant qu’un seul canal.</span><span class="sxs-lookup"><span data-stu-id="0609b-120">In many cases (as with POTS), a service provider will model a line as having only one channel.</span></span> <span data-ttu-id="0609b-121">D’autres technologies, comme RNIS, offrent davantage de canaux et le fournisseur de services doit les traiter en conséquence.</span><span class="sxs-lookup"><span data-stu-id="0609b-121">Other technologies, like ISDN, offer more channels, and the service provider should treat them accordingly.</span></span>

<span data-ttu-id="0609b-122">**TAPI 2. x :** Les applications découvrent les fonctionnalités de ligne à l’aide de la fonction [**lineGetDevCaps**](/windows/win32/api/tapi/nf-tapi-linegetdevcaps) .</span><span class="sxs-lookup"><span data-stu-id="0609b-122">**TAPI 2.x:** Applications discover line capabilities using the [**lineGetDevCaps**](/windows/win32/api/tapi/nf-tapi-linegetdevcaps) function.</span></span> <span data-ttu-id="0609b-123">La négociation de version à l’aide des fonctions [**lineNegotiateAPIVersion**](/windows/win32/api/tapi/nf-tapi-linenegotiateapiversion) [**lineNegotiateExtVersion**](/windows/win32/api/tapi/nf-tapi-linenegotiateextversion) doit avoir été appelée précédemment.</span><span class="sxs-lookup"><span data-stu-id="0609b-123">Version negotiation using the [**lineNegotiateAPIVersion**](/windows/win32/api/tapi/nf-tapi-linenegotiateapiversion) [**lineNegotiateExtVersion**](/windows/win32/api/tapi/nf-tapi-linenegotiateextversion) functions must have been previously called.</span></span>
