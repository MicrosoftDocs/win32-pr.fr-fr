# <a name="file-terminals"></a>Terminaux de fichiers

Dans TAPI 3,1, l’objet terminal est développé pour inclure un terminal de fichiers. Les terminaux de fichiers offrent un moyen simple d’effectuer des opérations courantes, par exemple l’enregistrement d’un flux audio ou de flux dans un fichier ou la lecture ultérieure de l’audio enregistré dans un flux.

Cinq objets de base sont utilisés pour manipuler les terminaux de fichiers :

-   Terminal de lecture de fichier
-   Terminal d’enregistrement de fichier
-   Piste de lecture de fichier
-   Piste d’enregistrement de fichier
-   Événement de fichier

Les objets terminaux de fichiers peuvent être considérés comme une abstraction pour le stockage, qui est généralement un fichier. Les pistes correspondent aux flux de données audio individuels dans ce stockage, par exemple lors de l’enregistrement de la voix.

Pour plus d’informations sur les terminaux de fichiers, consultez :

-   [Terminal de lecture de fichier et terminal d’enregistrement de fichier](file-playback-terminal-and-file-recording-terminal.md)
-   [Suivre les terminaux](track-terminals.md)

Pour plus d’informations et pour obtenir des exemples de code et des listes de tâches liés aux terminaux de fichiers, consultez [utilisation de terminaux de fichiers](using-file-terminals.md).
