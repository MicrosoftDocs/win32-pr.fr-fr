# <a name="error-checking"></a>Vérification des erreurs

Au niveau de l’interface TAPI, une application peut passer un grand nombre de paramètres différents, dont la plupart peuvent être non valides. TAPI vérifie les paramètres et retourne les erreurs à l’application sans appeler le fournisseur de services. Chaque description de fonction au niveau TSPI décrit les erreurs de paramètre déjà testées. Le fournisseur de services n’a pas besoin de répéter ces tests, même s’il doit effectuer des tests de validité supplémentaires appropriés à la fonction. Les titres et les descriptions des tests de validité de paramètres courants qui apparaissent dans de nombreuses fonctions sont répertoriés dans le tableau suivant.



| Test de validité       | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|---------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Validité du pointeur    | TAPI a déjà testé les pointeurs vers le stockage de données pour s’assurer qu’ils pointent vers une mémoire lisible ou accessible en écriture de la taille appropriée pour l’opération. En outre, pour les structures de données de taille variable commençant par un membre **dwTotalSize** , la structure de données a été vérifiée pour s’assurer que la taille totale indiquée est disponible.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Validité de la taille fixe | Pour les structures de données de taille variable, la structure de données a été vérifiée pour obtenir de l’espace pour la partie de taille fixe de la structure de données et cette **dwTotalSize** est suffisante pour la partie fixe.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Décalage/taille zéro  | Pour les structures de données de taille variable, le «... **Décalage**« et »... **Size**» les champs qui correspondent aux parties que le fournisseur de services définit ont été prédéfinis avec des valeurs null avant l’appel du fournisseur de services.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Validité du handle     | L’interface TAPI garantit que les handles de ligne, de téléphone et d’appel (des types définis [HDRVLINE](hdrvline.md), [HDRVPHONE](hdrvphone.md)et HDRVCALL) sont valides. Autrement dit, il s’agit de valeurs qui ont été retournées sans erreur en tant que handles dans [**TSPI \_ lineOpen**](/windows/win32/api/tspi/nf-tspi-tspi_lineopen), [**TSPI \_ phoneOpen**](/windows/win32/api/tspi/nf-tspi-tspi_phoneopen)ou l’un des éléments suivants qui démarre la durée de vie d’un handle d’appel : [**TSPI \_ lineMakeCall**](/windows/win32/api/tspi/nf-tspi-tspi_linemakecall)<br/> [**TSPI \_ lineCompleteTransfer**](/windows/win32/api/tspi/nf-tspi-tspi_linecompletetransfer)<br/> [**TSPI \_ lineForward**](/windows/win32/api/tspi/nf-tspi-tspi_lineforward)<br/> [**TSPI \_ linePickup**](/windows/win32/api/tspi/nf-tspi-tspi_linepickup)<br/> [**TSPI \_ linePrepareAddToConference**](/windows/win32/api/tspi/nf-tspi-tspi_lineprepareaddtoconference)<br/> [**TSPI \_ lineSetupConference**](/windows/win32/api/tspi/nf-tspi-tspi_linesetupconference)<br/> [**TSPI \_ lineSetupTransfer**](/windows/win32/api/tspi/nf-tspi-tspi_linesetuptransfer)<br/> [**TSPI \_ lineUnpark**](/windows/win32/api/tspi/nf-tspi-tspi_lineunpark)<br/> [**Ligne \_ Messages NEWCALL**](line-newcall.md)<br/> |
