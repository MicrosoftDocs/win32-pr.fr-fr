## <a name="remarks"></a>Remarques

Aucune extensibilité. Tous les 32 bits sont réservés.

Ces constantes sont utilisées dans [**LINEADDRESSCAPS**](/windows/desktop/api/Tapi/ns-tapi-lineaddresscaps) (retournées par [**lineGetAddressCaps**](/windows/desktop/api/Tapi/nf-tapi-linegetaddresscaps)) et dans [**LINECALLSTATUS**](/windows/desktop/api/Tapi/ns-tapi-linecallstatus) (retournées par [**lineGetCallStatus**](/windows/desktop/api/Tapi/nf-tapi-linegetcallstatus)). **LINEADDRESSCAPS** signale la disponibilité des fonctionnalités d’appel sur l’adresse spécifiée. Une application utilise ces informations lors de son initialisation afin de déterminer ce qu’elle peut faire ultérieurement lorsque des appels existent. Pour l’appel spécifié, **LINECALLSTATUS** signale les fonctionnalités d’appel qui peuvent être appelées pendant que l’appel est à l’état d’appel actuel. Ce dernier prend des privilèges d’appel en compte. Une application effectue cette détermination de manière dynamique, après modification de l’état de l’appel.

La \_ valeur LINECALLFEATURE RELEASEUSERUSERINFO est une nouveauté de TAPI 1,4. Il n’existe pas de considérations relatives à la compatibilité descendante. Un fournisseur de services peut choisir de retourner cette valeur dans les membres pertinents (dans [**LINEADDRESSCAPS**](/windows/desktop/api/Tapi/ns-tapi-lineaddresscaps) et [**LINECALLSTATUS**](/windows/desktop/api/Tapi/ns-tapi-linecallstatus)) même lorsque des versions antérieures de l’API ont été négociées sur le périphérique de ligne.
