# <a name="session-control"></a>Contrôle de session

Une session ou un appel représente une connexion entre deux adresses ou plus. Cette connexion est dynamique et les objets de programmation associés doivent être créés, manipulés et libérés si nécessaire. Dans le cas le plus simple, cela signifie créer et déconnecter un appel téléphonique. Pour les applications avancées, le contrôle de session peut impliquer la gestion de conférences multimédias sur un réseau IP jusqu’au niveau des participants individuels.

Le contrôle de session implique deux domaines de base :

-   Les [opérations de session](session-operations-ovr.md) sont des contrôles qui initient, maintiennent et mettent fin à une session de communication.
-   Les [informations de session](session-information-ovr.md) décrivent les données qui peuvent être obtenues concernant une session de communication.

Les deux types spécialisés de contrôle de session ne sont pas abordés dans cette section. Pour plus d’informations sur les centres de distribution d’appels automatiques, consultez [contrôle de centre d’appels](./call-center-control.md) (TAPI 2. x) ou [à propos des contrôles de centre d’appels](about-call-center-controls.md) (TAPI 3. x). Pour plus d’informations sur la conférence IP, consultez [à propos de la Conférence de téléphonie IP Rendezvous](about-rendezvous-ip-telephony-conferencing.md).
