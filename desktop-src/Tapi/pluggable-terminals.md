# <a name="pluggable-terminals"></a>Terminaux enfichables

Les terminaux enfichables permettent à une application de créer des terminaux au moment de l’exécution. Les terminaux sont normalement créés et contrôlés par une paire fournisseur de services de téléphonie/fournisseur de services de média (TSP/MSP), ce qui reste le moyen le plus polyvalent et le plus stable de manipuler les terminaux. Toutefois, certaines applications doivent utiliser des terminaux simples dans les situations où une paire TSP/MSP appropriée n’est pas disponible. Seuls les créateurs d’applications qui sont familiarisés avec les opérations TSP/MSP doivent utiliser des terminaux enfichables.

Dans TAPI version 3,1, les terminaux enfichables sont ajoutés au modèle TAPI. Les terminaux enfichables permettent à un tiers d’ajouter un nouvel objet terminal que n’importe quel MSP peut utiliser.

Pour plus d’informations, consultez [inscription de terminal enfichable](pluggable-terminal-registration.md) et [écriture d’un terminal enfichable](writing-a-pluggable-terminal.md) .
