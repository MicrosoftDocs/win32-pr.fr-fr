# <a name="s-rpc"></a>S (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) [D](d-glos.md) [e](e-glos.md) [F](f-glos.md) G H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) S [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="rpc.s_glos_1_gly"></span><span id="RPC.S_GLOS_1_GLY"></span>**évolutivité**
</dt> <dd>

Possibilité d’utiliser pleinement la puissance de traitement supplémentaire sur un système multiprocesseur (2, 4, 8, 32 ou plus). L’évolutivité est également la possibilité de traiter un grand nombre de clients.

</dd> <dt>

<span id="_rpcl_spp_glos"></span><span id="_RPCL_SPP_GLOS"></span>**Protocole de paquets séquencés (SPP)**
</dt> <dd>

Protocole de communication [*orienté connexion*](c-glos.md) Banyan VINES pour le routage des paquets d’informations sur des réseaux locaux.

</dd> <dt>

<span id="_rpc_spx_glos"></span><span id="_RPC_SPX_GLOS"></span>**Exchange de paquets séquencés (SPX)**
</dt> <dd>

Protocole de communication [*orienté connexion*](c-glos.md) Novell NetWare pour le routage des paquets d’informations sur des réseaux locaux et des réseaux étendus.

</dd> <dt>

<span id="_rpc_serialization_glos"></span><span id="_RPC_SERIALIZATION_GLOS"></span>**sérialisation**
</dt> <dd>

Processus de marshaling de données (encodage) et démarshaling de données à partir de mémoires tampons que vous contrôlez. Cela diffère de l’utilisation traditionnelle de RPC, où les stubs et le runtime RPC contrôlent les mémoires tampons de marshaling. Également appelé « picking ». Voir aussi [*sérialisation de procédure*](p-glos.md), [*sérialisation de type*](t-glos.md).

</dd> <dt>

<span id="_rpc_server_stub_glos"></span><span id="_RPC_SERVER_STUB_GLOS"></span>**stub serveur**
</dt> <dd>

Code source du langage C généré par MIDL qui contient toutes les fonctions nécessaires à l’application serveur pour gérer les demandes distantes à l’aide d’appels de procédure locale. Voir aussi [*stub client*](c-glos.md).

</dd> <dt>

<span id="_rpc_session_glos"></span><span id="_RPC_SESSION_GLOS"></span>**session**
</dt> <dd>

Relation établie entre une application cliente et une application serveur. Voir aussi [*lier*](b-glos.md), [*handle de liaison*](b-glos.md).

</dd> <dt>

<span id="_rpc_static_callback_function_glos"></span><span id="_RPC_STATIC_CALLBACK_FUNCTION_GLOS"></span>**fonction de rappel statique**
</dt> <dd>

Procédure distante qui fait partie du côté client d’une application distribuée, qu’un serveur peut appeler pour obtenir des informations à partir du client. L' \[ attribut [callback](/windows/desktop/Midl/callback) \] désigne une fonction de rappel statique.

</dd> <dt>

<span id="_rpc_static_identity_tracking_glos"></span><span id="_RPC_STATIC_IDENTITY_TRACKING_GLOS"></span>**suivi des identités statiques**
</dt> <dd>

Le suivi des identités statiques spécifie que le runtime RPC utilise les informations d’identification de sécurité dans le handle de liaison du client pour tous les appels RPC. Voir aussi [*suivi dynamique des identités*](d-glos.md).

</dd> <dt>

<span id="_rpc_string_binding_glos"></span><span id="_RPC_STRING_BINDING_GLOS"></span>**liaison de chaîne**
</dt> <dd>

Chaîne de caractères qui se compose de l' [*UUID*](u-glos.md)d’objet, de la [*séquence de protocole*](p-glos.md), de l' [*adresse réseau*](n-glos.md), du point de [*terminaison*](e-glos.md)et des options de point de terminaison, qui peuvent tous être utilisés pour créer un [*handle de liaison*](b-glos.md) vers le serveur spécifié.

</dd> <dt>

<span id="_rpc_strong_typing_glos"></span><span id="_RPC_STRONG_TYPING_GLOS"></span>**typage fort**
</dt> <dd>

Le compilateur applique un contrôle strict sur les types de données. Dans MIDL et RPC, le typage fort est utilisé pour s’assurer que les données sont interprétées de façon cohérente par différents ordinateurs dans un environnement distribué.

</dd> </dl>
