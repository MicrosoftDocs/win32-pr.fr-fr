# <a name="using-asynchronous-rpc-with-dce-pipes"></a>Utilisation de RPC asynchrone avec les canaux DCE

Microsoft RPC prend en charge l’utilisation de canaux DCE. Bien qu’ils partagent un nom similaire, les canaux DCE ne sont pas liés aux [canaux nommés](asynchronous-rpc-over-the-named-pipe-protocol.md). Les canaux nommés sont un protocole de transport. Les canaux DCE sont une méthode indépendante du protocole de communication client/serveur.

Cette section présente une discussion de RPC à l’aide de canaux DCE asynchrones. Elle est composée des rubriques suivantes :

-   [Canaux asynchrones](asynchronous-pipes.md)
-   [Déclaration de canaux asynchrones](declaring-asynchronous-pipes.md)
-   [Gestion de canal asynchrone côté client](client-side-asynchronous-pipe-handling.md)
-   [Gestion de canal asynchrone côté serveur](server-side-asynchronous-pipe-handling.md)

Pour obtenir un exemple d’application RPC de canal asynchrone, consultez l’exemple FileRep dans le kit de développement logiciel (SDK) de la plateforme.
