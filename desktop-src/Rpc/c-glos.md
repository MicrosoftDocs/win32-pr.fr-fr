<dl> <dt>

<span id="_rpc_cds_glos"></span><span id="_RPC_CDS_GLOS"></span>**Service d’annuaire de cellules/service d’annuaires de cellules (CDS)**
</dt> <dd>

Nom : fournisseur de services de l’environnement informatique distribué Open Software Foundation.

</dd> <dt>

<span id="_rpc_client_stub_glos"></span><span id="_RPC_CLIENT_STUB_GLOS"></span>**stub client**
</dt> <dd>

Code source du langage C généré par MIDL. Il contient toutes les fonctions nécessaires à l’application cliente pour effectuer des appels de procédure distante à l’aide du modèle d’appel de fonction traditionnel dans une application autonome. Le stub client est chargé de marshaler les paramètres d’entrée et de démarshaler les paramètres de sortie. Voir aussi [*stub de serveur*](s-glos.md), [*stub proxy*](p-glos.md).

</dd> <dt>

<span id="_rpc_conformant_array_glos"></span><span id="_RPC_CONFORMANT_ARRAY_GLOS"></span>**Tableau conforme**
</dt> <dd>

Tableau dont la taille est déterminée au moment de l’exécution par un autre paramètre, un champ de structure ou une expression.

</dd> <dt>

<span id="_rpc_connection_oriented_glos"></span><span id="_RPC_CONNECTION_ORIENTED_GLOS"></span>**orienté connexion**
</dt> <dd>

Protocole ou transport de communication qui fournit un circuit virtuel par le biais duquel les paquets de données sont reçus dans le même ordre que celui dans lequel ils ont été transmis. Si la connexion entre les ordinateurs échoue, l’application est avertie. [*TCP*](t-glos.md) et [*SPX*](s-glos.md) sont des exemples de protocoles orientés connexion. Voir aussi [*datagramme*](d-glos.md).

</dd> <dt>

<span id="_rpc_connectionless_glos"></span><span id="_RPC_CONNECTIONLESS_GLOS"></span>**sans connexion**
</dt> <dd>

Consultez [*datagramme*](d-glos.md).

</dd> <dt>

<span id="_rpc_context_rundown_glos"></span><span id="_RPC_CONTEXT_RUNDOWN_GLOS"></span>**arrêt du contexte**
</dt> <dd>

Notification de serveur qui résulte d’une fin inattendue de la liaison entre les applications client et serveur.

</dd> </dl>
