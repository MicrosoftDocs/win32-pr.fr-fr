# <a name="asynchronous-dcom"></a>DCOM asynchrone

Pour tirer parti du RPC asynchrone en toute transparence, les programmeurs COM peuvent utiliser les \* interfaces Async générées par le compilateur MIDL, comme décrit dans la rubrique d’attribut MIDL, [**\_ UUID Async**](../midl/async-uuid.md). Pour plus d’informations, consultez [clients et serveurs com](../com/com-clients-and-servers.md),[création et traitement des appels asynchrones](../com/making-and-processing-asynchronous-calls.md), et [appels de méthode asynchrones dans les clients et les serveurs com](/archive/msdn-magazine/2000/april/windows-2000-asynchronous-method-calls-eliminate-the-wait-for-com-clients-and-servers).
