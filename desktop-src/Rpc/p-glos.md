<dl> <dt>

<span id="_rpc_partially_bound_handle_glos"></span><span id="_RPC_PARTIALLY_BOUND_HANDLE_GLOS"></span>**handle partiellement lié**
</dt> <dd>

[*Handle de liaison*](b-glos.md) qui n’inclut pas les informations de [*point de terminaison*](e-glos.md) . Voir aussi [*handle entièrement lié*](f-glos.md).

</dd> <dt>

<span id="_rpc_pickling_glos"></span><span id="_RPC_PICKLING_GLOS"></span>**sélection**
</dt> <dd>

Consultez [*sérialisation*](s-glos.md).

</dd> <dt>

<span id="_rpc_pipe_glos"></span><span id="_RPC_PIPE_GLOS"></span>**conduit**
</dt> <dd>

Constructeur de type [*IDL*](i-glos.md)qui prend en charge la transmission d’un flux de données ouvert et ouvert entre les applications clientes et serveur.

</dd> <dt>

<span id="_rpc_primary_enterprise_controller_glos"></span><span id="_RPC_PRIMARY_ENTERPRISE_CONTROLLER_GLOS"></span>**contrôleur d’entreprise principal**
</dt> <dd>

Base de données principale MQIS (message queue Information Service) pour un réseau. Voir aussi [*service d’informations de file d’attente de messages (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_primary_site_controller_glos"></span><span id="_RPC_PRIMARY_SITE_CONTROLLER_GLOS"></span>**contrôleur de site principal**
</dt> <dd>

Base de données MQIS (message queue Information Service) pour un site particulier au sein d’une entreprise. Voir aussi [*service d’informations de file d’attente de messages (MQIS)*](m-glos.md).

</dd> <dt>

<span id="_rpc_procedure_serialization_glos"></span><span id="_RPC_PROCEDURE_SERIALIZATION_GLOS"></span>**sérialisation de procédure**
</dt> <dd>

[*Sérialisation*](s-glos.md) de données qui utilise un stub de sérialisation généré par MIDL pour effectuer l’encodage et le décodage d’un ou plusieurs types avec un appel de procédure unique. La sérialisation de procédure s’effectue en appliquant les \[ [](/windows/desktop/Midl/encode) \] attributs encode et \[ [Decode](/windows/desktop/Midl/decode) \] à un prototype de fonction dans le fichier ACF. Voir aussi [*sérialisation de type*](t-glos.md).

</dd> <dt>

<span id="_rpc_protocol_sequence_glos"></span><span id="_RPC_PROTOCOL_SEQUENCE_GLOS"></span>**séquence de protocole**
</dt> <dd>

Chaîne de caractères qui représente une combinaison valide d’un protocole RPC, d’un protocole de couche réseau et d’un protocole de couche de transport. Par exemple, la séquence de protocole NCACN \_ IP \_ TCP décrit une connexion [*NCA*](n-glos.md) sur un protocole Internet (IP) avec un protocole TCP (Transmission Control Protocol) en tant que transport.

</dd> <dt>

<span id="_rpc_proxy_stub_glos"></span><span id="_RPC_PROXY_STUB_GLOS"></span>**stub proxy**
</dt> <dd>

Langage C ou C++ généré par MIDL : code source qui contient toutes les fonctions nécessaires pour une interface OLE personnalisée.

</dd> </dl>
