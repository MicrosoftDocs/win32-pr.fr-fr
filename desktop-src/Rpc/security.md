# <a name="security-rpc"></a>Sécurité (RPC)

Avec l’utilisation accrue des applications distribuées, le besoin de communications sécurisées entre les parties client et serveur des applications est primordial. La bibliothèque Runtime de procédure distante (RPC) fournit une interface standardisée aux services d’authentification pour les clients et les serveurs. Les services d’authentification sur le système hôte du serveur fournissent l’authentification RPC. Les applications utilisent des appels de procédure distante authentifiés pour s’assurer que tous les appels proviennent de clients autorisés. Ils permettent également de s’assurer que toutes les réponses de serveur proviennent de serveurs authentifiés.

Cette section présente une vue d’ensemble des RPC authentifiés dans les sections suivantes :

-   [Notions fondamentales de la sécurité RPC](rpc-security-essentials.md)
-   [Méthodes de sécurité](security-methods.md)
