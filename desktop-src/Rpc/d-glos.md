# <a name="d-rpc"></a>D (RPC)

[A](a-glos.md) [B](b-glos.md) [C](c-glos.md) D [e](e-glos.md) [F](f-glos.md) G H [I](i-glos.md) J K [L](l-glos.md) [M](m-glos.md) [N](n-glos.md) [O](o-glos.md) [P](p-glos.md) [Q](q.md) [R](r-glos.md) [S](s-glos.md) [T](t-glos.md) [U](u-glos.md) [V](v-glos.md) [W](w-glos.md) X Y Z

<dl> <dt>

<span id="_rpc_datagram_glos"></span><span id="_RPC_DATAGRAM_GLOS"></span>**reçus**
</dt> <dd>

Protocole de communication ou transport dans lequel les paquets de données sont acheminés indépendamment. Ils peuvent suivre différents itinéraires et arriver dans un ordre différent de celui dans lequel ils ont été envoyés. [*UDP*](u-glos.md) et [*IPX*](i-glos.md) sont des exemples de protocoles de couche de transport. Voir aussi [*orienté connexion*](c-glos.md).

</dd> <dt>

<span id="_rpc_discriminant_glos"></span><span id="_RPC_DISCRIMINANT_GLOS"></span>**discriminante**
</dt> <dd>

Variable qui spécifie les types de données qui peuvent être stockés dans une Union.

</dd> <dt>

<span id="_rpc_discriminated_union_glos"></span><span id="_RPC_DISCRIMINATED_UNION_GLOS"></span>**Union discriminée**
</dt> <dd>

(ou *enregistrement variant*) Union qui comprend un discriminateur dans le cadre de la structure de données afin que le type de données actuellement valide soit transmis avec l’Union. Voir aussi Union [*encapsulée*](e-glos.md), Union qui n’est pas [*encapsulée*](n-glos.md).

</dd> <dt>

<span id="_rpc_dce_glos"></span><span id="_RPC_DCE_GLOS"></span>**Environnement de calcul distribué (DCE)**
</dt> <dd>

Spécification de la Fondation logicielle ouverte pour un ensemble de services intégrés, notamment les appels de procédure distante, les systèmes de fichiers distribués et les services de sécurité. La norme RPC OSF-DCE est la base de Microsoft RPC.

</dd> <dt>

<span id="_rpc_dynamic_endpoint_glos"></span><span id="_RPC_DYNAMIC_ENDPOINT_GLOS"></span>**point de terminaison dynamique**
</dt> <dd>

Spécifique au réseau : adresse de serveur demandée et assignée au moment de l’exécution. Voir aussi [*point de terminaison connu*](w-glos.md).

</dd> <dt>

<span id="_rpc_dynamic_identity_tracking_glos"></span><span id="_RPC_DYNAMIC_IDENTITY_TRACKING_GLOS"></span>**suivi dynamique des identités**
</dt> <dd>

Spécification que la bibliothèque Runtime RPC utilisera les informations d’identification du thread appelant, plutôt que le handle de liaison, pour l’authentification chaque fois que le client appelle une procédure distante. Voir aussi [*suivi statique des identités*](s-glos.md).

</dd> </dl>
