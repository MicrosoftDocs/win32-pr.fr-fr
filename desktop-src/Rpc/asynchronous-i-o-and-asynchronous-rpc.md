# <a name="asynchronous-io-and-asynchronous-rpc"></a><span data-ttu-id="fa894-103">E/s asynchrones et RPC asynchrone</span><span class="sxs-lookup"><span data-stu-id="fa894-103">Asynchronous I/O and Asynchronous RPC</span></span>

<span data-ttu-id="fa894-104">Les e/s asynchrones sont un moyen efficace pour qu’un seul thread gère simultanément plusieurs demandes d’e/s.</span><span class="sxs-lookup"><span data-stu-id="fa894-104">Asynchronous I/O is an efficient means for a single thread to manage multiple I/O requests simultaneously.</span></span> <span data-ttu-id="fa894-105">Le RPC asynchrone sur le serveur remplit un rôle similaire pour les demandes RPC.</span><span class="sxs-lookup"><span data-stu-id="fa894-105">Asynchronous RPC on the server accomplishes a similar purpose for RPC requests.</span></span> <span data-ttu-id="fa894-106">Dans les versions de Windows antérieures à Windows Vista, la publication de requêtes d’e/s asynchrones à partir de procédures serveur à l’aide de RPC asynchrone est déconseillée.</span><span class="sxs-lookup"><span data-stu-id="fa894-106">In versions of Windows prior to Windows Vista, posting asynchronous I/O requests from server procedures using asynchronous RPC is discouraged.</span></span> <span data-ttu-id="fa894-107">Toutefois, dans Windows Vista et les versions ultérieures de Windows, les demandes d’e/s asynchrones associées à un port de terminaison d’e/s sont prises en charge par RPC asynchrone.</span><span class="sxs-lookup"><span data-stu-id="fa894-107">However, in Windows Vista and later versions of Windows, asynchronous I/O requests that are associated with an I/O completion port are supported by asynchronous RPC.</span></span>

<span data-ttu-id="fa894-108">Avant Windows Vista, un appel de procédure distante asynchrone peut se terminer avant la fin de la requête d’e/s asynchrone.</span><span class="sxs-lookup"><span data-stu-id="fa894-108">Prior to Windows Vista, an asynchronous remote procedure call may complete before the asynchronous I/O request completes.</span></span> <span data-ttu-id="fa894-109">Lorsque l’appel asynchrone est terminé, son thread peut se terminer si le runtime RPC décide qu’il a suffisamment de threads disponibles pour traiter la charge de travail attendue.</span><span class="sxs-lookup"><span data-stu-id="fa894-109">When the asynchronous call completes, its thread may terminate if the RPC runtime decides it has enough threads available to service the expected workload.</span></span> <span data-ttu-id="fa894-110">Le système lie toutes les demandes d’e/s au thread qui les initie.</span><span class="sxs-lookup"><span data-stu-id="fa894-110">The system binds all I/O requests to the thread that initiates them.</span></span> <span data-ttu-id="fa894-111">Si le thread se termine, toutes les demandes d’e/s en attente sur ce thread sont abandonnées.</span><span class="sxs-lookup"><span data-stu-id="fa894-111">If the thread terminates, any I/O requests pending on that thread are aborted.</span></span> <span data-ttu-id="fa894-112">Les demandes d’e/s en attente ne peuvent pas être déplacées vers un autre thread.</span><span class="sxs-lookup"><span data-stu-id="fa894-112">Pending I/O requests cannot be moved to another thread.</span></span>

<span data-ttu-id="fa894-113">Par conséquent, les concepteurs d’applications ciblant des versions de Windows antérieures à Windows Vista peuvent utiliser des e/s synchrones dans les procédures serveur ou transférer toutes les requêtes qui impliquent des e/s asynchrones à des procédures qui s’exécutent sur un pool de threads géré par l’application.</span><span class="sxs-lookup"><span data-stu-id="fa894-113">Therefore, application designers targeting versions of Windows prior to Windows Vista can either use synchronous I/O in server procedures, or they can forward all requests that involve asynchronous I/O to procedures executing on a thread pool that the application manages.</span></span> <span data-ttu-id="fa894-114">L’API Windows fournit des fonctions pour la gestion des pools de threads.</span><span class="sxs-lookup"><span data-stu-id="fa894-114">The Windows API provides functions for thread-pool management.</span></span> <span data-ttu-id="fa894-115">Consultez [fonctions de processus et de thread](/windows/desktop/ProcThread/process-and-thread-functions).</span><span class="sxs-lookup"><span data-stu-id="fa894-115">See [Process and Thread Functions](/windows/desktop/ProcThread/process-and-thread-functions).</span></span>
