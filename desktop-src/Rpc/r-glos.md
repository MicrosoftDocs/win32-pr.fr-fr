# <a name="r-rpc"></a>R (RPC)

<dl> <dt>

<span id="_rpc_rpc_object_glos"></span><span id="_RPC_RPC_OBJECT_GLOS"></span>**Objet RPC**
</dt> <dd>

Les instances de serveur ou d’autres ressources, telles que les appareils, les bases de données et les files d’attente, qui sont exploitées et gérées par les applications serveur RPC. Chaque objet est identifié de manière unique par un ou plusieurs UUID d’objet.

</dd> <dt>

<span id="_rpc_rpcss_glos"></span><span id="_RPC_RPCSS_GLOS"></span>**Sous-système RPC (RPCSS)**
</dt> <dd>

Windows sous-système qui inclut divers services RPC et ole, y compris le mappeur de point de terminaison, le gestionnaire de contrôle des services ole et le programme de résolution d’objets COM. Ne confondez pas cela avec le package d’allocateur de mémoire spécifique à RPC, RpcSs.

</dd> <dt>

<span id="_rpc_reference_pointer_glos"></span><span id="_RPC_REFERENCE_POINTER_GLOS"></span>**pointeur de référence**
</dt> <dd>

Type de pointeur le plus simple. Un pointeur de référence pointe toujours vers un stockage valide et ce stockage ne change pas (bien que le contenu puisse changer). Un pointeur de référence ne peut pas être un alias. L' \[ attribut [ref](/windows/desktop/Midl/ref) \] désigne un pointeur de référence. Voir aussi [*pointeur unique*](u-glos.md) et [*pointeur complet*](f-glos.md).

</dd> </dl>
