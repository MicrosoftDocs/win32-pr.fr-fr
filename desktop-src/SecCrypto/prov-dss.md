# <a name="prov_dss"></a>DSS PROVen \_

Le \_ type de fournisseur DSS Proven, comme Proven \_ RSA \_ SIG, prend uniquement en charge les [*hachages*](../secgloss/h-gly.md) et les [*signatures numériques*](../secgloss/d-gly.md). L’algorithme de signature spécifié par le \_ type de fournisseur DSS Provy est l' [*algorithme de signature numérique*](../secgloss/d-gly.md) (DSA).

## <a name="algorithms-supported"></a>Algorithmes pris en charge

Pour obtenir une description de chacun de ces algorithmes, consultez le Glossaire.



| Objectif      | Algorithmes pris en charge                                                                                                              |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------|
| Exchange de clé | Aucun                                                                                                                              |
| Signature    | [*NORME*](../secgloss/d-gly.md)                                                                       |
| Chiffrement   | Aucun                                                                                                                              |
| Hashing      | [](../secgloss/m-gly.md)[*Hachage* MD5](../secgloss/s-gly.md)<br/> |



 

## <a name="related-documentation"></a>Documentations associées

Le DSA a été proposé par le [*National Institute of Standards and Technology*](../secgloss/n-gly.md) (NIST). Vous trouverez une description de l’algorithme dans la référence gouvernementale suivante : « normes FIPS (Federal Information Processing Standard) proposé pour la norme DSS (Digital Signature standard) », « registre fédéral, v ». 57, non. 21, 31 Jan 1992, pp. 3747-3749.
