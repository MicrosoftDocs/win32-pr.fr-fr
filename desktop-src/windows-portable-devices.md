# <a name="windows-portable-devices"></a>Appareils mobiles Windows

Les appareils mobiles Windows (WPD) permettent aux ordinateurs de communiquer avec les périphériques de stockage et les supports associés. WPD offre une méthode flexible et fiable pour les ordinateurs qui communiquent avec les lecteurs musicaux, les périphériques de stockage, les téléphones mobiles, les appareils photo et de nombreux autres types d’appareils connectés. Ce système remplace à la fois Windows Media Gestionnaire de périphériques et Windows image acquisition.

Les applications basées sur WPD peuvent explorer un appareil, envoyer et recevoir du contenu, et même contrôler l’appareil, par exemple, prendre une photo ou envoyer un SMS. Le système est conçu pour être flexible afin que de nombreux types d’appareils puissent être explorés et extensibles afin que les développeurs de pilotes puissent définir des propriétés personnalisées et des commandes pour les appareils personnalisés.

Vous pouvez écrire des applications Windows et des applications Web avec WPD. Vous utilisez l’interface de programmation d’applications WPD pour créer des applications Windows. Ces applications peuvent être écrites en C++, C \# .net ou Visual Basic .net. Vous utilisez le modèle objet Automation WPD pour écrire des applications Web. Ces applications sont écrites en JScript et HTML.

L’interface de programmation d’applications WPD est prise en charge dans les systèmes d’exploitation Windows 7, Windows Vista et Windows XP. Le modèle objet Automation WPD est pris en charge uniquement dans Windows 7.

Pour plus d’informations sur la création d’une application Windows WPD, reportez-vous à la documentation et aux exemples de l' [interface de programmation d’applications wpd](./wpd_sdk/wpd-application-programming-interface.md) .

Pour plus d’informations sur la création d’une application Automation WPD, reportez-vous à la documentation et aux exemples du [modèle objet Automation wpd](/previous-versions/windows/desktop/legacy/dd389295(v=vs.85)) .
