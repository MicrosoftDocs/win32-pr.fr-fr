# <a name="language-names"></a>Noms des langues

Un nom de langue utilise essentiellement la même mise en forme que le [nom des paramètres régionaux](locale-names.md). Par exemple, le nom de langue pour les paramètres régionaux anglais (Belize) est « en-via ». Chaque nom de langue correspond à un [identificateur de langue](language-identifiers.md).

Les principales différences entre la mise en forme d’un nom de langue et la mise en forme d’un nom de paramètres régionaux sont les suivantes :

-   Un nom de langue n’intègre jamais un ordre de tri. Par conséquent, le nom de langue correspondant au nom de paramètres régionaux « HU-HU \_ technl » (hongrois Language, Hongrie, avec le tri technique) est simplement « HU-HU ».
-   Un nom de langue peut être « neutre » en ce qui concerne les paramètres régionaux. Par exemple, « en » (anglais) est un nom de langage neutre valide, comme distinct de « en-via » (anglais, Belize).
