# <a name="locale_name-constants"></a>\_Constantes de nom de paramètres régionaux \*

Cette rubrique définit les constantes de nom de paramètres régionaux \_ \* utilisées par nls.



<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr class="header">
<th>Valeur</th>
<th>Signification</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td>LOCALE_NAME_INVARIANT</td>
<td>Nom d’un paramètre régional indifférent qui fournit des données de paramètres régionaux et de calendrier stables.</td>
</tr>
<tr class="even">
<td>LOCALE_NAME_MAX_LENGTH</td>
<td>Longueur maximale d’un nom de paramètres régionaux. Le nombre maximal de caractères autorisés pour cette chaîne est 85, y compris un caractère null de fin.
<blockquote>
[!Note]<br />
Votre application doit utiliser la constante pour la longueur maximale du nom de paramètres régionaux, au lieu de coder en dur la valeur &quot; 85 &quot; .
</blockquote>
<br/></td>
</tr>
<tr class="odd">
<td>LOCALE_NAME_SYSTEM_DEFAULT</td>
<td>Nom des paramètres régionaux du système d’exploitation actuel.</td>
</tr>
<tr class="even">
<td>LOCALE_NAME_USER_DEFAULT</td>
<td>Nom des paramètres régionaux de l’utilisateur actuel, correspondant à la préférence définie dans la partie Options régionales et linguistiques du panneau de configuration. Ces paramètres régionaux peuvent être différents des paramètres régionaux de la langue actuelle de l’interface utilisateur.</td>
</tr>
</tbody>
</table>
