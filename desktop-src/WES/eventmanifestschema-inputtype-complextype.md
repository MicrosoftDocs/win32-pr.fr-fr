# <a name="inputtype-complex-type"></a>Type complexe InputType

Définit un type de données d’entrée.

``` syntax
<xs:complexType name="InputType">
    <xs:sequence>
        <xs:element name="description"
            type="string"
         />
        <xs:element name="outType"
            type="OutputType"
            minOccurs="0"
            maxOccurs="unbounded"
         />
        <xs:any
            processContents="lax"
            namespace="##other"
         />
    </xs:sequence>
    <xs:attribute name="name"
        type="QName"
        use="required"
     />
    <xs:attribute name="value"
        type="string"
        use="required"
     />
    <xs:attribute name="symbol"
        type="CSymbolType"
        use="required"
     />
</xs:complexType>
```

## <a name="child-elements"></a>Éléments enfants



| Élément                                                                  | Type                                                             | Description                                                                                                                                                                                                                                                                                                                                                                                                   |
|--------------------------------------------------------------------------|------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**descriptive**](eventmanifestschema-description-inputtype-element.md) | string                                                           | Description du type d’entrée.<br/>                                                                                                                                                                                                                                                                                                                                                                   |
| [**Type de**](eventmanifestschema-outtype-inputtype-element.md)         | [**OutputType**](eventmanifestschema-outputtype-complextype.md) | Liste des types de données de sortie qui déterminent la façon dont les données sont restituées. Vous devez spécifier au moins un type de sortie. Spécifiez une liste si le type d’entrée peut être rendu en tant que types de sortie différents. Par exemple, le type d’entrée Win : UInt16 peut être rendu sous la forme d’un entier non signé, d’un numéro de port ou d’un nombre hexadécimal. L’attribut par défaut de l’un des types de sortie que vous spécifiez doit avoir la valeur true.<br/> |

## <a name="remarks"></a>Notes

La liste suivante répertorie les types d’entrées reconnus que vous pouvez utiliser dans votre manifeste. Les types sont définis dans le \\ fichier include \\Winmeta.xml inclus dans le SDK Windows. La colonne « type de sortie » répertorie les formats possibles dans lesquels les données peuvent être rendues. Si vous ne spécifiez pas de type de sortie pour l’élément de données, le service utilise le premier type de sortie listé pour restituer les données.
