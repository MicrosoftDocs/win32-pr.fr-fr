# <a name="i-isolated-applications-and-side-by-side-assemblies"></a>I (applications isolées et assemblys côte à côte)

<dl> <dt>

<span id="_win32_isolated_application_gly"></span><span id="_WIN32_ISOLATED_APPLICATION_GLY"></span>**application isolée**
</dt> <dd>

Application utilisant des assemblys côte à côte, mais qui ne partage pas ses assemblys. Une application isolée est toujours accompagnée d’un fichier manifeste d’application. Le manifeste spécifie les versions des assemblys partagés côte à côte auxquels l’application est liée au moment de l’exécution et peut contenir des métadonnées pour les assemblys côte à côte privés. Le manifeste contient des informations traditionnellement stockées dans le registre ; une application totalement isolée peut être indépendante du Registre.

</dd> </dl>
