<span data-ttu-id="bb8fb-104">L’API du serveur HTTP version 2,0 offre la possibilité de créer un service plus sûr et plus fiable en isolant les processus de travail qui traitent les demandes sur la file d’attente des demandes.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-104">The HTTP Server version 2.0 API provides the ability to build a safer, more reliable service by isolating de création qui contrôle strictement l’accès à celle-ci.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-105">The request queue is created and administrated by a controller or creator process that strictly controls access to it.</span></span> <span data-ttu-id="bb8fb-106">Le processus de contrôleur lance un ou plusieurs processus de travail distincts qui effectuent des e/s sur la file d’attente des demandes.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-106">The controller process launches one or more separate worker processes that perform I/O on the request queue.</span></span> <span data-ttu-id="bb8fb-107">Le processus du contrôleur s’exécute avec des privilèges d’administration et configure la file d’attente des demandes, tandis que le travail de <span data-ttu-id="bb8fb-109">L’accès à la file d’attente des demandes est accordé lorsque le processus du contrôleur crée la file d’attente des demandes avec un nom et une liste de Access Control (ACL).</span><span class="sxs-lookup"><span data-stu-id="bb8fb-109">Access to the request queue is granted when the controller process creates the request queue with a name and an Access Control List (ACL).</span></span> <span data-ttu-id="bb8fb-110">Les applications Web incluses dans la liste de contrôle d’accès peuvent ouvrir une file d’attente de demandes existante par nom.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-110">Web applications that are included in the ACL can open an existing request queue by name.</span></span> <span data-ttu-id="bb8fb-111">Le processus de création peut également être un processus de travail dans la file d’attente des demandes.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-111">The creator process may also be a worker process on the request queue.</span></span> <span data-ttu-id="bb8fb-112">Pour plus d’informations, consultez la rubrique [file d’attente des demandes nommées](named-request-queue.md) .</span><span class="sxs-lookup"><span data-stu-id="bb8fb-112">For more information, see the [Named Request Queue](named-request-queue.md) topic.</span></span> <span data-ttu-id="bb8fb-113">Le diagramme suivant illustre l’architecture d’une application HTTP standard exécutée avec le modèle de processus de travail :</span><span class="sxs-lookup"><span data-stu-id="bb8fb-113">The following diagram shows the architecture of a typical HTTP application running with the worker process model:</span></span>

<span data-ttu-id="bb8fb-115">Les processus de travail individuels au sein de l’application sont isolés des autres processus de travail, et l’intégrité de chaque processus de travail peut être surveillée par le processus du contrôleur.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-115">Individual worker processes within the application are isolated from other worker processes, and the health of each the worker processes can be monitored by the controller process.</span></span> <span data-ttu-id="bb8fb-116">Le processus du contrôleur est isolé des processus de travail.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-116">The controller process is isolated from the worker processes.</span></span> <span data-ttu-id="bb8fb-117">Les composants de l’architecture HTTP sont décrits ci-dessous :</span><span class="sxs-lookup"><span data-stu-id="bb8fb-117">The components of the HTTP architecture are described below:</span></span>

-   <span data-ttu-id="bb8fb-118">Processus du créateur ou du contrôleur : le processus du contrôleur peut s’exécuter avec ou sans privilèges d’administrateur pour surveiller l’intégrité et configurer le service.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-118">Creator or controller process: The controller process can run with, or without, administrative privileges to monitor the health and configure the service.</span></span> <span data-ttu-id="bb8fb-119">En général, le processus de contrôleur crée une session serveur unique pour le service et définit les groupes d’URL sous la session serveur.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-119">The controller process typically creates a single server session for the service and defines the URL groups under the server session.</span></span> <span data-ttu-id="bb8fb-120">Le groupe d’URL auquel une URL particulière est associée détermine la file d’attente de demandes qui sert d’espace de noms à l’espace de noms indiqué par l’URL spécifique.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-120">The URL group that a particular URL is associated with determines which request queue services the namespace denoted by the particular URL.</span></span> <span data-ttu-id="bb8fb-121">Le processus de contrôleur crée également la file d’attente des demandes et lance les processus de travail qui peuvent accéder à la file d’attente des demandes.</span><span class="sxs-lookup"><span data-stu-id="bb8fb-121">The controller process also creates the request queue and launches the worker processes that can access the request queue.</span></span>
-   <span data-ttu-id="bb8fb-122">Processus de travail : les processus de travail, lancés par le processus de contrôleur, effectuent des e/s sur la file d’attente de demandes qui est associée aux URL
