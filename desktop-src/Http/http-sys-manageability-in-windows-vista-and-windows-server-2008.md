# <a name="httpsys-manageability-in-windows-vista-and-windows-server-2008"></a>HTTP.sys de la facilité de gestion dans Windows Vista et Windows Server 2008

Avec Windows Vista et Windows Server 2008, les professionnels de l’informatique trouveront des fonctionnalités de facilité de gestion améliorées pour le composant de l’API du serveur HTTP, en plus de la prise en charge de journalisation HTTP introduite dans Windows Server 2003. Pour diagnostiquer et configurer le composant API du serveur HTTP, les professionnels de l’informatique peuvent utiliser trois outils :

-   Nouvelles fonctionnalités de suivi ETW
-   Compteurs de performance
-   Commandes de configuration netsh

L’API du serveur HTTP permet aux applications de communiquer via HTTP et prend en charge SSL afin que les applications puissent échanger des données via des connexions HTTP sécurisées. L’API du serveur HTTP est prise en charge sur les systèmes d’exploitation Windows Server 2003 et sur Windows XP SP2.
