# <a name="d1138-invalid-command-list"></a>D1138 : liste de commandes non valide

Une liste de commandes qui n’est pas valide a été référencée (une erreur s’est produite précédemment pendant le remplissage de la liste de commandes).






 

## <a name="possible-causes"></a>Causes possibles

Si des erreurs se produisent pendant le remplissage d’une liste de commandes, une application ne peut pas l’utiliser pour les opérations de dessin ou la diffusion en continu. Lorsque la liste de commandes est fermée, ID2D1CommandList :: Close retourne un code d’erreur pour indiquer que la liste de commandes n’a pas été remplie correctement. Souvent, ID2D1RenderTarget :: EndDraw ou les opérations de vidage utilisées pendant le remplissage renverront également des codes d’erreur, en fournissant des informations supplémentaires. Les problèmes qui provoquent une erreur pendant ID2D1RenderTarget :: EndDraw, tels que la tentative de dessin d’opérations qui utilisent les ressources d’un autre domaine ou une géométrie de chemin d’accès non fermée, empêchent généralement une liste de commandes de se fermer correctement.

## <a name="possible-fixes"></a>Corrections possibles

Pour garantir le remplissage correct des listes de commandes, corrigez les erreurs qui se produisent à partir de ID2D1CommandList :: Close et ID2D1RenderTarget :: EndDraw ou videz pendant le remplissage de la liste de commandes. Assurez-vous que l’application n’ignore pas le code de retour de ID2D1CommandList :: Close.
