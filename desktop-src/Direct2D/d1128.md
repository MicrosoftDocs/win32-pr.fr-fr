# <a name="d1128-dxgi-format-not-supported-in-any-api"></a>D1128 : format DXGI non pris en charge dans les API

Le format DXGI passé dans cette API n’est pas pris en charge par Direct2D.





 

## <a name="possible-causes"></a>Causes possibles

Certains formats de surface ne sont pas compatibles avec les opérations Direct2D, quelle que soit la façon dont ils sont utilisés. Ce message indique que l’application doit choisir un autre format d’aire.

## <a name="possible-fixes"></a>Corrections possibles

Consultez [formats de pixel pris en charge et modes alpha](supported-pixel-formats-and-alpha-modes.md) pour obtenir des conseils sur les formats pris en charge et ceux qui ne le sont pas. Si l’application doit effectuer un rendu sur un canal unique et tente d’utiliser un format de canal rouge qui n’est pas pris en charge, envisagez de la modifier pour qu’elle affiche un format de canal alpha uniquement.
