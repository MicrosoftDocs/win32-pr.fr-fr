# <a name="d1185-fillgeometry-opacity-mask-brush-restriction"></a>D1185 : restriction de pinceau de masque d’opacité FillGeometry

Si FillGeometry est utilisé avec un masque d’opacité, le pinceau principal doit être un pinceau bitmap ancré.






 

## <a name="possible-causes"></a>Causes possibles

Lors de l’utilisation de ID2D1RenderTarget :: FillGeometry, le pinceau principal était d’un type autre que ID2D1BitmapBrush. Ou le pinceau principal avait un mode d’extension x ou y autre que le mode D2D1 \_ Extend \_ \_ .

## <a name="possible-fixes"></a>Corrections possibles

Si l’application utilise FillGeometry avec un masque d’opacité, assurez-vous que le pinceau principal (à ne pas confondre avec le pinceau d’opacité) est un ID2D1BitmapBrush avec des modes d’extension fixés en mode clamp. Si le comportement de masquage d’opacité n’est pas important, laissez l’argument de pinceau d’opacité comme NULL pour supprimer ces restrictions sur le pinceau principal.
