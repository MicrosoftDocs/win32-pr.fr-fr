# <a name="d1149-print-detected-an-invalid-unicode-character"></a>D1149 : l’impression a détecté un caractère Unicode non valide

Impression Direct2D : caractère Unicode non valide détecté dans la description de l' \_ exécution du GLYPHE DWRITE \_ \_ . chaîne. Il a été remplacé par un espace (0x20).






 

## <a name="possible-causes"></a>Causes possibles

L’impression Direct2D prend en compte les caractères non valides notés dans la [documentation Unicode](https://unicode.org/faq/utf_bom.html#utf16-7), ainsi que certains caractères ASCII en dehors de la plage visible, pour être non imprimables. L’utilisation de ces caractères ne génère pas d’erreur. Au lieu de cela, l’impression Direct2D remplace ces caractères par des espaces.

## <a name="possible-fixes"></a>Corrections possibles

Pour éviter cette notification, n’utilisez pas de caractères Unicode incorrects ni de caractères ASCII non imprimables lors du dessin de texte pour l’impression.
