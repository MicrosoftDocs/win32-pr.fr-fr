# <a name="d1147-print-control-closed-without-addpage"></a>D1147 : contrôle d’impression fermé sans AddPage

Impression Direct2D : la fermeture est appelée sans appel de AddPage précédent






 

## <a name="possible-causes"></a>Causes possibles

Ce message peut survenir si un [**ID2D1PrintControl**](/windows/win32/api/d2d1_1/nn-d2d1_1-id2d1printcontrol) est fermé sans appel antérieur à [**ID2D1PrintControl :: AddPage**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-addpage). L’impression Direct2D prend en compte un contrôle d’impression sans aucune page erronée, et la commande [**ID2D1PrintControl :: Close**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-close) correspondante retourne une erreur.

## <a name="possible-fixes"></a>Corrections possibles

Pour adresser le message, ajoutez une page au contrôle d’impression avant de la fermer. Si l’application n’a pas besoin d’imprimer du contenu, évitez de fermer le contrôle d’impression ou de le créer.
