# <a name="d1136-text-rendering-mode-conflict"></a>D1136 : conflit du mode de rendu du texte

Le \_ mode de rendu DWRITE \_ et le \_ \_ mode d’anticrénelage de texte d2d1 \_ défini sur cette cible de rendu ne sont pas compatibles les uns avec les autres.






 

## <a name="possible-causes"></a>Causes possibles

pour les opérations basées sur du texte telles que ID2D1RenderTarget ::D rawglyphrun et ID2D1DeviceContext :: GetGlyphRunWorldBounds, Direct2D prend en compte l’option d’anticrénelage de texte actuelle du contexte de périphérique, ainsi que l’option d’anticrénelage DirectWrite. La sémantique d’anticrénelage diffère légèrement entre Direct2D et DirectWrite. Ce message s’affiche lorsque les deux options d’anticrénelage spécifiées ne correspondent pas.

## <a name="possible-fixes"></a>Corrections possibles

pour garantir que le mode d’anticrénelage du texte Direct2D accepte le mode de rendu DirectWrite, assurez-vous qu’ils remplissent les conditions suivantes : • pour le mode \_ \_ antialias du texte D2D1 \_ \_ CLEARTYPE, le mode de rendu de la DirectWrite peut ne pas être en mode plan ou avec alias. • pour le mode d’anticrénelage du \_ texte D2D1 \_ \_ en \_ nuances de gris, le mode \_ \_ \_ \_ de rendu du DirectWrite peut ne pas être un alias. DirectWrite \[NewLine \] pour modifier le mode d’anticrénelage du texte Direct2D actuel, utilisez ID2D1RenderTarget :: SetTextAntialiasMode.
