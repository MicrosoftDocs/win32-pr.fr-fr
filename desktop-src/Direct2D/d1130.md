# <a name="d1130-dxgi-format-disallowed-by-caps"></a>D1130 : format DXGI interdit par Cap

L’appareil n’a pas la possibilité de prendre en charge le format DXGI donné.






 

## <a name="possible-causes"></a>Causes possibles

Pour les opérations Direct2D exécutées sur le matériel et utilisant un format de surface, le format de surface doit être pris en charge par le niveau de fonctionnalité du matériel. Par exemple, si le matériel est de niveau de fonctionnalité 9, il ne prend peut-être pas en charge les formats 16 bpp ou SRGB, tels que le \_ format dxgi \_ B8G8R8A8 \_ UNORM \_ sRVB.

## <a name="possible-fixes"></a>Corrections possibles

Pour obtenir des conseils sur les différences entre les niveaux de fonctionnalité et leurs formats pris en charge, consultez [niveaux de fonctionnalité Direct3D](/previous-versions/windows/apps/hh994923(v=win.10)) . Envisagez d’utiliser la [**méthode ID3D11Device :: CheckFormatSupport**](/windows/desktop/api/d3d11/nf-d3d11-id3d11device-checkformatsupport) pour vérifier que le format requis est pris en charge par le matériel avant de l’utiliser avec Direct2D.
