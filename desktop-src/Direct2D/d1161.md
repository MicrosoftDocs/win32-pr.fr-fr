# <a name="d1161-copyfrommemory-requires-cpu-readable-bitmap"></a>D1161 : CopyFromMemory requiert une image bitmap de lecture de l’UC

Cette méthode n’est pas autorisée sur une bitmap créée avec les [**\_ options bitmap d2d1 \_ \_ \_ lecture**](/windows/desktop/api/D2d1_1/ne-d2d1_1-d2d1_bitmap_options)de l’UC.






 

## <a name="possible-causes"></a>Causes possibles

Ce message peut se produire lors d’un appel à [**ID2D1Bitmap :: CopyFromMemory**](/windows/win32/api/d2d1/nf-d2d1-id2d1bitmap-copyfrommemory). Si le bitmap a été créé à l’aide des \_ options de bitmap d2d1 lecture de l' \_ \_ UC \_ , Direct2D signale une erreur et la couche de débogage génère ce message.

## <a name="possible-fixes"></a>Corrections possibles

S’il est nécessaire pour une application d’appeler [**ID2D1Bitmap :: CopyFromMemory**](/windows/win32/api/d2d1/nf-d2d1-id2d1bitmap-copyfrommemory), vérifiez que l’image bitmap n’est pas créée comme étant lisible par le processeur. S’il n’est pas possible de modifier les attributs de surface de la bitmap, déterminez s’il existe une opération de rendu équivalente pour le résultat souhaité.
