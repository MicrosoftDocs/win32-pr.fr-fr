# <a name="d1158-pixels-unit-mode-unsupported-with-print"></a>D1158 : mode unité en pixels non pris en charge par l’impression

Impression Direct2D : la commande SetUnitMode (PIXELS) non prise en charge était présente dans la liste de commandes envoyée pour impression.



|             |             |
|-------------|-------------|
| Niveau d’erreur | Information |



 

## <a name="possible-causes"></a>Causes possibles

L’impression Direct2D n’autorise pas les PIXELS en mode unité. Le mode d’unité par défaut est DIP.

## <a name="possible-fixes"></a>Corrections possibles

Pour traiter ce message, évitez de définir le mode d’unité du contexte de périphérique sur PIXELS lors de la création de contenu imprimable. S’il est nécessaire d’appliquer une transformation d’échelle globale qui affecte toutes les opérations de dessin primitif, déterminez si un appel équivalent à ID2D1RenderTarget :: SetDpi ou ID2D1RenderTarget :: SetTransform est suffisant.
