# <a name="d1125-dxgi-format-mismatch"></a>D1125 : incompatibilité de format DXGI

Le format de pixel passé à cette API n’est pas compatible avec le format de pixel de la \[ surface \] .

## <a name="placeholders"></a>Espaces réservés

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*surfaces*
</dt> <dd>

La cible de rendu ou la bitmap spécifiée.

</dd> </dl>

## <a name="possible-causes"></a>Causes possibles

Les opérations telles que ID2D1RenderTarget :: CreateBitmapFromWicBitmap ou ID2D1RenderTarget :: CreateDxgiSurfaceRendertarget requièrent à la fois une surface et un format de surface spécifié. Direct2D vérifie que le type de surface spécifié est compatible avec la surface. S’il n’est pas compatible, Direct2D génère ce message.

## <a name="possible-fixes"></a>Corrections possibles

Assurez-vous que le format DXGI spécifié \_ correspond à, ou est compatible avec, le type de la surface sous-jacente. Pour les ressources de surface Direct3D créées par l’application, utilisez le même format DXGI que celui \_ utilisé pour créer la ressource. Pour obtenir des instructions supplémentaires, consultez [formats de pixel et modes alpha pris en charge](supported-pixel-formats-and-alpha-modes.md) .
