# <a name="d1120-invalid-attempt-to-draw-bitmap"></a>D1120 : tentative non valide de dessin de bitmap

Une opération a échoué, car la bitmap a les \_ options de bitmap d2d1 \_ \_ ne peut pas \_ dessiner l’option.





 

## <a name="possible-causes"></a>Causes possibles

Ce message s’affiche lorsqu’une application tente de dessiner une bitmap marquée avec des \_ options de bitmap d2d1 \_ \_ ne peut pas être \_ dessinée. Cela comprend les appels directs à DrawBitmap, ainsi que le dessin de primitives à l’aide d’un pinceau bitmap.

## <a name="possible-fixes"></a>Corrections possibles

L' \_ indicateur impossible de dessiner est utilisé pour annoter correctement les ressources qui sont utilisables en tant que cibles uniquement, telles que les caches de permutation de chaînes. Toutefois, il est également autorisé à omettre l' \_ indicateur de dessin impossible pour certaines bitmaps cibles s’ils peuvent également être utilisés comme source. Si c’est le cas, omettez simplement l’indicateur ne peut pas \_ dessiner. Si la bitmap doit être annotée avec ne peut pas être \_ dessinée en raison de son utilisation exclusive en tant que cible ou pour permettre la verrouillage de l’UC, envisagez d’utiliser une texture intermédiaire avec la même taille que la source souhaitée, mais sans la désignation ne peut pas \_ dessiner. Copiez la texture intermédiaire à l’aide de ID3D11DeviceContext :: CopyResource et utilisez la texture intermédiaire comme source pour l’opération de dessin. L’utilisation d’une texture intermédiaire engendre des implications en termes de performances ; elles peuvent être acceptables si la bitmap source ne change pas souvent.
