# <a name="d1127-alpha-mode-mismatch"></a>D1127 : incompatibilité du mode Alpha

Le mode Alpha passé dans cette API n’est pas compatible avec le format de pixel de la \[ *surface* \] .

## <a name="placeholders"></a>Espaces réservés

<dl> <dt>

<span id="surface"></span><span id="SURFACE"></span>*surfaces*
</dt> <dd>

Surface Direct3D qui ne prend pas en charge le mode Alpha.

</dd> </dl> 



 

## <a name="possible-causes"></a>Causes possibles

Les opérations telles que ID2D1RenderTarget :: CreateBitmapFromWicBitmap ou ID2D1RenderTarget :: CreateDxgiSurfaceRendertarget nécessitent à la fois une surface et un mode Alpha de surface spécifié. Direct2D vérifie que le mode Alpha spécifié est compatible avec l’aire de conception. S’il n’est pas compatible, Direct2D génère ce message.

## <a name="possible-fixes"></a>Corrections possibles

Modifiez l’application pour qu’elle utilise un \_ mode alpha qui correspond à, ou qui est compatible avec, le mode Alpha de la surface sous-jacente. Pour obtenir des conseils supplémentaires sur les modes alpha exposés par Direct2D, voir [formats de pixel et modes alpha pris en charge](supported-pixel-formats-and-alpha-modes.md) .
