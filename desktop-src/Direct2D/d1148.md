# <a name="d1148-invalid-operation-on-closed-print-control"></a>D1148 : opération non valide sur le contrôle d’impression fermé

Impression Direct2D : [**AddPage**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-addpage) ou [**Close**](/windows/win32/api/d2d1_1/nf-d2d1_1-id2d1printcontrol-close) est appelé après la fermeture de [**ID2D1PrintControl**](/windows/win32/api/d2d1_1/nn-d2d1_1-id2d1printcontrol)






 

## <a name="possible-causes"></a>Causes possibles

Ce message peut s’afficher si une application appelle ID2D1PrintControl :: AddPage sur un contrôle d’impression qui a été fermé. Direct2D considère que les objets de contrôle d’impression ne peuvent pas être modifiés une fois qu’ils ont été fermés, de la même façon que les géométries de tracés et les listes de commandes.

## <a name="possible-fixes"></a>Corrections possibles

Assurez-vous que l’application n’essaie pas d’ajouter des pages à un contrôle d’impression fermé. Modifiez la logique d’application afin que toutes les pages requises soient envoyées à un contrôle d’impression avant qu’elle ne soit fermée.
