<span data-ttu-id="5a062-113">Cette rubrique présente Direct2D, une API qui fournit aux développeurs Win32 la possibilité d’effectuer des tâches de rendu graphiques 2D avec des performances et une qualité visuelle supérieures.</span><span class="sxs-lookup"><span data-stu-id="5a062-113">This topic introduces Direct2D, an API that provides Win32 developers with the ability to perform 2-D graphics rendering tasks with superior performance and visual quality.</span></span>

<span data-ttu-id="5a062-115">Direct2D est une API graphique en 2D en mode immédiat, accélérée par le matériel, qui offre des performances élevées et un rendu de haute qualité pour la géométrie 2D, les bitmaps et le texte.</span><span class="sxs-lookup"><span data-stu-id="5a062-115">Direct2D is a hardware-accelerated, immediate-mode 2-D graphics API that provides high performance and high-quality rendering for 2-D geometry, bitmaps, and text.</span></span> <span data-ttu-id="5a062-116">L’API Direct2D est conçue pour interagir avec le code existant qui utilise GDI, GDI+ ou Direct3D.</span><span class="sxs-lookup"><span data-stu-id="5a062-116">The Direct2D API is designed to interoperate with existing code that uses GDI, GDI+, or Direct3D.</span></span>

-   <span data-ttu-id="5a062-118">Développeurs de grandes applications natives à l’échelle de l’entreprise.</span><span class="sxs-lookup"><span data-stu-id="5a062-118">Developers of large, enterprise-scale, native applications.</span></span>

-   <span data-ttu-id="5a062-120">Développeurs qui ont besoin d’un rendu côté serveur d’un graphique 2D.</span><span class="sxs-lookup"><span data-stu-id="5a062-120">Developers who require server-side rendering of 2-D graphics.</span></span>
-   <span data-ttu-id="5a062-121">Les développeurs qui utilisent les graphiques Direct3D et ont besoin d’un mode de rendu de texte et 2D simple et hautes performances pour les menus, les éléments d’interface utilisateur et les affichages de tête haut (HUDs).</span><span class="sxs-lookup"><span data-stu-id="5a062-121">Developers who use Direct3D graphics and need simple, high-performance 2-D and text rendering for menus, user interface (UI) elements, and Heads-up Displays (HUDs).</span></span>

## <a name="why-direct2d"></a><span data-ttu-id="5a062-122">Pourquoi Direct2D ?</span><span class="sxs-lookup"><span data-stu-id="5a062-122">Why Direct2D?</span></span>

<span data-ttu-id="5a062-123">Les principales motivations pour créer une nouvelle API graphique 2D dans Microsoft Windows sont les suivantes :</span><span class="sxs-lookup"><span data-stu-id="5a062-123">The primary motivations for creating a new 2-D graphics API in Microsoft Windows include the following:</span></span>

-   <span data-ttu-id="5a062-124">Pour suivre le niveau d’évolution de la richesse visuelle auquel les utilisateurs de Windows sont habitués.</span><span class="sxs-lookup"><span data-stu-id="5a062-124">To keep pace with the increasing level of visual richness that Windows users are accustomed to.</span></span>
-   <span data-ttu-id="5a062-125">Pour permettre aux développeurs d’écrire un code de rendu 2D qui s’adapte directement au matériel de traitement graphique du PC sur lequel il s’exécute.</span><span class="sxs-lookup"><span data-stu-id="5a062-125">To enable developers to write 2-D rendering code that scales directly with the graphics processing hardware of the PC it is running on.</span></span>
-   <span data-ttu-id="5a062-126">Pour permettre aux développeurs d’écrire du code pour afficher des graphiques 2D qui peuvent s’exécuter dans le contexte d’un service.</span><span class="sxs-lookup"><span data-stu-id="5a062-126">To enable developers to write code for rendering 2-D graphics that can run in the context of a service.</span></span>

<span data-ttu-id="5a062-127">Au cours des dernières années, les utilisateurs finaux ont commencé à s’attendre à une plus grande fidélité visuelle dans les expériences numériques.</span><span class="sxs-lookup"><span data-stu-id="5a062-127">In recent years, end users have begun to expect greater visual fidelity in digital experiences.</span></span> <span data-ttu-id="5a062-128">Cette tendance est reflétée dans l’électronique grand public.</span><span class="sxs-lookup"><span data-stu-id="5a062-128">This trend is reflected in consumer electronics.</span></span> <span data-ttu-id="5a062-129">Les périphériques GPS, les périphériques de lecture multimédia, les téléphones mobiles et les appareils photo numériques offrent des expériences plus riches en permanence à l’issue de l’année.</span><span class="sxs-lookup"><span data-stu-id="5a062-129">GPS devices, media playback devices, mobile phones, and digital cameras deliver consistently richer experiences year after year.</span></span> <span data-ttu-id="5a062-130">Cette tendance peut également être consultée dans la diversité du contenu graphique des films, des téléviseurs, des jeux vidéo et sur le Web.</span><span class="sxs-lookup"><span data-stu-id="5a062-130">This trend can also be seen in the diversity of graphical content in film, television, video games, and on the Web.</span></span> <span data-ttu-id="5a062-131">Pour faire face à ces changements, les développeurs sont invités à mettre leurs applications Windows existantes au niveau supérieur de la richesse visuelle.</span><span class="sxs-lookup"><span data-stu-id="5a062-131">To keep pace with these changes, developers are consistently asked to take their existing Windows applications to the next level of visual richness.</span></span>

<span data-ttu-id="5a062-132">Les processeurs graphiques des PC Windows modernes évoluent régulièrement, en passant par des avancées dans les graphiques de jeux vidéo et des parties de l’expérience Windows telles que Windows Media Center et Aero.</span><span class="sxs-lookup"><span data-stu-id="5a062-132">Graphics processors in modern Windows PCs have also been evolving steadily, driven by advances in video game graphics and parts of the Windows experience like Windows Media Center and Aero.</span></span> <span data-ttu-id="5a062-133">Certaines applications Windows peuvent tirer parti des GPU modernes à l’aide de Microsoft Direct3D et Windows Presentation Foundation (WPF).</span><span class="sxs-lookup"><span data-stu-id="5a062-133">Some Windows applications can take advantage of modern GPUs by using Microsoft Direct3D and Windows Presentation Foundation (WPF).</span></span> <span data-ttu-id="5a062-134">Bien que Direct3D serve des applications graphiques 3D haut de gamme et que WPF réponde aux besoins des développeurs .NET, il existe des lacunes pour les développeurs qui ont des codes base existants volumineux de rendu de code basés sur GDI et GDI+, ou qui souhaitent incorporer des graphiques 2D de haute qualité dans leurs applications basées sur Direct3D.</span><span class="sxs-lookup"><span data-stu-id="5a062-134">While Direct3D serves high-end 3-D graphics applications and WPF addresses the needs of .NET developers, there are gaps for developers who have large existing codebases of rendering code based on GDI and GDI+ or who want to incorporate high-quality 2-D graphics in their Direct3D-based applications.</span></span>

<span data-ttu-id="5a062-135">Enfin, la nécessité d’une API Graphics pouvant être utilisée dans un service est devenue une exigence émergente pour les développeurs qui travaillent dans des scénarios d’entreprise et de développement Web.</span><span class="sxs-lookup"><span data-stu-id="5a062-135">Finally, the need for a graphics API that can be used in a service has become an emerging requirement for developers working in enterprise and Web-development scenarios.</span></span> <span data-ttu-id="5a062-136">Les API de rendu existantes se concentrent sur le rendu côté client dans une session mono-utilisateur.</span><span class="sxs-lookup"><span data-stu-id="5a062-136">Existing rendering APIs focus on client-side rendering in a single user session.</span></span> <span data-ttu-id="5a062-137">Par conséquent, ils peuvent être très courts dans des domaines de robustesse et d’extensibilité lorsqu’ils sont utilisés dans un contexte de service.</span><span class="sxs-lookup"><span data-stu-id="5a062-137">As such, they can fall short in areas of robustness and scalability when used in a service context.</span></span> <span data-ttu-id="5a062-138">Une nouvelle API est requise pour résoudre ce cas de figure.</span><span class="sxs-lookup"><span data-stu-id="5a062-138">A new API is required to address this.</span></span>

## <a name="interoperability"></a><span data-ttu-id="5a062-163">Interopérabilité</span><span class="sxs-lookup"><span data-stu-id="5a062-163">Interoperability</span></span>

<span data-ttu-id="5a062-164">L’intégration du rendu basé sur Direct2D est facilitée pour les développeurs grâce à l’interopérabilité des niveaux de surface avec GDI et Direct3D.</span><span class="sxs-lookup"><span data-stu-id="5a062-164">Integrating Direct2D-based rendering is made easier for developers through surface-level interoperability with GDI and Direct3D.</span></span> <span data-ttu-id="5a062-165">Les applications qui restituent du contenu principalement avec GDI, GDI+ ou Direct3D peuvent commencer à utiliser Direct2D pour afficher des zones spécifiques de leur application, et au fil du temps, passer à un modèle où le rendu est réalisé principalement via Direct2D, en utilisant GDI principalement pour les plug-ins ou l’extensibilité héritée.</span><span class="sxs-lookup"><span data-stu-id="5a062-165">Applications that render content primarily with GDI, GDI+, or Direct3D, can begin by using Direct2D to render specific areas of their application, and over time move to a model where rendering is performed primarily via Direct2D, using GDI primarily for plug-ins or legacy extensibility.</span></span>
