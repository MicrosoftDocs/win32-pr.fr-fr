# <a name="d1154-invalid-bitmap-input-format"></a>D1154 : format d’entrée Bitmap non valide

Cette opération n’est pas compatible avec le format de pixel de la bitmap.






 

## <a name="possible-causes"></a>Causes possibles

Direct2D prend en charge les formats de surface alpha uniquement tels que le \_ format dxgi \_ a8 \_ UNORM, mais il ne peut être utilisé qu’en tant que pinceaux d’opacité ou pour le masquage. Si une application crée un pinceau bitmap dont la bitmap a un format alpha uniquement, il ne doit pas tenter d’utiliser le pinceau avec des opérations qui requièrent un pinceau de couleur. ID2D1RenderTarget ::D rawLine et ID2D1RenderTarget :: FillRectangle, par exemple, utilisez un pinceau de couleur.

## <a name="possible-fixes"></a>Corrections possibles

Pour traiter ce message, assurez-vous que les bitmaps avec des formats alpha uniquement sont utilisés uniquement pour les pinceaux d’opacité ou le masquage. Si une bitmap doit contenir des informations de couleur, créez-la à l’aide d’un format de surface qui comprend des canaux de couleur.
