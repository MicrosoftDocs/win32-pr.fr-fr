# <a name="d1181-enddraw-incorrect-state"></a>D1181 : état incorrect EndDraw

EndDraw a été appelé, mais le renderTarget n’était pas dans un état valide. Cela peut être dû à l’appel de EndDraw sans BeginDraw correspondant.






 

## <a name="possible-causes"></a>Causes possibles

L’application a effectué plusieurs appels à EndDraw sans appeler BeginDraw sur la même cible. Lors de l’utilisation de Direct2D, les applications doivent correspondre à chaque appel à EndDraw avec un appel antérieur à BeginDraw. Ce message peut également se produire si la cible est dans un État erroné et n’a pas pu être restituée.

## <a name="possible-fixes"></a>Corrections possibles

Vérifiez que chaque appel à BeginDraw est mis en correspondance avec un EndDraw suivant. Lors de l’envoi de commandes à une cible à l’aide de Flush ou de EndDraw, vérifiez que le résultat retourné est S \_ OK avant de tenter à nouveau d’utiliser la cible.
