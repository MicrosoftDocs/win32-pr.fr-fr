# <a name="d1251-d3d-state-issue"></a>D1251 : problème d’État D3D

L’état de l’appareil Direct3D a changé quand Direct2D l’utilise encore. Cela peut indiquer une violation de thread, car un seul thread peut utiliser le périphérique à la fois.






 

## <a name="possible-causes"></a>Causes possibles

L’application utilise peut-être une fabrique D2D à thread unique sur deux threads différents en même temps.

## <a name="possible-fixes"></a>Corrections possibles

Assurez-vous que l’application respecte correctement les stratégies de Threading Direct2d.
