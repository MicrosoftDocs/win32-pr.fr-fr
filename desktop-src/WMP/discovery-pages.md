<span data-ttu-id="ad865-113">Lorsque l’utilisateur interagit avec le contenu de la boutique en ligne dans le contrôle arborescence ou dans le volet d’informations, le lecteur Windows Media affiche les *pages* Web, appelées pages de découverte, fournies par le magasin en ligne.</span><span class="sxs-lookup"><span data-stu-id="ad865-113">As the user interacts with online store content in the tree-view control or in the details pane, Windows Media Player displays webpages, called *discovery pages*, provided by the online store.</span></span> <span data-ttu-id="ad865-114">Les pages de découverte fournissent des informations supplémentaires sur la musique au fur et à mesure que l’utilisateur parcourt le catalogue du magasin en ligne.</span><span class="sxs-lookup"><span data-stu-id="ad865-114">Discovery pages provide additional information about the music as the user browses the online store's catalog.</span></span> <span data-ttu-id="ad865-115">Les pages de découverte communiquent avec le lecteur Windows Media via les propriétés, les méthodes et les événements de l' [objet externe](external-object-for-type-1-online-stores.md).</span><span class="sxs-lookup"><span data-stu-id="ad865-115">Discovery pages communicate with Windows Media Player through the properties, methods, and events of the [External object](external-object-for-type-1-online-stores.md).</span></span>

<span data-ttu-id="ad865-116">Chaque fois que le lecteur Windows Media modifie son affichage du contenu du magasin en ligne, il appelle [IWMPContentPartner :: GetTemplate](/previous-versions/windows/desktop/api/contentpartner/nf-contentpartner-iwmpcontentpartner-gettemplate), implémenté par le plug-in du magasin en ligne, pour obtenir l’URL de la page de découverte à afficher avec la nouvelle vue.</span><span class="sxs-lookup"><span data-stu-id="ad865-116">Whenever Windows Media Player changes its view of the online store's content, it calls [IWMPContentPartner::GetTemplate](/previous-versions/windows/desktop/api/contentpartner/nf-contentpartner-iwmpcontentpartner-gettemplate), implemented by the online store's plug-in, to get the URL of the discovery page to display with the new view.</span></span>

<span data-ttu-id="ad865-117">L’affichage du contenu du magasin en ligne dans le lecteur Windows Media est caractérisé par cinq éléments d’information : tâche, type d’emplacement, ID d’emplacement, type d’élément sélectionné et ID d’élément sélectionné.</span><span class="sxs-lookup"><span data-stu-id="ad865-117">The view of online store content in Windows Media Player is characterized by five pieces of information: task, location type, location ID, selected item type, and selected item ID.</span></span> <span data-ttu-id="ad865-118">Le lecteur Windows Media fournit ces cinq éléments à la méthode **GetTemplate** dans les paramètres *Task*, *location*, *pContext*, *clickLocation* et *pClickContext* .</span><span class="sxs-lookup"><span data-stu-id="ad865-118">Windows Media Player supplies those five items to the **GetTemplate** method in the *task*, *location*, *pContext*, *clickLocation*, and *pClickContext* parameters.</span></span> <span data-ttu-id="ad865-119">Le lecteur Windows Media rend ces cinq éléments disponibles dans les pages de découverte des propriétés *Task*, *libraryLocationType*, *libraryLocationID*, *selectedItemType* et *SelectedItemId* de l’objet **externe** .</span><span class="sxs-lookup"><span data-stu-id="ad865-119">Windows Media Player makes those five items available to discovery pages in the *task*, *libraryLocationType*, *libraryLocationID*, *selectedItemType*, and *selectedItemID* properties of the **External** object.</span></span> <span data-ttu-id="ad865-120">Pour plus d’informations sur la façon dont le lecteur Windows Media spécifie son affichage du contenu de la boutique en ligne, consultez [emplacement et élément sélectionné](location-and-selected-item.md).</span><span class="sxs-lookup"><span data-stu-id="ad865-120">For more information about how Windows Media Player specifies its view of online store content, see [Location and Selected Item](location-and-selected-item.md).</span></span>

<span data-ttu-id="ad865-121">Outre l’activation d’une page de découverte pour communiquer avec le lecteur Windows Media, l’objet **externe** permet à une page de découverte de communiquer avec le plug-in du magasin en ligne.</span><span class="sxs-lookup"><span data-stu-id="ad865-121">In addition to enabling a discovery page to communicate with Windows Media Player, the **External** object enables a discovery page to communicate with the online store's plug-in.</span></span> <span data-ttu-id="ad865-122">Lorsque cela se produit, le lecteur Windows Media agit comme un pont entre la page de découverte et le plug-in.</span><span class="sxs-lookup"><span data-stu-id="ad865-122">When that happens, Windows Media Player acts as a bridge between the discovery page and the plug-in.</span></span> <span data-ttu-id="ad865-123">Par exemple, la page découverte peut appeler [External. SendMessage](external-sendmessage.md) pour envoyer un message personnalisé au plug-in.</span><span class="sxs-lookup"><span data-stu-id="ad865-123">For example, the discovery page can call [External.sendMessage](external-sendmessage.md) to send a custom message to the plug-in.</span></span> <span data-ttu-id="ad865-124">Le lecteur Windows Media reçoit cet appel de méthode et appelle à son tour [IWMPContentPartner :: SendMessage](/previous-versions/windows/desktop/api/contentpartner/nf-contentpartner-iwmpcontentpartner-sendmessage) pour transmettre le message au plug-in.</span><span class="sxs-lookup"><span data-stu-id="ad865-124">Windows Media Player receives this method call and in turn calls [IWMPContentPartner::SendMessage](/previous-versions/windows/desktop/api/contentpartner/nf-contentpartner-iwmpcontentpartner-sendmessage) to pass the message to the plug-in.</span></span> <span data-ttu-id="ad865-125">Lorsque le plug-in a terminé le traitement du message, il appelle [IWMPContentPartnerCallback :: SendMessageComplete](/previous-versions/windows/desktop/api/contentpartner/nf-contentpartner-iwmpcontentpartnercallback-sendmessagecomplete).</span><span class="sxs-lookup"><span data-stu-id="ad865-125">When the plug-in has finished processing the message, it calls [IWMPContentPartnerCallback::SendMessageComplete](/previous-versions/windows/desktop/api/contentpartner/nf-contentpartner-iwmpcontentpartnercallback-sendmessagecomplete).</span></span> <span data-ttu-id="ad865-126">Le lecteur Windows Media notifie ensuite la page de découverte en déclenchant l’événement [External. OnSendMessageComplete](external-onsendmessagecomplete-event.md) .</span><span class="sxs-lookup"><span data-stu-id="ad865-126">Windows Media Player then notifies the discovery page by raising the [External.OnSendMessageComplete](external-onsendmessagecomplete-event.md) event.</span></span>

<span data-ttu-id="ad865-127">L’objet **externe** permet également à une page de découverte de communiquer avec une autre page de découverte.</span><span class="sxs-lookup"><span data-stu-id="ad865-127">The **External** object also provides a way for a discovery page to communicate with another discovery page.</span></span> <span data-ttu-id="ad865-128">Lorsque le script d’une page de découverte appelle [External. changeView](external-changeview.md), le script peut fournir une chaîne dans le paramètre *ViewParams* .</span><span class="sxs-lookup"><span data-stu-id="ad865-128">When script on a discovery page calls [External.changeView](external-changeview.md), the script can supply a string in the *ViewParams* parameter.</span></span> <span data-ttu-id="ad865-129">Le lecteur Windows Media n’interprète pas la chaîne *ViewParams* , mais rend la chaîne disponible à la page de découverte suivante dans la propriété [External. viewParameters](external-viewparameters.md) .</span><span class="sxs-lookup"><span data-stu-id="ad865-129">Windows Media Player does not interpret the *ViewParams* string, but it makes the string available to the next discovery page in the [External.viewParameters](external-viewparameters.md) property.</span></span>
