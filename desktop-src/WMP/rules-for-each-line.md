# <a name="rules-for-each-line"></a>Règles pour chaque ligne

Chaque définition dans le fichier de définition d’apparence doit se trouver sur une ligne distincte. Vous devez entrer du texte dans les lignes à l’aide des règles suivantes :

-   Chaque ligne doit se terminer par un retour de paragraphe. Certaines lignes peuvent être plus longues que la longueur de ligne visible de votre éditeur de texte, mais pas les lignes. Vous souhaiterez peut-être désactiver le retour automatique à la ligne pour vous assurer que vos lignes ne s’arrêtent pas.
-   Utilisez uniquement des caractères visibles tels qu’ils sont définis dans la référence d’apparence.
-   Une ligne se compose d’un ou plusieurs éléments, séparés par des espaces.
-   Vous pouvez entrer autant d’espaces entre les éléments que vous le souhaitez pour une meilleure lisibilité.
-   N’utilisez pas de caractères de tabulation n’importe où dans le fichier.
-   Les lignes qui commencent par deux barres obliques consécutives (//) sont ignorées. Utilisez-les pour les commentaires. Les lignes vides sont également ignorées.
-   Insérez des espaces entre les crochets et le nom de la section.
