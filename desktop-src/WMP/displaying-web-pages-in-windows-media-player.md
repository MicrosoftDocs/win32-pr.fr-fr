# <a name="displaying-web-pages-in-windows-media-player"></a>affichage des Pages Web dans les Lecteur Windows Media

la fonctionnalité Lecteur Windows Media HTMLView permet aux fournisseurs de contenu Internet (icp) d’utiliser la fonctionnalité de **lecture maintenant** du lecteur en mode plein écran pour afficher le contenu Web lors de la lecture du contenu multimédia numérique. (HTMLView n’est pas pris en charge par le contrôle Lecteur Windows Media.) Par le passé, les ICP devaient afficher des publications dans des fenêtres de navigateur Web distinctes (appelées « publicité contextuelle ») ou devaient ajouter des lecteurs multimédias numériques personnalisés à leurs pages Web. avec HTMLView, les fournisseurs d’informations utilisateur peuvent fournir une expérience personnalisée et intégrée créée spécifiquement pour la série Lecteur Windows Media 9 et Lecteur Windows Media 10, tout en veillant à ce que le contenu multimédia numérique soit toujours lu dans les versions précédentes de Lecteur Windows Media.

Cette section fournit des informations sur l’utilisation de HTMLView, la création de contenu HTMLView et la résolution des problèmes potentiels que vous pouvez rencontrer lors de l’utilisation de la fonctionnalité HTMLView dans les rubriques suivantes :

-   [Avantages de l’utilisation de HTMLView](advantages-of-using-htmlview.md)
-   [Création d’une présentation HTMLView](creating-an-htmlview-presentation.md)
-   [ajout d’un contrôle de Lecteur Windows Media incorporé](adding-an-embedded-windows-media-player-control.md)
-   [s’assurer que Lecteur Windows Media ouvre le contenu HTMLView](ensuring-that-windows-media-player-opens-the-htmlview-content.md)
-   [Problèmes de page Web](web-page-issues.md)
-   [Contrôle de l’expérience de lecture](controlling-the-playback-experience.md)
-   [Problèmes de compatibilité](compatibility-issues.md)
