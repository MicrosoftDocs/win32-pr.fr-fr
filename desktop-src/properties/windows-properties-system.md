# <a name="windows-property-system"></a>Système de propriétés Windows

## <a name="purpose"></a>Objectif

Le système de propriétés Windows est un système extensible en lecture/écriture de définitions de données qui offre un moyen uniforme d’exprimer des métadonnées sur les éléments de l’interpréteur de commandes. Le système de propriétés Windows dans Windows Vista et les versions ultérieures vous permet de stocker et de récupérer des métadonnées pour les éléments de Shell. Un élément de Shell est un élément de contenu unique, tel qu’un fichier, un dossier, un message électronique ou un contact. Une propriété est un élément de métadonnées individuel associé à un élément de Shell.

## <a name="developer-audience"></a>Développeurs concernés

Avant de commencer à lire la documentation du kit de développement logiciel (SDK) du système de propriétés Windows, vous devez avoir une connaissance fondamentale des éléments suivants :

-   COM (Component Object Model)
-   Programmation de l’espace de noms Shell

Pour une introduction à COM, consultez [principes de base de com](../com/com-fundamentals.md). Pour une introduction à la programmation de l’espace de noms Shell, consultez [prise en main avec l’espace de noms Shell](../shell/namespace-intro.md).

Pour les utilisations du système de propriétés Windows, consultez [vue d’ensemble du système de propriétés : scénarios de développement](property-system-overview.md).

## <a name="run-time-requirements"></a>Conditions d’exécution

L’environnement d’exécution pris en charge pour l’utilisation du système de propriétés Windows est Windows Vista ou version ultérieure et le kit de développement logiciel (SDK) Windows. Windows XP et Microsoft Windows Desktop Search (WDS) 3,0 ou version ultérieure incluent également un sous-ensemble du système de propriétés Windows. Pour le téléchargement du kit de développement logiciel (SDK) Windows Vista ou mis à jour, consultez le [SDK Windows](https://msdn.microsoft.com/windowsvista/bb980924.aspx).
