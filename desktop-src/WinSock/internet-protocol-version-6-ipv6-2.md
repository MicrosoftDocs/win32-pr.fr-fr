# <a name="internet-protocol-version-6-ipv6"></a>Protocole Internet version 6 (IPv6)

Il est important de s’assurer que les nouvelles applications Winsock et les applications existantes sont entièrement compatibles avec IPv6. La disponibilité de l’espace d’adressage IPv4 pour les nouvelles allocations d’adresses IPv4 était épuisée pour l’Asie et le Pacifique dans 2011. D’autres parties du monde devraient être épuisées dans quelques années.

Un pourcentage croissant de nouveaux sites Web et services est disponible à l’aide d’adresses IPv6. De nombreux utilisateurs Internet sur les marchés émergents dépendent de IPv6 pour l’accès à Internet.

Microsoft s’engage à prendre en charge IPv6 de longue durée. La prise en charge complète d’IPv6 est fournie sur Windows XP avec Service Pack 1 (SP1) et versions ultérieures.

Ce document fournit des informations sur la prise en charge de Winsock pour IPv6 :

-   [Utilisation d'une adresse IPv6](using-ipv6-2.md)
-   [Utilisation d’Internet Explorer pour accéder aux sites Web IPv6](using-internet-explorer-to-access-ipv6-web-sites-2.md)
-   [Configurations recommandées pour IPv6](recommended-configurations-2.md)
-   [Autres rubriques IPv6](additional-topics-2.md)

Pour plus d’informations sur l’ajout de la fonctionnalité IPv6 à vos applications Windows Sockets, consultez le [Guide IPv6 pour les applications Windows Sockets](ipv6-guide-for-windows-sockets-applications-2.md).

Une introduction au protocole IPv6 ainsi que des vues d’ensemble sur le déploiement et les technologies de transition IPv6 sont disponibles sur TechNet au niveau de [Microsoft Internet Protocol version 6 (IPv6)](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/dd379473(v=ws.10)).
