## <a name="session-establishment-in-the-control-plane"></a><span data-ttu-id="cd425-105">Établissement de session dans le plan de contrôle</span><span class="sxs-lookup"><span data-stu-id="cd425-105">Session Establishment in the Control Plane</span></span>

<span data-ttu-id="cd425-109">Dans le cas d’un contrôle associé à une racine, il existe un participant spécial, \_ racine c, qui est différent du reste des membres de cette session multipoint, chacun d’eux étant appelé \_ feuille c.</span><span class="sxs-lookup"><span data-stu-id="cd425-109">In the case of rooted control, a special participant, c\_root, exists that is different from the rest of the members of this multipoint session, each of which are called a c\_leaf.</span></span> <span data-ttu-id="cd425-110">La \_ racine c doit rester présente pendant toute la durée de la session multipoint, car la session est décomposée en l’absence de la \_ racine c.</span><span class="sxs-lookup"><span data-stu-id="cd425-110">The c\_root must remain present for the whole duration of the multipoint session, as the session is broken up in the absence of the c\_root.</span></span> <span data-ttu-id="cd425-111">La \_ racine c initie généralement la session multipoint en configurant la connexion à une \_ feuille c, ou un nombre de \_ feuilles c.</span><span class="sxs-lookup"><span data-stu-id="cd425-111">The c\_root usually initiates the multipoint session by setting up the connection to a c\_leaf, or a number of c\_leafs.</span></span> <span data-ttu-id="cd425-112">La \_ racine c peut ajouter des feuilles c supplémentaires \_ ou (dans certains cas) une feuille c \_ peut joindre la racine c \_ à un moment ultérieur.</span><span class="sxs-lookup"><span data-stu-id="cd425-112">The c\_root may add more c\_leafs, or (in some cases) a c\_leaf can join the c\_root at a later time.</span></span> <span data-ttu-id="cd425-113">Vous trouverez des exemples de plan de contrôle enraciné dans ATM et ST-II.</span><span class="sxs-lookup"><span data-stu-id="cd425-113">Examples of the rooted control plane can be found in ATM and ST-II.</span></span>

<span data-ttu-id="cd425-116">Une autre façon de regarder ce problème est qu’il \_ existe toujours une racine c, mais elle peut être considérée comme étant dans le Cloud réseau plutôt que pour l’un des participants.</span><span class="sxs-lookup"><span data-stu-id="cd425-116">Another way to look at this is that a c\_root still exists, but can be considered to be in the network cloud as opposed to one of the participants.</span></span> <span data-ttu-id="cd425-117">Étant donné qu’il existe toujours une racine de contrôle, un plan de contrôle qui n’est pas enraciné peut également être considéré comme ayant une racine implicite.</span><span class="sxs-lookup"><span data-stu-id="cd425-117">Because a control root still exists, a nonrooted control plane could also be considered to be implicitly rooted.</span></span> <span data-ttu-id="cd425-118">Voici des exemples de ce type de schémas multipoint à racine implicite :</span><span class="sxs-lookup"><span data-stu-id="cd425-118">Examples for this kind of implicitly rooted multipoint schemes are:</span></span>

## <a name="data-transfer-in-the-data-plane"></a><span data-ttu-id="cd425-122">Transfert de données dans le plan de données</span><span class="sxs-lookup"><span data-stu-id="cd425-122">Data Transfer in the Data Plane</span></span>

<span data-ttu-id="cd425-123">Dans le plan de données, il existe deux types de styles de transfert de données :</span><span class="sxs-lookup"><span data-stu-id="cd425-123">In the data plane, there are two types of data transfer styles:</span></span>
