# <a name="sso-and-plap"></a>SSO et PLAP

L’API Single Sign-On (SSO) représente un ensemble de méthodes utilisées pour obtenir des informations d’identification spécifiques à la méthode EAP pour un compte d’utilisateur ou d’ordinateur réseau de manière sécurisée sans avoir à déclencher plusieurs instances d’interface utilisateur.

> [!Note]  
> Le terme fournisseur d’accès pré-ouverture de session (PLAP) est obsolète. Si « fournisseur d’accès avant ouverture de session » ou « PLAP » est rencontré dans le code ou la documentation, remplacez-le par « authentification unique » ou « SSO », respectivement.
