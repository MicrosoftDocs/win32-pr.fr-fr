# <a name="standards"></a>Standards

Windows Internet Explorer 8 contient un nouveau moteur de rendu et de disposition qui implémente entièrement CSS 2,1 et inclut une compatibilité HTML 4,0 supplémentaire (à la différence des versions précédentes de Windows Internet Explorer). La prise en charge plus large des normes Web dans Internet Explorer 8 peut entraîner un fonctionnement incorrect de certaines applications Web, car elles ne sont pas écrites pour Microsoft Internet Explorer 6 ou de manière non conforme aux normes. Les applications peuvent échouer de différentes façons, y compris des images ou du texte qui s’affichent dans des emplacements différents ou dans des applications entières qui sont rendues d’une manière qui rend l’application inutilisable.

Internet Explorer 8 comprend également un moteur JavaScript amélioré. Ce nouveau moteur de script améliore les performances, prend en charge de nouvelles méthodes en mode natif, offre une sécurité accrue et modifie certaines fonctions pour qu’elles fonctionnent de manière plus conforme aux normes. Les performances améliorées n’introduisent pas de problèmes de compatibilité des applications, mais d’autres modifications peuvent entraîner l’arrêt du fonctionnement de certaines applications Web. Pour plus d’informations sur les problèmes de compatibilité liés aux modifications de JavaScript, consultez [compatibilité de site et Internet Explorer 8](/archive/blogs/ie/site-compatibility-and-ie8).
