# <a name="windows-7-and-windows-server-2008-r2-application-quality-cookbook"></a>guide de qualité des applications Windows 7 et Windows Server 2008 R2

le guide de *qualité des applications Windows 7 et Windows Server 2008 R2* familiarise les développeurs d’applications avec la façon de vérifier la compatibilité de leurs applications avec le nouveau système d’exploitation et fournit une vue d’ensemble des quelques problèmes connus de compatibilité des applications dans Windows 7 et Windows Server 2008 R2. Il indique également les différences en matière de performances, de fiabilité et de facilité d’utilisation, et fournit des liens vers des livres blancs détaillés et d’autres conseils pour les développeurs.

## <a name="overview"></a>Vue d’ensemble

Windows 7 et Windows Server 2008 R2 présentent la dernière plate-forme de développement logiciel et de technologie de système d’exploitation pour une utilisation par les développeurs d’applications et les entreprises du monde entier. dans le cadre de l’amélioration de la sécurité, de la fiabilité, des performances et de l’expérience utilisateur des Windows, de nombreuses nouvelles fonctionnalités ont été introduites, les fonctionnalités existantes ont été améliorées et certaines fonctionnalités ont été supprimées.

bien que Windows 7 et Windows server 2008 R2 soient hautement compatibles avec la plupart de leurs applications respectives écrites pour Windows XP, Windows Server 2003, Windows Vista, Windows server 2008, Windows server 2008 R2 et leurs service packs, certains arrêts de compatibilité sont inévitables en raison des innovations, du renforcement de la sécurité et de la fiabilité accrue. globalement, la compatibilité des Windows 7 et Windows Server 2008 R2 avec les applications existantes est élevée.

ce document s’appuie sur les concepts décrits dans le guide de [compatibilité des applications Windows Vista et Windows Server 2008](/previous-versions/bb757005(v=msdn.10)). (Cette ressource n’est peut-être pas disponible dans certaines langues et pays/régions.) ce document vous permet de vous familiariser avec la façon de vérifier la compatibilité de vos applications avec le nouveau système d’exploitation et fournit une vue d’ensemble des problèmes d’incompatibilité d’applications connus dans Windows 7 et Windows Server 2008 R2. Mais plus que cela, il indique également les différences en matière de performances, de fiabilité et de facilité d’utilisation, et fournit des liens vers des livres blancs détaillés et d’autres conseils pour les développeurs.

en outre, Microsoft investit dans plusieurs outils et fonctionnalités nouveaux et améliorés pour vous permettre de créer des applications de qualité supérieure et de résoudre les problèmes lorsque les applications ne fonctionnent pas correctement sur Windows 7 et Windows Server 2008 R2. ces outils incluent des instructions sur la façon de qualifier vos applications clientes et serveur pour participer au programme Windows Logo.
