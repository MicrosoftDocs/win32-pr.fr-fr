# <a name="windows-7-and-windows-server-2008-r2-application-quality-cookbook"></a>Windows 7 et Windows Server 2008 R2-Guide pratique sur la qualité des applications

Le Guide de *qualité des applications Windows 7 et Windows Server 2008 R2* familiarise les développeurs d’applications avec la façon de vérifier la compatibilité de leurs applications avec le nouveau système d’exploitation et fournit une vue d’ensemble des quelques problèmes connus de compatibilité des applications dans Windows 7 et windows Server 2008 R2. Il indique également les différences en matière de performances, de fiabilité et de facilité d’utilisation, et fournit des liens vers des livres blancs détaillés et d’autres conseils pour les développeurs.

## <a name="overview"></a>Vue d’ensemble

Windows 7 et Windows Server 2008 R2 présentent la dernière plate-forme de développement logiciel et de technologie de système d’exploitation pour une utilisation par les développeurs d’applications et les entreprises du monde entier. Dans le cadre de l’amélioration de la sécurité, de la fiabilité, des performances et de l’expérience utilisateur de Windows, de nombreuses nouvelles fonctionnalités ont été introduites, les fonctionnalités existantes ont été améliorées et certaines fonctionnalités ont été supprimées.

Bien que Windows 7 et Windows Server 2008 R2 soient très compatibles avec la plupart de leurs applications respectives écrites pour Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008, Windows Server 2008 R2 et leurs service packs, certains arrêts de compatibilité sont inévitables en raison des innovations, du renforcement de la sécurité et de la fiabilité accrue. Globalement, la compatibilité de Windows 7 et de Windows Server 2008 R2 avec les applications existantes est élevée.

Ce document s’appuie sur les concepts intégrés au Guide de [compatibilité des applications Windows Vista et Windows Server 2008](/previous-versions/bb757005(v=msdn.10)). (Cette ressource n’est peut-être pas disponible dans certaines langues et pays/régions.) Comme c’est le cas, ce document vous permet de vous familiariser avec la façon de vérifier la compatibilité de vos applications avec le nouveau système d’exploitation et fournit une vue d’ensemble des problèmes d’incompatibilité d’applications connus dans Windows 7 et Windows Server 2008 R2. Mais plus que cela, il indique également les différences en matière de performances, de fiabilité et de facilité d’utilisation, et fournit des liens vers des livres blancs détaillés et d’autres conseils pour les développeurs.

En outre, Microsoft investit dans plusieurs outils et fonctionnalités nouveaux et améliorés pour vous permettre de créer des applications de qualité supérieure et de résoudre les problèmes lorsque les applications ne fonctionnent pas correctement sur Windows 7 et Windows Server 2008 R2. Ces outils incluent des instructions sur la façon de qualifier vos applications clientes et serveur pour participer au programme logo Windows.

Ce livre de *recettes* contient plus de trois douzaines de sujets répartis en trois sections principales :

-   [Compatibilité](compatibility.md) (générale et spécifique au serveur)
-   [Nouvelles fonctionnalités et améliorations](new-features-and-enhancements.md)
-   [Outils, meilleures pratiques et conseils](tools--resources--and-guidance.md)

Nous vous invitons à consulter ces rubriques pour apprendre à optimiser votre application afin de tirer parti de ce que propose Windows 7.
