# <a name="fiddler-web-debugger-tool"></a>Outil de débogage Web Fiddler

L’outil Fiddler vous aide à déboguer des applications Web en capturant le trafic réseau entre Internet et les ordinateurs de test. L’outil vous permet d’inspecter les données entrantes et sortantes afin de surveiller et de modifier les demandes et les réponses avant que le navigateur ne les reçoive. Fiddler comprend également un puissant sous-système de script basé sur les événements, que vous pouvez étendre à l’aide de n’importe quel langage de .NET Framework.

Fiddler et les options de relecture HTTP peuvent vous aider à résoudre les problèmes côté client avec les applications Web en effectuant une copie hors connexion du site de test. Grâce à ces outils, vous pouvez créer des images hors connexion de l’expérience de navigation, puis empaqueter et analyser les résultats pour obtenir des informations de débogage plus détaillées.

Pour télécharger le module complémentaire Fiddler, accédez à la page [modules complémentaires d’Internet Explorer](https://www.microsoft.com/iegallery) . Pour plus d’informations sur la résolution des problèmes à l’aide de Fiddler et des outils connexes, consultez le billet [de blog comment utiliser Fiddler et http Replay pour avoir une copie hors connexion de votre site](https://blogs.msdn.com/b/ricardo/archive/2010/06/22/how-to-user-fiddler-and-http-replay-to-have-an-offline-copy-of-your-site.aspx) et le billet de blog [Authentication with Fiddler](https://blogs.msdn.com/b/ieinternals/archive/2009/11/22/clipboard-paste-bug-in-windows-7-password-prompt.aspx) .
