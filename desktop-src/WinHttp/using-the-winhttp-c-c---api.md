# <a name="using-the-winhttp-cc-api"></a>Utilisation de l’API C/C++ WinHTTP

Cette section contient des informations spécifiques à l’utilisation de l’interface C/C++ WinHTTP.

-   Les [sessions WinHTTP](winhttp-sessions-overview.md) décrivent le déroulement de base d’une application WinHTTP, y compris l’ordre dans lequel les fonctions sont appelées.
-   Les [descripteurs HINTERNET dans WinHTTP](hinternet-handles-in-winhttp.md) décrivent les différents types de handles, les conventions d’affectation de noms pour ces handles et leur structure hiérarchique.
-   Les [URL (Uniform Resource Locators) dans WinHTTP](uniform-resource-locators--urls--in-winhttp.md) décrivent comment WinHTTP utilise des URL et fournissent un exemple de code pour illustrer l’utilisation des fonctions fournies.
-   [L’authentification dans WinHTTP](authentication-in-winhttp.md) décrit la prise en charge par WinHTTP pour l’authentification serveur et proxy des sessions http.
-   [L’authentification Passport dans WinHTTP](passport-authentication-in-winhttp.md) décrit la prise en charge par WinHTTP de l’authentification Passport 1,4.
-   [SSL dans WinHTTP](ssl-in-winhttp.md) décrit la prise en charge par WinHTTP du protocole SSL (Secure Sockets Layer) (SSL).
-   [L’utilisation de WinHTTP comme assembly côte à côte](using-winhttp-as-a-side-by-side-assembly.md) montre un exemple de manifeste d’application qui peut être utilisé pour la liaison à WinHTTP.
-   La [gestion des cookies dans WinHTTP](cookie-handling-in-winhttp.md) décrit plusieurs options que les applications clientes WinHTTP peuvent utiliser pour gérer les cookies.
-   [Gestion des erreurs dans WinHTTP](error-handling-in-winhttp.md) décrit comment définir des options WinHTTP par programmation à partir de votre application C/C++.
-   La prise en charge de la fonction de [proxy AutoProxy WinHTTP](winhttp-autoproxy-support.md) décrit les différentes façons dont les fonctions API WinHTTP signalent des erreurs.
