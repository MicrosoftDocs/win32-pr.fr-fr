# <a name="input-active-client"></a><span data-ttu-id="8dd1a-103">Client Input-Active</span><span class="sxs-lookup"><span data-stu-id="8dd1a-103">Input-Active Client</span></span>

<span data-ttu-id="8dd1a-105">Étant donné que plusieurs applications clientes peuvent partager le même caractère et que plusieurs clients peuvent utiliser des caractères différents en même temps, le serveur désigne un client en tant que client *d’entrée-actif* et envoie une entrée de la souris et de la voix uniquement à cette application cliente.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-105">Because multiple client applications can share the same character and because multiple clients can use different characters at the same time, the server designates one client as the *input-active* client and sends mouse and voice input only to that client application.</span></span> <span data-ttu-id="8dd1a-106">Cela permet de gérer la gestion ordonnée des entrées utilisateur, afin qu’un client approprié réponde à l’entrée.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-106">This maintains the orderly management of user input, so that an appropriate client responds to the input.</span></span>

<span data-ttu-id="8dd1a-107">En règle générale, l’interaction de l’utilisateur détermine l’application cliente qui devient active en entrée.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-107">Typically, user interaction determines which client application becomes input-active.</span></span> <span data-ttu-id="8dd1a-108">Par exemple, si l’utilisateur clique sur un caractère, l’application cliente de ce caractère devient entrée-active.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-108">For example, if the user clicks a character, that character's client application becomes input-active.</span></span> <span data-ttu-id="8dd1a-109">De même, si un utilisateur parle du nom d’un caractère, il devient en entrée-actif.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-109">Similarly, if a user speaks the name of a character, it becomes input-active.</span></span> <span data-ttu-id="8dd1a-110">En outre, lorsque le serveur traite la méthode [**Show**](show-method.md) d’un caractère, le client de ce caractère devient Input-active.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-110">Also, when the server processes a character's [**Show**](show-method.md) method, the client of that character becomes input-active.</span></span>

<span data-ttu-id="8dd1a-111">Lorsqu’un caractère est masqué, le client de ce caractère n’est plus en entrée-actif pour ce caractère.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-111">When a character is hidden, the client of that character will no longer be input-active for that character.</span></span> <span data-ttu-id="8dd1a-112">Le serveur rend automatiquement le client actif de tous les caractères restants en entrée-actif.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-112">The server automatically makes the active client of any remaining character(s) input-active.</span></span> <span data-ttu-id="8dd1a-113">Lorsque tous les caractères sont masqués, aucun client n’est en entrée-actif.</span><span class="sxs-lookup"><span data-stu-id="8dd1a-113">When all characters are hidden, no client is input-active.</span></span> <span data-ttu-id="8dd1a-114">Toutefois, dans cette situation, si l’utilisateur appuie sur la touche d’écoute, l’agent continue à écouter ses commandes (à l’aide du moteur de reconnaissance vocale qui correspond au premier caractère du dernier client d’entrée-actif).</span><span class="sxs-lookup"><span data-stu-id="8dd1a-114">However, in this situation, if the user presses the Listening hotkey, Agent will continue to listen for its commands (using the speech recognition engine matching the topmost character of the last input-active client).</span></span>
