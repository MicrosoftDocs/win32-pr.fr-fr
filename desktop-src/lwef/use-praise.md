# <a name="use-praise"></a>Utiliser l’compliment

\[Microsoft Agent est déconseillé à partir de Windows 7 et peut ne pas être disponible dans les versions ultérieures de Windows.\]

Les êtres humains répondent mieux aux louanges que critiques médias, même si l’compliment peut être non justifié. Bien que la plupart des interfaces logicielles visent à éviter les commentaires évaluation, l’compliment est manifestement plus efficace que de ne fournir aucune évaluation. De plus, de nombreuses interfaces conçues comme neutres sont souvent considérées comme critiques par les utilisateurs, car elles reçoivent rarement des commentaires positifs lorsque les choses fonctionnent normalement, et les messages d’erreur en cas de problème. De même, le libellé qui indique à l’utilisateur qu’il existe un meilleur moyen d’effectuer une tâche est de plus en plus critique pour l’utilisateur.

Étant donné que les caractères créent un contexte social, l’utilisation attentive des compliments et des critiques médias est encore plus importante que dans une interface utilisateur traditionnelle. Tandis que peu de personnes apprécient le comportement sycophantic, les limites de l’utilisation libérale des compliments n’ont pas encore été démontrées. L’compliment est particulièrement efficace dans les situations où les utilisateurs sont moins confiants quant à leurs performances d’une tâche. En revanche, critiques médias doit être utilisé avec modération. Même si vous pensez que critiques médias est approprié, sachez que les êtres humains ont tendance à les faire disparaître ou à les rediriger vers la source.

Toutefois, évitez l’auto-compliment, sauf s’il s’agit d’une partie humoristique de la personnalité des projets de caractères. Nous avons tendance à évaluer l’Conceit avec le son. Si vous souhaitez diriger l’approbation vers un caractère, la faire diriger vers une autre source, telle qu’un autre caractère ou une explication descriptive.
