## <a name="remarks"></a>Notes

Si vous chargez un caractère à l’aide du protocole HTTP (. ACF), vous devez **utiliser la méthode d’extraction pour** récupérer les données d’animation avant de pouvoir lire l’animation. Vous n’utilisez pas cette méthode si vous avez chargé le caractère à l’aide du protocole UNC (un. Fichier ACS). Vous ne pouvez pas non plus récupérer les données HTTP d’un caractère à l’aide de la ligne de recherche si vous avez chargé ce caractère à **l’aide du** protocole UNC (. Fichier de caractères ACS).

Si vous déclarez une référence d’objet et que vous la définissez sur cette méthode, elle retourne un objet de [**requête**](/windows/desktop/lwef/the-request-object) . Si le chargement de l’animation associée échoue, le serveur définit la propriété [**Status**](status-property.md) de l’objet **Request** sur « failed » avec un numéro d’erreur approprié. Vous pouvez utiliser l’événement [**RequestComplete**](requestcomplete-event.md) pour vérifier l’État et déterminer l’action à entreprendre.

Les données audio ou d’animation récupérées avec la méthode d' **extraction** sont stockées dans le cache du navigateur. Les appels suivants vérifient le cache et, si les données d’animation y sont déjà, le contrôle charge les données directement à partir du cache. Une fois chargées, les données de l’animation ou du son peuvent être lues avec les méthodes [**Play**](play-method.md) ou [**Speak**](speak-method.md) .
