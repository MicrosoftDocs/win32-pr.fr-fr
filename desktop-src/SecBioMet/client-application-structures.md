# <a name="client-application-structures"></a>Structures d’application cliente

Les structures suivantes sont prises en charge pour le développement d’applications clientes par l’API Windows Biometric Framework.

## <a name="in-this-section"></a>Contenu de cette section



| Rubrique                                                                                        | Description                                                                                                                     |
|----------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------|
| [**\_stratégie anti- \_ usurpation \_ WINBIO**](winbio-anti-spoof-policy.md)<br/>                   | Représente la stratégie d’antiusurpation pour un utilisateur.<br/>                                                                       |
| [**\_Résultat asynchrone \_ WINBIO**](/windows/desktop/api/Winbio/ns-winbio-winbio_async_result)<br/>                              | Contient les résultats d’une opération asynchrone.<br/>                                                                   |
| [**WINBIO \_ BDB \_ ANSI \_ 381 \_ en-tête**](winbio-bdb-ansi-381-header.md)<br/>              | Spécifie des informations sur une série d’exemples d’empreintes digitales ou Palm.<br/>                                                 |
| [**\_Enregistrement WINBIO BDB \_ ANSI \_ 381 \_**](winbio-bdb-ansi-381-record.md)<br/>              | Contient des informations sur une empreinte digitale ou un exemple Palm d’un utilisateur final.<br/>                                     |
| [**WINBIO \_ Bir**](winbio-bir.md)<br/>                                                 | Représente un enregistrement d’informations biométriques (BIR).<br/>                                                                     |
| [**WINBIO \_ Bir \_ données**](winbio-bir-data.md)<br/>                                      | Spécifie la taille, en octets, et le décalage d’un bloc d’informations biométriques.<br/>                                    |
| [**\_ \_ en-tête Bir WINBIO**](winbio-bir-header.md)<br/>                                  | Contient l’en-tête d’un enregistrement d’informations biométriques (BIR).<br/>                                                         |
| [**\_schéma BSP \_ WINBIO**](winbio-bsp-schema.md)<br/>                                  | Décrit les fonctionnalités d’un fournisseur de services biométriques.<br/>                                                          |
| [**\_ \_ informations sur le moteur étendu WINBIO \_**](winbio-extended-engine-info.md)<br/>             | Contient des informations sur les fonctionnalités et les exigences d’inscription de l’adaptateur de moteur pour une unité biométrique.<br/>  |
| [**WINBIO \_ \_ État d’inscription étendu \_**](winbio-extended-enrollment-status.md)<br/> | Contient des informations supplémentaires sur l’état d’une inscription en cours.<br/>                               |
| [**\_ \_ informations sur le capteur étendu WINBIO \_**](winbio-extended-sensor-info.md)<br/>             | Contient des informations sur les fonctionnalités et les exigences d’inscription de l’adaptateur de capteur pour une unité biométrique.<br/>  |
| [**\_informations de \_ stockage \_ étendu WINBIO**](winbio-extended-storage-info.md)<br/>           | Contient des informations sur les fonctionnalités et les exigences d’inscription de la carte de stockage pour une unité biométrique.<br/> |
| [**\_événement WINBIO**](winbio-event.md)<br/>                                             | Contient les informations d’État envoyées à la routine de rappel lorsqu’une notification d’événement est déclenchée.<br/>                             |
| [**\_identité WINBIO**](winbio-identity.md)<br/>                                       | Contient une valeur d’identification associée à un modèle biométrique.<br/>                                                  |
| [**WINBIO \_ présence**](winbio-presence.md)<br/>                                       | Contient des informations sur la présence d’une personne dont la présence est surveillée.<br/>                          |
| [**\_format enregistré \_ WINBIO**](winbio-registered-format.md)<br/>                    | Spécifie un format de données inscrit comme une paire propriétaire/format.<br/>                                                          |
| [**\_schéma de stockage WINBIO \_**](winbio-storage-schema.md)<br/>                          | Décrit les fonctionnalités d’un adaptateur de stockage biométrique.<br/>                                                           |
| [**\_schéma d’unité WINBIO \_**](winbio-unit-schema.md)<br/>                                | Décrit les fonctionnalités d’une unité biométrique.<br/>                                                                      |
| [**VERSION de WINBIO \_**](winbio-version.md)<br/>                                         | Contient le numéro de version du logiciel d’un composant du fournisseur de services biométriques.<br/>                                      |
