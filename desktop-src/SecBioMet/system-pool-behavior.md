# <a name="system-pool-behavior"></a><span data-ttu-id="b8e36-103">Comportement du pool système</span><span class="sxs-lookup"><span data-stu-id="b8e36-103">System Pool Behavior</span></span>

<span data-ttu-id="b8e36-104">La discussion suivante met en évidence les actions effectuées par le pool système lorsque les notifications d’événements sont envoyées et qu’aucune opération biométrique n’est en attente.</span><span class="sxs-lookup"><span data-stu-id="b8e36-104">The following discussion highlights the actions taken by the system pool when event notices are sent and when no biometric operations are pending.</span></span>

## <a name="event-dispatching"></a><span data-ttu-id="b8e36-105">Distribution d’événements</span><span class="sxs-lookup"><span data-stu-id="b8e36-105">Event Dispatching</span></span>

<span data-ttu-id="b8e36-106">Lorsqu’une unité biométrique génère un avis d’événement, le pool système utilise un filtre en cascade pour distribuer l’avis et lui attribuer l’un des niveaux de priorité suivants :</span><span class="sxs-lookup"><span data-stu-id="b8e36-106">When a biometric unit generates an event notice, the system pool uses a cascading filter to dispatch the notice and assign it one of the following priority levels:</span></span>

-   <span data-ttu-id="b8e36-107">La haute priorité est assignée aux demandes de correspondance et d’inscription explicites générées par les clients.</span><span class="sxs-lookup"><span data-stu-id="b8e36-107">High priority is assigned to explicit matching and enrollment requests generated by clients.</span></span>
-   <span data-ttu-id="b8e36-108">La priorité moyenne est assignée à des événements de correspondance ou d’inscription inattendus ou non revendiqués.</span><span class="sxs-lookup"><span data-stu-id="b8e36-108">Medium priority is assigned to unexpected or unclaimed matching or enrollment events.</span></span>
-   <span data-ttu-id="b8e36-109">La priorité basse est assignée aux événements de navigation.</span><span class="sxs-lookup"><span data-stu-id="b8e36-109">Low priority is assigned to navigation events.</span></span>

<span data-ttu-id="b8e36-110">Les événements de capture sont fournis dans l’ordre suivant :</span><span class="sxs-lookup"><span data-stu-id="b8e36-110">Capture events are delivered in the following sequence:</span></span>

-   <span data-ttu-id="b8e36-111">Si la fenêtre active active attend une opération de mise en correspondance ou d’inscription, l’exemple est traité et envoyé au client qui possède la fenêtre active active.</span><span class="sxs-lookup"><span data-stu-id="b8e36-111">If the current focus window is waiting for a matching or an enrollment operation, the sample is processed and sent to the client that owns the current focus window.</span></span>
-   <span data-ttu-id="b8e36-112">Si l’événement de capture n’est pas réclamé par la fenêtre active active et qu’un gestionnaire d’événements non réclamés a été inscrit auprès du service de biométrie Windows, l’événement de capture est envoyé à ce gestionnaire.</span><span class="sxs-lookup"><span data-stu-id="b8e36-112">If the capture event is unclaimed by the current focus window and an unclaimed event handler has been registered with the Windows Biometric service, the capture event is sent to this handler.</span></span>
-   <span data-ttu-id="b8e36-113">Si l’événement reste non réclamé, il est ignoré.</span><span class="sxs-lookup"><span data-stu-id="b8e36-113">If the event remains unclaimed, it is discarded.</span></span>

<span data-ttu-id="b8e36-114">Si l’événement est un événement de navigation et qu’un gestionnaire d’événements de navigation a été inscrit auprès du service de biométrie Windows, l’événement de capture est envoyé à ce gestionnaire.</span><span class="sxs-lookup"><span data-stu-id="b8e36-114">If the event is a navigation event and a navigation event handler has been registered with the Windows Biometric service, the capture event is sent to this handler.</span></span> <span data-ttu-id="b8e36-115">S’il n’existe aucun gestionnaire d’événements, l’événement est ignoré.</span><span class="sxs-lookup"><span data-stu-id="b8e36-115">If there is no event handler, the event is discarded.</span></span>

## <a name="idle-mode"></a><span data-ttu-id="b8e36-116">Mode inactif</span><span class="sxs-lookup"><span data-stu-id="b8e36-116">Idle Mode</span></span>

<span data-ttu-id="b8e36-117">Si aucun client n’attend la fin des demandes de correspondance explicite ou d’inscription, le pool système détermine s’il faut générer automatiquement des demandes de capture répétées et envoyer l’avis d’événement résultant au gestionnaire d’événements non réclamés ou attendre les événements de navigation et les envoyer au gestionnaire d’événements de navigation.</span><span class="sxs-lookup"><span data-stu-id="b8e36-117">When there are no clients waiting for explicit matching or enrollment requests to complete, the system pool determines whether to automatically generate repeated capture requests and send the resulting event notice to the unclaimed event handler or wait for navigation events and send them to the navigation event handler.</span></span>

<span data-ttu-id="b8e36-118">Si un gestionnaire d’événements non réclamé a été inscrit auprès du service de biométrie Windows, le pool système effectue les actions suivantes :</span><span class="sxs-lookup"><span data-stu-id="b8e36-118">If an unclaimed event handler has been registered with the Windows Biometric service, the system pool performs the following actions:</span></span>

-   <span data-ttu-id="b8e36-119">Le mode de navigation du capteur est désactivé.</span><span class="sxs-lookup"><span data-stu-id="b8e36-119">The navigation mode for the sensor is disabled.</span></span>
-   <span data-ttu-id="b8e36-120">Les opérations non réclamées sont envoyées au gestionnaire d’événements indépendamment du focus de la fenêtre.</span><span class="sxs-lookup"><span data-stu-id="b8e36-120">Unclaimed operations are sent to the event handler regardless of window focus.</span></span>
-   <span data-ttu-id="b8e36-121">S’il n’y a aucune demande en suspens pour une opération biométrique, une capture automatique est effectuée.</span><span class="sxs-lookup"><span data-stu-id="b8e36-121">If there are no outstanding requests for a biometric operation, an automatic capture is performed.</span></span>

<span data-ttu-id="b8e36-122">Si un gestionnaire de navigation a été inscrit auprès du service de biométrie Windows, le pool système effectue les opérations suivantes :</span><span class="sxs-lookup"><span data-stu-id="b8e36-122">If a navigation handler has been registered with the Windows Biometric service, the system pool does the following:</span></span>

-   <span data-ttu-id="b8e36-123">Les unités biométriques du pool système sont placées dans un état de navigation si aucune opération biométrique n’est en attente.</span><span class="sxs-lookup"><span data-stu-id="b8e36-123">The biometric units in the system pool are placed into a navigation state if no biometric operations are pending.</span></span>
-   <span data-ttu-id="b8e36-124">Les événements de navigation sont désactivés si une notification d’événement de correspondance ou d’inscription est envoyée par un client.</span><span class="sxs-lookup"><span data-stu-id="b8e36-124">Navigation events are disabled if a match or enrollment event notice is sent by a client.</span></span>
-   <span data-ttu-id="b8e36-125">Si un gestionnaire d’événements non réclamés a été enregistré, les événements de navigation sont désactivés.</span><span class="sxs-lookup"><span data-stu-id="b8e36-125">If an unclaimed events handler has been registered, navigation events are disabled.</span></span>
