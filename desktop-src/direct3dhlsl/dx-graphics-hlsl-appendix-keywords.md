## <a name="remarks"></a>Remarques

Ces types numériques ont des expansions de mots clés scalaires, vectorielles et de matrice :

-   float, int, uint, bool
-   min10float, min16float
-   min12int, min16int
-   min16uint

Les expansions de ces types numériques suivent ce modèle, qui utilise float comme exemple :

-   Scalaire

    <dl> float  
    </dl>

-   Vecteur

    <dl> float1, float2, float3, float4  
    </dl>

-   Matrice

    <dl> float1x1, float1x2, float1x3, float1x4  
    float2x1, float2x2, float2x3, float2x4  
    float3x1, float3x2, float3x3, float3x4  
    float4x1, float4x2, float4x3, float4x4  
    </dl>

Le langage HLSL prend en charge la [texture](dx-graphics-hlsl-texture.md) et l' [échantillonneur](dx-graphics-hlsl-sampler.md) en minuscules pour des raisons d’héritage. Au lieu de cela, pour vos nouvelles applications, nous vous recommandons d’utiliser les nouveaux objets de texture HLSL ([Texture2D](sm5-object-texture2d.md), [Texture3D](sm5-object-texture3d.md), etc.) et les objets d’échantillonneur ([SamplerState et SamplerComparisonState](dx-graphics-hlsl-sampler.md)).

### <a name="export"></a>export

Utilisez **Export** pour marquer les fonctions que vous empaquetez dans une bibliothèque.

Voici un exemple :

``` syntax
export float identity(float x)
{
    return x;
}
```

En marquant la fonction d' **identité** avec le mot clé **Export** , vous rendez la fonction d' **identité** disponible à partir d’une bibliothèque pour une liaison ultérieure. Sans le marquage d' **exportation** , la fonction **Identity** n’est pas disponible pour les liens ultérieurs.

Le compilateur ignore le mot clé **Export** pour la compilation non-bibliothèque.

> [!Note]  
> Le mot clé **Export** requiert la \_47.dll D3dcompiler ou une version ultérieure de la dll.
