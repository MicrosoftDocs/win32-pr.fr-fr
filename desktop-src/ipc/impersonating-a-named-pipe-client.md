<span data-ttu-id="8b68f-104">L' *emprunt d’identité* est la capacité d’un thread à s’exécuter dans un contexte de sécurité différent de celui du processus qui possède le thread.</span><span class="sxs-lookup"><span data-stu-id="8b68f-104">*Impersonation* is the ability of a thread to execute in a security context different from that of the process that owns the thread.</span></span> <span data-ttu-id="8b68f-105">L’emprunt d’identité permet au thread de serveur d’effectuer des actions pour le compte du client, mais dans les limites du contexte de sécurité du client.</span><span class="sxs-lookup"><span data-stu-id="8b68f-105">Impersonation enables the server thread to perform actions on behalf of the client, but within the limits of the client's security context.</span></span> <span data-ttu-id="8b68f-106">Le client dispose généralement d’un niveau de droits d’accès inférieur.</span><span class="sxs-lookup"><span data-stu-id="8b68f-106">The client typically has some lesser level of access rights.</span></span> <span data-ttu-id="8b68f-107">Pour plus d’informations, consultez [emprunt d’identité](/windows/desktop/SecAuthZ/client-impersonation).</span><span class="sxs-lookup"><span data-stu-id="8b68f-107">For more information, see [Impersonation](/windows/desktop/SecAuthZ/client-impersonation).</span></span>

<span data-ttu-id="8b68f-108">Un thread de serveur de canal nommé peut appeler la fonction [**ImpersonateNamedPipeClient**](/windows/desktop/api/namedpipeapi/nf-namedpipeapi-impersonatenamedpipeclient) pour supposer que le jeton d’accès de l’utilisateur est connecté à l’extrémité cliente du pipe.</span></span> <span data-ttu-id="8b68f-109">Par exemple, un serveur de canaux nommés peut fournir l’accès à une base de données ou à un système de fichiers sur lequel le serveur de canaux dispose d’un accès privilégié.</span><span class="sxs-lookup"><span data-stu-id="8b68f-109">For example, a named pipe server can provide access to a database or file system to which the pipe server <span data-ttu-id="8b68f-113">Le [niveau d’emprunt d’identité](/windows/desktop/SecAuthZ/impersonation-levels) détermine les opérations que le serveur peut effectuer en usurpant l’identité du client.</span><span class="sxs-lookup"><span data-stu-id="8b68f-113">The [impersonation level](/windows/desktop/SecAuthZ/impersonation-levels) determines the operations the server can perform while impersonating the client.</span></span> <span data-ttu-id="8b68f-114">Par défaut, un serveur emprunte une identité au niveau de l’emprunt d’identité SecurityImpersonation.</span><span class="sxs-lookup"><span data-stu-id="8b68f-114">By default, a server impersonates at the SecurityImpersonation impersonation level.</span></span> <span data-ttu-id="8b68f-115">Toutefois, lorsque le client appelle la fonction [**CreateFile**](/windows/desktop/api/fileapi/nf-fileapi-createfilea) pour ouvrir un handle à l’extrémité cliente du canal, le client peut utiliser l' \_ indicateur Security SQOS \_ present pour contrôler le niveau d’emprunt d’identité du serveur.</span><span class="sxs-lookup"><span data-stu-id="8b68f-115">However, when the client calls the [**CreateFile**](/windows/desktop/api/fileapi/nf-fileapi-createfilea) function to open a handle to the client end of the pipe, the client can use the SECURITY\_SQOS\_PRESENT flag to control the server's impersonation level.</span></span>
