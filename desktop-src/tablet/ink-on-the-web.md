# <a name="ink-on-the-web"></a>Entrée manuscrite sur le Web

Le kit de développement logiciel (SDK) Windows Vista et le kit de développement Windows XP Tablet PC Edition vous permettent de créer des applications Web pour les utilisateurs de Tablet PC qui doivent accéder à des applications distantes. Il existe deux techniques de base pour accomplir cela : l’un est un déploiement sans toucher, qui permet aux applications .NET d’être déployées à partir d’un serveur Web ou de fichiers, et l’autre est de créer des pages Web avec des contrôles de Windows Forms. Dans les deux cas, l’encre est gérée sur le client, plutôt que sur le serveur. Notez que l’API COM n’est pas prise en charge pour le Web.

Pour utiliser le traitement côté client sur le Web, vous devez comprendre le modèle de sécurité .NET et savoir comment l’utilisation de la confiance partielle affecte votre application. Pour cette raison, la sécurité et l’approbation des applications Tablet PC sont également abordées.

Les rubriques suivantes contiennent des remarques sur les différentes façons de créer des applications Web compatibles avec l’écriture manuscrite.

-   [Aucun déploiement tactile](no-touch-deployment.md)
-   [Contrôles Web](web-controls.md)
-   [Applications Web compatibles avec l’écriture manuscrite](ink-enabled-web-applications.md)
-   [Sécurité et confiance](security-and-trust.md)
