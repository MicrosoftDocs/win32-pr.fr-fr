# <a name="whats-new-in-tablet-pc-development"></a>Nouveautés du développement pour Tablet PC

Cette section décrit les nouveautés du développement de technologies Tablet PC.

## <a name="tablet-pc-development-kit-now-in-windows-sdk"></a>Kit de développement Tablet PC maintenant en SDK Windows

Depuis le printemps de 2006 et à l’avenir, le SDK Windows est le seul mécanisme de distribution pour la toute dernière technologie Tablet PC. Il n’existe plus de kit de développement logiciel (SDK) Tablet PC autonome. Les binaires de la plateforme, les en-têtes, les bibliothèques, la documentation et les exemples de code sont disponibles dans le SDK Windows. Cette distribution comprend les fichiers binaires de la plateforme Tablet PC 1,7 et les modules de fusion redistribuables pour ces fichiers binaires, ainsi que la documentation pour les fichiers binaires de la version 1,7 et les nouveaux fichiers binaires de la plateforme de technologie Tablet PC Windows Vista (version 6).

Reportez-vous à [l’environnement de développement](the-development-environment.md) pour plus d’informations sur la configuration de votre ordinateur pour le développement Tablet PC.

## <a name="windows-server-2008-r2-now-supports-handwriting-recognition-and-custom-dictionaries"></a>Windows Server 2008 R2 prend désormais en charge la reconnaissance de l’écriture manuscrite et les dictionnaires personnalisés

Windows Server 2008 R2 peut désormais effectuer la reconnaissance manuscrite et les fonctionnalités de dictionnaire personnalisé ont été ajoutées pour prendre en charge de nouveaux scénarios de reconnaissance.

Pour plus d’informations, consultez [reconnaissance de l’écriture manuscrite dans Windows Server 2008 R2](handwriting-recognition-in-windows-server-2008-r2.md) et [création de dictionnaires personnalisés pour la reconnaissance de l’écriture manuscrite](creating-custom-dictionaries-for-handwriting-recognition-in-windows-7-and-windows-server-2008-r2.md) .

## <a name="math-input"></a>Entrée mathématique

L’API d’entrée mathématique est désormais disponible dans une version COM. Pour plus d’informations, consultez [programmation du contrôle d’entrée mathématique](programming-the-math-input-control.md) et [Référence du contrôle d’entrée mathématique](math-input-control-reference.md) .

## <a name="ink-analysis"></a>Analyse des entrées manuscrites

De nouvelles fonctionnalités d’analyse d’encre ont été créées pour la plate-forme de technologie Tablet PC. Consultez [vue d’ensemble](ink-analysis-overview.md) de l’analyse de l’encre pour plus d’informations conceptuelles.

## <a name="stylus-input-api-com-version"></a>Version COM de l’API d’entrée du stylet

L’API d’entrée du stylet est désormais disponible dans une version COM. Consultez la [référence RealTimeStylus](realtimestylus-reference.md) pour la version com des rubriques de référence sur les API. Consultez [accès et manipulation de l’entrée du stylet](accessing-and-manipulating-stylus-input.md) pour obtenir des informations conceptuelles sur l’API d’entrée de stylet.

## <a name="textinputpanel-class"></a>TextInputPanel, classe

La classe [TextInputPanel](/previous-versions/ms573640(v=vs.100)) est disponible en tant qu’objet pouvant être attaché, ce qui vous permet d’ajouter des fonctionnalités du panneau de saisie Tablet PC à des contrôles existants. L’interface utilisateur est en grande partie mandatée par la langue d’entrée actuelle. Vous avez la possibilité de choisir l’écriture manuscrite ou le clavier comme méthode d’entrée par défaut pour l’objet TextInputPanel. L’utilisateur final peut basculer entre les méthodes d’entrée à l’aide des boutons de l’interface utilisateur.

## <a name="journal-reader-component"></a>Composant de lecteur de journal

Le composant lecteur de journal est disponible. Pour plus d’informations, consultez [utilisation du composant de lecture du journal](using-the-journal-reader-component.md) . Consultez la classe [JournalReader](/previous-versions/ms583815(v=vs.100)) pour les API managées. Consultez [Référence du composant de lecture du journal](journal-reader-component-reference.md) pour la version com des API.
