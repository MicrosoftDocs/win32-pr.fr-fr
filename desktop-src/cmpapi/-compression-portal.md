# <a name="compression-api"></a>API de compression

## <a name="purpose"></a>Objectif

l’API de compression expose les algorithmes de compression Windows MSZIP, xpress, xpress \_ HUFF et LZMS. cela permet aux développeurs de Windows applications de gérer les versions, le service et d’étendre les algorithmes de compression exposés.

## <a name="developer-audience"></a>Développeurs concernés

l’API de Compression est conçue pour être utilisée par les développeurs professionnels C/C++ de Windows applications. l’api de compression expose les algorithmes de compression sans perte utilisés par Windows via une interface publique et une API en mode utilisateur. l’API de compression est la méthode recommandée pour Windows aux développeurs de contrôler les algorithmes de compression. cette fonctionnalité est 64 bits sur 64 bits Windows et 32 bits sur 32 bits Windows.

## <a name="run-time-requirements"></a>Conditions d’exécution

l’API de Compression est disponible à partir de la Windows 8 ou Windows Server 2012.
