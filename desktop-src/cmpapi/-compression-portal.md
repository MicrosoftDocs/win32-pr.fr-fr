# <a name="compression-api"></a>API de compression

## <a name="purpose"></a>Objectif

L’API de compression expose les algorithmes de compression Windows MSZIP, XPRESS, XPRESS \_ Huff et LZMS. Cela permet aux développeurs d’applications Windows de gérer les versions, le service et d’étendre les algorithmes de compression exposés.

## <a name="developer-audience"></a>Développeurs concernés

L’API de compression est conçue pour être utilisée par les développeurs professionnels C/C++ des applications Windows. L’API de compression expose les algorithmes de compression sans perte utilisés par Windows par le biais d’une interface publique et d’une API en mode utilisateur. L’API de compression est la méthode recommandée pour les développeurs Windows qui contrôlent les algorithmes de compression. Cette fonctionnalité est 64 bits sur Windows 64 bits et 32 bits sur Windows 32 bits.

## <a name="run-time-requirements"></a>Conditions d’exécution

L’API de compression est disponible à partir de Windows 8 ou Windows Server 2012.
