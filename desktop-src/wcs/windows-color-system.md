# <a name="windows-color-system"></a>Système de couleurs Windows

## <a name="purpose"></a>Objectif

les schémas de gestion des couleurs sont mis en œuvre dans les systèmes d’exploitation Microsoft Windows pour améliorer le rendu du contenu en couleurs dans toutes les formes de communication numérique.

le schéma de gestion des couleurs utilisé à partir de Windows 2000, Windows XP et Windows Server 2003 est appelé gestion des couleurs des images (ICM) 2,0. le schéma de gestion des couleurs utilisé à partir de Windows Vista est appelé Windows système de couleurs (WCS) 1,0. le schéma de gestion des couleurs WCS 1,0 est un sur-ensemble des api et des fonctionnalités de ICM 2,0.

## <a name="where-applicable"></a>Le cas échéant

ICM peut être utilisé dans toutes les applications sur les systèmes d’exploitation Windows 2000 et versions ultérieures. WCS peut être utilisé dans toutes les applications sur les systèmes d’exploitation Windows Vista et versions ultérieures.

## <a name="developer-audience"></a>Développeurs concernés

L’API WCS est conçue pour être utilisée par les programmeurs C/C++. vous devez vous familiariser avec l’interface utilisateur graphique de Windows, l’architecture pilotée par les messages et des connaissances pratiques sur les concepts de gestion des couleurs.

## <a name="run-time-requirements"></a>Conditions d’exécution

les Applications qui utilisent l’API ICM requièrent Windows 2000 ou une version ultérieure. les Applications qui utilisent l’API WCS requièrent Windows Vista ou version ultérieure. Pour obtenir des informations spécifiques au moment de l’exécution sur une fonction, consultez la section Configuration requise de la page de référence pour cette fonction.
