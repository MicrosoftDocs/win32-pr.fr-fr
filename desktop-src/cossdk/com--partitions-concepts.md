# <a name="com-partitions-concepts"></a>Concepts des partitions COM+

Dans les environnements informatiques où il est nécessaire d’utiliser différentes versions ou configurations d’une application sur le même ordinateur, des conflits peuvent survenir quand une version de l’application nécessite des composants différents de ceux de l’autre, ou lorsqu’une version requiert un composant sur l’ordinateur qui est incompatible avec l’autre version de l’application. Par le passé, le seul moyen de résoudre ce problème consistait à gérer plusieurs ordinateurs et à exécuter une version différente de l’application sur chaque ordinateur. Une telle approche est coûteuse et inefficace, car elle augmente les coûts matériels et la surcharge administrative.

COM+ résout ce problème en fournissant la fonctionnalité partitions COM+. Vous pouvez utiliser des partitions COM+ pour installer différentes versions d’une application COM+ sur un ordinateur et les exécuter simultanément.

Pour comprendre et utiliser cette fonctionnalité, consultez les rubriques suivantes :

-   [Que sont les partitions COM+ ?](what-are-com--partitions-.md)
-   [Implémentation de la partition](partition-implementation.md)
-   [Inscription et activation de composants dans des partitions](registering-and-activating-components-in-partitions.md)
-   [Composants et partitions COM+ en file d’attente](com--queued-components-and-partitions.md)
-   [Restrictions relatives à la conception d’applications](application-design-restrictions.md)
