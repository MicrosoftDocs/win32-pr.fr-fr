## <a name="enforcing-security-in-the-middle-tier"></a><span data-ttu-id="2be9e-108">Application de la sécurité dans la couche intermédiaire</span><span class="sxs-lookup"><span data-stu-id="2be9e-108">Enforcing Security in the Middle Tier</span></span>

<span data-ttu-id="2be9e-109">La règle générale à suivre avec les applications multiniveaux évolutives à l’aide de COM+ est que, chaque fois que cela est possible, vous devez appliquer une sécurité détaillée dans l’application COM+ au niveau intermédiaire.</span><span class="sxs-lookup"><span data-stu-id="2be9e-109">The general rule to follow with scalable multi-tier applications using COM+ is that, whenever possible, you should enforce detailed security in the COM+ application at the middle tier.</span></span> <span data-ttu-id="2be9e-110">Cela vous permet de tirer pleinement parti des services évolutifs fournis par COM+.</span><span class="sxs-lookup"><span data-stu-id="2be9e-110">Doing so enables you to fully leverage the scalable services provided by COM+.</span></span> <span data-ttu-id="2be9e-111">Appliquez l’authentification au niveau de la base de données uniquement lorsque vous en avez absolument besoin et évaluez entièrement les implications en matière de performances.</span><span class="sxs-lookup"><span data-stu-id="2be9e-111">Enforce authentication at the database only when you absolutely need to, and fully weigh the performance implications of doing so.</span></span>

<span data-ttu-id="2be9e-112">La situation optimale est de pouvoir sécuriser les tables de base de données afin que l’application COM+ puisse y accéder sous sa propre identité.</span><span class="sxs-lookup"><span data-stu-id="2be9e-112">The optimal situation is to be able to secure database tables so that the COM+ application is able to access them under its own identity.</span></span> <span data-ttu-id="2be9e-113">La base de données doit simplement authentifier l’application COM+ et l’approuver pour authentifier et autoriser correctement les clients qui accèdent aux données.</span><span class="sxs-lookup"><span data-stu-id="2be9e-113">The database should just authenticate the COM+ application and trust it to correctly authenticate and authorize clients that will be accessing data.</span></span> <span data-ttu-id="2be9e-114">Les avantages de cette approche sont les suivants :</span><span class="sxs-lookup"><span data-stu-id="2be9e-114">The benefits of this approach are as follows:</span></span>

-   <span data-ttu-id="2be9e-115">Il permet le regroupement de connexions entre tous les clients, car les connexions sont entièrement génériques.</span><span class="sxs-lookup"><span data-stu-id="2be9e-115">It enables connection pooling among all clients, as connections are completely generic.</span></span>
-   <span data-ttu-id="2be9e-116">Il optimise généralement l’accès aux données, souvent un point d’amaigrissement problématique lors de la mise à l’échelle des applications.</span><span class="sxs-lookup"><span data-stu-id="2be9e-116">It generally optimizes data access, often a problematic choke point when scaling applications.</span></span>
-   <span data-ttu-id="2be9e-117">Cela peut réduire la surcharge logique pour l’application de la sécurité, en particulier si la sécurité déclarative basée sur les rôles peut être utilisée.</span><span class="sxs-lookup"><span data-stu-id="2be9e-117">It can minimize the logic overhead for enforcing security, particularly if declarative role-based security can be used.</span></span>

<span data-ttu-id="2be9e-120">Toutefois, comme nous l’avons vu dans [conception de rôles de manière efficace](designing-roles-effectively.md), bien que les rôles puissent être appliqués facilement et efficacement à certaines relations utilisateur-données, ils ne constituent pas une solution universelle pour prendre des décisions d’accès à la sécurité.</span><span class="sxs-lookup"><span data-stu-id="2be9e-120">But, as discussed in [Designing Roles Effectively](designing-roles-effectively.md), while roles can be easily and effectively applied to some user-data relationships, they are not a universal solution for security access decisions.</span></span>

## <a name="enforcing-security-at-the-database"></a><span data-ttu-id="2be9e-121">Application de la sécurité au niveau de la base de données</span><span class="sxs-lookup"><span data-stu-id="2be9e-121">Enforcing Security at the Database</span></span>

<span data-ttu-id="2be9e-122">Malgré l’application de la sécurité au niveau intermédiaire, il existe plusieurs raisons pour mettre en œuvre la sécurité dans la base de données, notamment les suivantes :</span><span class="sxs-lookup"><span data-stu-id="2be9e-122">Despite the preferability of enforcing security at the middle tier, there are a number of reasons to enforce security at the database, such as the following:</span></span>

-   <span data-ttu-id="2be9e-123">Vous n’avez pas de choix, peut-être pour des raisons d’héritage ou peut-être parce que la décision n’est tout simplement pas dans vos mains.</span><span class="sxs-lookup"><span data-stu-id="2be9e-123">You don't have a choice, perhaps for legacy reasons or perhaps because the decision is simply not in your hands.</span></span>
-   <span data-ttu-id="2be9e-124">La base de données est accessible par un large éventail d’applications et sa sécurité ne peut pas être configurée sur la base d’une seule application.</span><span class="sxs-lookup"><span data-stu-id="2be9e-124">The database is accessed by a wide variety of applications, and its security can't be configured on the basis of a single application.</span></span>
-   <span data-ttu-id="2be9e-125">Une base de données peut être rendue protégée de manière prévisible.</span><span class="sxs-lookup"><span data-stu-id="2be9e-125">A database can be made predictably protected.</span></span> <span data-ttu-id="2be9e-126">Si vous conservez les données critiques de l’entreprise dans une base de données, vous pouvez dessiner un petit wagon et aider à contrôler qui le voit.</span><span class="sxs-lookup"><span data-stu-id="2be9e-126">If you keep enterprise-critical data in a database, you can draw a tight wagon around it and help control who gets to see it.</span></span>
-   <span data-ttu-id="2be9e-127">L’authentification des clients d’origine dans la base de données permet une journalisation détaillée des personnes qui ont fait quoi dans la base de données.</span><span class="sxs-lookup"><span data-stu-id="2be9e-127">Authenticating original clients at the database enables detailed logging of who has done what in the database.</span></span>

<span data-ttu-id="2be9e-129">Lorsque vous avez le luxe de concevoir la sécurité de la base de données et de l’application COM+ en tandem, la plupart de ces problèmes se rapportent aux caractéristiques des données elles-mêmes et à leur relation avec les utilisateurs qui y accèdent.</span><span class="sxs-lookup"><span data-stu-id="2be9e-129">When you do have the luxury of designing the security of the database and the COM+ application in tandem, most of these issues pertain to characteristics of the data itself and its relationship to the users accessing it.</span></span> <span data-ttu-id="2be9e-130">Avec les données accessibles par des catégories d’utilisateurs prévisibles, il est efficace de contrôler l’accès au niveau intermédiaire à l’aide de rôles.</span><span class="sxs-lookup"><span data-stu-id="2be9e-130">With data that can be accessed by predictable categories of users, it is efficient to control access in the middle tier using roles.</span></span> <span data-ttu-id="2be9e-131">Avec les données qui sont liées de manière complexe aux très petites catégories d’utilisateurs, ces relations doivent souvent être exprimées avec les données elles-mêmes et, par conséquent, vous devez appliquer une sécurité à la base de données.</span><span class="sxs-lookup"><span data-stu-id="2be9e-131">With data that is intricately bound to very small classes of users, those relationships must often be expressed with the data itself and hence you must enforce some security at the database.</span></span>

### <a name="performance-implications-of-enforcing-security-at-the-database"></a><span data-ttu-id="2be9e-132">Implications sur les performances de l’application de la sécurité au niveau de la base de données</span><span class="sxs-lookup"><span data-stu-id="2be9e-132">Performance Implications of Enforcing Security at the Database</span></span>

<span data-ttu-id="2be9e-133">Si vous authentifiez ou autorisez des utilisateurs au niveau de la base de données, l’identité de l’utilisateur doit être propagée à la base de données pour prendre en charge cela.</span><span class="sxs-lookup"><span data-stu-id="2be9e-133">If you authenticate or authorize users at the database, the user identity needs to be propagated through to the database to support this.</span></span> <span data-ttu-id="2be9e-134">Si la base de données approuve l’application de couche intermédiaire pour ce faire, il est possible de transmettre les informations de sécurité utilisateur dans les paramètres.</span><span class="sxs-lookup"><span data-stu-id="2be9e-134">If the database trusts the middle-tier application to do so, it is possible to pass in user security information in parameters.</span></span> <span data-ttu-id="2be9e-135">Dans le cas contraire, l’appel à la base de données doit être effectué sous l’identité de l’utilisateur, ce qui signifie que l’application serveur doit emprunter l’identité du client.</span><span class="sxs-lookup"><span data-stu-id="2be9e-135">Otherwise, the call through to the database must be made under the user's identity, meaning that the server application must impersonate the client.</span></span> <span data-ttu-id="2be9e-136">Cela peut avoir des répercussions sur les performances, notamment les suivantes :</span><span class="sxs-lookup"><span data-stu-id="2be9e-136">This can have performance implications such as the following:</span></span>

-   <span data-ttu-id="2be9e-137">L’emprunt d’identité du client peut être beaucoup plus lent que l’appel direct sous l’identité de l’application.</span><span class="sxs-lookup"><span data-stu-id="2be9e-137">Impersonating the client can be much slower than making the call directly under the application identity.</span></span> <span data-ttu-id="2be9e-138">Pour plus d’informations, consultez [emprunt d’identité et délégation du client](client-impersonation-and-delegation.md).</span><span class="sxs-lookup"><span data-stu-id="2be9e-138">For more detail, see [Client Impersonation and Delegation](client-impersonation-and-delegation.md).</span></span>
-   <span data-ttu-id="2be9e-139">Une connexion de base de données effectuée sous une identité d’utilisateur spécifique ne peut pas être regroupée.</span><span class="sxs-lookup"><span data-stu-id="2be9e-139">A database connection made under a specific user identity cannot be pooled.</span></span>
-   <span data-ttu-id="2be9e-140">L’ajout de logique dans la base de données elle-même exécute le risque de créer un goulot d’étranglement de mise à l’échelle.</span><span class="sxs-lookup"><span data-stu-id="2be9e-140">Adding logic in the database itself runs the risk of creating a scaling bottleneck.</span></span>
