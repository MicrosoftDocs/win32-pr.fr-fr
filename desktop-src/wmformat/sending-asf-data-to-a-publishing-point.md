<span data-ttu-id="3cbfc-111">Vous pouvez utiliser le kit de développement logiciel (SDK) Windows Media format pour envoyer des données ASF à un point de publication sur un serveur Windows Media.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-111">You can use the Windows Media Format SDK to push ASF data to a publishing point on a Windows Media server.</span></span> <span data-ttu-id="3cbfc-112">Le serveur diffuse ensuite les données à partir de ce point de publication.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-112">The server then broadcasts the data from that publishing point.</span></span> <span data-ttu-id="3cbfc-113">Ce scénario est utile si vous capturez ou ré-encodez du contenu sur un ordinateur et que vous souhaitez distribuer le contenu à partir d’un autre ordinateur (ou de plusieurs ordinateurs).</span><span class="sxs-lookup"><span data-stu-id="3cbfc-113">This scenario is useful if you are capturing or re-encoding content on one computer, and wish to distribute the content from another computer (or several computers).</span></span> <span data-ttu-id="3cbfc-114">Cela est également utile si vous devez déplacer le contenu d’un ordinateur à l’intérieur d’un pare-feu vers un serveur Windows Media à l’extérieur du pare-feu, car la distribution Push utilise le protocole HTTP.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-114">It is also useful if you need to move content from a computer inside a firewall to a Windows Media server outside the firewall, because push distribution uses the HTTP protocol.</span></span>

> [!Note]  
> <span data-ttu-id="3cbfc-115">Un *point de publication* agit essentiellement comme un redirecteur.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-115">A *publishing point* acts essentially like a redirector.</span></span> <span data-ttu-id="3cbfc-116">Le client spécifie le point de publication dans l’URL (par exemple, mms://MyServer/MyPublishingPoint) et le serveur convertit ce point en requête de contenu.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-116">The client specifies the publishing point in the URL (for example, mms://MyServer/MyPublishingPoint) and the server translates this into a request for content.</span></span>

 

<span data-ttu-id="3cbfc-117">Pour envoyer des données au point de publication, attachez l’objet récepteur Push à l’objet Writer.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-117">To push data to the publishing point, attach the push sink object to the writer object.</span></span> <span data-ttu-id="3cbfc-118">Le récepteur push est utilisé pour ouvrir la connexion au serveur et gérer la session push.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-118">The push sink is used to open the connection to the server and manage the push session.</span></span> <span data-ttu-id="3cbfc-119">L’objet writer gère tous les autres aspects de la création du fichier.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-119">The writer object handles all the other aspects of creating the file.</span></span>

<span data-ttu-id="3cbfc-120">Procédez comme suit :</span><span class="sxs-lookup"><span data-stu-id="3cbfc-120">Perform the following steps:</span></span>

1.  <span data-ttu-id="3cbfc-121">Créez l’objet writer en appelant la fonction [**WMCreateWriter**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-wmcreatewriter) , qui retourne un pointeur [**IWMWriter**](/previous-versions/windows/desktop/api/wmsdkidl/nn-wmsdkidl-iwmwriter) .</span><span class="sxs-lookup"><span data-stu-id="3cbfc-121">Create the writer object by calling the [**WMCreateWriter**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-wmcreatewriter) function, which returns an [**IWMWriter**](/previous-versions/windows/desktop/api/wmsdkidl/nn-wmsdkidl-iwmwriter) pointer.</span></span>
2.  <span data-ttu-id="3cbfc-122">Créez l’objet récepteur Push en appelant la fonction [**WMCreateWriterPushSink**](/previous-versions/windows/desktop/api/wmsdkidl/nf-wmsdkidl-wmcreatewriterpushsink) , qui retourne un pointeur [**IWMWriterPushSink**](/previous-versions/windows/desktop/api/wmsdkidl/nn-wmsdkidl-iwmwriterpushsink) .</span><span class="sxs-lookup"><span data-stu-id="3cbfc-122">Create the push sink object by calling the [**WMCreateWriterPushSink**](/previous-versions/windows/desktop/api/wmsdkidl/nf-wmsdkidl-wmcreatewriterpushsink) function, which returns an [**IWMWriterPushSink**](/previous-versions/windows/desktop/api/wmsdkidl/nn-wmsdkidl-iwmwriterpushsink) pointer.</span></span>
3.  <span data-ttu-id="3cbfc-123">Attachez le récepteur réseau au writer en appelant [**IWMWriterAdvanced :: AddSink**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriteradvanced-addsink) sur le writer, avec un pointeur vers l’interface **IWMWriterPushSink** du récepteur réseau.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-123">Attach the network sink to the writer by calling [**IWMWriterAdvanced::AddSink**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriteradvanced-addsink) on the writer, with a pointer to the network sink's **IWMWriterPushSink** interface.</span></span>
4.  <span data-ttu-id="3cbfc-124">Connectez-vous au serveur en appelant [**IWMWriterPushSink :: Connect**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriterpushsink-connect).</span><span class="sxs-lookup"><span data-stu-id="3cbfc-124">Connect to the server by calling [**IWMWriterPushSink::Connect**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriterpushsink-connect).</span></span>
5.  <span data-ttu-id="3cbfc-125">Écrire le flux.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-125">Write the stream.</span></span> <span data-ttu-id="3cbfc-126">Cette étape implique la définition du profil sur l’objet Writer, l’envoi d’exemples au writer, et éventuellement d’autres tâches.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-126">This step involves setting the profile on the writer object, sending samples to the writer, and possibly other tasks.</span></span> <span data-ttu-id="3cbfc-127">Pour plus d’informations, consultez [écriture de fichiers ASF](writing-asf-files.md).</span><span class="sxs-lookup"><span data-stu-id="3cbfc-127">For more information, see [Writing ASF Files](writing-asf-files.md).</span></span> <span data-ttu-id="3cbfc-128">Des tâches supplémentaires peuvent inclure la définition des attributs de métadonnées (comme décrit dans [utilisation de métadonnées](working-with-metadata.md)) ou la définition de la gestion des droits numériques (DRM) sur le flux (comme décrit dans activation de la [prise en charge de DRM](enabling-drm-support.md)).</span><span class="sxs-lookup"><span data-stu-id="3cbfc-128">Additional tasks might include setting metadata attributes (as described in [Working with Metadata](working-with-metadata.md)) or setting live-DRM on the stream (as described in [Enabling DRM Support](enabling-drm-support.md)).</span></span> <span data-ttu-id="3cbfc-129">Ces tâches sont exécutées exactement telles qu’elles sont pour l’écriture de fichiers ASF.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-129">These tasks are performed exactly as they are for ASF file writing.</span></span>
6.  <span data-ttu-id="3cbfc-130">Une fois que vous avez terminé d’écrire, appelez [**IWMWriterAdvanced :: RemoveSink**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriteradvanced-removesink) sur le writer pour détacher l’objet récepteur push.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-130">After you are done writing, call [**IWMWriterAdvanced::RemoveSink**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriteradvanced-removesink) on the writer to detach the push sink object.</span></span>
7.  <span data-ttu-id="3cbfc-131">Appelez [**IWMWriterPushSink :: EndSession**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriterpushsink-endsession) sur le récepteur de notifications push pour mettre fin à la session avec le serveur.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-131">Call [**IWMWriterPushSink::EndSession**](/previous-versions/windows/desktop/api/Wmsdkidl/nf-wmsdkidl-iwmwriterpushsink-endsession) on the push sink to end the session with the server.</span></span>

<span data-ttu-id="3cbfc-132">Ces étapes sont illustrées dans l’exemple d’application WMVNetWrite.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-132">These steps are illustrated in the WMVNetWrite sample application.</span></span>

> [!Note]  
> <span data-ttu-id="3cbfc-133">Si vous envoyez un fichier vidéo uniquement à faible débit, il est possible qu’il ne démarre pas sur le point de publication pendant plusieurs secondes.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-133">If you are sending a very-low-bit-rate video-only file, it might not start playing on the publishing point for several seconds.</span></span> <span data-ttu-id="3cbfc-134">Cela peut se produire dans différents cas, par exemple lorsqu’un paquet contient de nombreuses petites images vidéo et pas de son, ou lorsqu’il y a un laps de temps important entre le premier paquet et le deuxième paquet dans un fichier vidéo uniquement à faible débit.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-134">This can happen in various cases, for example when a single packet contains many small video frames and no audio, or when there is a long time gap between the first packet and the second packet in a low-bit-rate video-only file.</span></span> <span data-ttu-id="3cbfc-135">Pour éviter ce problème, insérez un flux audio silencieux dans le fichier.</span><span class="sxs-lookup"><span data-stu-id="3cbfc-135">To avoid this problem, insert a silent audio stream into the file.</span></span>
