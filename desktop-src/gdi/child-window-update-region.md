<span data-ttu-id="3f22f-104">Une fenêtre enfant est une fenêtre avec le \_ style WS Child ou WS \_ CHILDWINDOW.</span><span class="sxs-lookup"><span data-stu-id="3f22f-104">A child window is a window with the WS\_CHILD or WS\_CHILDWINDOW style.</span></span> <span data-ttu-id="3f22f-105">Comme les autres styles de fenêtre, les fenêtres enfants reçoivent des messages de [**\_ peinture WM**](wm-paint.md) pour demander une mise à jour.</span><span class="sxs-lookup"><span data-stu-id="3f22f-105">Like other window styles, child windows receive [**WM\_PAINT**](wm-paint.md) messages to prompt updating.</span></span> <span data-ttu-id="3f22f-106">Chaque fenêtre enfant a une région de mise à jour, que le système ou l’application peut définir pour générer des messages de **\_ peinture WM** éventuels.</span><span class="sxs-lookup"><span data-stu-id="3f22f-106">Each child window has an update region, which either the system or the application can set to generate eventual **WM\_PAINT** messages.</span></span>

<span data-ttu-id="3f22f-107">Les régions visibles et mises à jour d’une fenêtre enfant sont affectées par la fenêtre parente de l’enfant ; Cela n’est pas vrai pour les fenêtres d’autres styles.</span><span class="sxs-lookup"><span data-stu-id="3f22f-107">A child window's update and visible regions are affected by the child's parent window; this is not true for windows of other styles.</span></span> <span data-ttu-id="3f22f-108">Le système définit souvent la région de mise à jour de la fenêtre enfant lorsqu’elle définit la région de mise à jour de la fenêtre parente, ce qui amène la fenêtre enfant à recevoir les messages de [**\_ peinture WM**](wm-paint.md) lorsque la fenêtre parente les reçoit.</span><span class="sxs-lookup"><span data-stu-id="3f22f-108">The system often sets the child window's update region when it sets the parent window's update region, causing the child window to receive [**WM\_PAINT**](wm-paint.md) messages when the parent window receives them.</span></span> <span data-ttu-id="3f22f-109">Le système limite l’emplacement de la zone visible de la fenêtre enfant à dans la zone cliente de la fenêtre parente et découpe toute partie de la fenêtre enfant déplacée en dehors de la fenêtre parente.</span><span class="sxs-lookup"><span data-stu-id="3f22f-109">The system limits the location of the child window's visible region to within the client area of the parent window and clips any portion of the child window moved outside the parent window.</span></span>

<span data-ttu-id="3f22f-110">Le système définit la région de mise à jour pour une fenêtre enfant lorsqu’une partie de la région de mise à jour de la fenêtre parente comprend une partie de la fenêtre enfant.</span><span class="sxs-lookup"><span data-stu-id="3f22f-110">The system sets the update region for a child window whenever part of the parent window's update region includes a portion of the child window.</span></span> <span data-ttu-id="3f22f-111">Dans ce cas, le système envoie d’abord un message de [**\_ peinture WM**](wm-paint.md) à la fenêtre parente, puis envoie un message à la fenêtre enfant, ce qui permet à l’enfant de restaurer toutes les parties de la fenêtre sur lesquelles le parent peut avoir été dessiné.</span><span class="sxs-lookup"><span data-stu-id="3f22f-111">In such cases, the system first sends a [**WM\_PAINT**](wm-paint.md) message to the parent window and then sends a message to the child window, allowing the child to restore any portions of the window that the parent may have drawn over.</span></span>

<span data-ttu-id="3f22f-112">Le système ne définit pas la région de mise à jour du parent lorsque la valeur de l’enfant est définie.</span><span class="sxs-lookup"><span data-stu-id="3f22f-112">The system does not set the parent's update region when the child's is set.</span></span> <span data-ttu-id="3f22f-113">Une application ne peut pas générer un message de [**\_ peinture WM**](wm-paint.md) pour la fenêtre parente en invalidant la fenêtre enfant.</span><span class="sxs-lookup"><span data-stu-id="3f22f-113">An application cannot generate a [**WM\_PAINT**](wm-paint.md) message for the parent window by invalidating the child window.</span></span> <span data-ttu-id="3f22f-114">De même, une application ne peut pas générer un message de **\_ peinture WM** pour l’enfant en invalidant une partie de la zone cliente du parent qui se trouve entièrement sous la fenêtre enfant.</span><span class="sxs-lookup"><span data-stu-id="3f22f-114">Similarly, an application cannot generate a **WM\_PAINT** message for the child by invalidating a portion of the parent's client area that lies entirely under the child window.</span></span> <span data-ttu-id="3f22f-115">Dans ce cas, aucune fenêtre ne reçoit un message de **\_ peinture WM** .</span><span class="sxs-lookup"><span data-stu-id="3f22f-115">In such cases, neither window receives a **WM\_PAINT** message.</span></span>

<span data-ttu-id="3f22f-116">Une application peut empêcher la définition de la région de mise à jour d’une fenêtre enfant lorsque la fenêtre parente est définie en spécifiant le \_ style WS CLIPCHILDREN lors de la création de la fenêtre parente.</span><span class="sxs-lookup"><span data-stu-id="3f22f-116">An application can prevent a child window's update region from being set when the parent window's is set by specifying the WS\_CLIPCHILDREN style when creating the parent window.</span></span> <span data-ttu-id="3f22f-117">Quand ce style est défini, le système exclut les fenêtres enfants de la région visible du parent et ignore donc toutes les parties de la région de mise à jour qui peuvent contenir les fenêtres enfants.</span><span class="sxs-lookup"><span data-stu-id="3f22f-117">When this style is set, the system excludes the child windows from the parent's visible region and therefore ignores any portion of the update region that may contain the child windows.</span></span> <span data-ttu-id="3f22f-118">Lorsque l’application peint dans la fenêtre parente, tout dessin qui couvre la fenêtre enfant est coupé, ce qui rend un message de [**\_ peinture WM**](wm-paint.md) ultérieur à la fenêtre enfant inutile.</span><span class="sxs-lookup"><span data-stu-id="3f22f-118">When the application paints in the parent window, any drawing that would cover the child window is clipped, making a subsequent [**WM\_PAINT**](wm-paint.md) message to the child window unnecessary.</span></span>
