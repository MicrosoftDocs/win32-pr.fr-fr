# <a name="physical-address-extension"></a><span data-ttu-id="9e596-103">Extension d’adresse physique</span><span class="sxs-lookup"><span data-stu-id="9e596-103">Physical Address Extension</span></span>

<span data-ttu-id="9e596-104">L’extension d’adresse physique (PAE) est une fonctionnalité de processeur qui permet aux processeurs x86 d’accéder à plus de 4 Go de mémoire physique sur les versions de Windows.</span><span class="sxs-lookup"><span data-stu-id="9e596-104">Physical Address Extension (PAE) is a processor feature that enables x86 processors to access more than 4 GB of physical memory on capable versions of Windows.</span></span> <span data-ttu-id="9e596-105">Certaines versions 32 bits de Windows Server s’exécutant sur des systèmes x86 peuvent utiliser PAE pour accéder à jusqu’à 64 Go ou 128 Go de mémoire physique, en fonction de la taille d’adresse physique du processeur.</span><span class="sxs-lookup"><span data-stu-id="9e596-105">Certain 32-bit versions of Windows Server running on x86-based systems can use PAE to access up to 64 GB or 128 GB of physical memory, depending on the physical address size of the processor.</span></span> <span data-ttu-id="9e596-106">Pour plus d’informations, consultez [limites de mémoire pour les versions de Windows](memory-limits-for-windows-releases.md).</span><span class="sxs-lookup"><span data-stu-id="9e596-106">For details, see [Memory Limits for Windows Releases](memory-limits-for-windows-releases.md).</span></span>

<span data-ttu-id="9e596-107">Les architectures de processeur Intel Itanium et x64 peuvent accéder à plus de 4 Go de mémoire physique en mode natif et, par conséquent, ne fournissent pas l’équivalent d’PAE.</span><span class="sxs-lookup"><span data-stu-id="9e596-107">The Intel Itanium and x64 processor architectures can access more than 4 GB of physical memory natively and therefore do not provide the equivalent of PAE.</span></span> <span data-ttu-id="9e596-108">PAE est utilisé uniquement par les versions 32 bits de Windows s’exécutant sur des systèmes x86.</span><span class="sxs-lookup"><span data-stu-id="9e596-108">PAE is used only by 32-bit versions of Windows running on x86-based systems.</span></span>

<span data-ttu-id="9e596-109">Avec PAE, le système d’exploitation passe de la traduction d’adresses linéaires à deux niveaux à la traduction d’adresses à trois niveaux.</span><span class="sxs-lookup"><span data-stu-id="9e596-109">With PAE, the operating system moves from two-level linear address translation to three-level address translation.</span></span> <span data-ttu-id="9e596-110">Au lieu d’une adresse linéaire qui est divisée en trois champs distincts pour l’indexation dans des tables de mémoire, elle est divisée en quatre champs distincts : un champ de bits de 2 bits, de 2 9 bits et un champ de bits de 12 bits qui correspond à la taille de page implémentée par l’architecture Intel (4 Ko).</span><span class="sxs-lookup"><span data-stu-id="9e596-110">Instead of a linear address being split into three separate fields for indexing into memory tables, it is split into four separate fields: a 2-bit bitfield, two 9-bit bitfields, and a 12-bit bitfield that corresponds to the page size implemented by Intel architecture (4 KB).</span></span> <span data-ttu-id="9e596-111">La taille des entrées de table de pages (PTE) et des entrées de répertoire de page (PDEs) en mode PAE est augmentée de 32 à 64 bits.</span><span class="sxs-lookup"><span data-stu-id="9e596-111">The size of page table entries (PTEs) and page directory entries (PDEs) in PAE mode is increased from 32 to 64 bits.</span></span> <span data-ttu-id="9e596-112">Les bits supplémentaires permettent à une PTE ou PDE de système d’exploitation de référencer la mémoire physique au-delà de 4 Go.</span><span class="sxs-lookup"><span data-stu-id="9e596-112">The additional bits allow an operating system PTE or PDE to reference physical memory above 4 GB.</span></span>

<span data-ttu-id="9e596-113">Dans Windows 32 bits s’exécutant sur des systèmes x64, PAE offre également plusieurs fonctionnalités avancées du système et du processeur, notamment la [prévention de l’exécution des données](data-execution-prevention.md) (DEP), l' [accès non uniforme à la mémoire (NUMA](../procthread/numa-support.md)) et la possibilité d’ajouter de la mémoire à un système pendant qu’il est en cours d’exécution (ajout de mémoire à chaud).</span><span class="sxs-lookup"><span data-stu-id="9e596-113">In 32-bit Windows running on x64-based systems, PAE also enables several advanced system and processor features, including hardware-enabled [Data Execution Prevention](data-execution-prevention.md) (DEP), [non-uniform memory access (NUMA)](../procthread/numa-support.md), and the ability to add memory to a system while it is running (hot-add memory).</span></span>

<span data-ttu-id="9e596-114">PAE ne modifie pas la quantité d’espace d’adressage virtuel disponible pour un processus.</span><span class="sxs-lookup"><span data-stu-id="9e596-114">PAE does not change the amount of virtual address space available to a process.</span></span> <span data-ttu-id="9e596-115">Chaque processus s’exécutant dans Windows 32 bits est toujours limité à un espace d’adressage virtuel de 4 Go.</span><span class="sxs-lookup"><span data-stu-id="9e596-115">Each process running in 32-bit Windows is still limited to a 4 GB virtual address space.</span></span>
