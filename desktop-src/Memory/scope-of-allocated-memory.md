# <a name="scope-of-allocated-memory"></a><span data-ttu-id="418cd-103">Portée de la mémoire allouée</span><span class="sxs-lookup"><span data-stu-id="418cd-103">Scope of Allocated Memory</span></span>

<span data-ttu-id="418cd-104">Toute la mémoire allouée par un processus à l’aide des fonctions d’allocation de mémoire ( [**HeapAlloc**](/windows/desktop/api/HeapApi/nf-heapapi-heapalloc), [**VirtualAlloc**](/windows/win32/api/memoryapi/nf-memoryapi-virtualalloc), [**GlobalAlloc**](/windows/desktop/api/WinBase/nf-winbase-globalalloc)ou [**LocalAlloc**](/windows/desktop/api/WinBase/nf-winbase-localalloc)) est accessible uniquement au processus.</span><span class="sxs-lookup"><span data-stu-id="418cd-104">All memory a process allocates by using the memory allocation functions ( [**HeapAlloc**](/windows/desktop/api/HeapApi/nf-heapapi-heapalloc), [**VirtualAlloc**](/windows/win32/api/memoryapi/nf-memoryapi-virtualalloc), [**GlobalAlloc**](/windows/desktop/api/WinBase/nf-winbase-globalalloc), or [**LocalAlloc**](/windows/desktop/api/WinBase/nf-winbase-localalloc)) is accessible only to the process.</span></span> <span data-ttu-id="418cd-105">Toutefois, la mémoire allouée par une DLL est allouée dans l’espace d’adressage du processus qui a appelé la DLL et n’est pas accessible aux autres processus à l’aide de la même DLL.</span><span class="sxs-lookup"><span data-stu-id="418cd-105">However, memory allocated by a DLL is allocated in the address space of the process that called the DLL and is not accessible to other processes using the same DLL.</span></span> <span data-ttu-id="418cd-106">Pour créer de la mémoire partagée, vous devez utiliser le mappage de fichiers.</span><span class="sxs-lookup"><span data-stu-id="418cd-106">To create shared memory, you must use file mapping.</span></span>

<span data-ttu-id="418cd-107">Le mappage de fichier nommé offre un moyen simple de créer un bloc de mémoire partagée.</span><span class="sxs-lookup"><span data-stu-id="418cd-107">Named file mapping provides an easy way to create a block of shared memory.</span></span> <span data-ttu-id="418cd-108">Un processus peut spécifier un nom lorsqu’il utilise la fonction [**CreateFileMapping**](/windows/desktop/api/WinBase/nf-winbase-createfilemappinga) pour créer un objet de mappage de fichiers.</span><span class="sxs-lookup"><span data-stu-id="418cd-108">A process can specify a name when it uses the [**CreateFileMapping**](/windows/desktop/api/WinBase/nf-winbase-createfilemappinga) function to create a file-mapping object.</span></span> <span data-ttu-id="418cd-109">D’autres processus peuvent spécifier le même nom pour la fonction **CreateFileMapping** ou [**OpenFileMapping**](/windows/desktop/api/WinBase/nf-winbase-openfilemappinga) pour obtenir un descripteur de l’objet de mappage.</span><span class="sxs-lookup"><span data-stu-id="418cd-109">Other processes can specify the same name to either the **CreateFileMapping** or [**OpenFileMapping**](/windows/desktop/api/WinBase/nf-winbase-openfilemappinga) function to obtain a handle to the mapping object.</span></span>

<span data-ttu-id="418cd-110">Chaque processus spécifie son descripteur de l’objet de mappage de fichiers dans la fonction [**MapViewOfFile**](/windows/win32/api/memoryapi/nf-memoryapi-mapviewoffile) pour mapper une vue du fichier dans son propre espace d’adressage.</span><span class="sxs-lookup"><span data-stu-id="418cd-110">Each process specifies its handle to the file-mapping object in the [**MapViewOfFile**](/windows/win32/api/memoryapi/nf-memoryapi-mapviewoffile) function to map a view of the file into its own address space.</span></span> <span data-ttu-id="418cd-111">Les vues de tous les processus pour un objet de mappage de fichier unique sont mappées dans les mêmes pages partageables de stockage physique.</span><span class="sxs-lookup"><span data-stu-id="418cd-111">The views of all processes for a single file-mapping object are mapped into the same sharable pages of physical storage.</span></span> <span data-ttu-id="418cd-112">Toutefois, les adresses virtuelles des vues mappées peuvent varier d’un processus à un autre, sauf si la fonction [**MapViewOfFileEx**](/windows/win32/api/memoryapi/nf-memoryapi-mapviewoffileex) est utilisée pour mapper la vue à une adresse spécifiée.</span><span class="sxs-lookup"><span data-stu-id="418cd-112">However, the virtual addresses of the mapped views can vary from one process to another, unless the [**MapViewOfFileEx**](/windows/win32/api/memoryapi/nf-memoryapi-mapviewoffileex) function is used to map the view at a specified address.</span></span> <span data-ttu-id="418cd-113">Bien qu’elles soient partageables, les pages de stockage physique utilisées pour une vue de fichier mappée ne sont pas globales ; ils ne sont pas accessibles aux processus qui n’ont pas mappé une vue du fichier.</span><span class="sxs-lookup"><span data-stu-id="418cd-113">Although sharable, the pages of physical storage used for a mapped file view are not global; they are not accessible to processes that have not mapped a view of the file.</span></span>

<span data-ttu-id="418cd-114">Toutes les pages validées par le mappage d’une vue d’un fichier sont libérées lorsque le dernier processus avec une vue de l’objet de mappage se termine ou annule la mise en correspondance de sa vue en appelant la fonction [**UnmapViewOfFile**](/windows/win32/api/memoryapi/nf-memoryapi-unmapviewoffile) .</span><span class="sxs-lookup"><span data-stu-id="418cd-114">Any pages committed by mapping a view of a file are released when the last process with a view of the mapping object either terminates or unmaps its view by calling the [**UnmapViewOfFile**](/windows/win32/api/memoryapi/nf-memoryapi-unmapviewoffile) function.</span></span> <span data-ttu-id="418cd-115">À ce stade, le fichier spécifié (le cas échéant) associé à l’objet de mappage est mis à jour.</span><span class="sxs-lookup"><span data-stu-id="418cd-115">At this time, the specified file (if any) associated with the mapping object is updated.</span></span> <span data-ttu-id="418cd-116">Un fichier spécifié peut également être forcé à être mis à jour en appelant la fonction [**FlushViewOfFile**](/windows/win32/api/memoryapi/nf-memoryapi-flushviewoffile) .</span><span class="sxs-lookup"><span data-stu-id="418cd-116">A specified file can also be forced to update by calling the [**FlushViewOfFile**](/windows/win32/api/memoryapi/nf-memoryapi-flushviewoffile) function.</span></span>

<span data-ttu-id="418cd-117">Pour plus d’informations, consultez [mappage de fichiers](file-mapping.md).</span><span class="sxs-lookup"><span data-stu-id="418cd-117">For more information, see [File Mapping](file-mapping.md).</span></span> <span data-ttu-id="418cd-118">Pour obtenir un exemple de mémoire partagée dans une DLL, consultez [utilisation de la mémoire partagée dans une bibliothèque de Dynamic-Link](../dlls/using-shared-memory-in-a-dynamic-link-library.md).</span><span class="sxs-lookup"><span data-stu-id="418cd-118">For an example of shared memory in a DLL, see [Using Shared Memory in a Dynamic-Link Library](../dlls/using-shared-memory-in-a-dynamic-link-library.md).</span></span>
